Visão geral do CodeIgniter¶

CodeIgniter é uma estrutura de aplicativos¶

O CodeIgniter é um kit de ferramentas para pessoas que criam aplicativos da Web usando PHP. Its goal is to enable you to develop projects much faster than you poderia se você estivesse escrevendo código do zero, fornecendo um rico conjunto de librarys para tarefas comumente necessárias, bem como uma interface simples e logical structure to access these libraries. CodeIgniter lets you creatively focus on your project by minimizing the amount of code needed for a given task.

CodeIgniter is Free¶

CodeIgniter is licensed under the MIT license so you can use it however you please. For more information please read the Contrato de licença.

CodeIgniter é peso leve¶

Verdadeiramente leve peso. O sistema central requer apenas alguns poucos libraries. This is in stark contrast to many frameworks that require significantly more resources. Additional libraries are loaded dynamically upon request, based on your needs for a given process, so o sistema básico é muito enxuto e bastante rápido.

O CodeIgniter é rápido¶

Really fast. We challenge you to find a framework that has better performance than CodeIgniter.

CodeIgniter usa MVC¶

CodeIgniter uses the Model-View-Controller approach, which allows great separação entre lógica e apresentação. Isso é particularmente bom para projects in which designers are working with your template files, as the code these files contain will be minimized. We describe MVC in more detail on its own page.

CodeIgniter Generates Clean URLs¶

The URLs generated by CodeIgniter are clean and search-engine friendly. Em vez de usar a abordagem padrão de "string de consulta" para URLs synonymous with dynamic systems, CodeIgniter uses a segment-based approach:

example.com/news/article/345

Nota

By default the index.php file is included in the URL but it can be removed using a simple .htaccess file.

CodeIgniter Packs a Punch¶

CodeIgniter comes with full-range of libraries that enable the most commonly needed web development tasks, like accessing a database, sending email, validating form data, maintaining sessions, manipulating images, working with XML-RPC data and much more.

CodeIgniter é extensível¶

The system can be easily extended through the use of your own libraries, helpers, or through class extensions or system hooks.

O CodeIgniter não requer um mecanismo de modelo¶

Although CodeIgniter does come with a simple template parser that can be optionally used, it does not force you to use one. Template engines simplesmente não pode igualar o desempenho do PHP nativo, e a sintaxe que must be learned to use a template engine is usually only marginally easier than learning the basics of PHP. Consider this block of PHP code:

<ul>
<?php foreach ($addressbook as $name):?>
        <li><?=$name?></li>
<?php endforeach; ?>
</ul>

Compare isso com o pseudo-código usado por um mecanismo de modelo:

<ul>
{foreach from=$addressbook item="name"}
        <li>{$name}</li>
{/foreach}
</ul>

Sim, o exemplo do motor de modelo é um pouco mais limpo, mas vem no price of performance, as the pseudo-code must be converted back into PHP to run. Since one of our goals is maximum performance, we opted to not require the use of a template engine.

CodeIgniter is Thoroughly Documented¶

Os programadores adoram codificar e odeiam escrever documentação. Nós não somos diferente, é claro, mas como a documentação é tão importante as the code itself, we are committed to doing it. Our source code is extremely limpo e bem comentado também.

CodeIgniter has a Friendly Community of Users¶

Nossa crescente comunidade de usuários pode ser vista participando ativamente Community Forums.

View geral do CodeIgniter¶

CodeIgniter é uma estrutura de aplicativos¶

O CodeIgniter é um kit de ferramentas para pessoas que criam aplicativos da Web usando PHP Seu objetivo é permitir que você desenvolva projetos muito mais rápido do que você poderia se você estivesse escrevendo código do zero, fornecendo um rico conjunto de librarys para tarefas comumente necessárias, bem como uma interface simples e estrutura lógica para acessar essas Librarys. O CodeIgniter permite que você focar de forma criativa em seu projeto, minimizando a quantidade de código necessária para uma determinada tarefa.

O CodeIgniter é gratuito¶

O CodeIgniter é licenciado sob a licença do MIT para que você possa usá-lo no entanto Você por favor. Para mais informações, por favor leia o Contrato de licença.

CodeIgniter é peso leve¶

Verdadeiramente leve peso. O sistema central requer apenas alguns poucos Librarys. Isto está em contraste com muitos frameworks que requerem significativamente mais recursos. Librarys adicionais são carregadas dinamicamente a pedido, com base nas suas necessidades de um determinado processo, o sistema básico é muito enxuto e bastante rápido.

O CodeIgniter é rápido¶

Muito depressa. Nós desafiamos você a encontrar uma estrutura que tenha melhor desempenho do que o CodeIgniter.

CodeIgniter usa MVC¶

O CodeIgniter usa a abordagem Model-View-Controller, que permite grande separação entre lógica e apresentação. Isso é particularmente bom para projetos nos quais designers estão trabalhando com seus arquivos de Model, código que esses arquivos contêm será minimizado. Nós descrevemos MVC em mais detalhe na sua própria página.

CodeIgniter gera URLs limpas¶

As URLs geradas pelo CodeIgniter são limpas e amigáveis ao mecanismo de busca. Em vez de usar a abordagem padrão de "string de Query" para URLs sinônimo de sistemas dinâmicos, o CodeIgniter usa um sistema baseado em segmentos. approach:

example.com/news/article/345

Nota

By default the index.php file is included in the URL but it can be removed using a simple .htaccess file.

CodeIgniter embala um soco¶

O CodeIgniter vem com uma gama completa de Librarys que permitem o máximo tarefas de desenvolvimento da web normalmente necessárias, como acessar um Database, envio de e-mail, validação de dados de formulários, manutenção de sessões, manipulação de imagens, trabalhando com dados XML-RPC e muito mais.

CodeIgniter é extensível¶

O sistema pode ser facilmente estendido através do uso de suas próprias Librarys, Helpers, ou através de extensões de classe ou ganchos de sistema.

O CodeIgniter não requer um mecanismo de Model¶

Although CodeIgniter does come with a simple template parser that can ser usado opcionalmente, não o força a usar um. Motores de Models simplesmente não pode igualar o desempenho do PHP nativo, e a sintaxe que deve ser aprendido a usar um motor de Model é geralmente apenas marginalmente mais fácil do que aprender o básico do PHP. Considere este bloco de código PHP:

<ul>
<?php foreach ($addressbook as $name):?>
        <li><?=$name?></li>
<?php endforeach; ?>
</ul>

Compare isso com o pseudo-código usado por um mecanismo de Model:

<ul>
{foreach from=$addressbook item="name"}
        <li>{$name}</li>
{/foreach}
</ul>

Sim, o exemplo do motor de Model é um pouco mais limpo, mas vem no preço do desempenho, como o pseudo-código deve ser convertido de volta para o PHP to run. Since one of our goals is maximum performance, we opted to not requerem o uso de um mecanismo de Model.

O CodeIgniter está bem documentado¶

Os programadores adoram codificar e odeiam escrever documentação. Nós não somos diferente, é claro, mas como a documentação é tão importante Enquanto o código em si, estamos comprometidos em fazê-lo. Nosso código-fonte é extremamente limpo e bem comentado também.

O CodeIgniter tem uma comunidade amigável de usuários¶

Nossa crescente comunidade de usuários pode ser vista participando ativamente Fóruns da comunidade.

View geral do CodeIgniter¶

CodeIgniter é uma estrutura de aplicativos¶

O CodeIgniter é um kit de ferramentas para pessoas que criam aplicativos da Web usando PHP Seu objetivo é permitir que você desenvolva projetos muito mais rápido do que você poderia se você estivesse escrevendo código do zero, fornecendo um rico conjunto de librarys para tarefas comumente necessárias, bem como uma interface simples e estrutura lógica para acessar essas Librarys. O CodeIgniter permite que você focar de forma criativa em seu projeto, minimizando a quantidade de código necessária para uma determinada tarefa.

O CodeIgniter é gratuito¶

O CodeIgniter é licenciado sob a licença do MIT para que você possa usá-lo no entanto Você por favor. Para mais informações, por favor leia o Contrato de licença.

CodeIgniter é peso leve¶

Verdadeiramente leve peso. O sistema central requer apenas alguns poucos Librarys. Isto está em contraste com muitos frameworks que requerem significativamente mais recursos. Librarys adicionais são carregadas dinamicamente a pedido, com base nas suas necessidades de um determinado processo, o sistema básico é muito enxuto e bastante rápido.

O CodeIgniter é rápido¶

Muito depressa. Nós desafiamos você a encontrar uma estrutura que tenha melhor desempenho do que o CodeIgniter.

CodeIgniter usa MVC¶

O CodeIgniter usa a abordagem Model-View-Controller, que permite grande separação entre lógica e apresentação. Isso é particularmente bom para projetos nos quais designers estão trabalhando com seus arquivos de Model, código que esses arquivos contêm será minimizado. Nós descrevemos MVC em mais detalhe na sua própria página.

CodeIgniter gera URLs limpas¶

As URLs geradas pelo CodeIgniter são limpas e amigáveis ao mecanismo de busca. Em vez de usar a abordagem padrão de "string de Query" para URLs sinônimo de sistemas dinâmicos, o CodeIgniter usa um sistema baseado em segmentos. approach:

example.com/news/article/345

Nota

By default the index.php file is included in the URL but it can be removed using a simple .htaccess file.

CodeIgniter embala um soco¶

O CodeIgniter vem com uma gama completa de Librarys que permitem o máximo tarefas de desenvolvimento da web normalmente necessárias, como acessar um Database, envio de e-mail, validação de dados de formulários, manutenção de sessões, manipulação de imagens, trabalhando com dados XML-RPC e muito mais.

CodeIgniter é extensível¶

O sistema pode ser facilmente estendido através do uso de suas próprias Librarys, Helpers, ou através de extensões de classe ou ganchos de sistema.

O CodeIgniter não requer um mecanismo de Model¶

Although CodeIgniter does come with a simple template parser that can ser usado opcionalmente, não o força a usar um. Motores de Models simplesmente não pode igualar o desempenho do PHP nativo, e a sintaxe que deve ser aprendido a usar um motor de Model é geralmente apenas marginalmente mais fácil do que aprender o básico do PHP. Considere este bloco de código PHP:

<ul>
<?php foreach ($addressbook as $name):?>
        <li><?=$name?></li>
<?php endforeach; ?>
</ul>

Compare isso com o pseudo-código usado por um mecanismo de Model:

<ul>
{foreach from=$addressbook item="name"}
        <li>{$name}</li>
{/foreach}
</ul>

Sim, o exemplo do motor de Model é um pouco mais limpo, mas vem no preço do desempenho, como o pseudo-código deve ser convertido de volta para o PHP to run. Since one of our goals is maximum performance, we opted to not requerem o uso de um mecanismo de Model.

O CodeIgniter está bem documentado¶

Os programadores adoram codificar e odeiam escrever documentação. Nós não somos diferente, é claro, mas como a documentação é tão importante as the código em si, estamos comprometidos em fazê-lo. Nosso código-fonte é extremamente limpo e bem comentado também.

O CodeIgniter tem uma comunidade amigável de usuários¶

Nossa crescente comunidade de usuários pode ser vista participando ativamente Fóruns da comunidade.
