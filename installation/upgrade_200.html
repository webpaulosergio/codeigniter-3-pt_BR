Atualizando de 1.7.2 para 2.0.0¶

Antes de executar uma atualização, você deve colocar seu site off-line replacing the index.php file with a static one.

Update Instructions¶

Etapa 1: atualize seus arquivos do CodeIgniter¶

Replace all files and directories in your “system” folder except sua pasta de aplicativos.

Nota

Se você tem algum arquivo personalizado desenvolvido nestas pastas, por favor make copies of them first.

Step 2: Adjust get_dir_file_info() where necessary¶

A versão 2.0.0 traz uma alteração não compatível get_dir_file_info() no Arquivo Helper. Non-backwards compatible changes are extremely rare in CodeIgniter, but this one we feel was warranted due to how easy it was to create serious server performance issues. If you need recursiveness where you are using this helper function, change such instances, setting the second parameter, $top_level_only para FALSE:

get_dir_file_info('/path/to/directory', FALSE);

Etapa 3: converter seus plug-ins para os auxiliares¶

2.0.0 gets rid of the “Plugin” system as their functionality was idêntico aos Helpers, mas não extensível. Você precisará renomear seu plugin files from filename_pi.php to filename_helper.php, move them to your helpers folder, and change all instances of:

$this->load->plugin('foo');

to

$this->load->helper('foo');

Etapa 4: atualizar dados criptografados armazenados¶

Nota

If your application does not use the Encrypt library, does não armazena dados criptografados permanentemente ou está em um ambiente que não não suporta Mcrypt, você pode pular esta etapa.

o Encrypt library has had a number of improvements, some for encryption strength and some for performance, that has an unavoidable consequence of making it no longer possible to decode encrypted data produzido pela versão original desta library. Para ajudar com o transition, a new method has been added, encode_from_legacy() that irá decodificar os dados com o algoritmo original e retornar uma codificação string usando os métodos aprimorados. Isso permitirá que você facilmente substituir dados criptografados obsoletos por novos em seus aplicativos, a mosca ou em massa.

Por favor leia como usar isso method in the Encrypt library documentation.

Step 5: Remove loading calls for the compatibility helper.¶

O auxiliar de compatibilidade foi removido do núcleo do CodeIgniter. Todos methods in it should be natively available in supported PHP versions.

Step 6: Update Class extension¶

Todas as classes principais agora são prefixadas com CI_. Atualizar modelos e Controllers to extend CI_Model and CI_Controller, respectively.

Etapa 7: Atualizar Chamadas do Construtor Pai¶

Todas as classes nativas do CodeIgniter agora usam o PHP 5 __construct() convention. Please update extended libraries to call parent::__construct().

Step 8: Move any core extensions to application/core¶

Any extensions to core classes (e.g. MY_Controller.php) in your application/libraries folder must be moved to the new pasta application/core.

Step 9: Update your user guide¶

Por favor, substitua sua cópia local do guia do usuário pela nova versão, including the image files.

Update Notas¶

Please refer to the 2.0.0 Change Log for full details, but here are some of the larger changes that are more likely to impactar seu código:

  • O andaime foi removido.
  • o CAPTCHA plugin in now a helper.
  • O plug-in de calendário JavaScript foi removido.
  • o system/cache and system/logs directories are now in the application directory.
  • o Validation class has been removed. Please see the Validação de Formulário library
  • “default” is now a reserved name.
  • o xss_clean() function has moved to the Segurança Class.
  • do_xss_clean() agora retorna FALSE se o arquivo enviado falhar nas verificações de XSS.
  • o Session Class requires now the use of an encryption key set in the config file.
  • o following deprecated Active Record functions have been removed: orwhere, orlike, groupby, orhaving, orderby, getwhere.
  • _drop_database() and _create_database() functions have been removed from the db utility drivers.
  • o dohash() function of the Segurança helper has been renamed to do_hash() for naming consistency.

o config folder¶

o following files have been changed:

  • config.php
  • database.php
  • mimes.php
  • routes.php
  • user_agents.php

o following files have been added:

  • foreign_chars.php
  • profiler.php

Atualizando de 1.7.2 para 2.0.0¶

Antes de executar uma atualização, você deve colocar seu site off-line substituindo o arquivo index.php por um estático.

Instruções de atualização¶

Etapa 1: atualize seus arquivos do CodeIgniter¶

Substitua todos os arquivos e diretórios na sua pasta "system" except sua pasta de aplicativos.

Nota

Se você tem algum arquivo personalizado desenvolvido nestas pastas, por favor faça cópias deles primeiro.

Etapa 2: ajuste get_dir_file_info() onde for necessário¶

A versão 2.0.0 traz uma alteração não compatível get_dir_file_info() no File Helper. Alterações não compatíveis com versões anteriores são extremamente raros no CodeIgniter, mas este que nos sentimos foi garantido devido à facilidade de criar sérios problemas de desempenho do servidor. E se you need recursiveness where you are using this helper function, altere tais instâncias, configurando o segundo parâmetro, $top_level_only para FALSE:

get_dir_file_info('/path/to/directory', FALSE);

Etapa 3: converter seus plug-ins para os Helperes¶

2.0.0 se livra do sistema "Plugin", pois sua funcionalidade era idêntico aos Helpers, mas não extensível. Você precisará renomear seu arquivos plugin de filename_pi.php para filename_helper.php, mova-os para sua pasta de Helpers e altere todas as instâncias de:

$this->load->plugin('foo');

to

$this->load->helper('foo');

Etapa 4: atualizar dados criptografados armazenados¶

Nota

Se o seu aplicativo não usar a Library Criptografar, não armazena dados criptografados permanentemente ou está em um ambiente que não não suporta Mcrypt, você pode pular esta etapa.

A Library Encrypt teve várias melhorias, algumas para força de criptografia e alguns para o desempenho, que tem um inevitável conseqüência de tornar impossível a decodificação de dados criptografados produzido pela versão original desta library. Para ajudar com o transição, um novo método foi adicionado, encode_from_legacy() que irá decodificar os dados com o algoritmo original e retornar uma codificação string usando os métodos aprimorados. Isso permitirá que você facilmente substituir dados criptografados obsoletos por novos em seus aplicativos, a mosca ou em massa.

Por favor leia como usar isso method na Library Criptografar documentation.

Etapa 5: Remova as chamadas de carregamento do Helper de compatibilidade.¶

O Helper de compatibilidade foi removido do núcleo do CodeIgniter. Todos métodos devem estar nativamente disponíveis nas versões suportadas do PHP.

Etapa 6: atualizar extensão de classe¶

Todas as classes principais agora são prefixadas com CI_. Atualizar Models e Controlleres para estender CI_Model e CI_Controller, respectivamente.

Etapa 7: Atualizar Chamadas do Construtor Pai¶

Todas as classes nativas do CodeIgniter agora usam o PHP 5 __construct() convenção. Atualize as Librarys estendidas para chamar parent::__construct().

Etapa 8: Mova as extensões principais para o aplicativo/núcleo¶

Quaisquer extensões para as classes principais (por exemplo, MY_Controller.php) no seu pasta de aplicativos/Librarys deve ser movida para o novo pasta application/core.

Etapa 9: atualize seu guia do usuário¶

Por favor, substitua sua cópia local do guia do usuário pela nova versão, incluindo os arquivos de imagem.

Update Notas¶

Por favor, consulte o 2.0.0 Change Log por completo detalhes, mas aqui estão algumas das maiores mudanças que são mais prováveis impactar seu código:

  • O andaime foi removido.
  • O plugin CAPTCHA agora em um helper.
  • O plug-in de calendário JavaScript foi removido.
  • o system/cache and system/logs directories are now in the application directory.
  • A classe de validação foi removida. por favor veja o Validação de Formulário library
  • "Padrão" agora é um nome reservado.
  • A função xss_clean() foi movida para o Segurança Class.
  • do_xss_clean() agora retorna FALSE se o arquivo enviado falhar nas verificações de XSS.
  • o Classe de sessão requer agora o uso de um chave de criptografia definida no arquivo de configuração.
  • o following deprecated Active Record functions have been removed: orwhere, orlike, groupby, orhaving, orderby, getwhere.
  • _drop_database() and _create_database() functions have been removed dos drivers de utilitário do Database.
  • o dohash() function of the Segurança helper has been renamed to do_hash() for naming consistency.

o config folder¶

o following files have been changed:

  • config.php
  • database.php
  • mimes.php
  • routes.php
  • user_agents.php

o following files have been added:

  • foreign_chars.php
  • profiler.php

Atualizando de 1.7.2 para 2.0.0¶

Antes de executar uma atualização, você deve colocar seu site off-line substituindo o arquivo index.php por um estático.

Instruções de atualização¶

Etapa 1: atualize seus arquivos do CodeIgniter¶

Substitua todos os arquivos e diretórios na sua pasta "system" except sua pasta de aplicativos.

Nota

Se você tem algum arquivo personalizado desenvolvido nestas pastas, por favor faça cópias deles primeiro.

Etapa 2: ajuste get_dir_file_info() onde for necessário¶

A versão 2.0.0 traz uma alteração não compatível get_dir_file_info() no File Helper. Alterações não compatíveis com versões anteriores são extremamente raros no CodeIgniter, mas este que nos sentimos foi garantido devido à facilidade de criar sérios problemas de desempenho do servidor. E se you need recursiveness where you are using this helper function, altere tais instâncias, configurando o segundo parâmetro, $top_level_only to FALSE:

get_dir_file_info('/path/to/directory', FALSE);

Etapa 3: converter seus plug-ins para os Helperes¶

2.0.0 se livra do sistema "Plugin", pois sua funcionalidade era idêntico aos Helpers, mas não extensível. Você precisará renomear seu arquivos plugin de filename_pi.php para filename_helper.php, mova-os para sua pasta de Helpers e altere todas as instâncias de:

$this->load->plugin('foo');

to

$this->load->helper('foo');

Etapa 4: atualizar dados criptografados armazenados¶

Nota

Se o seu aplicativo não usar a Library Criptografar, não armazena dados criptografados permanentemente ou está em um ambiente que não não suporta Mcrypt, você pode pular esta etapa.

A Library Encrypt teve várias melhorias, algumas para força de criptografia e alguns para o desempenho, que tem um inevitável conseqüência de tornar impossível a decodificação de dados criptografados produzido pela versão original desta library. Para ajudar com o transição, um novo método foi adicionado, encode_from_legacy() que irá decodificar os dados com o algoritmo original e retornar uma codificação string usando os métodos aprimorados. Isso permitirá que você facilmente substituir dados criptografados obsoletos por novos em seus aplicativos, a mosca ou em massa.

Por favor leia como usar isso method na Library Criptografar documentation.

Etapa 5: Remova as chamadas de carregamento do Helper de compatibilidade.¶

O Helper de compatibilidade foi removido do núcleo do CodeIgniter. Todos métodos devem estar nativamente disponíveis nas versões suportadas do PHP.

Etapa 6: atualizar extensão de classe¶

Todas as classes principais agora são prefixadas com CI_. Atualizar Models e Controlleres para estender CI_Model e CI_Controller, respectivamente.

Etapa 7: Atualizar Chamadas do Construtor Pai¶

Todas as classes nativas do CodeIgniter agora usam o PHP 5 __construct() convenção. Atualize as Librarys estendidas para chamar parent::__construct().

Etapa 8: Mova as extensões principais para o aplicativo/núcleo¶

Quaisquer extensões para as classes principais (por exemplo, MY_Controller.php) no seu pasta de aplicativos/Librarys deve ser movida para o novo pasta application/core.

Etapa 9: atualize seu guia do usuário¶

Por favor, substitua sua cópia local do guia do usuário pela nova versão, incluindo os arquivos de imagem.

Update Notas¶

Por favor, consulte o 2.0.0 Change Log for full detalhes, mas aqui estão algumas das maiores mudanças que são mais prováveis impactar seu código:

  • O andaime foi removido.
  • O plugin CAPTCHA agora em um helper.
  • O plug-in de calendário JavaScript foi removido.
  • o system/cache and system/logs directories are now in the application directory.
  • A classe de validação foi removida. por favor veja o Validação de Formulário library
  • "Padrão" agora é um nome reservado.
  • A função xss_clean() foi movida para o Segurança Class.
  • do_xss_clean() agora retorna FALSE se o arquivo enviado falhar nas verificações de XSS.
  • o Classe de sessão requer agora o uso de um chave de criptografia definida no arquivo de configuração.
  • o following deprecated Active Record functions have been removed: orwhere, orlike, groupby, orhaving, orderby, getwhere.
  • _drop_database() and _create_database() functions have been removed dos drivers de utilitário do Database.
  • o dohash() function of the Segurança helper has been renamed to do_hash() for naming consistency.

o config folder¶

o following files have been changed:

  • config.php
  • database.php
  • mimes.php
  • routes.php
  • user_agents.php

o following files have been added:

  • foreign_chars.php
  • profiler.php
