Páginas estáticas¶

Nota: This tutorial assumes you’ve downloaded CodeIgniter and installed the framework in your ambiente de desenvolvimento.

A primeira coisa que você vai fazer é configurar um controller lidar páginas estáticas. Um controlador é simplesmente uma classe que ajuda a delegar trabalho. It is the glue of your web application.

For example, when a call is made to:

http://example.com/news/latest/10

Podemos imaginar que existe um controlador chamado “news”. O método being called on news would be “latest”. o news method’s job could be to pegue 10 itens de notícias e os renderize na página. Muitas vezes no MVC, you’ll see URL patterns that match:

http://example.com/[controller-class]/[controller-method]/[arguments]

As URL schemes become more complex, this may change. But for now, this is all we will need to know.

Create a file at application/controllers/Pages.php with the following code.

<?php
class Pages extends CI_Controller {

        public function view($page = 'home')
        {
        }
}

You have created a class named Pages, with a view method that accepts one argument named $page. o Pages class is extending the CI_Controller class. This means that the new pages class can access the methods and variables defined in the CI_Controller class (system/core/Controller.php).

o controlador é o que vai se tornar o centro de cada pedido to sua aplicação web. Em discussões muito técnicas do CodeIgniter, pode be referred to as the super object. Like any php class, you refer to it within your controllers as $this. Referring to $this is how you will load libraries, views, and generally command the framework.

Agora você criou seu primeiro método, é hora de fazer uma página básica templates. We will be creating two “views” (page templates) that act as our page footer and header.

Create the header at application/views/templates/header.php and add the following code:

<html>
        <head>
                <title>CodeIgniter Tutorial</title>
        </head>
        <body>

                <h1><?php echo $title; ?></h1>

o header contains the basic HTML code that you’ll want to display before loading the main view, together with a heading. It will also output the $title variable, which we’ll define later in the controller. Now, create a footer at application/views/templates/footer.php that includes the following code:

                <em>&copy; 2015</em>
        </body>
</html>

Adding logic to the controller¶

Earlier you set up a controller with a view() method. o method accepts one parameter, which is the name of the page to be loaded. o static page templates will be located in the application/views/pages/ directory.

In that directory, create two files named home.php and about.php. Within those files, type some text − anything you’d like − and save them. Se você gosta de ser particularmente original, tente “Hello World!”.

In order to load those pages, you’ll have to check whether the requested page actually exists:

public function view($page = 'home')
{
        if ( ! file_exists(APPPATH.'views/pages/'.$page.'.php'))
        {
                // Whoops, we don't have a page for that!
                show_404();
        }

        $data['title'] = ucfirst($page); // Capitalize the first letter

        $this->load->view('templates/header', $data);
        $this->load->view('pages/'.$page, $data);
        $this->load->view('templates/footer', $data);
}

Now, when the page does exist, it is loaded, including the header and footer, and displayed to the user. If the page doesn’t exist, a “404 Page not found” error is shown.

o first line in this method checks whether the page actually exists. PHP’s native file_exists() function is used to check whether the file is where it’s expected to be. show_404() is a built-in CodeIgniter function that renders the default error page.

In the header template, the $title variable was used to customize the page title. o value of title is defined in this method, but instead of assigning the value to a variable, it is assigned to the title element in the $data array.

o last thing that has to be done is loading the views in the order they should be displayed. o second parameter in the view() method is used to pass values to the view. Each value in the $data array is assigned to a variable with the name of its key. So the value of $data['title'] in the controller is equivalent to $title in the view.

Roteamento¶

o controller is now functioning! Point your browser to [your-site-url]index.php/pages/view para ver sua página. Quando você visita index.php/pages/view/about você verá a página sobre, novamente incluindo the header and footer.

Using custom routing rules, you have the power to map any URI to any controller and method, and break free from the normal convention: http://example.com/[controller-class]/[controller-method]/[arguments]

Let’s do that. Open the routing file located at application/config/routes.php and add the following two lines. Remove all other code that sets any element in the $route array.

$route['default_controller'] = 'pages/view';
$route['(:any)'] = 'pages/view/$1';

CodeIgniter reads its routing rules from top to bottom and routes the request to the first matching rule. Each rule is a regular expression (lado esquerdo) mapeado para um nome de controlador e método separado por barras (right-side). When a request comes in, CodeIgniter looks for the first corresponder e chama o controlador e método apropriado, possivelmente com arguments.

More information about routing can be found in the Roteamento de URI documentation.

Here, the second rule in the $routes array matches any request using the wildcard string (:any). and passes the parameter to the view() method of the Pages class.

Now visit index.php/about. Did it get routed correctly to the view() method in the pages controller? Awesome!

Páginas estáticas¶

Note: Este tutorial assume que você baixou o CodeIgniter e instalou o quadro in your ambiente de desenvolvimento.

A primeira coisa que você vai fazer é configurar um controller to handle páginas estáticas. Um Controller é simplesmente uma classe que ajuda a delegar trabalho. É a cola do seu aplicativo da web.

Por exemplo, quando uma chamada é feita para:

http://example.com/news/latest/10

Podemos imaginar que existe um Controller chamado "news". O método ser chamado em notícias seria "mais recente". O trabalho do método de notícias poderia ser pegue 10 itens de notícias e os renderize na página. Muitas vezes no MVC, você verá padrões de URL que correspondem:

http://example.com/[controller-class]/[controller-method]/[arguments]

medida que os esquemas de URL se tornam mais complexos, isso pode mudar. Mas por enquanto, isso é tudo que precisamos saber.

Create a file at application/controllers/Pages.php with the following code.

<?php
class Pages extends CI_Controller {

        public function view($page = 'home')
        {
        }
}

You have created a class named Pages, with a view method that accepts one argument named $page. o Pages class is extending the CI_Controller classe. Isso significa que a nova classe de páginas pode acessar o methods and variables defined in the CI_Controller class (system/core/Controller.php).

o Controller é o que vai se tornar o centro de cada pedido to sua aplicação web. Em discussões muito técnicas do CodeIgniter, pode be referred to as the super object. Like any php class, you refer to it within your controllers as $this. Referring to $this is how você carregará Librarys, visualizações e geralmente comandará a estrutura.

Agora você criou seu primeiro método, é hora de fazer uma página básica Models. Nós estaremos criando duas "visões" (Models de página) que funcionam como nosso rodapé de página e cabeçalho.

Create the header at application/views/templates/header.php and add o seguinte código:

<html>
        <head>
                <title>CodeIgniter Tutorial</title>
        </head>
        <body>

                <h1><?php echo $title; ?></h1>

o header contains the basic HTML code that you’ll want to display antes de carregar a vista principal, junto com um cabeçalho. Ele também output the $title variable, which we’ll define later in the controller. Now, create a footer at application/views/templates/footer.php that includes o seguinte código:

                <em>&copy; 2015</em>
        </body>
</html>

Adicionando lógica ao Controller¶

Earlier you set up a controller with a view() method. o method accepts one parameter, which is the name of the page to be loaded. o Models de página estáticos estarão localizados no application/views/pages/ directory.

In that directory, create two files named home.php and about.php. Within those files, type some text − anything you’d like − and save them. Se você gosta de ser particularmente original, tente "Hello World!".

Para carregar essas páginas, você terá que verificar se o arquivo solicitado página realmente existe:

public function view($page = 'home')
{
        if ( ! file_exists(APPPATH.'views/pages/'.$page.'.php'))
        {
                // Whoops, we don't have a page for that!
                show_404();
        }

        $data['title'] = ucfirst($page); // Capitalize the first letter

        $this->load->view('templates/header', $data);
        $this->load->view('pages/'.$page, $data);
        $this->load->view('templates/footer', $data);
}

Agora, quando a página existe, ela é carregada, incluindo o cabeçalho e rodapé e exibido ao usuário. Se a página não existir, um "404 Página não encontrada "erro é mostrado.

o first line in this method checks whether the page actually exists. PHP’s native file_exists() function is used to check whether the file is where it’s expected to be. show_404() is a built-in CodeIgniter função que renderiza a página de erro padrão.

In the header template, the $title variable was used to customize the page title. o value of title is defined in this method, but instead of atribuindo o valor a uma variável, ele é atribuído ao elemento title in the $data array.

o last thing that has to be done is loading the views in the order they should be displayed. o second parameter in the view() method is used to pass values to the view. Each value in the $data array is atribuído a uma variável com o nome da sua chave. Então o valor de $data['title'] in the controller is equivalent to $title in the view.

Roteamento¶

o controller is now functioning! Point your browser to [your-site-url]index.php/pages/view para ver sua página. Quando você visita index.php/pages/view/about você verá a página sobre, novamente incluindo o cabeçalho e rodapé.

Usando regras de roteamento personalizadas, você tem o poder de mapear qualquer URI para qualquer Controller e método, e libertar-se da convenção normal: http://example.com/[controller-class]/[controller-method]/[arguments]

Vamos fazer isso. Abra o arquivo de roteamento localizado em application/config/routes.php e adicione as duas linhas seguintes. Remove all other code that sets any element in the $route array.

$route['default_controller'] = 'pages/view';
$route['(:any)'] = 'pages/view/$1';

O CodeIgniter lê suas regras de roteamento de cima para baixo e direciona o pedido para a primeira regra de correspondência. Cada regra é uma expressão regular (lado esquerdo) mapeado para um nome de Controller e método separado por barras (lado direito). Quando um pedido chega, o CodeIgniter procura o primeiro corresponder e chama o Controller e método apropriado, possivelmente com arguments.

More information about routing can be found in the Roteamento de URI documentation.

Here, the second rule in the $routes array matches any request using the wildcard string (:any). and passes the parameter to the view() method of the Pages class.

Now visit index.php/about. Did it get routed correctly to the view() método no Controller de páginas? Impressionante!
