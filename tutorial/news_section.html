Seção de notícias¶

Na última seção, examinamos alguns conceitos básicos do framework by writing a class that includes static pages. We cleaned up the URI by adding custom routing rules. Now it’s time to introduce dynamic content and start using a database.

Setting up your model¶

Instead of writing database operations right in the controller, queries devem ser colocados em um modelo, para que possam ser facilmente reutilizados posteriormente. Modelos are the place where you retrieve, insert, and update information in your database or other data stores. They represent your data.

Open up the application/models/ directory and create a new file called News_model.php e adicione o seguinte código. Certifique-se de que você configurou your database properly as described here.

<?php
class News_model extends CI_Model {

        public function __construct()
        {
                $this->load->database();
        }
}

This code looks similar to the controller code that was used earlier. It creates a new model by extending CI_Model and loads the database library. This will make the database class available through the $this->db object.

Before querying the database, a database schema has to be created. Connect to your database and run the SQL command below (MySQL). Adicione também alguns registros de sementes.

CREATE TABLE news (
        id int(11) NOT NULL AUTO_INCREMENT,
        title varchar(128) NOT NULL,
        slug varchar(128) NOT NULL,
        text text NOT NULL,
        PRIMARY KEY (id),
        KEY slug (slug)
);

Now that the database and a model have been set up, you’ll need a method to get all of our posts from our database. To do this, the database abstraction layer that is included with CodeIgniter — Query Builder — is used. This makes it possível escrever suas 'consultas' uma vez e fazê-las funcionar all sistemas de Database suportados. Add the seguindo o código para o seu modelo.

public function get_news($slug = FALSE)
{
        if ($slug === FALSE)
        {
                $query = $this->db->get('news');
                return $query->result_array();
        }

        $query = $this->db->get_where('news', array('slug' => $slug));
        return $query->row_array();
}

With this code you can perform two different queries. You can get all registros de notícias, ou obter uma notícia por sua slug. You might have noticed that the $slug variable wasn’t sanitized before running the query; Query Builder faz isso por você.

Display the news¶

Agora que as consultas estão escritas, o modelo deve estar vinculado às visualizações que vão exibir os itens de notícias para o usuário. Isso poderia ser feito in our Pages controller created earlier, but for the sake of clarity, a new News controller is defined. Create the new controller at application/controllers/News.php.

<?php
class News extends CI_Controller {

        public function __construct()
        {
                parent::__construct();
                $this->load->model('news_model');
                $this->load->helper('url_helper');
        }

        public function index()
        {
                $data['news'] = $this->news_model->get_news();
        }

        public function view($slug = NULL)
        {
                $data['news_item'] = $this->news_model->get_news($slug);
        }
}

Looking at the code, you may see some similarity with the files we created earlier. First, the __construct() method: it calls the constructor of its parent class (CI_Controller) and loads the model, so it can be used in all other methods in this controller. Também carrega uma coleção de Helper de URL functions, because we’ll use one of them in a view later.

Next, there are two methods to view all news items and one for a specific news item. You can see that the $slug variable is passed to the model’s método no segundo método. O modelo está usando essa lesma para identificar item de notícias a ser devolvido.

Agora os dados são recuperados pelo controlador através do nosso modelo, mas nada é exibido ainda. A próxima coisa a fazer é passar esses dados para as visualizações.

public function index()
{
        $data['news'] = $this->news_model->get_news();
        $data['title'] = 'News archive';

        $this->load->view('templates/header', $data);
        $this->load->view('news/index', $data);
        $this->load->view('templates/footer');
}

The code above gets all news records from the model and assigns it to a variable. The value for the title is also assigned to the $data['title'] element and all data is passed to as visualizações. You now need to create a view to render the news items. Create application/views/news/index.php and add the next piece of code.

<h2><?php echo $title; ?></h2>

<?php foreach ($news as $news_item): ?>

        <h3><?php echo $news_item['title']; ?></h3>
        <div class="main">
                <?php echo $news_item['text']; ?>
        </div>
        <p><a href="<?php echo site_url('news/'.$news_item['slug']); ?>">View article</a></p>

<?php endforeach; ?>

Aqui, cada notícia é colocada em loop e exibida ao usuário. Você pode ver nós wrote our template in PHP mixed with HTML. If you prefer to use a template language, you can use CodeIgniter’s Template Analisador classe ou um analisador de terceiros.

The news overview page is now done, but a page to display individual notícias ainda estão ausentes. O modelo criado anteriormente é feito em maneira que ele pode ser facilmente usado para essa funcionalidade. Você só precisa adicione algum código ao controlador e crie uma nova visão. Volte para o News controller and update view() with the following:

public function view($slug = NULL)
{
        $data['news_item'] = $this->news_model->get_news($slug);

        if (empty($data['news_item']))
        {
                show_404();
        }

        $data['title'] = $data['news_item']['title'];

        $this->load->view('templates/header', $data);
        $this->load->view('news/view', $data);
        $this->load->view('templates/footer');
}

Instead of calling the get_news() method without a parameter, the $slug variable is passed, so it will return the specific news item. As únicas coisas que resta a fazer é criar a visão correspondente em application/views/news/view.php. Put the following code in this file.

<?php
echo '<h2>'.$news_item['title'].'</h2>';
echo $news_item['text'];

Roteamento¶

Because of the wildcard routing rule created earlier, you need an extra route to view the controller that you just made. Modify your routing file (application/config/routes.php) so it looks as follows. This makes sure the requests reaches the News controller instead of going directly to the Pages controller. The first line routes URI’s with a slug to the view() method in the News controller.

$route['news/(:any)'] = 'news/view/$1';
$route['news'] = 'news';
$route['(:any)'] = 'pages/view/$1';
$route['default_controller'] = 'pages/view';

Point your browser to your document root, followed by index.php/news and watch your news page.

Configurando seu Model¶

Em vez de escrever as operações do Database no Controller, as Querys devem ser colocados em um Model, para que possam ser facilmente reutilizados posteriormente. Models é o lugar onde você recupera, insere e atualiza informações em seu Database ou outros armazenamentos de dados. Eles representam seus dados.

Open up the application/models/ directory and create a new file called News_model.php e adicione o seguinte código. Certifique-se de que você configurou seu Database corretamente conforme descrito here.

<?php
class News_model extends CI_Model {

        {
                $this->load->database();
        }
}

Esse código é semelhante ao código do Controller usado anteriormente. isto creates a new model by extending CI_Model and loads the database Library. Isso tornará a classe do Database disponível através do $this->db object.

Antes de Queryr o Database, um esquema de Database deve ser criado. Conecte-se ao seu Database e execute o comando SQL abaixo (MySQL). Adicione também alguns registros de sementes.

CREATE TABLE news (
        id int(11) NOT NULL AUTO_INCREMENT,
        title varchar(128) NOT NULL,
        slug varchar(128) NOT NULL,
        text text NOT NULL,
        PRIMARY KEY (id),
        KEY slug (slug)
);

Agora que o Database e um Model foram configurados, você precisará de um método para obter todos os nossos posts do nosso Database. Para fazer isso, o Database camada de abstração incluída no CodeIgniter - Construtor de Querys - é usado. Isso faz com que possível escrever suas 'Querys' uma vez e fazê-las funcionar all sistemas de Database suportados. Adicione o seguindo o código para o seu Model.

public function get_news($slug = FALSE)
{

        {
                $query = $this->db->get('news');
                return $query->result_array();
        }

        $query = $this->db->get_where('news', array('slug' => $slug));
        return $query->row_array();

Com este código, você pode realizar duas Querys diferentes. Você pode obter tudo registros de notícias, ou obter uma notícia por sua slug. Você talvez tenha noticed that the $slug variable wasn’t sanitized before running the query; Construtor de Querys faz isso por você.

Mostrar as notícias¶

Agora que as Querys estão escritas, o Model deve estar vinculado às visualizações que vão exibir os itens de notícias para o usuário. Isso poderia ser feito in our Pages controller created earlier, but for the sake of clarity, a new News controller is defined. Create the new controller at application/controllers/News.php.

<?php
class News extends CI_Controller {

        public function __construct()
        {
                parent::__construct();
                $this->load->model('news_model');
                $this->load->helper('url_helper');
        }

        public function index()
        {
                $data['news'] = $this->news_model->get_news();
        }

        public function view($slug = NULL)
        {
                $data['news_item'] = $this->news_model->get_news($slug);
        }
}

Olhando para o código, você pode ver alguma semelhança com os arquivos que created earlier. Também carrega uma coleção de Helper de URL funções, porque vamos usar um deles em uma View posterior.

Em seguida, existem dois métodos para visualizar todos os itens de notícias e um para um item específico. news item. You can see that the $slug variable is passed to the model’s método no segundo método. O Model está usando essa lesma para identificar item de notícias a ser devolvido.

Agora os dados são recuperados pelo Controller através do nosso Model, mas nada é exibido ainda. A próxima coisa a fazer é passar esses dados para as visualizações.

public function index()
{

Seção de notícias¶

Na última seção, examinamos alguns conceitos básicos do framework escrevendo uma classe que inclui páginas estáticas. Nós limpamos o URI adicionando regras de roteamento personalizadas. Agora é hora de introduzir conteúdo dinâmico e comece a usar um Database.

Configurando seu Model¶

Em vez de escrever as operações do Database no Controller, as Querys devem ser colocados em um Model, para que possam ser facilmente reutilizados posteriormente. Models é o lugar onde você recupera, insere e atualiza informações em seu Database ou outros armazenamentos de dados. Eles representam seus dados.

Open up the application/models/ directory and create a new file called News_model.php e adicione o seguinte código. Certifique-se de que você configurou seu Database corretamente conforme descrito here.

<?php
class News_model extends CI_Model {

        public function __construct()
        {
                $this->load->database();
        }
}

Esse código é semelhante ao código do Controller usado anteriormente. isto creates a new model by extending CI_Model and loads the database Library. Isso tornará a classe do Database disponível através do $this->db object.

Antes de Queryr o Database, um esquema de Database deve ser criado. Conecte-se ao seu Database e execute o comando SQL abaixo (MySQL). Adicione também alguns registros de sementes.

CREATE TABLE news (
        id int(11) NOT NULL AUTO_INCREMENT,
        title varchar(128) NOT NULL,
        slug varchar(128) NOT NULL,
        text text NOT NULL,
        PRIMARY KEY (id),
        KEY slug (slug)
);

Agora que o Database e um Model foram configurados, você precisará de um método para obter todos os nossos posts do nosso Database. Para fazer isso, o Database camada de abstração incluída no CodeIgniter - Construtor de Querys - é usado. Isso faz com que possível escrever suas 'Querys' uma vez e fazê-las funcionar all sistemas de Database suportados. Add the seguindo o código para o seu Model.

public function get_news($slug = FALSE)
{
        if ($slug === FALSE)
        {
                $query = $this->db->get('news');
                return $query->result_array();
        }

        $query = $this->db->get_where('news', array('slug' => $slug));
        return $query->row_array();
}

Com este código, você pode realizar duas Querys diferentes. Você pode obter tudo registros de notícias, ou obter uma notícia por sua slug. Você talvez tenha noticed that the $slug variable wasn’t sanitized before running the query; Construtor de Querys faz isso por você.

Mostrar as notícias¶

Agora que as Querys estão escritas, o Model deve estar vinculado às visualizações que vão exibir os itens de notícias para o usuário. Isso poderia ser feito in our Pages controller created earlier, but for the sake of clarity, a new News controller is defined. Create the new controller at application/controllers/News.php.

<?php
class News extends CI_Controller {

        public function __construct()
        {
                parent::__construct();
                $this->load->model('news_model');
                $this->load->helper('url_helper');
        }

        public function index()
        {
                $data['news'] = $this->news_model->get_news();
        }

        public function view($slug = NULL)
        {
                $data['news_item'] = $this->news_model->get_news($slug);
        }
}

Olhando para o código, você pode ver alguma semelhança com os arquivos que created earlier. First, the __construct() method: it calls the constructor of its parent class (CI_Controller) and loads the model, para que possa ser usado em todos os outros métodos neste Controller. Também carrega uma coleção de Helper de URL funções, porque vamos usar um deles em uma View posterior.

Em seguida, existem dois métodos para visualizar todos os itens de notícias e um para um item específico. news item. You can see that the $slug variable is passed to the model’s método no segundo método. O Model está usando essa lesma para identificar item de notícias a ser devolvido.

Agora os dados são recuperados pelo Controller através do nosso Model, mas nada é exibido ainda. A próxima coisa a fazer é passar esses dados para the views.

public function index()
{
        $data['news'] = $this->news_model->get_news();
        $data['title'] = 'News archive';

        $this->load->view('templates/header', $data);
        $this->load->view('news/index', $data);
        $this->load->view('templates/footer');
}
