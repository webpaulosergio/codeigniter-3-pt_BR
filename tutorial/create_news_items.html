Criar itens de notícias¶

Agora você sabe como pode ler dados de um database usando o CodeIgniter, mas you haven’t written any information to the database yet. In this section you’ll expand your news controller and model created earlier to include this functionality.

Create a form¶

Para inserir dados no database, você precisa criar um formulário onde possa input the information to be stored. This means you’ll be needing a form with two fields, one for the title and one for the text. You’ll derive the slug from our title in the model. Create the new view at application/views/news/create.php.

<h2><?php echo $title; ?></h2>

<?php echo validation_errors(); ?>

<?php echo form_open('news/create'); ?>

    <label for="title">Title</label>
    <input type="input" name="title" /><br />

    <label for="text">Text</label>
    <textarea name="text"></textarea><br />

    <input type="submit" name="submit" value="Create news item" />

</form>

Há apenas duas coisas aqui que provavelmente parecem estranhas para você: o form_open() function and the validation_errors() function.

A primeira função é fornecida pelo form helper e renderiza o elemento form e adds extra functionality, like adding a hidden Prevenção do CSRF field. The latter is used to report erros relacionados à validação de formulários.

Go back to your news controller. You’re going to do two things here, verificar se o formulário foi submetido e se os dados submetidos passou as regras de validação. Você vai usar o form validation library to do this.

public function create()
{
    $this->load->helper('form');
    $this->load->library('form_validation');

    $data['title'] = 'Create a news item';

    $this->form_validation->set_rules('title', 'Title', 'required');
    $this->form_validation->set_rules('text', 'Text', 'required');

    if ($this->form_validation->run() === FALSE)
    {
        $this->load->view('templates/header', $data);
        $this->load->view('news/create');
        $this->load->view('templates/footer');

    }
    else
    {
        $this->news_model->set_news();
        $this->load->view('news/success');
    }
}

The code above adds a lot of functionality. The first few lines load the form helper and the form validation library. After that, rules for the form validation are set. The set_rules() method takes three arguments; o nome do campo de entrada, o nome a ser usado em mensagens de erro e a regra. Nesse caso, os campos de título e texto são obrigatórios.

O CodeIgniter possui uma poderosa library de validação de formulários, conforme demonstrado acima. Você pode ler more about this library here.

Continuing down, you can see a condition that checks whether the form validation ran successfully. If it did not, the form is displayed, if it foi submetido and passed all the rules, the model is called. After this, a view is loaded to display a success message. Create a view at application/views/news/success.php e escreva uma mensagem de sucesso.

Model¶

The only thing that remains is writing a method that writes the data to the database. You’ll use the Query Builder class to insert the information and use the input library to get the posted data. Open up the model created earlier and add the following:

public function set_news()
{
    $this->load->helper('url');

    $slug = url_title($this->input->post('title'), 'dash', TRUE);

    $data = array(
        'title' => $this->input->post('title'),
        'slug' => $slug,
        'text' => $this->input->post('text')
    );

    return $this->db->insert('news', $data);
}

Esse novo método cuida de inserir o item de notícias no database. The third line contains a new function, url_title(). This function - provided by the Helper de URL - desmonta the string you pass it, replacing all spaces by dashes (-) and makes sure everything is in lowercase characters. This leaves you with a nice slug, perfect for creating URIs.

Let’s continue with preparing the record that is going to be inserted later, inside the $data array. Each element corresponds with a column in the database table created earlier. You might notice a new method here, namely the post() method from the input library. Esse método garante que os dados sejam sanitized, protecting you from nasty attacks from others. The input library is loaded by default. At last, you insert our $data array into nosso database.

Roteamento¶

Antes de começar a adicionar itens de notícias ao seu aplicativo CodeIgniter you have to add an extra rule to config/routes.php file. Make sure your arquivo contém o seguinte. Isso garante que o CodeIgniter veja "criar" as a method instead of a news item’s slug.

$route['news/create'] = 'news/create';
$route['news/(:any)'] = 'news/view/$1';
$route['news'] = 'news';
$route['(:any)'] = 'pages/view/$1';
$route['default_controller'] = 'pages/view';

Now point your browser to your local development environment where you Instale o CodeIgniter e adicione index.php/news/create ao URL. Congratulations, you just created your first CodeIgniter application! Add some news and check out the different pages you made.

Crie um formulário¶

Para inserir dados no database, você precisa criar um formulário onde possa introduza a informação a ser armazenada. Isso significa que você vai precisar de um formulário com dois campos, um para o título e outro para o texto. Você vai derivar a lesma do nosso título no Model. Crie a nova View em application/views/news/create.php.

<h2><?php echo $title; ?></h2>

<?php echo validation_errors(); ?>

<?php echo form_open('news/create'); ?>

    <label for="title">Title</label>

public function create()
{

O código acima adiciona muita funcionalidade. As primeiras linhas carregam o formulário Helper e a Library de validação de formulários. Depois disso, regras para o form validation are set. The set_rules() method takes three arguments; o nome do campo de entrada, o nome a ser usado em mensagens de erro e a regra. Nesse caso, os campos de título e texto são obrigatórios.

O CodeIgniter possui uma poderosa library de validação de formulários, conforme demonstrado acima. Você pode ler mais sobre esta Library here.

Continuando para baixo, você pode ver uma condição que verifica se o formulário validação executada com sucesso. Se não, o formulário é exibido, se foi submetido and passou todas as regras, o Model é chamado. Depois de isso, uma View é carregada para exibir uma mensagem de sucesso. Crie uma View em application/views/news/success.php e escreva uma mensagem de sucesso.

Model¶

A única coisa que resta é escrever um método que grava os dados para o Database. Você usará a classe Query Builder para inserir o informações e usar a Library de entrada para obter os dados publicados. Abrir o Model criado anteriormente e adicione o seguinte:

public function set_news()
{
    $this->load->helper('url');

    $slug = url_title($this->input->post('title'), 'dash', TRUE);

    $data = array(
        'title' => $this->input->post('title'),
        'slug' => $slug,
        'text' => $this->input->post('text')
    );

    return $this->db->insert('news', $data);
}

Esse novo método cuida de inserir o item de notícias no database. A terceira linha contém uma nova função, url_title(). Esta função - fornecida pelo Helper de URL - desmonta a string que você passa, substituindo todos os espaços por traços (-) e faz Certifique-se tudo está em caracteres minúsculos. Isso deixa você com um bom Slug, perfeito para criar URIs.

Vamos continuar com a preparação do registro que será inserido later, inside the $data array. Each element corresponds with a column in a tabela do Database criada anteriormente. Você pode notar um novo método aqui, namely the post() method from the input library. Esse método garante que os dados sejam Sanitized, protegendo-o de ataques desagradáveis de outros. A entrada library is loaded by default. At last, you insert our $data array into nosso database.

Roteamento¶

Antes de começar a adicionar itens de notícias ao seu aplicativo CodeIgniter you have to add an extra rule to config/routes.php file. Make sure your arquivo contém o seguinte. Isso garante que o CodeIgniter veja "criar" como um método em vez da lesma de um item de notícias.

$route['news/create'] = 'news/create';

$route['news'] = 'news';
$route['(:any)'] = 'pages/view/$1';

Agora aponte seu navegador para o ambiente de desenvolvimento local onde você Instale o CodeIgniter e adicione index.php/news/create ao URL. Parabéns, você acabou de criar sua primeira aplicação CodeIgniter! Adicione algumas novidades e confira as diferentes páginas que você fez.

Criar itens de notícias¶

Agora você sabe como pode ler dados de um database usando o CodeIgniter, mas você ainda não escreveu nenhuma informação no Database. Nesta secção você expandirá seu Controller de notícias e Model criado anteriormente para incluir essa funcionalidade.

Crie um formulário¶

Para inserir dados no database, você precisa criar um formulário onde possa introduza a informação a ser armazenada. Isso significa que você vai precisar de um formulário com dois campos, um para o título e outro para o texto. Você vai derivar a lesma do nosso título no Model. Crie a nova View em application/views/news/create.php.

<h2><?php echo $title; ?></h2>

<?php echo validation_errors(); ?>

<?php echo form_open('news/create'); ?>

    <label for="title">Title</label>
    <input type="input" name="title" /><br />

    <label for="text">Text</label>
    <textarea name="text"></textarea><br />

    <input type="submit" name="submit" value="Create news item" />

</form>

Há apenas duas coisas aqui que provavelmente parecem estranhas para você: o form_open() function and the validation_errors() function.

A primeira função é fornecida pelo form helper e renderiza o elemento form e adiciona funcionalidade extra, como adicionar um oculto Prevenção do CSRF field. Este último é usado para relatar erros relacionados à validação de formulários.

Volte para o seu Controller de notícias. Você vai fazer duas coisas aqui verificar se o formulário foi submetido e se os dados submetidos passou as regras de validação. Você vai usar o form validation Library para fazer isso.

public function create()
{
    $this->load->helper('form');
    $this->load->library('form_validation');

    $data['title'] = 'Create a news item';

    $this->form_validation->set_rules('title', 'Title', 'required');
    $this->form_validation->set_rules('text', 'Text', 'required');

    if ($this->form_validation->run() === FALSE)
    {
        $this->load->view('templates/header', $data);
        $this->load->view('news/create');
        $this->load->view('templates/footer');

    }
    else
    {
        $this->news_model->set_news();
        $this->load->view('news/success');
    }
}

O código acima adiciona muita funcionalidade. As primeiras linhas carregam o formulário Helper e a Library de validação de formulários. Depois disso, regras para o form validation are set. The set_rules() method takes three arguments; o nome do campo de entrada, o nome a ser usado em mensagens de erro e a regra. Nesse caso, os campos de título e texto são obrigatórios.

O CodeIgniter possui uma poderosa library de validação de formulários, conforme demonstrado acima. Você pode ler mais sobre esta Library here.

Continuando para baixo, você pode ver uma condição que verifica se o formulário validação executada com sucesso. Se não, o formulário é exibido, se foi submetido and passou todas as regras, o Model é chamado. Depois de isso, uma View é carregada para exibir uma mensagem de sucesso. Crie uma View em application/views/news/success.php e escreva uma mensagem de sucesso.

Model¶

A única coisa que resta é escrever um método que grava os dados para o Database. Você usará a classe Query Builder para inserir o informações e usar a Library de entrada para obter os dados publicados. Abrir o Model criado anteriormente e adicione o seguinte:

public function set_news()
{
    $this->load->helper('url');

    $slug = url_title($this->input->post('title'), 'dash', TRUE);

    $data = array(
        'title' => $this->input->post('title'),
        'slug' => $slug,
        'text' => $this->input->post('text')
    );

    return $this->db->insert('news', $data);
}

Esse novo método cuida de inserir o item de notícias no database. A terceira linha contém uma nova função, url_title(). Esta função - fornecida pelo Helper de URL - desmonta a string que você passa, substituindo todos os espaços por traços (-) e faz Certifique-se tudo está em caracteres minúsculos. Isso deixa você com um bom Slug, perfeito para criar URIs.

Vamos continuar com a preparação do registro que será inserido later, inside the $data array. Each element corresponds with a column in a tabela do Database criada anteriormente. Você pode notar um novo método aqui, namely the post() method from the input library. Esse método garante que os dados sejam Sanitized, protegendo-o de ataques desagradáveis de outros. A entrada library is loaded by default. At last, you insert our $data array into nosso database.

Roteamento¶

Antes de começar a adicionar itens de notícias ao seu aplicativo CodeIgniter you have to add an extra rule to config/routes.php file. Make sure your arquivo contém o seguinte. Isso garante que o CodeIgniter veja "criar" como um método em vez da lesma de um item de notícias.

$route['news/create'] = 'news/create';
$route['news/(:any)'] = 'news/view/$1';
$route['news'] = 'news';
$route['(:any)'] = 'pages/view/$1';
$route['default_controller'] = 'pages/view';

Agora aponte seu navegador para o ambiente de desenvolvimento local onde você Instale o CodeIgniter e adicione index.php/news/create ao URL. Parabéns, você acabou de criar sua primeira aplicação CodeIgniter! Adicione algumas novidades e confira as diferentes páginas que você fez.
