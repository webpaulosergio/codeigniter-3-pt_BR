Classe do Construtor de Consultas¶

CodeIgniter gives you access to a Query Builder class. This pattern allows information to be retrieved, inserted, and updated in your database com scripts mínimos. Em alguns casos, apenas uma ou duas linhas of code are necessary to perform a database action. O CodeIgniter não requer que cada tabela de database seja sua própria classe file. It instead provides a more simplified interface.

Além da simplicidade, um grande benefício para usar os recursos do Query Builder is that it allows you to create database independent applications, since the query syntax is generated by each database adapter. It also allows para consultas mais seguras, já que os valores são escapados automaticamente pelo system.

Nota

If you intend to write your own queries you can disable this classe no seu arquivo de configuração do database, permitindo que a library do database núcleo e adaptador para utilizar menos recursos.

Selecting Data¶

As seguintes funções permitem que você construa SQL SELECIONAR statements.

$this->db->get()

Runs the selection query and returns the result. Can be used by itself to retrieve all records from a table:

$query = $this->db->get('mytable');  // Produces: SELECIONAR * FROM mytable

O segundo e terceiro parâmetros permitem que você defina um limite e um deslocamento clause:

$query = $this->db->get('mytable', 10, 20);

// Executes: SELECIONAR * FROM mytable LIMIT 20, 10
// (in MySQL. Other databases have slightly different syntax)

You’ll notice that the above function is assigned to a variable named $query, que pode ser usado para mostrar os resultados:

$query = $this->db->get('mytable');

foreach ($query->result() as $row)
{
        echo $row->title;
}

Por favor visite o funções resultantes página para um completo discussão sobre geração de resultados.

$this->db->get_compiled_select()

Compila a consulta de seleção como $this->db->get() but does not run the query. This method simply returns the SQL query as a string.

Example:

$sql = $this->db->get_compiled_select('mytable');
echo $sql;

// Prints string: SELECIONAR * FROM mytable

The second parameter enables you to set whether or not the query builder query will be reset (by default it will be reset, just like when using $this->db->get()):

echo $this->db->limit(10,20)->get_compiled_select('mytable', FALSE);

// Prints string: SELECIONAR * FROM mytable LIMIT 20, 10
// (in MySQL. Other databases have slightly different syntax)

echo $this->db->select('title, content, date')->get_compiled_select();

// Prints string: SELECIONAR title, content, date FROM mytable LIMIT 20, 10

The key thing to notice in the above example is that the second query did not utilize $this->db->from() and did not pass a table name into the first parâmetro. A razão para este resultado é porque a consulta não foi executed using $this->db->get() que redefine valores ou redefine diretamente using $this->db->reset_query().

$this->db->get_where()

Identical to the above function except that it permits you to add a “where” clause in the second parameter, instead of using the db->where() function:

$query = $this->db->get_where('mytable', array('id' => $id), $limit, $offset);

Por favor, leia sobre a função where abaixo para mais informações.

Nota

get_where() era conhecido anteriormente como getwhere(), que foi removido

$this->db->select()

Permits you to write the SELECIONAR portion of your query:

$this->db->select('title, content, date');
$query = $this->db->get('mytable');

// Executes: SELECIONAR title, content, date FROM mytable

Nota

If you are selecting all (*) from a table you do not need to use esta função. Quando omitido, o CodeIgniter assume que você deseja to select all fields and automatically adds ‘SELECIONAR *’.

$this->db->select() accepts an optional second parameter. If you set it to FALSE, CodeIgniter will not try to protect your field or table names. This is useful if you need a compound select statement where automatic escaping of fields may break them.

$this->db->select('(SELECIONAR SUM(payments.amount) FROM payments WHERE payments.invoice_id=4) AS amount_paid', FALSE);
$query = $this->db->get('mytable');

$this->db->select_max()

Writes a SELECIONAR MAX(field) portion for your query. You can optionally include a second parameter to rename the resulting field.

$this->db->select_max('age');
$query = $this->db->get('members');  // Produces: SELECIONAR MAX(age) as age FROM members

$this->db->select_max('age', 'member_age');
$query = $this->db->get('members'); // Produces: SELECIONAR MAX(age) as member_age FROM members

$this->db->select_min()

Writes a “SELECIONAR MIN(field)” portion for your query. As with select_max(), You can optionally include a second parameter to rename the resulting field.

$this->db->select_min('age');
$query = $this->db->get('members'); // Produces: SELECIONAR MIN(age) as age FROM members

$this->db->select_avg()

Writes a “SELECIONAR AVG(field)” portion for your query. As with select_max(), You can optionally include a second parameter to rename the resulting field.

$this->db->select_avg('age');
$query = $this->db->get('members'); // Produces: SELECIONAR AVG(age) as age FROM members

$this->db->select_sum()

Writes a “SELECIONAR SUM(field)” portion for your query. As with select_max(), You can optionally include a second parameter to rename the resulting field.

$this->db->select_sum('age');
$query = $this->db->get('members'); // Produces: SELECIONAR SUM(age) as age FROM members

$this->db->from()

Permits you to write the FROM portion of your query:

$this->db->select('title, content, date');
$this->db->from('mytable');
$query = $this->db->get();  // Produces: SELECIONAR title, content, date FROM mytable

Nota

Como mostrado anteriormente, a parte FROM da sua consulta pode ser especificada in the $this->db->get() function, so use whichever method you prefer.

$this->db->join()

Permits you to write the JOIN portion of your query:

$this->db->select('*');
$this->db->from('blogs');
$this->db->join('comments', 'comments.id = blogs.id');
$query = $this->db->get();

// Produces:
// SELECIONAR * FROM blogs JOIN comments ON comments.id = blogs.id

Multiple function calls can be made if you need several joins in one query.

If you need a specific type of JOIN you can specify it via the third parâmetro da função. As opções são: esquerda, direita, externa, interna, esquerda exterior e direito exterior.

$this->db->join('comments', 'comments.id = blogs.id', 'left');
// Produces: LEFT JOIN comments ON comments.id = blogs.id

Procurando por dados específicos¶

$this->db->where()

This function enables you to set WHERE cláusulas usando um dos quatro methods:

Nota

All values passed to this function are escaped automatically, producing safer queries.

  1. Simple key/value method:

    $this->db->where('name', $name); // Produces: WHERE name = 'Joe'

    Notice that the equal sign is added for you.

    If you use multiple function calls they will be chained together with E between them:

    $this->db->where('name', $name);
$this->db->where('title', $title);
$this->db->where('status', $status);
// WHERE name = 'Joe' E title = 'boss' E status = 'active'

  2. Custom key/value method:

    You can include an operator in the first parameter in order to controlar a comparação:

    $this->db->where('name !=', $name);
$this->db->where('id <', $id); // Produces: WHERE name != 'Joe' E id < 45

  3. Método de matriz associativa:

    $array = array('name' => $name, 'title' => $title, 'status' => $status);
$this->db->where($array);
// Produces: WHERE name = 'Joe' E title = 'boss' E status = 'active'

    You can include your own operators using this method as well:

    $array = array('name !=' => $name, 'id <' => $id, 'date >' => $date);
$this->db->where($array);
  4. Seqüência personalizada:

    Você pode escrever suas próprias cláusulas manualmente:

    $where = "name='Joe' E status='boss' OR status='active'";
$this->db->where($where);

$this->db->where() aceita um terceiro parâmetro opcional. Se você configurá-lo para FALSE, CodeIgniter will not try to protect your field or table names.

$this->db->where('MATCH (field) AGAINST ("value")', NULL, FALSE);

$this->db->or_where()

This function is identical to the one above, except that multiple instances are joined by OR:

$this->db->where('name !=', $name);
$this->db->or_where('id >', $id);  // Produces: WHERE name != 'Joe' OR id > 50

Nota

ou_where() era conhecido anteriormente como orwhere(), que foi removed.

$this->db->where_in()

Gera um WHERE field IN ('item', 'item') consulta SQL associada a E se appropriate

$names = array('Frank', 'Todd', 'James');
$this->db->where_in('username', $names);
// Produces: WHERE username IN ('Frank', 'Todd', 'James')

$this->db->or_where_in()

Gera um WHERE field IN ('item', 'item') consulta SQL associada a OR se appropriate

$names = array('Frank', 'Todd', 'James');
$this->db->or_where_in('username', $names);
// Produces: OR username IN ('Frank', 'Todd', 'James')

$this->db->where_not_in()

Generates a WHERE field NOT IN (‘item’, ‘item’) SQL query joined with E if appropriate

$names = array('Frank', 'Todd', 'James');
$this->db->where_not_in('username', $names);
// Produces: WHERE username NOT IN ('Frank', 'Todd', 'James')

$this->db->or_where_not_in()

Gera um WHERE field NOT IN ('item', 'item') consulta SQL associada a OR if appropriate

$names = array('Frank', 'Todd', 'James');
$this->db->or_where_not_in('username', $names);
// Produces: OR username NOT IN ('Frank', 'Todd', 'James')

Procurando por dados semelhantes¶

$this->db->like()

Esse método permite gerar GOSTAR clauses, useful for doing searches.

Nota

All values passed to this method are escaped automatically.

  1. Simple key/value method:

    $this->db->like('title', 'match');
// Produces: WHERE `title` GOSTAR '%match%' ESCAPE '!'

    If you use multiple method calls they will be chained together with E between them:

    $this->db->like('title', 'match');
$this->db->like('body', 'match');
// WHERE `title` GOSTAR '%match%' ESCAPE '!' E  `body` GOSTAR '%match% ESCAPE '!'

    If you want to control where the wildcard (%) is placed, you can use an optional third argument. Your options are ‘before’, ‘after’ and ‘both’ (which is the default).

    $this->db->like('title', 'match', 'before');    // Produces: WHERE `title` GOSTAR '%match' ESCAPE '!'
$this->db->like('title', 'match', 'after');     // Produces: WHERE `title` GOSTAR 'match%' ESCAPE '!'
$this->db->like('title', 'match', 'both');      // Produces: WHERE `title` GOSTAR '%match%' ESCAPE '!'

  2. Método de matriz associativa:

    $array = array('title' => $match, 'page1' => $match, 'page2' => $match);
$this->db->like($array);
// WHERE `title` GOSTAR '%match%' ESCAPE '!' E  `page1` GOSTAR '%match%' ESCAPE '!' E  `page2` GOSTAR '%match%' ESCAPE '!'

$this->db->or_like()

Este método é idêntico ao acima, exceto que múltiplos instances are joined by OR:

$this->db->like('title', 'match'); $this->db->or_like('body', $match);
// WHERE `title` GOSTAR '%match%' ESCAPE '!' OR  `body` GOSTAR '%match%' ESCAPE '!'

Nota

or_like() was formerly known as orlike(), which has been removed.

$this->db->not_like()

This method is identical to like(), except that it generates NOT GOSTAR statements:

$this->db->not_like('title', 'match');  // WHERE `title` NOT GOSTAR '%match% ESCAPE '!'

$this->db->or_not_like()

This method is identical to not_like(), except that multiple instances are joined by OR:

$this->db->like('title', 'match');
$this->db->or_not_like('body', 'match');
// WHERE `title` GOSTAR '%match% OR  `body` NOT GOSTAR '%match%' ESCAPE '!'

$this->db->group_by()

Permits you to write the GROUP BY portion of your query:

$this->db->group_by("title"); // Produces: GROUP BY title

Você também pode passar uma matriz de vários valores:

$this->db->group_by(array("title", "date"));  // Produces: GROUP BY title, date

Nota

group_by() was formerly known as groupby(), which has been removed.

$this->db->distinct()

Adds the “DISTINCT” keyword to a query

$this->db->distinct();
$this->db->get('table'); // Produces: SELECIONAR DISTINCT * FROM table

$this->db->having()

Permite que você escreva a parte HAVING da sua consulta. Há 2 possible syntaxes, 1 argument or 2:

$this->db->having('user_id = 45');  // Produces: HAVING user_id = 45
$this->db->having('user_id',  45);  // Produces: HAVING user_id = 45

Você também pode passar uma matriz de vários valores:

$this->db->having(array('title =' => 'My Title', 'id <' => $id));
// Produces: HAVING title = 'My Title', id < 45

If you are using a database that CodeIgniter escapes queries for, you pode impedir o escape de conteúdo passando um terceiro argumento opcional e setting it to FALSE.

$this->db->having('user_id',  45);  // Produces: HAVING `user_id` = 45 in some databases such as MySQL
$this->db->having('user_id',  45, FALSE);  // Produces: HAVING user_id = 45

$this->db->or_having()

Identical to having(), only separates multiple clauses with “OR”.

Ordenação de resultados¶

$this->db->order_by()

Lets you set an ORDER BY clause.

The first parameter contains the name of the column you would like to order by.

The second parameter lets you set the direction of the result. Opções são ASC, DESC E REOM.

$this->db->order_by('title', 'DESC');
// Produces: ORDER BY `title` DESC

You can also pass your own string in the first parameter:

$this->db->order_by('title DESC, name ASC');
// Produces: ORDER BY `title` DESC, `name` ASC

Or multiple function calls can be made if you need multiple fields.

$this->db->order_by('title', 'DESC');
$this->db->order_by('name', 'ASC');
// Produces: ORDER BY `title` DESC, `name` ASC

Se você escolher o REOM direction option, then the first parameters will be ignored, unless you specify a numeric seed value.

$this->db->order_by('title', 'REOM');
// Produces: ORDER BY RE()

$this->db->order_by(42, 'REOM');
// Produces: ORDER BY RE(42)

Nota

order_by() was formerly known as orderby(), which has been removed.

Nota

A ordenação aleatória não é atualmente suportada no Oracle e O padrão será ASC.

Limiting or Counting Results¶

$this->db->limit()

Lets you limit the number of rows you would like returned by the query:

$this->db->limit(10);  // Produces: LIMIT 10

The second parameter lets you set a result offset.

$this->db->limit(10, 20);  // Produces: LIMIT 20, 10 (in MySQL.  Other databases have slightly different syntax)

$this->db->count_all_results()

Permite que você determine o número de linhas em um determinado Active Consulta de registro. As consultas aceitarão os restritores do Query Builder, como where(), or_where(), like(), or_like(), etc. Example:

echo $this->db->count_all_results('my_table');  // Produces an integer, like 25
$this->db->like('title', 'match');
$this->db->from('my_table');
echo $this->db->count_all_results(); // Produces an integer, like 17

However, this method also resets any field values that you may have passed to select(). If you need to keep them, you can pass FALSE as the segundo parâmetro:

echo $this->db->count_all_results('my_table', FALSE);

$this->db->count_all()

Permits you to determine the number of rows in a particular table. Envie o nome da tabela no primeiro parâmetro. Exemplo:

echo $this->db->count_all('my_table');  // Produces an integer, like 25

Query grouping¶

O agrupamento de consultas permite criar grupos de cláusulas WHERE, colocando-os entre parênteses. Isso permitirá you to create queries with complex WHERE clauses. Nested groups are supported. Example:

$this->db->select('*')->from('my_table')
        ->group_start()
                ->where('a', 'a')
                ->or_group_start()
                        ->where('b', 'b')
                        ->where('c', 'c')
                ->group_end()
        ->group_end()
        ->where('d', 'd')
->get();

// Generates:
// SELECIONAR * FROM (`my_table`) WHERE ( `a` = 'a' OR ( `b` = 'b' E `c` = 'c' ) ) E `d` = 'd'

Nota

grupos precisam ser equilibrados, certifique-se de que cada group_start() seja correspondido por um group_end().

$this->db->group_start()

Starts a new group by adding an opening parenthesis to the WHERE clause of the query.

$this->db->or_group_start()

Starts a new group by adding an opening parenthesis to the WHERE clause of the query, prefixing it with ‘OR’.

$this->db->not_group_start()

Starts a new group by adding an opening parenthesis to the WHERE clause of the query, prefixing it with ‘NOT’.

$this->db->or_not_group_start()

Starts a new group by adding an opening parenthesis to the WHERE clause of the query, prefixing it with ‘OR NOT’.

$this->db->group_end()

Termina o grupo atual adicionando um parêntese de fechamento à cláusula WHERE da consulta.

Inserting Data¶

$this->db->insert()

Gera uma sequência de inserção com base nos dados que você fornece e executa o inquerir. Você pode passar um array or an object to the função. Aqui está um exemplo usando uma matriz:

$data = array(
        'title' => 'My title',
        'name' => 'My Name',
        'date' => 'My date'
);

$this->db->insert('mytable', $data);
// Produces: INSERT INTO mytable (title, name, date) VALUES ('My title', 'My name', 'My date')

O primeiro parâmetro conterá o nome da tabela, o segundo é um associative array of values.

Aqui está um exemplo usando um objeto:

/*
class Myclass {
        public $title = 'My Title';
        public $content = 'My Content';
        public $date = 'My Date';
}
*/

$object = new Myclass;
$this->db->insert('mytable', $object);
// Produces: INSERT INTO mytable (title, content, date) VALUES ('My Title', 'My Content', 'My Date')

O primeiro parâmetro conterá o nome da tabela, o segundo é um object.

Nota

All values are escaped automatically producing safer queries.

$this->db->get_compiled_insert()

Compiles the insertion query just like $this->db->insert() but does not run the query. This method simply returns the SQL query as a string.

Example:

$data = array(
        'title' => 'My title',
        'name'  => 'My Name',
        'date'  => 'My date'
);

$sql = $this->db->set($data)->get_compiled_insert('mytable');
echo $sql;

// Produces string: INSERT INTO mytable (`title`, `name`, `date`) VALUES ('My title', 'My name', 'My date')

The second parameter enables you to set whether or not the query builder query will be reset (by default it will be–just like $this->db->insert()):

echo $this->db->set('title', 'My Title')->get_compiled_insert('mytable', FALSE);

// Produces string: INSERT INTO mytable (`title`) VALUES ('My Title')

echo $this->db->set('content', 'My Content')->get_compiled_insert();

// Produces string: INSERT INTO mytable (`title`, `content`) VALUES ('My Title', 'My Content')

The key thing to notice in the above example is that the second query did not utilize $this->db->from() nem passou um nome de tabela para o primeiro parâmetro. A razão pela qual isso funcionou é porque a consulta não foi executada using $this->db->insert() que redefine valores ou redefine diretamente using $this->db->reset_query().

Nota

This method doesn’t work for batched inserts.

$this->db->insert_batch()

Gera uma sequência de inserção com base nos dados que você fornece e executa o inquerir. Você pode passar um array or an object to the função. Aqui está um exemplo usando uma matriz:

$data = array(
        array(
                'title' => 'My title',
                'name' => 'My Name',
                'date' => 'My date'
        ),
        array(
                'title' => 'Another title',
                'name' => 'Another Name',
                'date' => 'Another date'
        )
);

$this->db->insert_batch('mytable', $data);
// Produces: INSERT INTO mytable (title, name, date) VALUES ('My title', 'My name', 'My date'),  ('Another title', 'Another name', 'Another date')

O primeiro parâmetro conterá o nome da tabela, o segundo é um associative array of values.

Nota

All values are escaped automatically producing safer queries.

Atualizando Dados¶

$this->db->replace()

This method executes a REPLACE statement, which is basically the SQL standard for (optional) DELETE + INSERT, using PRIMARY and UNIQUE chaves como o fator determinante. In our case, it will save you from the need to implement complex logics with different combinations of select(), update(), delete() and insert() calls.

Example:

$data = array(
        'title' => 'My title',
        'name'  => 'My Name',
        'date'  => 'My date'
);

$this->db->replace('table', $data);

// Executes: REPLACE INTO mytable (title, name, date) VALUES ('My title', 'My name', 'My date')

In the above example, if we assume that the title field is our primary key, then if a row containing ‘My title’ as the title value, that row will be deleted with our new row data replacing it.

Usage of the set() method is also allowed and all fields are automatically escaped, just like with insert().

$this->db->set()

Esta função permite que você defina valores para inserções ou atualizações.

Pode ser usado em vez de passar um array de dados diretamente para a inserção or update functions:

$this->db->set('name', $name);
$this->db->insert('mytable');  // Produces: INSERT INTO mytable (`name`) VALUES ('{$name}')

If you use multiple function called they will be assembled properly based on whether you are doing an insert or an update:

$this->db->set('name', $name);
$this->db->set('title', $title);
$this->db->set('status', $status);
$this->db->insert('mytable');

set() will also accept an optional third parameter ($escape), that will prevent data from being escaped if set to FALSE. To illustrate the difference, here is set() used both with and without the escape parameter.

$this->db->set('field', 'field+1', FALSE);
$this->db->where('id', 2);
$this->db->update('mytable'); // gives UPDATE mytable SET field = field+1 WHERE id = 2

$this->db->set('field', 'field+1');
$this->db->where('id', 2);
$this->db->update('mytable'); // gives UPDATE `mytable` SET `field` = 'field+1' WHERE `id` = 2

Você também pode passar um array associativo para esta função:

$array = array(
        'name' => $name,
        'title' => $title,
        'status' => $status
);

$this->db->set($array);
$this->db->insert('mytable');

Or an object:

/*
class Myclass {
        public $title = 'My Title';
        public $content = 'My Content';
        public $date = 'My Date';
}
*/

$object = new Myclass;
$this->db->set($object);
$this->db->insert('mytable');

$this->db->update()

Generates an update string and runs the query based on the data you supply. You can pass an array or an object para a função. Here is an example using an array:

$data = array(
        'title' => $title,
        'name' => $name,
        'date' => $date
);

$this->db->where('id', $id);
$this->db->update('mytable', $data);
// Produces:
//
//      UPDATE mytable
//      SET title = '{$title}', name = '{$name}', date = '{$date}'
//      WHERE id = $id

Ou você pode fornecer um objeto:

/*
class Myclass {
        public $title = 'My Title';
        public $content = 'My Content';
        public $date = 'My Date';
}
*/

$object = new Myclass;
$this->db->where('id', $id);
$this->db->update('mytable', $object);
// Produces:
//
// UPDATE `mytable`
// SET `title` = '{$title}', `name` = '{$name}', `date` = '{$date}'
// WHERE id = `$id`

Nota

All values are escaped automatically producing safer queries.

You’ll notice the use of the $this->db->where() function, enabling you para definir a cláusula WHERE. Você pode, opcionalmente, passar essas informações diretamente na função de atualização como uma string:

$this->db->update('mytable', $data, "id = 4");

Or as an array:

$this->db->update('mytable', $data, array('id' => $id));

You may also use the $this->db->set() function described above when performing updates.

$this->db->update_batch()

Gera uma cadeia de atualização com base nos dados que você fornece e executa a consulta. You can either pass an array or an object para a função. Here is an example using an array:

$data = array(
   array(
      'title' => 'My title' ,
      'name' => 'My Name 2' ,
      'date' => 'My date 2'
   ),
   array(
      'title' => 'Another title' ,
      'name' => 'Another Name 2' ,
      'date' => 'Another date 2'
   )
);

$this->db->update_batch('mytable', $data, 'title');

// Produces:
// UPDATE `mytable` SET `name` = CASE
// WHEN `title` = 'My title' THEN 'My Name 2'
// WHEN `title` = 'Another title' THEN 'Another Name 2'
// ELSE `name` END,
// `date` = CASE
// WHEN `title` = 'My title' THEN 'My date 2'
// WHEN `title` = 'Another title' THEN 'Another date 2'
// ELSE `date` END
// WHERE `title` IN ('My title','Another title')

O primeiro parâmetro conterá o nome da tabela, o segundo é um associative array of values, the third parameter is the where key.

Nota

All values are escaped automatically producing safer queries.

Nota

affected_rows() won’t give you proper results with this method, due to the very nature of how it works. Instead, update_batch() returns the number of rows affected.

$this->db->get_compiled_update()

This works exactly the same way as $this->db->get_compiled_insert() except that it produces an UPDATE SQL string instead of an INSERT SQL string.

For more information view documentation for $this->db->get_compiled_insert().

Nota

This method doesn’t work for batched updates.

Excluindo dados¶

$this->db->delete()

Gera uma string SQL de exclusão e executa a consulta.

$this->db->delete('mytable', array('id' => $id));  // Produces: // DELETE FROM mytable  // WHERE id = $id

The first parameter is the table name, the second is the where clause. Você também pode usar as funções where() ou or_where() ao invés de passar the data to the second parameter of the function:

$this->db->where('id', $id);
$this->db->delete('mytable');

// Produces:
// DELETE FROM mytable
// WHERE id = $id

Uma matriz de nomes de tabelas pode ser passada para delete() se você quiser Exclua dados de mais de uma tabela.

$tables = array('table1', 'table2', 'table3');
$this->db->where('id', '5');
$this->db->delete($tables);

If you want to delete all data from a table, you can use the truncate() função, ou empty_table().

$this->db->empty_table()

Gera uma string SQL de exclusão e executa o query.:

$this->db->empty_table('mytable'); // Produces: DELETE FROM mytable

$this->db->truncate()

Generates a truncate SQL string and runs the query.

$this->db->from('mytable');
$this->db->truncate();

// or

$this->db->truncate('mytable');

// Produce:
// TRUNCATE mytable

Nota

If the TRUNCATE command isn’t available, truncate() will execute as “DELETE FROM table”.

$this->db->get_compiled_delete()

This works exactly the same way as $this->db->get_compiled_insert() except that it produces a DELETE SQL string instead of an INSERT SQL string.

For more information view documentation for $this->db->get_compiled_insert().

Method chaining allows you to simplify your syntax by connecting multiple functions. Consider this example:

$query = $this->db->select('title')
                ->where('id', $id)
                ->limit(10, 20)
                ->get('mytable');

Query Builder Cache¶

Embora não seja cache "true", o Query Builder permite salvar (ou "cache") certain parts of your queries for reuse at a later point in your execução do script. Normalmente, quando uma chamada do Construtor de Consultas é concluída, todas as informações armazenadas são redefinidas para a próxima chamada. Com o cache, você pode Impeça essa redefinição e reutilize as informações facilmente.

Chamadas em cache são cumulativas. Se você fizer 2 chamadas select() em cache e em seguida, 2 chamadas select() sem cache, isso resultará em 4 chamadas select(). There are three Cache functions available:

$this->db->start_cache()

Essa função deve ser chamada para iniciar o armazenamento em cache. Todas as consultas do Query Builder of the correct type (see below for supported queries) are stored for later use.

$this->db->stop_cache()

Essa função pode ser chamada para parar o armazenamento em cache.

$this->db->flush_cache()

Essa função exclui todos os itens do cache do Query Builder.

An example of caching¶

Here’s a usage example:

$this->db->start_cache();
$this->db->select('field1');
$this->db->stop_cache();
$this->db->get('tablename');
//Generates: SELECIONAR `field1` FROM (`tablename`)

$this->db->select('field2');
$this->db->get('tablename');
//Generates:  SELECIONAR `field1`, `field2` FROM (`tablename`)

$this->db->flush_cache();
$this->db->select('field2');
$this->db->get('tablename');
//Generates:  SELECIONAR `field2` FROM (`tablename`)

Nota

The following statements can be cached: select, from, join, where, like, group_by, having, order_by

Resetting Query Builder¶

$this->db->reset_query()

A redefinição do Query Builder permite que você inicie de novo com sua consulta sem executing it first using a method like $this->db->get() or $this->db->insert(). Just like the methods that execute a query, this will not reset items you’ve cached using Query Builder Cache.

This is useful in situations where you are using Query Builder to generate SQL (ex. $this->db->get_compiled_select()) but then choose to, for instance, run the query:

// Nota that the second parameter of the get_compiled_select method is FALSE
$sql = $this->db->select(array('field1','field2'))
                                ->where('field3',5)
                                ->get_compiled_select('mytable', FALSE);

// ...
// Do something crazy with the SQL code... like add it to a cron script for
// later execution or something...
// ...

$data = $this->db->get()->result_array();

// Would execute and return an array of results of the following query:
// SELECIONAR field1, field1 from mytable where field3 = 5;

Nota

Double calls to get_compiled_select() while you’re using the Query Builder Cache functionality and NOT resetting your queries will results in the cache being merged twice. That in turn will i.e. if you’re caching a select() - select the same field twice.

Classe do Construtor de Querys¶

O CodeIgniter lhe dá acesso a uma classe do Query Builder. Esse padrão permite que as informações sejam recuperadas, inseridas e atualizadas em sua database com scripts mínimos. Em alguns casos, apenas uma ou duas linhas de código são necessários para executar uma ação de Database. O CodeIgniter não requer que cada tabela de database seja sua própria classe Arquivo. Em vez disso, fornece uma interface mais simplificada.

Além da simplicidade, um grande benefício para usar os recursos do Query Builder é que ele permite criar aplicativos independentes de Database, a sintaxe da Query é gerada por cada adaptador de Database. Também permite para Querys mais seguras, já que os valores são escapados automaticamente pelo system.

Nota

Se você pretende escrever suas próprias perguntas, você pode desabilitar classe no seu arquivo de configuração do database, permitindo que a library do database núcleo e adaptador para utilizar menos recursos.

Selecionando dados¶

As seguintes funções permitem que você construa SQL SELECIONAR statements.

$this->db->get()

Executa a Query de seleção e retorna o resultado. Pode ser usado por si para recuperar todos os registros de uma tabela:

$query = $this->db->get('mytable');  // Produces: SELECIONAR * FROM mytable

O segundo e terceiro parâmetros permitem que você defina um limite e um deslocamento clause:

$query = $this->db->get('mytable', 10, 20);

// Executes: SELECIONAR * FROM mytable LIMIT 20, 10
// (in MySQL. Other databases have slightly different syntax)

Você notará que a função acima é atribuída a uma variável chamada $query, que pode ser usado para mostrar os resultados:

$query = $this->db->get('mytable');

foreach ($query->result() as $row)
{
        echo $row->title;
}

Por favor visite o funções resultantes página para um completo discussão sobre geração de resultados.

$this->db->get_compiled_select()

Compila a Query de seleção como $this->db->get() mas não run A pergunta. Esse método simplesmente retorna a Query SQL como uma string.

Exemplo:

$sql = $this->db->get_compiled_select('mytable');
echo $sql;

// Prints string: SELECIONAR * FROM mytable

O segundo parâmetro permite definir se a Query do construtor de Querys é ou não será redefinido (por padrão, ele será redefinido, assim como ao usar $this->db->get()):

echo $this->db->limit(10,20)->get_compiled_select('mytable', FALSE);

// Prints string: SELECIONAR * FROM mytable LIMIT 20, 10
// (in MySQL. Other databases have slightly different syntax)

echo $this->db->select('title, content, date')->get_compiled_select();

// Prints string: SELECIONAR title, content, date FROM mytable LIMIT 20, 10

A principal coisa a notar no exemplo acima é que a segunda Query não utilize $this->db->from() e não passou um nome de tabela para o primeiro parâmetro. A razão para este resultado é porque a Query não foi executado usando $this->db->get() que redefine valores ou redefine diretamente using $this->db->reset_query().

$this->db->get_where()

Idêntico à função acima, exceto que permite adicionar um “where” clause in the second parameter, instead of using the db->where() function:

$query = $this->db->get_where('mytable', array('id' => $id), $limit, $offset);

Por favor, leia sobre a função where abaixo para mais informações.

Nota

get_where() era conhecido anteriormente como getwhere(), que foi removido

$this->db->select()

Permits you to write the SELECIONAR portion of your query:

$this->db->select('title, content, date');
$query = $this->db->get('mytable');

// Executes: SELECIONAR title, content, date FROM mytable

Nota

Se você estiver selecionando todos (*) de uma tabela, não será necessário use esta função. Quando omitido, o CodeIgniter assume que você deseja to select all fields and automatically adds ‘SELECIONAR *’.

$this->db->select() aceita um segundo parâmetro opcional. Se você definir para FALSE, o CodeIgniter não tentará proteger seus nomes de campo ou tabela. Isso é útil se você precisar de uma instrução de seleção composta onde o escape de campos pode quebrá-los.

$this->db->select('(SELECIONAR SUM(payments.amount) FROM payments WHERE payments.invoice_id=4) AS amount_paid', FALSE);
$query = $this->db->get('mytable');

$this->db->select_max()

Writes a SELECIONAR MAX(field) portion for your query. You can optionally inclua um segundo parâmetro para renomear o campo resultante.

$this->db->select_max('age');
$query = $this->db->get('members');  // Produces: SELECIONAR MAX(age) as age FROM members

$this->db->select_max('age', 'member_age');
$query = $this->db->get('members'); // Produces: SELECIONAR MAX(age) as member_age FROM members

$this->db->select_min()

Writes a “SELECIONAR MIN(field)” portion for your query. As with select_max(), Você pode, opcionalmente, incluir um segundo parâmetro para renomear o campo resultante.

$this->db->select_min('age');
$query = $this->db->get('members'); // Produces: SELECIONAR MIN(age) as age FROM members

$this->db->select_avg()

Writes a “SELECIONAR AVG(field)” portion for your query. As with select_max(), Você pode, opcionalmente, incluir um segundo parâmetro para renomear o campo resultante.

$this->db->select_avg('age');
$query = $this->db->get('members'); // Produces: SELECIONAR AVG(age) as age FROM members

$this->db->select_sum()

Writes a “SELECIONAR SUM(field)” portion for your query. As with select_max(), Você pode, opcionalmente, incluir um segundo parâmetro para renomear o campo resultante.

$this->db->select_sum('age');
$query = $this->db->get('members'); // Produces: SELECIONAR SUM(age) as age FROM members

$this->db->from()

Permite que você escreva a parte FROM de sua Query:

$this->db->select('title, content, date');
$this->db->from('mytable');
$query = $this->db->get();  // Produces: SELECIONAR title, content, date FROM mytable

Nota

Como mostrado anteriormente, a parte FROM da sua Query pode ser especificada in the $this->db->get() function, so use whichever method you prefer.

$this->db->join()

Permite que você escreva a parte JOIN de sua Query:

$this->db->select('*');
$this->db->from('blogs');
$this->db->join('comments', 'comments.id = blogs.id');
$query = $this->db->get();

// Produces:
// SELECIONAR * FROM blogs JOIN comments ON comments.id = blogs.id

Várias chamadas de função podem ser feitas se você precisar de várias junções em um query.

Se você precisa de um tipo específico de JOIN, você pode especificá-lo através do terceiro parâmetro da função. As opções são: esquerda, direita, externa, interna, esquerda exterior e direito exterior.

$this->db->join('comments', 'comments.id = blogs.id', 'left');
// Produces: LEFT JOIN comments ON comments.id = blogs.id

Procurando por dados específicos¶

$this->db->where()

Esta função permite-lhe definir WHERE cláusulas usando um dos quatro methods:

Nota

All values passed to this function are escaped automatically, produzindo Querys mais seguras.

  1. Método chave/valor simples:

    $this->db->where('name', $name); // Produces: WHERE name = 'Joe'

    Observe que o sinal de igual é adicionado para você.

    Se você usar várias chamadas de função, elas serão encadeadas junto com E entre eles:

    $this->db->where('name', $name);
$this->db->where('title', $title);
$this->db->where('status', $status);
// WHERE name = 'Joe' AND title = 'boss' AND status = 'active'

  2. Método de chave/valor personalizado:

    Você pode incluir um operador no primeiro parâmetro para controlar a comparação:

    $this->db->where('name !=', $name);
$this->db->where('id <', $id); // Produces: WHERE name != 'Joe' AND id < 45

  3. Método de matriz associativa:

    $array = array('name' => $name, 'title' => $title, 'status' => $status);
$this->db->where($array);
// Produces: WHERE name = 'Joe' AND title = 'boss' AND status = 'active'

    Você pode incluir seus próprios operadores usando este método também:

    $array = array('name !=' => $name, 'id <' => $id, 'date >' => $date);
$this->db->where($array);
  4. Seqüência personalizada:

    Você pode escrever suas próprias cláusulas manualmente:

    $where = "name='Joe' AND status='boss' OR status='active'";
$this->db->where($where);

$this->db->where() aceita um terceiro parâmetro opcional. Se você configurá-lo para FALSE, o CodeIgniter não tentará proteger seus nomes de campo ou tabela.

$this->db->where('MATCH (field) AGAINST ("value")', NULL, FALSE);

$this->db->or_where()

Esta função é idêntica à acima, exceto que múltiplos instâncias são unidas por OR:

$this->db->where('name !=', $name);
$this->db->or_where('id >', $id);  // Produces: WHERE name != 'Joe' OR id > 50

Nota

ou_where() era conhecido anteriormente como orwhere(), que foi removed.

$this->db->where_in()

Gera um WHERE field IN ('item', 'item') Query SQL associada a AND se appropriate

$names = array('Frank', 'Todd', 'James');
$this->db->where_in('username', $names);
// Produces: WHERE username IN ('Frank', 'Todd', 'James')

$this->db->or_where_in()

Gera um WHERE field IN ('item', 'item') Query SQL associada a OR se appropriate

$names = array('Frank', 'Todd', 'James');
$this->db->or_where_in('username', $names);
// Produces: OR username IN ('Frank', 'Todd', 'James')

$this->db->where_not_in()

Gera um WHERE field NOT IN ('item', 'item') Query SQL associada a E se apropriado

$names = array('Frank', 'Todd', 'James');
$this->db->where_not_in('username', $names);
// Produces: WHERE username NOT IN ('Frank', 'Todd', 'James')

$this->db->or_where_not_in()

Gera um WHERE field NOT IN ('item', 'item') Query SQL associada a OR se apropriado

$names = array('Frank', 'Todd', 'James');
$this->db->or_where_not_in('username', $names);
// Produces: OR username NOT IN ('Frank', 'Todd', 'James')

Procurando por dados semelhantes¶

$this->db->like()

Esse método permite gerar LIKE cláusulas úteis para fazer searches.

Nota

All values passed to this method are escaped automatically.

  1. Método chave/valor simples:

    $this->db->like('title', 'match');
// Produces: WHERE `title` LIKE '%match%' ESCAPE '!'

    Se você usar várias chamadas de método, elas serão encadeadas junto com E entre eles:

    $this->db->like('title', 'match');
$this->db->like('body', 'match');
// WHERE `title` LIKE '%match%' ESCAPE '!' AND  `body` LIKE '%match% ESCAPE '!'

    Se você quiser controlar onde o curinga (%) é colocado, você pode usar um terceiro argumento opcional. Suas opções são 'antes', 'depois' e 'both' (que é o padrão).

    $this->db->like('title', 'match', 'before');    // Produces: WHERE `title` LIKE '%match' ESCAPE '!'
$this->db->like('title', 'match', 'after');     // Produces: WHERE `title` LIKE 'match%' ESCAPE '!'
$this->db->like('title', 'match', 'both');      // Produces: WHERE `title` LIKE '%match%' ESCAPE '!'

  2. Método de matriz associativa:

    $array = array('title' => $match, 'page1' => $match, 'page2' => $match);
$this->db->like($array);
// WHERE `title` LIKE '%match%' ESCAPE '!' AND  `page1` LIKE '%match%' ESCAPE '!' AND  `page2` LIKE '%match%' ESCAPE '!'

$this->db->or_like()

Este método é idêntico ao acima, exceto que múltiplos instâncias são unidas por OR:

$this->db->like('title', 'match'); $this->db->or_like('body', $match);
// WHERE `title` LIKE '%match%' ESCAPE '!' OR  `body` LIKE '%match%' ESCAPE '!'

Nota

or_like() was formerly known as orlike(), which has been removed.

$this->db->not_like()

This method is identical to like(), except that it generates NÃO FAZER declarações:

$this->db->not_like('title', 'match');  // WHERE `title` NOT LIKE '%match% ESCAPE '!'

$this->db->or_not_like()

This method is identical to not_like(), except that multiple instâncias são unidas por OR:

$this->db->like('title', 'match');
$this->db->or_not_like('body', 'match');
// WHERE `title` LIKE '%match% OR  `body` NOT LIKE '%match%' ESCAPE '!'

$this->db->group_by()

Permite que você escreva a parte GROUP BY da sua Query:

$this->db->group_by("title"); // Produces: GROUP BY title

Você também pode passar uma matriz de vários valores:

$this->db->group_by(array("title", "date"));  // Produces: GROUP BY title, date

Nota

group_by() era conhecido anteriormente como groupby(), que tem sido removed.

$this->db->distinct()

Adiciona a palavra-chave "DISTINCT" a uma Query

$this->db->distinct();
$this->db->get('table'); // Produces: SELECIONAR DISTINCT * FROM table

$this->db->having()

Permite que você escreva a parte HAVING da sua Query. Há 2 possíveis sintaxes, 1 argumento ou 2:

$this->db->having('user_id = 45');  // Produces: HAVING user_id = 45
$this->db->having('user_id',  45);  // Produces: HAVING user_id = 45

Você também pode passar uma matriz de vários valores:

$this->db->having(array('title =' => 'My Title', 'id <' => $id));
// Produces: HAVING title = 'My Title', id < 45

Se você estiver usando um Database para o qual o CodeIgniter escapa, você pode impedir o escape de conteúdo passando um terceiro argumento opcional e configurando para FALSE.

$this->db->having('user_id',  45);  // Produces: HAVING `user_id` = 45 in some databases such as MySQL
$this->db->having('user_id',  45, FALSE);  // Produces: HAVING user_id = 45

$this->db->or_having()

Idêntico a ter(), separa apenas várias cláusulas com “OR”.

Ordenação de resultados¶

$this->db->order_by()

Permite que você defina uma cláusula ORDER BY.

O primeiro parâmetro contém o nome da coluna que você gostaria de pedir.

O segundo parâmetro permite definir a direção do resultado. Opções são ASC, DESC AND ALEATÓRIA.

$this->db->order_by('title', 'DESC');
// Produces: ORDER BY `title` DESC

Você também pode passar sua própria string no primeiro parâmetro:

$this->db->order_by('title DESC, name ASC');
// Produces: ORDER BY `title` DESC, `name` ASC

Ou várias chamadas de função podem ser feitas se você precisar de vários campos.

$this->db->order_by('title', 'DESC');
$this->db->order_by('name', 'ASC');
// Produces: ORDER BY `title` DESC, `name` ASC

Se você escolher o ALEATÓRIA opção de direção, os primeiros parâmetros serão ser ignorado, a menos que você especifique um valor de semente numérico.

$this->db->order_by('title', 'ALEATÓRIA');
// Produces: ORDER BY RAND()

$this->db->order_by(42, 'ALEATÓRIA');
// Produces: ORDER BY RAND(42)

Nota

order_by() era conhecido anteriormente como orderby(), que foi removed.

Nota

A ordenação aleatória não é atualmente suportada no Oracle e O padrão será ASC.

Limitando ou contando resultados¶

$this->db->limit()

Permite limitar o número de linhas que você gostaria de retornar pela Query:

$this->db->limit(10);  // Produces: LIMIT 10

O segundo parâmetro permite definir um deslocamento de resultado.

$this->db->limit(10, 20);  // Produces: LIMIT 20, 10 (in MySQL.  Other databases have slightly different syntax)

$this->db->count_all_results()

Permite que você determine o número de linhas em um determinado Active Query de registro. As Querys aceitarão os restritores do Query Builder, como where(), or_where(), like(), or_like(), etc. Exemplo:

echo $this->db->count_all_results('my_table');  // Produces an integer, like 25
$this->db->like('title', 'match');
$this->db->from('my_table');
echo $this->db->count_all_results(); // Produces an integer, like 17

No entanto, esse método também redefine todos os valores de campo que você pode ter passado to select(). If you need to keep them, you can pass FALSE as the segundo parâmetro:

echo $this->db->count_all_results('my_table', FALSE);

$this->db->count_all()

Permite que você determine o número de linhas em uma tabela específica. Envie o nome da tabela no primeiro parâmetro. Exemplo:

echo $this->db->count_all('my_table');  // Produces an integer, like 25

Agrupamento de Query¶

Agrupamento de Query allows you to create groups of WHERE clauses by enclosing them in parentheses. This will allow você criar Querys com cláusulas WHERE complexas. Grupos aninhados são suportados. Exemplo:

$this->db->select('*')->from('my_table')
        ->group_start()
                ->where('a', 'a')
                ->or_group_start()
                        ->where('b', 'b')
                        ->where('c', 'c')
                ->group_end()
        ->group_end()
        ->where('d', 'd')
->get();

// Generates:
// SELECIONAR * FROM (`my_table`) WHERE ( `a` = 'a' OR ( `b` = 'b' AND `c` = 'c' ) ) AND `d` = 'd'

Nota

grupos precisam ser equilibrados, certifique-se de que cada group_start() seja correspondido por um group_end().

$this->db->group_start()

Inicia um novo grupo adicionando um parêntese de abertura à cláusula WHERE da Query.

$this->db->or_group_start()

Inicia um novo grupo adicionando um parêntese de abertura à cláusula WHERE da Query, prefixando-o com 'OR'.

$this->db->not_group_start()

Inicia um novo grupo adicionando um parêntese de abertura à cláusula WHERE da Query, prefixando-a com 'NOT'.

$this->db->or_not_group_start()

Inicia um novo grupo adicionando um parêntese de abertura à cláusula WHERE da Query, prefixando-a com 'OR NOT'.

$this->db->group_end()

Termina o grupo atual adicionando um parêntese de fechamento à cláusula WHERE da Query.

Inserindo Dados¶

$this->db->insert()

Gera uma sequência de inserção com base nos dados que você fornece e executa o inquerir. Você pode passar um array or an object ao função. Aqui está um exemplo usando uma matriz:

$data = array(
        'title' => 'My title',
        'name' => 'My Name',
        'date' => 'My date'
);

$this->db->insert('mytable', $data);
// Produces: INSERT INTO mytable (title, name, date) VALUES ('My title', 'My name', 'My date')

O primeiro parâmetro conterá o nome da tabela, o segundo é um matriz associativa de valores.

Aqui está um exemplo usando um objeto:

/*
class Myclass {
        public $title = 'My Title';
        public $content = 'My Content';
        public $date = 'My Date';
}
*/

$object = new Myclass;
$this->db->insert('mytable', $object);
// Produces: INSERT INTO mytable (title, content, date) VALUES ('My Title', 'My Content', 'My Date')

O primeiro parâmetro conterá o nome da tabela, o segundo é um object.

Nota

All values are escaped automatically produzindo Querys mais seguras.

$this->db->get_compiled_insert()

Compiles the insertion query just like $this->db->insert() mas não run A pergunta. Esse método simplesmente retorna a Query SQL como uma string.

Exemplo:

$data = array(
        'title' => 'My title',
        'name'  => 'My Name',
        'date'  => 'My date'
);

$sql = $this->db->set($data)->get_compiled_insert('mytable');
echo $sql;

// Produces string: INSERT INTO mytable (`title`, `name`, `date`) VALUES ('My title', 'My name', 'My date')

O segundo parâmetro permite definir se a Query do construtor de Querys é ou não will be reset (by default it will be–just like $this->db->insert()):

echo $this->db->set('title', 'My Title')->get_compiled_insert('mytable', FALSE);

// Produces string: INSERT INTO mytable (`title`) VALUES ('My Title')

echo $this->db->set('content', 'My Content')->get_compiled_insert();

// Produces string: INSERT INTO mytable (`title`, `content`) VALUES ('My Title', 'My Content')

A principal coisa a notar no exemplo acima é que a segunda Query não utilize $this->db->from() nem passou um nome de tabela para o primeiro parâmetro. A razão pela qual isso funcionou é porque a Query não foi executada using $this->db->insert() que redefine valores ou redefine diretamente using $this->db->reset_query().

Nota

Este método não funciona para inserções em lote.

$this->db->insert_batch()

Gera uma sequência de inserção com base nos dados que você fornece e executa o inquerir. Você pode passar um array or an object ao função. Aqui está um exemplo usando uma matriz:

$data = array(
        array(
                'title' => 'My title',
                'name' => 'My Name',
                'date' => 'My date'
        ),
        array(
                'title' => 'Another title',
                'name' => 'Another Name',
                'date' => 'Another date'
        )
);

$this->db->insert_batch('mytable', $data);
// Produces: INSERT INTO mytable (title, name, date) VALUES ('My title', 'My name', 'My date'),  ('Another title', 'Another name', 'Another date')

O primeiro parâmetro conterá o nome da tabela, o segundo é um matriz associativa de valores.

Nota

All values are escaped automatically produzindo Querys mais seguras.

Atualizando Dados¶

$this->db->replace()

Este método executa uma instrução REPLACE, que é basicamente o SQL standard for (optional) DELETE + INSERT, using PRIMARY and UNIQUE chaves como o fator determinante. No nosso caso, você vai economizar a necessidade de implementar logics with different combinations of select(), update(), delete() and insert() calls.

Exemplo:

$data = array(
        'title' => 'My title',
        'name'  => 'My Name',
        'date'  => 'My date'
);

$this->db->replace('table', $data);

// Executes: REPLACE INTO mytable (title, name, date) VALUES ('My title', 'My name', 'My date')

In the above example, if we assume that the title field is our primary key, then if a row containing ‘My title’ as the title value, that row será excluído com nossos novos dados de linha substituindo-os.

Usage of the set() method is also allowed and all fields are automatically escaped, just like with insert().

$this->db->set()

Esta função permite-lhe definir values for inserts or updates.

It can be used instead of passing a data array directly ao insert ou atualizar funções:

$this->db->set('name', $name);
$this->db->insert('mytable');  // Produces: INSERT INTO mytable (`name`) VALUES ('{$name}')

Se você usar várias funções, elas serão montadas corretamente com base em se você está fazendo uma inserção ou uma atualização:

$this->db->set('name', $name);
$this->db->set('title', $title);
$this->db->set('status', $status);
$this->db->insert('mytable');

set() will also accept an optional third parameter ($escape), that impedirá que os dados sejam escapados se definidos como FALSE. Para ilustrar o difference, here is set() used both with and without the escape parameter.

$this->db->set('field', 'field+1', FALSE);
$this->db->where('id', 2);
$this->db->update('mytable'); // gives UPDATE mytable SET field = field+1 WHERE id = 2

$this->db->set('field', 'field+1');
$this->db->where('id', 2);
$this->db->update('mytable'); // gives UPDATE `mytable` SET `field` = 'field+1' WHERE `id` = 2

Você também pode passar um array associativo para esta função:

$array = array(
        'name' => $name,
        'title' => $title,
        'status' => $status
);

$this->db->set($array);
$this->db->insert('mytable');

Ou um objeto:

/*
class Myclass {
        public $title = 'My Title';
        public $content = 'My Content';
        public $date = 'My Date';
}
*/

$object = new Myclass;
$this->db->set($object);
$this->db->insert('mytable');

$this->db->update()

Gera uma string de atualização e executa a Query com base nos dados que você fornecem. Você pode passar um array or an object ao function. Here é um exemplo usando uma matriz:

$data = array(
        'title' => $title,
        'name' => $name,
        'date' => $date
);

$this->db->where('id', $id);
$this->db->update('mytable', $data);
// Produces:
//
//      UPDATE mytable
//      SET title = '{$title}', name = '{$name}', date = '{$date}'
//      WHERE id = $id

Ou você pode fornecer um objeto:

/*
class Myclass {
        public $title = 'My Title';
        public $content = 'My Content';
        public $date = 'My Date';
}
*/

$object = new Myclass;
$this->db->where('id', $id);
$this->db->update('mytable', $object);
// Produces:
//
// UPDATE `mytable`
// SET `title` = '{$title}', `name` = '{$name}', `date` = '{$date}'
// WHERE id = `$id`

Nota

All values are escaped automatically produzindo Querys mais seguras.

You’ll notice the use of the $this->db->where() function, enabling you para definir a cláusula WHERE. Você pode, opcionalmente, passar essas informações directly inao update function as a string:

$this->db->update('mytable', $data, "id = 4");

Ou como uma matriz:

$this->db->update('mytable', $data, array('id' => $id));

You may also use the $this->db->set() function described above when executando atualizações.

$this->db->update_batch()

Gera uma cadeia de atualização com base nos dados que você fornece e executa a Query. Você pode passar um array or an object ao function. Here é um exemplo usando uma matriz:

$data = array(
   array(
      'title' => 'My title' ,
      'name' => 'My Name 2' ,
      'date' => 'My date 2'
   ),
   array(
      'title' => 'Another title' ,
      'name' => 'Another Name 2' ,
      'date' => 'Another date 2'
   )
);

$this->db->update_batch('mytable', $data, 'title');

// Produces:
// UPDATE `mytable` SET `name` = CASE
// WHEN `title` = 'My title' THEN 'My Name 2'
// WHEN `title` = 'Another title' THEN 'Another Name 2'
// ELSE `name` END,
// `date` = CASE
// WHEN `title` = 'My title' THEN 'My date 2'
// WHEN `title` = 'Another title' THEN 'Another date 2'
// ELSE `date` END
// WHERE `title` IN ('My title','Another title')

O primeiro parâmetro conterá o nome da tabela, o segundo é um associative matriz de valores, o terceiro parâmetro é a chave where.

Nota

All values are escaped automatically produzindo Querys mais seguras.

Nota

affected_rows() não lhe dará resultados adequados com este método, due ao very nature of how it works. Instead, update_batch() retorna o número de linhas afetadas.

$this->db->get_compiled_update()

This works exactly the same way as $this->db->get_compiled_insert() except que produz uma string SQL UPDATE em vez de uma string SQL INSERT.

Para mais informações, consulte a documentação para $this->db->get_compiled_insert().

Nota

Este método não funciona para atualizações em lote.

Excluindo dados¶

$this->db->delete()

Gera uma string SQL de exclusão e executa a Query.

$this->db->delete('mytable', array('id' => $id));  // Produces: // DELETE FROM mytable  // WHERE id = $id

O primeiro parâmetro é o nome da tabela, o segundo é a cláusula where. Você também pode usar as funções where() ou or_where() ao invés de passar the data ao second parameter of the function:

$this->db->where('id', $id);
$this->db->delete('mytable');

// Produces:
// DELETE FROM mytable
// WHERE id = $id

Uma matriz de nomes de tabelas pode ser passada para delete() se você quiser Exclua dados de mais de uma tabela.

$tables = array('table1', 'table2', 'table3');
$this->db->where('id', '5');
$this->db->delete($tables);

Se você quiser apagar todos os dados de uma tabela, você pode usar o truncate() função, ou empty_table().

$this->db->empty_table()

Gera uma string SQL de exclusão e executa o query.:

$this->db->empty_table('mytable'); // Produces: DELETE FROM mytable

$this->db->truncate()

Gera uma string SQL truncada e executa a Query.

$this->db->from('mytable');
$this->db->truncate();

// or

$this->db->truncate('mytable');

// Produce:
// TRUNCATE mytable

Nota

Se o comando TRUNCATE não estiver disponível, truncate() será executar como "DELETE FROM table".

$this->db->get_compiled_delete()

This works exactly the same way as $this->db->get_compiled_insert() except que produz uma string SQL DELETE em vez de uma string SQL INSERT.

Para mais informações, consulte a documentação para $this->db->get_compiled_insert().

$query = $this->db->select('title')

Query Builder Cache¶

Embora não seja cache "true", o Query Builder permite salvar (ou "cache") certas partes de suas Querys para reutilização em um ponto posterior em sua execução do script. Normalmente, quando uma chamada do Construtor de Querys é concluída, todas as informações armazenadas são redefinidas para a próxima chamada. Com o cache, você pode Impeça essa redefinição e reutilize as informações facilmente.

Chamadas em cache são cumulativas. Se você fizer 2 chamadas select() em cache e em seguida, 2 chamadas select() sem cache, isso resultará em 4 chamadas select(). There are three Cache functions available:

$this->db->start_cache()

Essa função deve ser chamada para iniciar o armazenamento em cache. Todas as Querys do Query Builder do tipo correto (veja abaixo as Querys suportadas) são armazenadas para mais tarde use.

$this->db->stop_cache()

Essa função pode ser chamada para parar o armazenamento em cache.

$this->db->flush_cache()

Essa função exclui todos os itens do cache do Query Builder.

Um exemplo de armazenamento em cache¶

Aqui está um exemplo de uso:

$this->db->start_cache();
$this->db->select('field1');
$this->db->stop_cache();
$this->db->get('tablename');
//Generates: SELECIONAR `field1` FROM (`tablename`)

$this->db->select('field2');
$this->db->get('tablename');
//Generates:  SELECIONAR `field1`, `field2` FROM (`tablename`)

$this->db->flush_cache();
$this->db->select('field2');
$this->db->get('tablename');
//Generates:  SELECIONAR `field2` FROM (`tablename`)

Nota

As seguintes instruções podem ser armazenadas em cache: select, from, join, onde, como, group_by, tendo, order_by

Redefinindo o Construtor de Querys¶

$this->db->reset_query()

Redefinindo o Construtor de Querys allows you to start fresh with your query without executing it first using a method like $this->db->get() or $this->db->insert(). Just like the methods that execute a query, this will not reset items you’ve em cache usando Query Builder Cache.

Isso é útil em situações em que você está usando o Query Builder para gerar SQL (ex. $this->db->get_compiled_select()) but then choose to, for instance, execute a Query:

// Nota that the second parameter of the get_compiled_select method is FALSE
$sql = $this->db->select(array('field1','field2'))
                                ->where('field3',5)
                                ->get_compiled_select('mytable', FALSE);

// ...
// Do something crazy with the SQL code... like add it to a cron script for
// later execution or something...
// ...

$data = $this->db->get()->result_array();

// Would execute and return an array of results of the following query:
// SELECIONAR field1, field1 from mytable where field3 = 5;

Nota

Double calls to get_compiled_select() while you’re using the Query Builder Cache functionality and NOT resetting your queries resultará no cache sendo mesclado duas vezes. Isso, por sua vez, i.e. if you’re caching a select() - select the same field twice.

Classe do Construtor de Querys¶

O CodeIgniter lhe dá acesso a uma classe do Query Builder. Esse padrão permite que as informações sejam recuperadas, inseridas e atualizadas em sua database com scripts mínimos. Em alguns casos, apenas uma ou duas linhas de código são necessários para executar uma ação de Database. O CodeIgniter não requer que cada tabela de database seja sua própria classe Arquivo. Em vez disso, fornece uma interface mais simplificada.

Além da simplicidade, um grande benefício para usar os recursos do Query Builder é que ele permite criar aplicativos independentes de Database, a sintaxe da Query é gerada por cada adaptador de Database. Também permite para Querys mais seguras, já que os valores são escapados automaticamente pelo system.

Nota

Se você pretende escrever suas próprias perguntas, você pode desabilitar classe no seu arquivo de configuração do database, permitindo que a library do database núcleo e adaptador para utilizar menos recursos.

Selecionando dados¶

As seguintes funções permitem que você construa SQL SELECT statements.

$this->db->get()

Executa a Query de seleção e retorna o resultado. Pode ser usado por si para recuperar todos os registros de uma tabela:

$query = $this->db->get('mytable');  // Produces: SELECT * FROM mytable

O segundo e terceiro parâmetros permitem que você defina um limite e um deslocamento clause:

$query = $this->db->get('mytable', 10, 20);

// Executes: SELECT * FROM mytable LIMIT 20, 10
// (in MySQL. Other databases have slightly different syntax)

Você notará que a função acima é atribuída a uma variável chamada $query, que pode ser usado para mostrar os resultados:

$query = $this->db->get('mytable');

foreach ($query->result() as $row)
{
        echo $row->title;
}

Por favor visite o funções resultantes página para um completo discussão sobre geração de resultados.

$this->db->get_compiled_select()

Compila a Query de seleção como $this->db->get() mas não run A pergunta. Esse método simplesmente retorna a Query SQL como uma string.

Exemplo:

$sql = $this->db->get_compiled_select('mytable');
echo $sql;

// Prints string: SELECT * FROM mytable

O segundo parâmetro permite definir se a Query do construtor de Querys é ou não será redefinido (por padrão, ele será redefinido, assim como ao usar $this->db->get()):

echo $this->db->limit(10,20)->get_compiled_select('mytable', FALSE);

// Prints string: SELECT * FROM mytable LIMIT 20, 10
// (in MySQL. Other databases have slightly different syntax)

echo $this->db->select('title, content, date')->get_compiled_select();

// Prints string: SELECT title, content, date FROM mytable LIMIT 20, 10

A principal coisa a notar no exemplo acima é que a segunda Query não utilize $this->db->from() e não passou um nome de tabela para o primeiro parâmetro. A razão para este resultado é porque a Query não foi executado usando $this->db->get() que redefine valores ou redefine diretamente using $this->db->reset_query().

$this->db->get_where()

Idêntico à função acima, exceto que permite adicionar um "Where" cláusula no segundo parâmetro, em vez de usar o db-> onde() function:

$query = $this->db->get_where('mytable', array('id' => $id), $limit, $offset);

Por favor, leia sobre a função where abaixo para mais informações.

Nota

get_where() era conhecido anteriormente como getwhere(), que foi removido

$this->db->select()

Permite que você escreva a parte SELECT da sua Query:

$this->db->select('title, content, date');
$query = $this->db->get('mytable');

// Executes: SELECT title, content, date FROM mytable

Nota

Se você estiver selecionando todos (*) de uma tabela, não será necessário use esta função. Quando omitido, o CodeIgniter assume que você deseja para selecionar todos os campos e adiciona automaticamente 'SELECT *'.

$this->db->select() aceita um segundo parâmetro opcional. Se você definir para FALSE, o CodeIgniter não tentará proteger seus nomes de campo ou tabela. Isso é útil se você precisar de uma instrução de seleção composta onde o escape de campos pode quebrá-los.

$this->db->select('(SELECT SUM(payments.amount) FROM payments WHERE payments.invoice_id=4) AS amount_paid', FALSE);
$query = $this->db->get('mytable');

$this->db->select_max()

Writes a SELECT MAX(field) portion for your query. You can optionally inclua um segundo parâmetro para renomear o campo resultante.

$this->db->select_max('age');
$query = $this->db->get('members');  // Produces: SELECT MAX(age) as age FROM members

$this->db->select_max('age', 'member_age');
$query = $this->db->get('members'); // Produces: SELECT MAX(age) as member_age FROM members

$this->db->select_min()

Escreve uma porção "SELECT MIN (field)" para sua Query. Como com select_max(), Você pode, opcionalmente, incluir um segundo parâmetro para renomear o campo resultante.

$this->db->select_min('age');
$query = $this->db->get('members'); // Produces: SELECT MIN(age) as age FROM members

$this->db->select_avg()

Escreve uma porção "SELECT AVG (field)" para sua Query. Como com select_max(), Você pode, opcionalmente, incluir um segundo parâmetro para renomear o campo resultante.

$this->db->select_avg('age');
$query = $this->db->get('members'); // Produces: SELECT AVG(age) as age FROM members

$this->db->select_sum()

Escreve uma porção "SELECT SUM (field)" para sua Query. Como com select_max(), Você pode, opcionalmente, incluir um segundo parâmetro para renomear o campo resultante.

$this->db->select_sum('age');
$query = $this->db->get('members'); // Produces: SELECT SUM(age) as age FROM members

$this->db->from()

Permite que você escreva a parte FROM de sua Query:

$this->db->select('title, content, date');
$this->db->from('mytable');
$query = $this->db->get();  // Produces: SELECT title, content, date FROM mytable

Nota

Como mostrado anteriormente, a parte FROM da sua Query pode ser especificada in the $this->db->get() function, so use whichever method you prefer.

$this->db->join()

Permite que você escreva a parte JOIN de sua Query:

$this->db->select('*');
$this->db->from('blogs');
$this->db->join('comments', 'comments.id = blogs.id');
$query = $this->db->get();

// Produces:
// SELECT * FROM blogs JOIN comments ON comments.id = blogs.id

Várias chamadas de função podem ser feitas se você precisar de várias junções em um query.

Se você precisa de um tipo específico de JOIN, você pode especificá-lo através do terceiro parâmetro da função. As opções são: esquerda, direita, externa, interna, esquerda exterior e direito exterior.

$this->db->join('comments', 'comments.id = blogs.id', 'left');
// Produces: LEFT JOIN comments ON comments.id = blogs.id

Procurando por dados específicos¶

$this->db->where()

Esta função permite-lhe definir WHERE cláusulas usando um dos quatro methods:

Nota

Todos os valores passados para esta função são escapados automaticamente, produzindo Querys mais seguras.

  1. Método chave/valor simples:

    $this->db->where('name', $name); // Produces: WHERE name = 'Joe'

    Observe que o sinal de igual é adicionado para você.

    Se você usar várias chamadas de função, elas serão encadeadas junto com E entre eles:

    $this->db->where('name', $name);
$this->db->where('title', $title);
$this->db->where('status', $status);
// WHERE name = 'Joe' AND title = 'boss' AND status = 'active'

  2. Método de chave/valor personalizado:

    Você pode incluir um operador no primeiro parâmetro para controlar a comparação:

    $this->db->where('name !=', $name);
$this->db->where('id <', $id); // Produces: WHERE name != 'Joe' AND id < 45

  3. Método de matriz associativa:

    $array = array('name' => $name, 'title' => $title, 'status' => $status);
$this->db->where($array);
// Produces: WHERE name = 'Joe' AND title = 'boss' AND status = 'active'

    Você pode incluir seus próprios operadores usando este método também:

    $array = array('name !=' => $name, 'id <' => $id, 'date >' => $date);
$this->db->where($array);
  4. Seqüência personalizada:

    Você pode escrever suas próprias cláusulas manualmente:

    $where = "name='Joe' AND status='boss' OR status='active'";
$this->db->where($where);

$this->db->where() aceita um terceiro parâmetro opcional. Se você configurá-lo para FALSE, o CodeIgniter não tentará proteger seus nomes de campo ou tabela.

$this->db->where('MATCH (field) AGAINST ("value")', NULL, FALSE);

$this->db->or_where()

Esta função é idêntica à acima, exceto que múltiplos instâncias são unidas por OR:

$this->db->where('name !=', $name);
$this->db->or_where('id >', $id);  // Produces: WHERE name != 'Joe' OR id > 50

Nota

ou_where() era conhecido anteriormente como orwhere(), que foi removed.

$this->db->where_in()

Gera um WHERE field IN ('item', 'item') Query SQL associada a AND se appropriate

$names = array('Frank', 'Todd', 'James');
$this->db->where_in('username', $names);
// Produces: WHERE username IN ('Frank', 'Todd', 'James')

$this->db->or_where_in()

Gera um WHERE field IN ('item', 'item') Query SQL associada a OR se appropriate

$names = array('Frank', 'Todd', 'James');
$this->db->or_where_in('username', $names);
// Produces: OR username IN ('Frank', 'Todd', 'James')

$this->db->where_not_in()

Gera um WHERE field NOT IN ('item', 'item') Query SQL associada a E se apropriado

$names = array('Frank', 'Todd', 'James');
$this->db->where_not_in('username', $names);
// Produces: WHERE username NOT IN ('Frank', 'Todd', 'James')

$this->db->or_where_not_in()

Gera um WHERE field NOT IN ('item', 'item') Query SQL associada a OR se apropriado

$names = array('Frank', 'Todd', 'James');
$this->db->or_where_not_in('username', $names);
// Produces: OR username NOT IN ('Frank', 'Todd', 'James')

Procurando por dados semelhantes¶

$this->db->like()

Esse método permite gerar LIKE cláusulas úteis para fazer searches.

Nota

Todos os valores passados para este método são escapados automaticamente.

  1. Método chave/valor simples:

    $this->db->like('title', 'match');
// Produces: WHERE `title` LIKE '%match%' ESCAPE '!'

    Se você usar várias chamadas de método, elas serão encadeadas junto com E entre eles:

    $this->db->like('title', 'match');
$this->db->like('body', 'match');
// WHERE `title` LIKE '%match%' ESCAPE '!' AND  `body` LIKE '%match% ESCAPE '!'

    Se você quiser controlar onde o curinga (%) é colocado, você pode usar um terceiro argumento opcional. Suas opções são 'antes', 'depois' e 'both' (que é o padrão).

    $this->db->like('title', 'match', 'before');    // Produces: WHERE `title` LIKE '%match' ESCAPE '!'
$this->db->like('title', 'match', 'after');     // Produces: WHERE `title` LIKE 'match%' ESCAPE '!'
$this->db->like('title', 'match', 'both');      // Produces: WHERE `title` LIKE '%match%' ESCAPE '!'

  2. Método de matriz associativa:

    $array = array('title' => $match, 'page1' => $match, 'page2' => $match);
$this->db->like($array);
// WHERE `title` LIKE '%match%' ESCAPE '!' AND  `page1` LIKE '%match%' ESCAPE '!' AND  `page2` LIKE '%match%' ESCAPE '!'

$this->db->or_like()

Este método é idêntico ao acima, exceto que múltiplos instâncias são unidas por OR:

$this->db->like('title', 'match'); $this->db->or_like('body', $match);
// WHERE `title` LIKE '%match%' ESCAPE '!' OR  `body` LIKE '%match%' ESCAPE '!'

Nota

or_like() was formerly known as orlike(), which has been removed.

$this->db->not_like()

This method is identical to like(), except that it generates NÃO FAZER declarações:

$this->db->not_like('title', 'match');  // WHERE `title` NOT LIKE '%match% ESCAPE '!'

$this->db->or_not_like()

This method is identical to not_like(), except that multiple instâncias são unidas por OR:

$this->db->like('title', 'match');
$this->db->or_not_like('body', 'match');
// WHERE `title` LIKE '%match% OR  `body` NOT LIKE '%match%' ESCAPE '!'

$this->db->group_by()

Permite que você escreva a parte GROUP BY da sua Query:

$this->db->group_by("title"); // Produces: GROUP BY title

Você também pode passar uma matriz de vários valores:

$this->db->group_by(array("title", "date"));  // Produces: GROUP BY title, date

Nota

group_by() era conhecido anteriormente como groupby(), que tem sido removed.

$this->db->distinct()

Adiciona a palavra-chave "DISTINCT" a uma Query

$this->db->distinct();
$this->db->get('table'); // Produces: SELECT DISTINCT * FROM table

$this->db->having()

Permite que você escreva a parte HAVING da sua Query. Há 2 possíveis sintaxes, 1 argumento ou 2:

$this->db->having('user_id = 45');  // Produces: HAVING user_id = 45
$this->db->having('user_id',  45);  // Produces: HAVING user_id = 45

Você também pode passar uma matriz de vários valores:

$this->db->having(array('title =' => 'My Title', 'id <' => $id));
// Produces: HAVING title = 'My Title', id < 45

Se você estiver usando um Database para o qual o CodeIgniter escapa, você pode impedir o escape de conteúdo passando um terceiro argumento opcional e configurando para FALSE.

$this->db->having('user_id',  45);  // Produces: HAVING `user_id` = 45 in some databases such as MySQL
$this->db->having('user_id',  45, FALSE);  // Produces: HAVING user_id = 45

$this->db->or_having()

Idêntico a ter(), separa apenas várias cláusulas com "OR".

Ordenação de resultados¶

$this->db->order_by()

Permite que você defina uma cláusula ORDER BY.

O primeiro parâmetro contém o nome da coluna que você gostaria de pedir.

O segundo parâmetro permite definir a direção do resultado. Opções são ASC, DESC AND RANDOM.

$this->db->order_by('title', 'DESC');
// Produces: ORDER BY `title` DESC

Você também pode passar sua própria string no primeiro parâmetro:

$this->db->order_by('title DESC, name ASC');
// Produces: ORDER BY `title` DESC, `name` ASC

Ou várias chamadas de função podem ser feitas se você precisar de vários campos.

$this->db->order_by('title', 'DESC');
$this->db->order_by('name', 'ASC');
// Produces: ORDER BY `title` DESC, `name` ASC

Se você escolher o RANDOM opção de direção, os primeiros parâmetros serão ser ignorado, a menos que você especifique um valor de semente numérico.

$this->db->order_by('title', 'RANDOM');
// Produces: ORDER BY RAND()

$this->db->order_by(42, 'RANDOM');
// Produces: ORDER BY RAND(42)

Nota

order_by() era conhecido anteriormente como orderby(), que foi removed.

Nota

A ordenação aleatória não é atualmente suportada no Oracle e O padrão será ASC.

Limitando ou contando resultados¶

$this->db->limit()

Permite limitar o número de linhas que você gostaria de retornar pela Query:

$this->db->limit(10);  // Produces: LIMIT 10

O segundo parâmetro permite definir um deslocamento de resultado.

$this->db->limit(10, 20);  // Produces: LIMIT 20, 10 (in MySQL.  Other databases have slightly different syntax)

$this->db->count_all_results()

Permite que você determine o número de linhas em um determinado Active Query de registro. As Querys aceitarão os restritores do Query Builder, como where(), or_where(), like(), or_like(), etc. Exemplo:

echo $this->db->count_all_results('my_table');  // Produces an integer, like 25
$this->db->like('title', 'match');
$this->db->from('my_table');
echo $this->db->count_all_results(); // Produces an integer, like 17

No entanto, esse método também redefine todos os valores de campo que você pode ter passado to select(). If you need to keep them, you can pass FALSE as the segundo parâmetro:

echo $this->db->count_all_results('my_table', FALSE);

$this->db->count_all()

Permite que você determine o número de linhas em uma tabela específica. Envie o nome da tabela no primeiro parâmetro. Exemplo:

echo $this->db->count_all('my_table');  // Produces an integer, like 25

Agrupamento de Query¶

Agrupamento de Query allows you to create groups of WHERE clauses by enclosing them in parentheses. This will allow você criar Querys com cláusulas WHERE complexas. Grupos aninhados são suportados. Exemplo:

$this->db->select('*')->from('my_table')
        ->group_start()
                ->where('a', 'a')
                ->or_group_start()
                        ->where('b', 'b')
                        ->where('c', 'c')
                ->group_end()
        ->group_end()
        ->where('d', 'd')
->get();

// Generates:
// SELECT * FROM (`my_table`) WHERE ( `a` = 'a' OR ( `b` = 'b' AND `c` = 'c' ) ) AND `d` = 'd'

Nota

grupos precisam ser equilibrados, certifique-se de que cada group_start() seja correspondido por um group_end().

$this->db->group_start()

Inicia um novo grupo adicionando um parêntese de abertura à cláusula WHERE da Query.

$this->db->or_group_start()

Inicia um novo grupo adicionando um parêntese de abertura à cláusula WHERE da Query, prefixando-o com 'OR'.

$this->db->not_group_start()

Inicia um novo grupo adicionando um parêntese de abertura à cláusula WHERE da Query, prefixando-a com 'NOT'.

$this->db->or_not_group_start()

Inicia um novo grupo adicionando um parêntese de abertura à cláusula WHERE da Query, prefixando-a com 'OR NOT'.

$this->db->group_end()

Termina o grupo atual adicionando um parêntese de fechamento à cláusula WHERE da Query.

Inserindo Dados¶

$this->db->insert()

Gera uma sequência de inserção com base nos dados que você fornece e executa o inquerir. Você pode passar um array or an object to the função. Aqui está um exemplo usando uma matriz:

$data = array(
        'title' => 'My title',
        'name' => 'My Name',
        'date' => 'My date'
);

$this->db->insert('mytable', $data);
// Produces: INSERT INTO mytable (title, name, date) VALUES ('My title', 'My name', 'My date')

O primeiro parâmetro conterá o nome da tabela, o segundo é um matriz associativa de valores.

Aqui está um exemplo usando um objeto:

/*
class Myclass {
        public $title = 'My Title';
        public $content = 'My Content';
        public $date = 'My Date';
}
*/

$object = new Myclass;
$this->db->insert('mytable', $object);
// Produces: INSERT INTO mytable (title, content, date) VALUES ('My Title', 'My Content', 'My Date')

O primeiro parâmetro conterá o nome da tabela, o segundo é um object.

Nota

All values are escaped automatically produzindo Querys mais seguras.

$this->db->get_compiled_insert()

Compiles the insertion query just like $this->db->insert() mas não run A pergunta. Esse método simplesmente retorna a Query SQL como uma string.

Exemplo:

$data = array(
        'title' => 'My title',
        'name'  => 'My Name',
        'date'  => 'My date'
);

$sql = $this->db->set($data)->get_compiled_insert('mytable');
echo $sql;

// Produces string: INSERT INTO mytable (`title`, `name`, `date`) VALUES ('My title', 'My name', 'My date')

O segundo parâmetro permite definir se a Query do construtor de Querys é ou não will be reset (by default it will be–just like $this->db->insert()):

echo $this->db->set('title', 'My Title')->get_compiled_insert('mytable', FALSE);

// Produces string: INSERT INTO mytable (`title`) VALUES ('My Title')

echo $this->db->set('content', 'My Content')->get_compiled_insert();

// Produces string: INSERT INTO mytable (`title`, `content`) VALUES ('My Title', 'My Content')

A principal coisa a notar no exemplo acima é que a segunda Query não utilize $this->db->from() nem passou um nome de tabela para o primeiro parâmetro. A razão pela qual isso funcionou é porque a Query não foi executada using $this->db->insert() que redefine valores ou redefine diretamente using $this->db->reset_query().

Nota

Este método não funciona para inserções em lote.

$this->db->insert_batch()

Gera uma sequência de inserção com base nos dados que você fornece e executa o inquerir. Você pode passar um array or an object to the função. Aqui está um exemplo usando uma matriz:

$data = array(
        array(
                'title' => 'My title',
                'name' => 'My Name',
                'date' => 'My date'
        ),
        array(
                'title' => 'Another title',
                'name' => 'Another Name',
                'date' => 'Another date'
        )
);

$this->db->insert_batch('mytable', $data);
// Produces: INSERT INTO mytable (title, name, date) VALUES ('My title', 'My name', 'My date'),  ('Another title', 'Another name', 'Another date')

O primeiro parâmetro conterá o nome da tabela, o segundo é um matriz associativa de valores.

Nota

All values are escaped automatically produzindo Querys mais seguras.

Atualizando Dados¶

$this->db->replace()

Este método executa uma instrução REPLACE, que é basicamente o SQL standard for (optional) DELETE + INSERT, using PRIMARY and UNIQUE chaves como o fator determinante. No nosso caso, você vai economizar a necessidade de implementar logics with different combinations of select(), update(), delete() and insert() calls.

Exemplo:

$data = array(
        'title' => 'My title',
        'name'  => 'My Name',
        'date'  => 'My date'
);

$this->db->replace('table', $data);

// Executes: REPLACE INTO mytable (title, name, date) VALUES ('My title', 'My name', 'My date')

In the above example, if we assume that the title field is our primary key, then if a row containing ‘My title’ as the title value, that row será excluído com nossos novos dados de linha substituindo-os.

Usage of the set() method is also allowed and all fields are automatically escaped, just like with insert().

$this->db->set()

Esta função permite-lhe definir values for inserts or updates.

Pode ser usado em vez de passar um array de dados diretamente para a inserção ou atualizar funções:

$this->db->set('name', $name);
$this->db->insert('mytable');  // Produces: INSERT INTO mytable (`name`) VALUES ('{$name}')

Se você usar várias funções, elas serão montadas corretamente com base em se você está fazendo uma inserção ou uma atualização:

$this->db->set('name', $name);
$this->db->set('title', $title);
$this->db->set('status', $status);
$this->db->insert('mytable');

set() will also accept an optional third parameter ($escape), that impedirá que os dados sejam escapados se definidos como FALSE. Para ilustrar o difference, here is set() used both with and without the escape parameter.

$this->db->set('field', 'field+1', FALSE);
$this->db->where('id', 2);
$this->db->update('mytable'); // gives UPDATE mytable SET field = field+1 WHERE id = 2

$this->db->set('field', 'field+1');
$this->db->where('id', 2);
$this->db->update('mytable'); // gives UPDATE `mytable` SET `field` = 'field+1' WHERE `id` = 2

Você também pode passar um array associativo para esta função:

$array = array(
        'name' => $name,
        'title' => $title,
        'status' => $status
);

$this->db->set($array);
$this->db->insert('mytable');

Ou um objeto:

/*
class Myclass {
        public $title = 'My Title';
        public $content = 'My Content';
        public $date = 'My Date';
}
*/

$object = new Myclass;
$this->db->set($object);
$this->db->insert('mytable');

$this->db->update()

Gera uma string de atualização e executa a Query com base nos dados que você fornecem. Você pode passar um array or an object para a função. Aqui é um exemplo usando uma matriz:

$data = array(
        'title' => $title,
        'name' => $name,
        'date' => $date
);

$this->db->where('id', $id);
$this->db->update('mytable', $data);
// Produces:
//
//      UPDATE mytable
//      SET title = '{$title}', name = '{$name}', date = '{$date}'
//      WHERE id = $id

Ou você pode fornecer um objeto:

/*
class Myclass {
        public $title = 'My Title';
        public $content = 'My Content';
        public $date = 'My Date';
}
*/

$object = new Myclass;
$this->db->where('id', $id);
$this->db->update('mytable', $object);
// Produces:
//
// UPDATE `mytable`
// SET `title` = '{$title}', `name` = '{$name}', `date` = '{$date}'
// WHERE id = `$id`

Nota

All values are escaped automatically produzindo Querys mais seguras.

You’ll notice the use of the $this->db->where() function, enabling you para definir a cláusula WHERE. Você pode, opcionalmente, passar essas informações diretamente na função de atualização como uma string:

$this->db->update('mytable', $data, "id = 4");

Ou como uma matriz:

$this->db->update('mytable', $data, array('id' => $id));

You may also use the $this->db->set() function described above when executando atualizações.

$this->db->update_batch()

Gera uma cadeia de atualização com base nos dados que você fornece e executa a Query. Você pode passar um array or an object para a função. Here é um exemplo usando uma matriz:

$data = array(
   array(
      'title' => 'My title' ,
      'name' => 'My Name 2' ,
      'date' => 'My date 2'
   ),
   array(
      'title' => 'Another title' ,
      'name' => 'Another Name 2' ,
      'date' => 'Another date 2'
   )
);

$this->db->update_batch('mytable', $data, 'title');

// Produces:
// UPDATE `mytable` SET `name` = CASE
// WHEN `title` = 'My title' THEN 'My Name 2'
// WHEN `title` = 'Another title' THEN 'Another Name 2'
// ELSE `name` END,
// `date` = CASE
// WHEN `title` = 'My title' THEN 'My date 2'
// WHEN `title` = 'Another title' THEN 'Another date 2'
// ELSE `date` END
// WHERE `title` IN ('My title','Another title')

O primeiro parâmetro conterá o nome da tabela, o segundo é um associative matriz de valores, o terceiro parâmetro é a chave where.

Nota

All values are escaped automatically produzindo Querys mais seguras.

Nota

affected_rows() não lhe dará resultados adequados com este método, devido à própria natureza de como funciona. Em vez de, update_batch() retorna o número de linhas afetadas.

Excluindo dados¶

$this->db->delete()

Gera uma string SQL de exclusão e executa a Query.

$this->db->delete('mytable', array('id' => $id));  // Produces: // DELETE FROM mytable  // WHERE id = $id

O primeiro parâmetro é o nome da tabela, o segundo é a cláusula where. Você também pode usar as funções where() ou or_where() ao invés de passar os dados para o segundo parâmetro da função:

$this->db->where('id', $id);
$this->db->delete('mytable');

// Produces:
// DELETE FROM mytable
// WHERE id = $id

Uma matriz de nomes de tabelas pode ser passada para delete() se você quiser Exclua dados de mais de uma tabela.

$tables = array('table1', 'table2', 'table3');
$this->db->where('id', '5');
$this->db->delete($tables);

Se você quiser apagar todos os dados de uma tabela, você pode usar o truncate() função, ou empty_table().

$this->db->empty_table()

Gera uma string SQL de exclusão e executa o query.:

$this->db->empty_table('mytable'); // Produces: DELETE FROM mytable

$this->db->truncate()

Gera uma string SQL truncada e executa a Query.

$this->db->from('mytable');
$this->db->truncate();

// or

$this->db->truncate('mytable');

// Produce:
// TRUNCATE mytable

Nota

Se o comando TRUNCATE não estiver disponível, truncate() será executar como "DELETE FROM table".

$query = $this->db->select('title')

Query Builder Cache¶

Embora não seja cache "true", o Query Builder permite salvar (ou "cache") certas partes de suas Querys para reutilização em um ponto posterior em sua execução do script. Normalmente, quando uma chamada do Construtor de Querys é concluída, todas as informações armazenadas são redefinidas para a próxima chamada. Com o cache, você pode Impeça essa redefinição e reutilize as informações facilmente.

Chamadas em cache são cumulativas. Se você fizer 2 chamadas select() em cache e em seguida, 2 chamadas select() sem cache, isso resultará em 4 chamadas select(). There are three Cache functions available:

$this->db->start_cache()

Essa função deve ser chamada para iniciar o armazenamento em cache. Todas as Querys do Query Builder do tipo correto (veja abaixo as Querys suportadas) são armazenadas para later use.

$this->db->stop_cache()

Essa função pode ser chamada para parar o armazenamento em cache.

$this->db->flush_cache()

Essa função exclui todos os itens do cache do Query Builder.

Um exemplo de armazenamento em cache¶

Aqui está um exemplo de uso:

$this->db->start_cache();
$this->db->select('field1');
$this->db->stop_cache();
$this->db->get('tablename');
//Generates: SELECT `field1` FROM (`tablename`)

$this->db->select('field2');
$this->db->get('tablename');
//Generates:  SELECT `field1`, `field2` FROM (`tablename`)

$this->db->flush_cache();
$this->db->select('field2');
$this->db->get('tablename');
//Generates:  SELECT `field2` FROM (`tablename`)

Nota

As seguintes instruções podem ser armazenadas em cache: select, from, join, onde, como, group_by, tendo, order_by

Redefinindo o Construtor de Querys¶

$this->db->reset_query()

Redefinindo o Construtor de Querys allows you to start fresh with your query without executing it first using a method like $this->db->get() or $this->db->insert(). Just like the methods that execute a query, this will not reset items you’ve em cache usando Query Builder Cache.

Isso é útil em situações em que você está usando o Query Builder para gerar SQL (ex. $this->db->get_compiled_select()) but then choose to, for instance, execute a Query:

// Nota that the second parameter of the get_compiled_select method is FALSE
$sql = $this->db->select(array('field1','field2'))
                                ->where('field3',5)
                                ->get_compiled_select('mytable', FALSE);

// ...
// Do something crazy with the SQL code... like add it to a cron script for
// later execution or something...
// ...

$data = $this->db->get()->result_array();

// Would execute and return an array of results of the following query:
// SELECT field1, field1 from mytable where field3 = 5;

Nota

Double calls to get_compiled_select() while you’re using the Query Builder Cache functionality and NOT resetting your queries resultará no cache sendo mesclado duas vezes. Isso, por sua vez, i.e. if you’re caching a select() - select the same field twice.
