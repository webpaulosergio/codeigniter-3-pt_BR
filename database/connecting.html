Conectando-se ao seu database¶

Existem duas maneiras de se conectar a um database:

Automatically Connecting¶

O recurso “auto connect” irá carregar e instanciar a classe do Database with every page load. To enable “auto connecting”, add the word database to the library array, as indicated in the following file:

application/config/autoload.php

Manually Connecting¶

Se apenas algumas de suas páginas exigirem conectividade de Database, manually connect to your database by adding this line of code in any function where it is needed, or in your class constructor to make the database available globally in that class.

$this->load->database();

If the above function does not contain any information in the first parâmetro irá se conectar ao grupo especificado na configuração do seu database Arquivo. Para a maioria das pessoas, esse é o método preferido de uso.

Parâmetros Disponíveis¶

  1. The database connection values, passed either as an array or a DSN string.
  2. TRUE/FALSE (boolean). Whether to return the connection ID (see Conectando-se a vários databases abaixo).
  3. TRUE/FALSE (boolean). Whether to enable the Query Builder class. Set para TRUE por padrão.

Manually Conectando-se a um database¶

The first parameter of this function can optionally ser usado para especificar um determinado grupo de Database do seu arquivo de configuração, ou você pode even submit connection values for a database that is not specified in seu arquivo de configuração. Exemplos:

To choose a specific group from your config file you can do this:

$this->load->database('group_name');

Onde group_name é o nome do grupo de conexão da sua configuração file.

To connect manually to a desired database you can pass an array of values:

$config['hostname'] = 'localhost';
$config['username'] = 'myusername';
$config['password'] = 'mypassword';
$config['database'] = 'mydatabase';
$config['dbdriver'] = 'mysqli';
$config['dbprefix'] = '';
$config['pconnect'] = FALSE;
$config['db_debug'] = TRUE;
$config['cache_on'] = FALSE;
$config['cachedir'] = '';
$config['char_set'] = 'utf8';
$config['dbcollat'] = 'utf8_general_ci';
$this->load->database($config);

Para obter informações sobre cada um desses valores, consulte o configuration page.

Nota

For the PDO driver, you should use the $config[‘dsn’] setting em vez de 'hostname' e 'database':


$config ['dsn'] = 'mysql: host = localhost; nome_bd = mydatabase';

Ou você pode enviar seus valores de Database como um Nome da fonte de dados. Os DSNs devem have this prototype:

$dsn = 'dbdriver://username:password@hostname/database';
$this->load->database($dsn);

To override default config values when connecting with a DSN string, add the config variables as a query string.

$dsn = 'dbdriver://username:password@hostname/database?char_set=utf8&dbcollat=utf8_general_ci&cache_on=true&cachedir=/path/to/cache';
$this->load->database($dsn);

Conectando-se a vários databases¶

If you need to connect to more than one database simultaneously you can do so as follows:

$DB1 = $this->load->database('group_one', TRUE);
$DB2 = $this->load->database('group_two', TRUE);

Nota: Mude as palavras “group_one” e “group_two” para o específico nomes de grupo aos quais você está se conectando (ou pode passar os valores de conexão as indicated above).

By setting the second parameter to TRUE (boolean) the function will retornar o objeto do database.

Nota

When you connect this way, you will use your object name to issue comandos em vez da sintaxe usada em todo este guia. Em outro words, rather than issuing commands with:


$this->db->query();
$this->db->result();
etc…

Você irá usar:

$DB1->query();
$DB1->result();
etc…

Nota

You don’t need to create separate database configurations if you só precisa usar um Database diferente na mesma conexão. Você pode mudar para um database diferente quando você precisar, assim:

$this->db->db_select($database2_name);

Reconnecting / Keeping the Connection Alive¶

If the database server’s idle timeout is exceeded while you’re doing algum levantamento pesado de PHP (processamento de uma imagem, por exemplo), você deve considere fazer ping no servidor usando o método reconnect() antes sending further queries, which can gracefully keep the connection alive or re-establish it.

$this->db->reconnect();

Manually closing the Connection¶

While CodeIgniter intelligently takes care of closing your database connections, you can explicitly close the connection.

$this->db->close();

Conectando-se ao seu database¶

Existem duas maneiras de se conectar a um database:

Conectando automaticamente¶

O recurso “auto connect” irá carregar e instanciar a classe do Database com cada carregamento de página. Para ativar a "conexão automática", adicione o Database de palavras para o array da Library, conforme indicado no seguinte arquivo:

application/config/autoload.php

Conectando Manualmente¶

Se apenas algumas de suas páginas exigirem conectividade de Database, conectar-se manualmente ao seu Database adicionando essa linha de código em qualquer função onde é necessário, ou no seu construtor de classe para fazer o Database disponível globalmente nessa classe.

$this->load->database();

Se a função acima não not contém qualquer informação no primeiro parâmetro irá se conectar ao grupo especificado na configuração do seu database Arquivo. Para a maioria das pessoas, esse é o método preferido de uso.

Parâmetros Disponíveis¶

  1. Os valores de conexão do Database, transmitidos como uma matriz ou um DSN string.
  2. TRUE/FALSE (booleano). Se deve retornar o ID de conexão (consulte Conectando-se a vários databases abaixo).
  3. TRUE/FALSE (booleano). Se deseja ativar a classe Query Builder. Conjunto para TRUE por padrão.

Manually Conectando-se a um database¶

O primeiro parâmetro desta função pode optionally ser usado para especificar um determinado grupo de Database do seu arquivo de configuração, ou você pode até mesmo enviar valores de conexão para um Database que não esteja especificado em seu arquivo de configuração. Exemplos:

Para escolher um grupo específico do seu arquivo de configuração, você pode fazer isso:

$this->load->database('group_name');

Onde group_name é o nome do grupo de conexão da sua configuração file.

Para se conectar manualmente a um Database desejado, você pode passar uma matriz de values:

$config['hostname'] = 'localhost';
$config['username'] = 'myusername';
$config['password'] = 'mypassword';
$config['database'] = 'mydatabase';
$config['dbdriver'] = 'mysqli';
$config['dbprefix'] = '';
$config['pconnect'] = FALSE;
$config['db_debug'] = TRUE;
$config['cache_on'] = FALSE;
$config['cachedir'] = '';
$config['char_set'] = 'utf8';
$config['dbcollat'] = 'utf8_general_ci';
$this->load->database($config);

Para obter informações sobre cada um desses valores, consulte o configuration page.

Nota

Para o driver PDO, você deve usar a configuração $config ['dsn'] em vez de 'hostname' e 'database':


$config ['dsn'] = 'mysql: host = localhost; nome_bd = mydatabase';

Ou você pode enviar seus valores de Database como um Nome da fonte de dados. Os DSNs devem tem esse protótipo:

$dsn = 'dbdriver://username:password@hostname/database';
$this->load->database($dsn);

Para substituir os valores de configuração padrão ao conectar-se a uma string DSN, adicione the config variables as a query string.

$dsn = 'dbdriver://username:password@hostname/database?char_set=utf8&dbcollat=utf8_general_ci&cache_on=true&cachedir=/path/to/cache';
$this->load->database($dsn);

Conectando-se a vários databases¶

Se você precisar se conectar a mais de um Database simultaneamente, faça da seguinte maneira:

$DB1 = $this->load->database('group_one', TRUE);
$DB2 = $this->load->database('group_two', TRUE);

Nota: Mude as palavras “group_one” e “group_two” para o específico nomes de grupo aos quais você está se conectando (ou pode passar os valores de conexão como indicado acima).

Ao definir o segundo parâmetro como TRUE (booleano), a função retornar o objeto do database.

Nota

Quando você se conectar dessa maneira, usará o nome do objeto para emitir comandos em vez da sintaxe usada em todo este guia. Em outro palavras, em vez de emitir comandos com:


$this->db->query();
$this->db->result();
etc…

Você irá usar:

$DB1->query();
$DB1->result();
etc…

Nota

Você não precisa criar configurações de Database separadas se só precisa usar um Database diferente na mesma conexão. Você pode mudar para um database diferente quando você precisar, assim:

$this->db->db_select($database2_name);

Reconectando/mantendo a conexão viva¶

Se o tempo limite ocioso do servidor de Database for excedido enquanto você estiver fazendo algum levantamento pesado de PHP (processamento de uma imagem, por exemplo), você deve considere fazer ping no servidor usando o método reconnect() antes enviando mais Querys, que podem graciosamente manter a conexão viva ou restabelecer.

$this->db->reconnect();

Fechando manualmente a conexão¶

Enquanto o CodeIgniter cuida inteligentemente de fechar seu Database conexões, você pode fechar explicitamente a conexão.

$this->db->close();

Conectando-se ao seu database¶

Existem duas maneiras de se conectar a um database:

Conectando automaticamente¶

O recurso "auto connect" irá carregar e instanciar a classe do Database com cada carregamento de página. Para ativar a "conexão automática", adicione o Database de palavras para o array da Library, conforme indicado no seguinte arquivo:

application/config/autoload.php

Conectando Manualmente¶

Se apenas algumas de suas páginas exigirem conectividade de Database, conectar-se manualmente ao seu Database adicionando essa linha de código em qualquer função onde é necessário, ou no seu construtor de classe para fazer o Database disponível globalmente nessa classe.

$this->load->database();

Se a função acima não not contém qualquer informação no primeiro parâmetro irá se conectar ao grupo especificado na configuração do seu database Arquivo. Para a maioria das pessoas, esse é o método preferido de uso.

Parâmetros Disponíveis¶

  1. Os valores de conexão do Database, transmitidos como uma matriz ou um DSN string.
  2. TRUE/FALSE (booleano). Se deve retornar o ID de conexão (consulte Conectando-se a vários databases abaixo).
  3. TRUE/FALSE (booleano). Se deseja ativar a classe Query Builder. Conjunto para TRUE por padrão.

Manually Conectando-se a um database¶

O primeiro parâmetro desta função pode optionally be used to especificar um determinado grupo de Database do seu arquivo de configuração, ou você pode até mesmo enviar valores de conexão para um Database que não esteja especificado em seu arquivo de configuração. Exemplos:

Para escolher um grupo específico do seu arquivo de configuração, você pode fazer isso:

$this->load->database('group_name');

Onde group_name é o nome do grupo de conexão da sua configuração file.

Para se conectar manualmente a um Database desejado, você pode passar uma matriz de values:

$config['hostname'] = 'localhost';
$config['username'] = 'myusername';
$config['password'] = 'mypassword';
$config['database'] = 'mydatabase';
$config['dbdriver'] = 'mysqli';
$config['dbprefix'] = '';
$config['pconnect'] = FALSE;
$config['db_debug'] = TRUE;
$config['cache_on'] = FALSE;
$config['cachedir'] = '';
$config['char_set'] = 'utf8';
$config['dbcollat'] = 'utf8_general_ci';
$this->load->database($config);

Para obter informações sobre cada um desses valores, consulte o configuration page.

Nota

Para o driver PDO, você deve usar a configuração $config ['dsn'] em vez de 'hostname' e 'database':


$config ['dsn'] = 'mysql: host = localhost; nome_bd = mydatabase';

Ou você pode enviar seus valores de Database como um Nome da fonte de dados. Os DSNs devem tem esse protótipo:

$dsn = 'dbdriver://username:password@hostname/database';
$this->load->database($dsn);

Para substituir os valores de configuração padrão ao conectar-se a uma string DSN, adicione as variáveis de configuração como uma string de Query.

$dsn = 'dbdriver://username:password@hostname/database?char_set=utf8&dbcollat=utf8_general_ci&cache_on=true&cachedir=/path/to/cache';
$this->load->database($dsn);

Conectando-se a vários databases¶

Se você precisar se conectar a mais de um Database simultaneamente, faça da seguinte maneira:

$DB1 = $this->load->database('group_one', TRUE);
$DB2 = $this->load->database('group_two', TRUE);

Nota: Mude as palavras "group_one" e "group_two" para o específico nomes de grupo aos quais você está se conectando (ou pode passar os valores de conexão como indicado acima).

Ao definir o segundo parâmetro como TRUE (booleano), a função retornar o objeto do database.

Nota

Quando você se conectar dessa maneira, usará o nome do objeto para emitir comandos em vez da sintaxe usada em todo este guia. Em outro palavras, em vez de emitir comandos com:


$this->db->query();
$this->db->result();
etc…

Você irá usar:

$DB1->query();
$DB1->result();
etc…

Nota

Você não precisa criar configurações de Database separadas se só precisa usar um Database diferente na mesma conexão. Você pode mudar para um database diferente quando você precisar, assim:

$this->db->db_select($database2_name);

Reconectando/mantendo a conexão viva¶

Se o tempo limite ocioso do servidor de Database for excedido enquanto você estiver fazendo algum levantamento pesado de PHP (processamento de uma imagem, por exemplo), você deve considere fazer ping no servidor usando o método reconnect() antes enviando mais Querys, que podem graciosamente manter a conexão viva ou restabelecer.

$this->db->reconnect();

Fechando manualmente a conexão¶

Enquanto o CodeIgniter cuida inteligentemente de fechar seu Database conexões, você pode fechar explicitamente a conexão.

$this->db->close();
