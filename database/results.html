Gerando Resultados da Consulta¶

There are several ways to generate query results:

  • Result Arrays
  • Result Rows
  • Objetos de resultado personalizados
  • Métodos auxiliares de resultados
  • Class Reference

Result Arrays¶

result()

Esse método retorna o resultado da consulta como uma matriz de objects, or an empty array no fracasso. Normalmente você vai usar isso em um foreach loop, like this:

$query = $this->db->query("YOUR QUERY");

foreach ($query->result() as $row)
{
        echo $row->title;
        echo $row->name;
        echo $row->body;
}

The above method is an alias of result_object().

You can also pass a string to result() which represents a class to instantiate for each result object (note: this class must be loaded)

$query = $this->db->query("SELECT * FROM users;");

foreach ($query->result('User') as $user)
{
        echo $user->name; // access attributes
        echo $user->reverse_name(); // or methods defined on the 'User' class
}

result_array()

This method returns the query result as a pure array, or an empty array when no result is produced. Typically you’ll use this in a foreach loop, like this:

$query = $this->db->query("YOUR QUERY");

foreach ($query->result_array() as $row)
{
        echo $row['title'];
        echo $row['name'];
        echo $row['body'];
}

Result Rows¶

row()

This method returns a single result row. If your query has more than one row, it returns only the first row. The result is returned as an object. Here’s a usage example:

$query = $this->db->query("YOUR QUERY");

$row = $query->row();

if (isset($row))
{
        echo $row->title;
        echo $row->name;
        echo $row->body;
}

Se você quiser uma linha específica retornada, você pode enviar o número da linha como um digit in the first parameter:

$row = $query->row(5);

Você também pode adicionar um segundo parâmetro String, que é o nome de uma classe to instantiate the row with:

$query = $this->db->query("SELECT * FROM users LIMIT 1;");
$row = $query->row(0, 'User');

echo $row->name; // access attributes
echo $row->reverse_name(); // or methods defined on the 'User' class

row_array()

Identical to the above row() method, except it returns an array. Example:

$query = $this->db->query("YOUR QUERY");

$row = $query->row_array();

if (isset($row))
{
        echo $row['title'];
        echo $row['name'];
        echo $row['body'];
}

Se você quiser uma linha específica retornada, você pode enviar o número da linha como um digit in the first parameter:

$row = $query->row_array(5);

In addition, you can walk forward/backwards/first/last through your results using these variations:

$row = $query->first_row()
$row = $query->last_row()
$row = $query->next_row()
$row = $query->previous_row()

By default they return an object unless you put the word “array” in the parameter:

$row = $query->first_row(‘array’)
$row = $query->last_row(‘array’)
$row = $query->next_row(‘array’)
$row = $query->previous_row(‘array’)

Nota

Todos os métodos acima irão carregar todo o resultado na memória (prefetching). Use unbuffered_row() for processing large conjuntos de resultados.

unbuffered_row()

Esse método retorna uma única linha de resultado sem pré-buscar o todo result in memory as row() does. If your query has more than one row, it returns the current row and moves the internal data pointer ahead.

$query = $this->db->query("YOUR QUERY");

while ($row = $query->unbuffered_row())
{
        echo $row->title;
        echo $row->name;
        echo $row->body;
}

You can optionally pass ‘object’ (default) or ‘array’ in order to specify o tipo do valor retornado:

$query->unbuffered_row();               // object
$query->unbuffered_row('object');       // object
$query->unbuffered_row('array');        // associative array

Objetos de resultado personalizados¶

You can have the results returned as an instance of a custom class instead of a stdClass or array, as the result() and result_array() métodos permitem. Isso requer que a classe já esteja carregada na memória. The object will have all values returned from the database set as properties. Se estes foram declarados e não são públicos, então você deve fornecer um __set() method to allow them to be set.

Example:

class User {

        public $id;
        public $email;
        public $username;

        protected $last_login;

        public function last_login($format)
        {
                return $this->last_login->format($format);
        }

        public function __set($name, $value)
        {
                if ($name === 'last_login')
                {
                        $this->last_login = DateTime::createFromFormat('U', $value);
                }
        }

        public function __get($name)
        {
                if (isset($this->$name))
                {
                        return $this->$name;
                }
        }
}

In addition to the two methods listed below, the following methods also can take a class name to return the results as: first_row(), last_row(), next_row(), and previous_row().

custom_result_object()

Returns the entire result set as an array of instances of the class requested. O único parâmetro é o nome da classe para instanciar.

Example:

$query = $this->db->query("YOUR QUERY");

$rows = $query->custom_result_object('User');

foreach ($rows as $row)
{
        echo $row->id;
        echo $row->email;
        echo $row->last_login('Y-m-d');
}

custom_row_object()

Returns a single row from your query results. The first parameter is the row number of the results. The second parameter is the class name to instantiate.

Example:

$query = $this->db->query("YOUR QUERY");

$row = $query->custom_row_object(0, 'User');

if (isset($row))
{
        echo $row->email;   // access attributes
        echo $row->last_login('Y-m-d');   // access class methods
}

You can also use the row() method in exactly the same way.

Example:

$row = $query->custom_row_object(0, 'User');

Métodos auxiliares de resultados¶

num_rows()

O número de linhas retornadas pela consulta. Nota: Neste exemplo, $query é a variável à qual o objeto de resultado da consulta é atribuído:

$query = $this->db->query('SELECT * FROM my_table');

echo $query->num_rows();

Nota

Not all database drivers have a native way of getting the total número de linhas para um conjunto de resultados. Quando este é o caso, todos the data is prefetched and count() is manually called on the matriz resultante para obter o mesmo resultado.

num_fields()

The number of FIELDS (columns) returned by the query. Make sure to call o método usando seu objeto de resultado da consulta:

$query = $this->db->query('SELECT * FROM my_table');

echo $query->num_fields();

free_result()

It frees the memory associated with the result and deletes the result resource ID. Normally PHP frees its memory automatically at the end of script execution. However, if you are running a lot of queries in a particular script you might want to free the result after each query result has been generated in order to cut down on memory consumption.

Example:

$query = $this->db->query('SELECT title FROM my_table');

foreach ($query->result() as $row)
{
        echo $row->title;
}

$query->free_result();  // The $query result object will no longer be available

$query2 = $this->db->query('SELECT name FROM some_table');

$row = $query2->row();
echo $row->name;
$query2->free_result(); // The $query2 result object will no longer be available

data_seek()

This method sets the internal pointer for the next result row to be fetched. It is only useful in combination with unbuffered_row().

It accepts a positive integer value, which defaults to 0 and returns TRUE on success or FALSE on failure.

$query = $this->db->query('SELECT `field_name` FROM `table_name`');
$query->data_seek(5); // Skip the first 5 rows
$row = $query->unbuffered_row();

Nota

Nem todos os drivers de database suportam esse recurso e retornarão FALSE. Mais notavelmente - você não poderá usá-lo com o PDO.

Gerando Resultados da Query¶

Existem várias maneiras de gerar resultados de Query:

  • Matrizes de resultados
  • Linhas de Resultado
  • Objetos de resultado personalizados
  • Métodos Helperes de resultados
  • Referência de Classe

Matrizes de resultados¶

result()

Esse método retorna o resultado da Query como uma matriz de objects, or uma matriz vazia no fracasso. Normalmente você vai usar isso em um foreach loop, assim:

$query = $this->db->query("YOUR QUERY");

foreach ($query->result() as $row)
{
        echo $row->title;
        echo $row->name;
        echo $row->body;
}

The above method is an alias of result_object().

You can also pass a string to result() which represents a class to instanciar para cada objeto de resultado (nota: esta classe deve ser carregada)

$query = $this->db->query("SELECT * FROM users;");

foreach ($query->result('User') as $user)
{
        echo $user->name; // access attributes
        echo $user->reverse_name(); // or methods defined on the 'User' class
}

result_array()

Esse método retorna o resultado da Query como uma matriz pura ou um vazio array quando nenhum resultado é produzido. Normalmente você vai usar isso em um foreach loop, assim:

$query = $this->db->query("YOUR QUERY");

foreach ($query->result_array() as $row)
{
        echo $row['title'];
        echo $row['name'];
        echo $row['body'];
}

Linhas de Resultado¶

row()

Este método retorna uma única linha de resultado. Se a sua Query tiver mais de uma linha, retorna apenas a primeira linha. O resultado é retornado como um object. Aqui está um exemplo de uso:

$query = $this->db->query("YOUR QUERY");

$row = $query->row();

if (isset($row))
{
        echo $row->title;
        echo $row->name;
        echo $row->body;
}

Se você quiser uma linha específica retornada, você pode enviar o número da linha como um dígito no primeiro parâmetro:

$row = $query->row(5);

Você também pode adicionar um segundo parâmetro String, que é o nome de uma classe para instanciar a linha com:

$query = $this->db->query("SELECT * FROM users LIMIT 1;");
$row = $query->row(0, 'User');

echo $row->name; // access attributes
echo $row->reverse_name(); // or methods defined on the 'User' class

row_array()

Identical to the above row() method, except it returns an array. Exemplo:

$query = $this->db->query("YOUR QUERY");

$row = $query->row_array();

if (isset($row))
{
        echo $row['title'];
        echo $row['name'];
        echo $row['body'];
}

Se você quiser uma linha específica retornada, você pode enviar o número da linha como um dígito no primeiro parâmetro:

$row = $query->row_array(5);

Além disso, você pode andar para frente/para trás/primeiro/último através do seu resultados usando estas variações:

$row = $query->first_row()
$row = $query->last_row()
$row = $query->next_row()
$row = $query->previous_row()

Por padrão, eles retornam um objeto, a menos que você coloque a palavra “array” no parameter:

$row = $query->first_row(‘array’)
$row = $query->last_row(‘array’)
$row = $query->next_row(‘array’)
$row = $query->previous_row(‘array’)

Nota

Todos os métodos acima irão carregar todo o resultado na memória (prefetching). Use unbuffered_row() for processing large conjuntos de resultados.

unbuffered_row()

Esse método retorna uma única linha de resultado sem pré-buscar o todo result in memory as row() does. If your query has more than one row, ele retorna a linha atual e move o ponteiro de dados interno adiante.

$query = $this->db->query("YOUR QUERY");

while ($row = $query->unbuffered_row())
{
        echo $row->title;
        echo $row->name;
        echo $row->body;
}

Você pode, opcionalmente, passar 'objeto' (padrão) ou 'matriz' para especificar o tipo do valor retornado:

$query->unbuffered_row();               // object
$query->unbuffered_row('object');       // object
$query->unbuffered_row('array');        // associative array

Objetos de resultado personalizados¶

Você pode ter os resultados retornados como uma instância de uma classe personalizada of a stdClass or array, as the result() and result_array() métodos permitem. Isso requer que a classe já esteja carregada na memória. O objeto terá todos os valores retornados do conjunto de Databases como propriedades. Se estes foram declarados e não são públicos, então você deve fornecer um __set() método para permitir que eles sejam definidos.

Exemplo:

class User {

        public $id;
        public $email;
        public $username;

        protected $last_login;

        public function last_login($format)
        {
                return $this->last_login->format($format);
        }

        public function __set($name, $value)
        {
                if ($name === 'last_login')
                {
                        $this->last_login = DateTime::createFromFormat('U', $value);
                }
        }

        public function __get($name)
        {
                if (isset($this->$name))
                {
                        return $this->$name;
                }
        }
}

Além dos dois métodos listados abaixo, os seguintes métodos também take a class name to return the results as: first_row(), last_row(), next_row(), and previous_row().

custom_result_object()

Retorna todo o conjunto de resultados como uma matriz de instâncias da classe solicitada. O único parâmetro é o nome da classe para instanciar.

Exemplo:

$query = $this->db->query("YOUR QUERY");

$rows = $query->custom_result_object('User');

foreach ($rows as $row)
{
        echo $row->id;
        echo $row->email;
        echo $row->last_login('Y-m-d');
}

custom_row_object()

Retorna uma única linha dos resultados da sua Query. O primeiro parâmetro é a linha número dos resultados. O segundo parâmetro é o nome da classe para instanciar.

Exemplo:

$query = $this->db->query("YOUR QUERY");

$row = $query->custom_row_object(0, 'User');

if (isset($row))
{
        echo $row->email;   // access attributes
        echo $row->last_login('Y-m-d');   // access class methods
}

You can also use the row() method in exactly the same way.

Exemplo:

$row = $query->custom_row_object(0, 'User');

Métodos Helperes de resultados¶

num_rows()

O número de linhas retornadas pela Query. Nota: Neste exemplo, $query é a variável à qual o objeto de resultado da Query é atribuído:

$query = $this->db->query('SELECT * FROM my_table');

echo $query->num_rows();

Nota

Nem todos os drivers de Database têm uma maneira nativa de obter o total número de linhas para um conjunto de resultados. Quando este é o caso, todos the data is prefetched and count() is manually called on the matriz resultante para obter o mesmo resultado.

num_fields()

O número de FIELDS (colunas) retornados pela Query. Certifique-se de ligar o método usando seu objeto de resultado da Query:

$query = $this->db->query('SELECT * FROM my_table');

echo $query->num_fields();

free_result()

Liberta a memória associada ao resultado e apaga o resultado ID do recurso. Normalmente PHP libera sua memória automaticamente no final do execução de script. No entanto, se você estiver executando muitas Querys em um determinado script que você pode querer libertar o resultado depois de cada Query resultado foi gerado a fim de reduzir o consumo de memória.

Exemplo:

$query = $this->db->query('SELECT title FROM my_table');

foreach ($query->result() as $row)
{
        echo $row->title;
}

$query->free_result();  // The $query result object will no longer be available

$query2 = $this->db->query('SELECT name FROM some_table');

$row = $query2->row();
echo $row->name;
$query2->free_result(); // The $query2 result object will no longer be available

data_seek()

Este método define o ponteiro interno para a próxima linha de resultado a ser fetched. It is only useful in combination with unbuffered_row().

Ele aceita um valor inteiro positivo, cujo padrão é 0 e retorna Verdadeiro em sucesso ou FALSE em falha.

$query = $this->db->query('SELECT `field_name` FROM `table_name`');
$query->data_seek(5); // Skip the first 5 rows
$row = $query->unbuffered_row();

Nota

Nem todos os drivers de database suportam esse recurso e retornarão FALSE. Mais notavelmente - você não poderá usá-lo com o PDO.

Existem várias maneiras de gerar resultados de Query:

Matrizes de resultados¶

result()

Esse método retorna o resultado da Query como uma matriz de objects, or uma matriz vazia no fracasso. Normalmente você vai usar isso em um foreach loop, assim:

$query = $this->db->query("YOUR QUERY");

{

        echo $row->name;
        echo $row->body;
}

$query = $this->db->query("SELECT * FROM users;");

foreach ($query->result('User') as $user)
{
        echo $user->name; // access attributes
        echo $user->reverse_name(); // or methods defined on the 'User' class
}

result_array()

Esse método retorna o resultado da Query como uma matriz pura ou um vazio array quando nenhum resultado é produzido. Normalmente você vai usar isso em um foreach loop, assim:

$query = $this->db->query("YOUR QUERY");

foreach ($query->result_array() as $row)
{
        echo $row['title'];
        echo $row['name'];
        echo $row['body'];
}

Linhas de Resultado¶

row()

Este método retorna uma única linha de resultado. Se a sua Query tiver mais de uma linha, retorna apenas a primeira linha. O resultado é retornado como um object. Aqui está um exemplo de uso:

$query = $this->db->query("YOUR QUERY");

$row = $query->row();

if (isset($row))
{
        echo $row->title;
        echo $row->name;
        echo $row->body;
}

Se você quiser uma linha específica retornada, você pode enviar o número da linha como um dígito no primeiro parâmetro:

$row = $query->row(5);

Você também pode adicionar um segundo parâmetro String, que é o nome de uma classe para instanciar a linha com:

$query = $this->db->query("SELECT * FROM users LIMIT 1;");
$row = $query->row(0, 'User');

echo $row->name; // access attributes
echo $row->reverse_name(); // or methods defined on the 'User' class

row_array()

Identical to the above row() method, except it returns an array. Exemplo:

$query = $this->db->query("YOUR QUERY");

$row = $query->row_array();

if (isset($row))
{
        echo $row['title'];
        echo $row['name'];
        echo $row['body'];
}

Se você quiser uma linha específica retornada, você pode enviar o número da linha como um dígito no primeiro parâmetro:

$row = $query->row_array(5);

Além disso, você pode andar para frente/para trás/primeiro/último através do seu resultados usando estas variações:

$row = $query->first_row()
$row = $query->last_row()
$row = $query->next_row()
$row = $query->previous_row()

Por padrão, eles retornam um objeto, a menos que você coloque a palavra "array" no parameter:

$row = $query->first_row(‘array’)
$row = $query->last_row(‘array’)
$row = $query->next_row(‘array’)
$row = $query->previous_row(‘array’)

Nota

Todos os métodos acima irão carregar todo o resultado na memória (prefetching). Use unbuffered_row() for processing large conjuntos de resultados.

unbuffered_row()

Esse método retorna uma única linha de resultado sem pré-buscar o todo result in memory as row() does. If your query has more than one row, ele retorna a linha atual e move o ponteiro de dados interno adiante.

$query = $this->db->query("YOUR QUERY");

while ($row = $query->unbuffered_row())
{
        echo $row->title;
        echo $row->name;
        echo $row->body;
}

Você pode, opcionalmente, passar 'objeto' (padrão) ou 'matriz' para especificar o tipo do valor retornado:

$query->unbuffered_row();               // object
$query->unbuffered_row('object');       // object
$query->unbuffered_row('array');        // associative array

Objetos de resultado personalizados¶

Você pode ter os resultados retornados como uma instância de uma classe personalizada of a stdClass or array, as the result() and result_array() métodos permitem. Isso requer que a classe já esteja carregada na memória. O objeto terá todos os valores retornados do conjunto de Databases como propriedades. Se estes foram declarados e não são públicos, então você deve fornecer um __set() método para permitir que eles sejam definidos.

Exemplo:

class User {

        public $id;
        public $email;
        public $username;

        protected $last_login;

        public function last_login($format)
        {
                return $this->last_login->format($format);
        }

        public function __set($name, $value)
        {
                if ($name === 'last_login')
                {
                        $this->last_login = DateTime::createFromFormat('U', $value);
                }
        }

        public function __get($name)
        {
                if (isset($this->$name))
                {
                        return $this->$name;
                }
        }
}

Além dos dois métodos listados abaixo, os seguintes métodos também take a class name to return the results as: first_row(), last_row(), next_row(), and previous_row().

custom_result_object()

Retorna todo o conjunto de resultados como uma matriz de instâncias da classe solicitada. O único parâmetro é o nome da classe para instanciar.

Exemplo:

$query = $this->db->query("YOUR QUERY");

$rows = $query->custom_result_object('User');

foreach ($rows as $row)
{
        echo $row->id;
        echo $row->email;
        echo $row->last_login('Y-m-d');
}

custom_row_object()

Retorna uma única linha dos resultados da sua Query. O primeiro parâmetro é a linha número dos resultados. O segundo parâmetro é o nome da classe para instanciar.

Exemplo:

$query = $this->db->query("YOUR QUERY");

$row = $query->custom_row_object(0, 'User');

if (isset($row))
{
        echo $row->email;   // access attributes
        echo $row->last_login('Y-m-d');   // access class methods
}

You can also use the row() method in exactly the same way.

Exemplo:

$row = $query->custom_row_object(0, 'User');

Métodos Helperes de resultados¶

num_rows()

O número de linhas retornadas pela Query. Nota: Neste exemplo, $query é a variável à qual o objeto de resultado da Query é atribuído:

$query = $this->db->query('SELECT * FROM my_table');

echo $query->num_rows();

Nota

Nem todos os drivers de Database têm uma maneira nativa de obter o total número de linhas para um conjunto de resultados. Quando este é o caso, todos the data is prefetched and count() is manually called on the matriz resultante para obter o mesmo resultado.

num_fields()

O número de FIELDS (colunas) retornados pela Query. Certifique-se de ligar o método usando seu objeto de resultado da Query:

$query = $this->db->query('SELECT * FROM my_table');

echo $query->num_fields();

free_result()

Liberta a memória associada ao resultado e apaga o resultado ID do recurso. Normalmente PHP libera sua memória automaticamente no final do execução de script. No entanto, se você estiver executando muitas Querys em um determinado script que você pode querer libertar o resultado depois de cada Query resultado foi gerado a fim de reduzir o consumo de memória.

Exemplo:

$query = $this->db->query('SELECT title FROM my_table');

foreach ($query->result() as $row)
{
        echo $row->title;
}

$query->free_result();  // The $query result object will no longer be available

$query2 = $this->db->query('SELECT name FROM some_table');

$row = $query2->row();
echo $row->name;
$query2->free_result(); // The $query2 result object will no longer be available

data_seek()

Este método define o ponteiro interno para a próxima linha de resultado a ser fetched. It is only useful in combination with unbuffered_row().

Ele aceita um valor inteiro positivo, cujo padrão é 0 e retorna Verdadeiro em sucesso ou FALSE em falha.

$query = $this->db->query('SELECT `field_name` FROM `table_name`');
$query->data_seek(5); // Skip the first 5 rows
$row = $query->unbuffered_row();

Nota

Nem todos os drivers de database suportam esse recurso e retornarão FALSE. Mais notavelmente - você não poderá usá-lo com o PDO.
