Transações¶

CodeIgniter’s database abstraction allows you to use transactions with databases that support transaction-safe table types. In MySQL, you’ll need to be running InnoDB or BDB table types rather than the more common MyISAM. Most other database platforms support transactions natively.

If you are not familiar with transactions we recommend you find a good recurso on-line para aprender sobre eles para seu database específico. o information below assumes you have a basic understanding of transactions.

CodeIgniter’s Approach to Transações¶

CodeIgniter utilizes an approach to transactions that is very similar to o processo usado pela popular classe de database ADODB. Nós escolhemos isso abordagem porque simplifica muito o processo de execução transações. Na maioria dos casos, tudo o que é necessário são duas linhas de código.

Traditionally, transactions have required a fair amount of work to implement since they demand that you keep track of your queries and determine whether to commit or rollback based on the success or failure de suas consultas. Isso é particularmente complicado com consultas aninhadas. Em Em contraste, implementamos um sistema de transação inteligente que faz tudo isso para você automaticamente (você também pode gerenciar suas transações manually if you choose to, but there’s really no benefit).

Running Transações¶

To run your queries using transactions you will use the $this->db->trans_start() and $this->db->trans_complete() functions as follows:

$this->db->trans_start();
$this->db->query('AN SQL QUERY...');
$this->db->query('ANÃOHER QUERY...');
$this->db->query('AND YET ANÃOHER QUERY...');
$this->db->trans_complete();

Você pode executar quantas consultas quiser entre o início/conclusão funções e todos eles serão cometidos ou revertidos com base no sucesso ou falha de qualquer consulta dada.

modo estrito¶

By default CodeIgniter runs all transactions in modo estrito. When strict mode is enabled, if you are running multiple groups of transactions, if um grupo falha, todos os grupos serão revertidos. Se o modo estrito é desativado, cada grupo é tratado de forma independente, ou seja, uma falha de um grupo não afetará os outros.

modo estrito can be disabled as follows:

$this->db->trans_strict(FALSE);

Managing Errors¶

If you have error reporting enabled in your config/database.php file você verá uma mensagem de erro padrão se a confirmação não for bem sucedida. E se depuração está desativada, você pode gerenciar seus próprios erros como este:

$this->db->trans_start();
$this->db->query('AN SQL QUERY...');
$this->db->query('ANÃOHER QUERY...');
$this->db->trans_complete();

if ($this->db->trans_status() === FALSE)
{
        // generate an error... or use the log_message() function to log your error
}

Disabling Transações¶

If you would like to disable transactions you can do so using $this->db->trans_off():

$this->db->trans_off();

$this->db->trans_start();
$this->db->query('AN SQL QUERY...');
$this->db->trans_complete();

Quando as transações são desativadas, suas consultas serão confirmadas automaticamente, assim como they are when running queries without transactions, practically ignoring any calls to trans_start(), trans_complete(), etc.

Test Mode¶

Opcionalmente, você pode colocar o sistema de transação no "modo de teste", que will cause your queries to be rolled back – even if the queries produce a valid result. To use test mode simply set the first parameter in the $this->db->trans_start() function to TRUE:

$this->db->trans_start(TRUE); // Query will be rolled back
$this->db->query('AN SQL QUERY...');
$this->db->trans_complete();

Running Transações Manually¶

If you would like to run transactions manually you can do so as follows:

$this->db->trans_begin();

$this->db->query('AN SQL QUERY...');
$this->db->query('ANÃOHER QUERY...');
$this->db->query('AND YET ANÃOHER QUERY...');

if ($this->db->trans_status() === FALSE)
{
        $this->db->trans_rollback();
}
else
{
        $this->db->trans_commit();
}

Nota

Make sure to use $this->db->trans_begin() when running manual transactions, NÃO $this->db->trans_start().

Transações¶

A abstração de Database do CodeIgniter permite que você use transações com Databases que suportam tipos de tabela seguros para transações. No MySQL, você vai precisa estar executando tipos de tabelas InnoDB ou BDB, em vez do mais comum MyISAM A maioria das outras plataformas de Database suporta transações nativamente.

Se você não está familiarizado com transações, recomendamos que você encontre uma boa recurso on-line para aprender sobre eles para seu database específico. o A informação abaixo assume que você tem uma compreensão básica de transactions.

CodeIgniter’s Approach to Transações¶

O CodeIgniter utiliza uma abordagem para transações que é muito semelhante a o processo usado pela popular classe de database ADODB. Nós escolhemos isso abordagem porque simplifica muito o processo de execução transações. Na maioria dos casos, tudo o que é necessário são duas linhas de código.

Tradicionalmente, as transações exigiam uma quantidade razoável de trabalho para implementar desde que eles exigem que você mantenha o controle de suas Querys e determinar se deve confirmar ou reverter com base no sucesso ou fracasso de suas Querys. Isso é particularmente complicado com Querys aninhadas. Em Em contraste, implementamos um sistema de transação inteligente que faz tudo isso para você automaticamente (você também pode gerenciar suas transações manualmente se você escolher, mas não há realmente nenhum benefício).

Running Transações¶

Para executar suas Querys usando transações, você usará $this->db->trans_start() and $this->db->trans_complete() functions as follows:

$this->db->trans_start();
$this->db->query('AN SQL QUERY...');
$this->db->query('ANOTHER QUERY...');
$this->db->query('AND YET ANOTHER QUERY...');
$this->db->trans_complete();

Você pode executar quantas Querys quiser entre o início/conclusão funções e todos eles serão cometidos ou revertidos com base no sucesso ou falha de qualquer Query dada.

modo estrito¶

By default CodeIgniter runs all transactions in modo estrito. When strict O modo está ativado, se você estiver executando vários grupos de transações, se um grupo falha, todos os grupos serão revertidos. Se o modo estrito é desativado, cada grupo é tratado de forma independente, ou seja, uma falha de um grupo não afetará os outros.

modo estrito can be disabled as follows:

$this->db->trans_strict(FALSE);

Gerenciando Erros¶

Se você tem relatórios de erros habilitados em seu arquivo config/database.php você verá uma mensagem de erro padrão se a confirmação não for bem sucedida. E se depuração está desativada, você pode gerenciar seus próprios erros como este:

$this->db->trans_start();
$this->db->query('AN SQL QUERY...');
$this->db->query('ANOTHER QUERY...');
$this->db->trans_complete();

if ($this->db->trans_status() === FALSE)
{
        // generate an error... or use the log_message() function to log your error
}

Disabling Transações¶

Se você gostaria de desabilitar as transações, pode fazê-lo usando $this->db->trans_off():

$this->db->trans_off();

$this->db->trans_start();
$this->db->query('AN SQL QUERY...');
$this->db->trans_complete();

Quando as transações são desativadas, suas Querys serão confirmadas automaticamente, assim como eles são quando executando Querys sem transações, praticamente ignorando any calls to trans_start(), trans_complete(), etc.

Modo de teste¶

Opcionalmente, você pode colocar o sistema de transação no "modo de teste", que fará com que suas Querys sejam revertidas - mesmo que as Querys produzam um resultado válido. Para usar o modo de teste, basta definir o primeiro parâmetro no $this->db->trans_start() function to TRUE:

$this->db->trans_start(TRUE); // Query will be rolled back
$this->db->query('AN SQL QUERY...');
$this->db->trans_complete();

Running Transações Manually¶

Se você gostaria de executar transações manualmente, você pode fazer da seguinte maneira:

$this->db->trans_begin();

$this->db->query('AN SQL QUERY...');
$this->db->query('ANOTHER QUERY...');
$this->db->query('AND YET ANOTHER QUERY...');

if ($this->db->trans_status() === FALSE)
{
        $this->db->trans_rollback();
}
else
{
        $this->db->trans_commit();
}

Nota

Make sure to use $this->db->trans_begin() when running manual transactions, NOT $this->db->trans_start().

Transações¶

A abstração de Database do CodeIgniter permite que você use transações com Databases que suportam tipos de tabela seguros para transações. No MySQL, você vai precisa estar executando tipos de tabelas InnoDB ou BDB, em vez do mais comum MyISAM A maioria das outras plataformas de Database suporta transações nativamente.

Se você não está familiarizado com transações, recomendamos que você encontre uma boa recurso on-line para aprender sobre eles para seu database específico. o A informação abaixo assume que você tem uma compreensão básica de transactions.

CodeIgniter’s Approach to Transações¶

O CodeIgniter utiliza uma abordagem para transações que é muito semelhante a o processo usado pela popular classe de database ADODB. Nós escolhemos isso abordagem porque simplifica muito o processo de execução transações. Na maioria dos casos, tudo o que é necessário são duas linhas de código.

Tradicionalmente, as transações exigiam uma quantidade razoável de trabalho para implementar desde que eles exigem que você mantenha o controle de suas Querys e determinar se deve confirmar ou reverter com base no sucesso ou fracasso de suas Querys. Isso é particularmente complicado com Querys aninhadas. Em Em contraste, implementamos um sistema de transação inteligente que faz tudo isso para você automaticamente (você também pode gerenciar suas transações manualmente se você escolher, mas não há realmente nenhum benefício).

Running Transações¶

Para executar suas Querys usando transações, você usará $this->db->trans_start() and $this->db->trans_complete() functions as follows:

$this->db->trans_start();
$this->db->query('AN SQL QUERY...');
$this->db->query('ANOTHER QUERY...');
$this->db->query('AND YET ANOTHER QUERY...');
$this->db->trans_complete();

Você pode executar quantas Querys quiser entre o início/conclusão funções e todos eles serão cometidos ou revertidos com base no sucesso ou falha de qualquer Query dada.

modo estrito¶

By default CodeIgniter runs all transactions in modo estrito. When strict O modo está ativado, se você estiver executando vários grupos de transações, se um grupo falha, todos os grupos serão revertidos. Se o modo estrito é desativado, cada grupo é tratado de forma independente, ou seja, uma falha de um grupo não afetará os outros.

modo estrito can be disabled as follows:

$this->db->trans_strict(FALSE);

Gerenciando Erros¶

Se você tem relatórios de erros habilitados em seu arquivo config/database.php você verá uma mensagem de erro padrão se a confirmação não for bem sucedida. E se depuração está desativada, você pode gerenciar seus próprios erros como este:

$this->db->trans_start();
$this->db->query('AN SQL QUERY...');
$this->db->query('ANOTHER QUERY...');
$this->db->trans_complete();

if ($this->db->trans_status() === FALSE)
{
        // generate an error... or use the log_message() function to log your error
}

Disabling Transações¶

Se você gostaria de desabilitar as transações, pode fazê-lo usando $this->db->trans_off():

$this->db->trans_off();

$this->db->trans_start();
$this->db->query('AN SQL QUERY...');
$this->db->trans_complete();

Quando as transações são desativadas, suas Querys serão confirmadas automaticamente, assim como eles são quando executando Querys sem transações, praticamente ignorando any calls to trans_start(), trans_complete(), etc.

Test Mode¶

Opcionalmente, você pode colocar o sistema de transação no "modo de teste", que fará com que suas Querys sejam revertidas - mesmo que as Querys produzam um resultado válido. Para usar o modo de teste, basta definir o primeiro parâmetro no $this->db->trans_start() function to TRUE:

$this->db->trans_start(TRUE); // Query will be rolled back
$this->db->query('AN SQL QUERY...');
$this->db->trans_complete();

Running Transações Manually¶

Se você gostaria de executar transações manualmente, você pode fazer da seguinte maneira:

$this->db->trans_begin();

$this->db->query('AN SQL QUERY...');
$this->db->query('ANOTHER QUERY...');
$this->db->query('AND YET ANOTHER QUERY...');

if ($this->db->trans_status() === FALSE)
{
        $this->db->trans_rollback();
}
else
{
        $this->db->trans_commit();
}

Nota

Make sure to use $this->db->trans_begin() when running manual transactions, NOT $this->db->trans_start().
