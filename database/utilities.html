Classe de Utilitário de Database¶

The Classe de Utilitário de Database contains methods that help you manage your database.

Initializing the Utility Class¶

Importante

In order to initialize the Utility class, your database driver must already be running, since the utilities class relies on it.

Carregue a Classe de Utilitário da seguinte maneira:

$this->load->dbutil();

Você também pode passar outro objeto de database para o carregador DB Utility, caso the database you want to manage isn’t the default one:

$this->myutil = $this->load->dbutil($this->other_db, TRUE);

In the above example, we’re passing a custom database object as the first parameter and then tell it to return the dbutil object, instead of assigning it directly to $this->dbutil.

Nota

Both of the parameters can be used individually, just pass an empty value as the first one if you wish to skip it.

Uma vez inicializado, você acessará os métodos usando o $this->dbutil object:

$this->dbutil->some_method();

Using the Database Utilities¶

Retrieve list of database names¶

Returns an array of database names:

$dbs = $this->dbutil->list_databases();

foreach ($dbs as $db)
{
        echo $db;
}

Determine If a Database Exists¶

Às vezes é útil saber se existe um database específico. Returns a boolean TRUE FALSE. Usage example:

if ($this->dbutil->database_exists('database_name'))
{
        // some code...
}

Nota

Replace database_name with the name of the database you are procurando por. Este método faz distinção entre maiúsculas e minúsculas.

Optimize a Table¶

Permits you to optimize a table using the table name specified in the first parameter. Returns TRUE FALSE based on success or failure:

if ($this->dbutil->optimize_table('table_name'))
{
        echo 'Success!';
}

Nota

Not all database platforms support table optimization. It is mostly for use with MySQL.

Reparar uma tabela¶

Permits you to repair a table using the table name specified in the first parameter. Returns TRUE FALSE based on success or failure:

if ($this->dbutil->repair_table('table_name'))
{
        echo 'Success!';
}

Nota

Nem todas as plataformas de database suportam reparos de tabelas.

Otimizar um database¶

Permite que você otimize o database que sua classe DB está atualmente connected to. Returns an array containing the DB status messages or FALSE em falha.

$result = $this->dbutil->optimize_database();

if ($result !== FALSE)
{
        print_r($result);
}

Nota

Not all database platforms support database optimization. It é principalmente para uso com o MySQL.

Export a Query Result as a CSV File¶

Permits you to generate a CSV file from a query result. The first parâmetro do método deve conter o objeto de resultado do seu inquerir. Exemplo:

$this->load->dbutil();

$query = $this->db->query("SELECT * FROM mytable");

echo $this->dbutil->csv_from_result($query);

The second, third, and fourth parameters allow you to set the delimiter newline, and enclosure characters respectively. By default commas are usado como delimitador, “n” é usado como uma nova linha e um aspas duplas é usado como o gabinete. Exemplo:

$delimiter = ",";
$newline = "\r\n";
$enclosure = '"';

echo $this->dbutil->csv_from_result($query, $delimiter, $newline, $enclosure);

Importante

Esse método NÃO gravará o arquivo CSV para você. isto simplesmente cria o layout CSV. Se você precisa escrever o arquivo use o Arquivo auxiliar.

Export a Query Result as an XML Document¶

Permits you to generate an XML file from a query result. The first parâmetro espera um objeto de resultado de consulta, o segundo pode conter um optional array of config parameters. Example:

$this->load->dbutil();

$query = $this->db->query("SELECT * FROM mytable");

$config = array (
        'root'          => 'root',
        'element'       => 'element',
        'newline'       => "\n",
        'tab'           => "\t"
);

echo $this->dbutil->xml_from_result($query, $config);

Importante

Este método NÃO gravará o arquivo XML para você. isto simply creates the XML layout. If you need to write the file use o Arquivo auxiliar.

Backup Your Database¶

Database Backup Notas¶

Permits you to backup your full database or individual tables. The backup data can be compressed in either Zip or Gzip format.

Nota

This feature is only available for MySQL and Interbase/Firebird databases.

Nota

Para databases Interbase/Firebird, o nome do arquivo de backup é o único parâmetro.

$this->dbutil->backup(‘db_backup_filename’);

Nota

Due to the limited execution time and memory available to PHP, O backup de databases muito grandes pode não ser possível. Se o seu database é very large you might need to backup directly from your SQL server via a linha de comando, ou faça com que o administrador do seu servidor faça isso por você, se você não have root privileges.

Exemplo de uso¶

// Load the DB utility class
$this->load->dbutil();

// Backup your entire database and assign it to a variable
$backup = $this->dbutil->backup();

// Load the file helper and write the file to your server
$this->load->helper('file');
write_file('/path/to/mybackup.gz', $backup);

// Load the download helper and send the file to your desktop
$this->load->helper('download');
force_download('mybackup.gz', $backup);

Setting Backup Preferências¶

As preferências de backup são definidas enviando uma matriz de valores para o primeiro parameter of the backup() method. Example:

$prefs = array(
        'tables'        => array('table1', 'table2'),   // Array of tables to backup.
        'ignore'        => array(),                     // List of tables to omit from the backup
        'format'        => 'txt',                       // gzip, zip, txt
        'filename'      => 'mybackup.sql',              // File name - NEEDED ONLY WITH ZIP FILES
        'add_drop'      => TRUE,                        // Whether to add DROP TABLE statements to backup file
        'add_insert'    => TRUE,                        // Whether to add INSERT data to backup file
        'newline'       => "\n"                         // Newline character used in backup file
);

$this->dbutil->backup($prefs);

Descrição of Backup Preferências¶

Preferência Default Value Opções Descrição
tables empty array None An array of tables you want backed up. If left blank all tables will be exported.
ignore empty array None Uma matriz de tabelas que você deseja que a rotina de backup ignore.
format gzip gzip, zip, txt The file format of the export file.
filename a data/hora atual None O nome do arquivo de backup. O nome é necessário apenas se você estiver usando compactação zip.
add_drop TRUE TRUE FALSE Whether to include DROP TABLE statements in your SQL export file.
add_insert TRUE TRUE FALSE Se incluir instruções INSERT no seu arquivo de exportação SQL.
newline “\n” "\ N", "\ r", "\ r \ n" Tipo de nova linha para usar no seu arquivo de exportação do SQL.
foreign_key_checks TRUE TRUE FALSE Whether output should keep foreign key checks enabled.

Inicializando a classe de utilitário¶

Importante

Para inicializar a classe Utility, seu Database O driver já deve estar em execução, já que a classe de utilitários depende dele.

Carregue a Classe de Utilitário da seguinte maneira:

$this->load->dbutil();

Você também pode passar outro objeto de database para o carregador DB Utility, caso o Database que você deseja gerenciar não é o padrão:

$this->myutil = $this->load->dbutil($this->other_db, TRUE);

No exemplo acima, estamos passando um objeto de Database personalizado como o primeiro parâmetro e, em seguida, diz-lhe para retornar o objeto dbutil, em vez de assigning it directly to $this->dbutil.

Nota

Both of the parameters can be used individually, just pass an empty valor como o primeiro se você quiser pular.

Uma vez inicializado, você acessará os métodos usando o $this->dbutil object:

$this->dbutil->some_method();

Usando os utilitários de Database¶

Recuperar lista de nomes de Databases¶

Retorna uma matriz de nomes de Database:

$dbs = $this->dbutil->list_databases();

foreach ($dbs as $db)
{
        echo $db;
}

Determinar se existe um Database¶

Às vezes é útil saber se existe um database específico. Retorna um booleano TRUE FALSE. Exemplo de uso:

if ($this->dbutil->database_exists('database_name'))
{
        // some code...
}

Nota

Replace database_name with the name of the database you are procurando por. Este método faz distinção entre maiúsculas e minúsculas.

Otimizar uma tabela¶

Permite que você otimize uma tabela usando o nome da tabela especificado no primeiro parâmetro. Retorna TRUE FALSE com base no sucesso ou falha:

if ($this->dbutil->optimize_table('table_name'))
{
        echo 'Success!';
}

Nota

Nem todas as plataformas de Database suportam otimização de tabelas. Isto é principalmente para uso com o MySQL.

Reparar uma tabela¶

Permite que você conserte uma tabela usando o nome da tabela especificado no primeiro parâmetro. Retorna TRUE FALSE com base no sucesso ou falha:

if ($this->dbutil->repair_table('table_name'))
{
        echo 'Success!';
}

Nota

Nem todas as plataformas de database suportam reparos de tabelas.

Otimizar um database¶

Permite que você otimize o database que sua classe DB está atualmente conectado a. Retorna uma matriz contendo as mensagens de status do DB ou FALSE em falha.

$result = $this->dbutil->optimize_database();

if ($result !== FALSE)
{
        print_r($result);
}

Nota

Nem todas as plataformas de Database suportam otimização de Database. isto it is principalmente para uso com o MySQL.

Exportar um resultado da Query como um arquivo CSV¶

Permite que você gere um arquivo CSV a partir de um resultado de Query. O primeiro parâmetro do método deve conter o objeto de resultado do seu inquerir. Exemplo:

$this->load->dbutil();

$query = $this->db->query("SELECT * FROM mytable");

echo $this->dbutil->csv_from_result($query);

O segundo, terceiro e quarto parâmetros permitem que você defina o delimitador nova linha e caracteres de caixa respectivamente. Por padrão, as vírgulas são usado como delimitador, “n” é usado como uma nova linha e um aspas duplas é usado como o gabinete. Exemplo:

$delimiter = ",";
$newline = "\r\n";
$enclosure = '"';

echo $this->dbutil->csv_from_result($query, $delimiter, $newline, $enclosure);

Importante

Esse método NÃO gravará o arquivo CSV para você. isto simplesmente cria o layout CSV. Se você precisa escrever o arquivo use o Arquivo Helper.

Exportar um resultado da Query como um documento XML¶

Permite que você gere um arquivo XML a partir de um resultado de Query. O primeiro parâmetro espera um objeto de resultado de Query, o segundo pode conter um matriz opcional de parâmetros de configuração. Exemplo:

$this->load->dbutil();

$query = $this->db->query("SELECT * FROM mytable");

$config = array (
        'root'          => 'root',
        'element'       => 'element',
        'newline'       => "\n",
        'tab'           => "\t"
);

echo $this->dbutil->xml_from_result($query, $config);

Importante

Este método NÃO gravará o arquivo XML para você. isto simplesmente cria o layout XML. Se você precisa escrever o arquivo use o Arquivo Helper.

Backup de seu Database¶

Notas de backup do Database¶

Permite que você faça backup de seu Database completo ou de tabelas individuais. o os dados de backup podem ser compactados no formato Zip ou Gzip.

Nota

Este recurso está disponível apenas para Databases MySQL e Interbase/Firebird.

Nota

Para databases Interbase/Firebird, o nome do arquivo de backup é o único parâmetro.

$this->dbutil->backup(‘db_backup_filename’);

Nota

Devido ao tempo de execução limitado e memória disponível para PHP, O backup de databases muito grandes pode não ser possível. Se o seu database é muito grande você pode precisar fazer backup diretamente do seu servidor SQL via a linha de comando, ou faça com que o administrador do seu servidor faça isso por você, se você não tem privilégios de root.

Exemplo de uso¶

// Load the DB utility class
$this->load->dbutil();

// Backup your entire database and assign it to a variable
$backup = $this->dbutil->backup();

// Load the file helper and write the file to your server
$this->load->helper('file');
write_file('/path/to/mybackup.gz', $backup);

// Load the download helper and send the file to your desktop
$this->load->helper('download');
force_download('mybackup.gz', $backup);

Configurando Preferências de Backup¶

As preferências de backup são definidas enviando uma matriz de valores para o primeiro parameter of the backup() method. Example:

$prefs = array(
        'tables'        => array('table1', 'table2'),   // Array of tables to backup.
        'ignore'        => array(),                     // List of tables to omit from the backup
        'format'        => 'txt',                       // gzip, zip, txt
        'filename'      => 'mybackup.sql',              // File name - NEEDED ONLY WITH ZIP FILES
        'add_drop'      => TRUE,                        // Whether to add DROP TABLE statements to backup file
        'add_insert'    => TRUE,                        // Whether to add INSERT data to backup file
        'newline'       => "\n"                         // Newline character used in backup file
);

$this->dbutil->backup($prefs);

Descrição das Preferências de Backup¶

Preferência Valor padrão Opções Descrição
tables matriz vazia None Uma matriz de tabelas que você deseja fazer backup. Se deixado em branco, todas as tabelas serão exported.
ignore matriz vazia None Uma matriz de tabelas que você deseja que a rotina de backup ignore.
format gzip gzip, zip, txt O formato de arquivo do arquivo de exportação.
filename a data/hora atual None O nome do arquivo de backup. O nome é necessário apenas se você estiver usando compactação zip.
add_drop TRUE TRUE FALSE Se deve incluir instruções DROP TABLE no seu arquivo de exportação SQL.
add_insert TRUE TRUE FALSE Se incluir instruções INSERT no seu arquivo de exportação SQL.
newline “\n” "\ N", "\ r", "\ r \ n" Tipo de nova linha para usar no seu arquivo de exportação do SQL.
foreign_key_checks TRUE TRUE FALSE Se a saída deve manter as verificações de chave estrangeira ativadas.

Inicializando a classe de utilitário¶

Important

Para inicializar a classe Utility, seu Database O driver já deve estar em execução, já que a classe de utilitários depende dele.

Carregue a Classe de Utilitário da seguinte maneira:

$this->load->dbutil();

Você também pode passar outro objeto de database para o carregador DB Utility, caso o Database que você deseja gerenciar não é o padrão:

$this->myutil = $this->load->dbutil($this->other_db, TRUE);

No exemplo acima, estamos passando um objeto de Database personalizado como o primeiro parâmetro e, em seguida, diz-lhe para retornar o objeto dbutil, em vez de assigning it directly to $this->dbutil.

Nota

Ambos os parâmetros podem ser usados individualmente, basta passar um vazio valor como o primeiro se você quiser pular.

Uma vez inicializado, você acessará os métodos usando o $this->dbutil object:

$this->dbutil->some_method();
