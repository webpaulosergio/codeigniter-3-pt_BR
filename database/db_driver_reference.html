DB Driver Reference¶

This is the platform-independent base DB implementation class. This class will not be called directly. Rather, the adapter classe para o Database específico irá estender e instanciá-lo.

The how-to material for this has been split over several articles. Este artigo pretende ser uma referência para eles.

Importante

Nem todos os métodos são suportados por todos os drivers de database, alguns deles podem falhar (e retornar FALSE) se o driver does not support them.

class CI_DB_driver¶
initialize()¶
Returns:Verdadeiro sucesso, FALSE em falha
Return type:bool

Initialize database settings, establish a connection to the database.

db_connect($persistent = TRUE)¶
Parâmetros:
  • $persistent (bool) - Se estabelecer uma conexão persistente ou regular
Returns:

Recurso/objeto de conexão do database ou FALSE na falha
Return type:

mixed

Establish a connection with the database.

Nota

O valor retornado depende do subjacente driver in use. For example, a mysqli instance will be returned with the ‘mysqli’ driver.

db_pconnect()¶
Returns:Recurso/objeto de conexão do database ou FALSE na falha
Return type:mixed

Estabelecer uma conexão persistente com o database.

Nota

This method is just an alias for db_connect(TRUE).

reconnect()¶
Returns:Verdadeiro sucesso, FALSE em falha
Return type:bool

Manter/restabelecer a conexão com o database se não houver consultas have been sent for a length of time exceeding the server’s idle timeout.

db_select([$database = ''])¶
Parâmetros:
  • $database (string) - Nome do database
Returns:

Verdadeiro sucesso, FALSE em falha
Return type:

bool

Select / switch the current database.

db_set_charset($charset)¶
Parâmetros:
  • $charset (string) - nome do conjunto de caracteres
Returns:

Verdadeiro sucesso, FALSE em falha
Return type:

bool

Set client character set.

platform()¶
Returns:Platform name
Return type:string

O nome da plataforma em uso (mysql, mssql, etc…).

version()¶
Returns:The version of the database being used
Return type:string

Database version number.

query($sql[, $binds = FALSE[, $return_object = NULL]])¶
Parâmetros:
  • $sql (string) - A instrução SQL para executar
  • $binds (array) – An array of binding data
  • $return_object (bool) – Whether to return a result object or not
Returns:

TRUE for successful “write-type” queries, CI_DB_result instance (method chaining) on “query” success, FALSE on failure
Return type:

mixed

Execute an SQL query.

Aceita uma string SQL como entrada e retorna um objeto de resultado upon successful execution of a “read” type query.

Returns:

  • Boolean TRUE upon successful execution of a “write type” queries
  • Booleano FALSE em caso de falha
  • CI_DB_result object for “read type” queries
simple_query($sql)¶
Parâmetros:
  • $sql (string) - A instrução SQL para executar
Returns:

Whatever the underlying driver’s “query” function returns
Return type:

mixed

A simplified version of the query() method, appropriate for use when you don’t need to get a result object or to basta enviar uma consulta para o database e não se importar com o resultado.

affected_rows()¶
Returns:Número de linhas afetadas
Return type:int

Returns the number of rows changed by the last executed query.

Useful for checking how much rows were created, updated or deleted during the last executed query.

trans_strict([$mode = TRUE])¶
Parâmetros:
  • $mode (bool) - Sinalizador de modo estrito
Return type:

void

Enable/disable transaction “strict” mode.

When strict mode is enabled, if you are running multiple groups of transactions and one group fails, all subsequent grupos serão revertidos.

If strict mode is disabled, each group is treated autonomously, meaning a failure of one group will not affect any others.

trans_off()¶
Return type:void

Disables transactions at run-time.

trans_start([$test_mode = FALSE])¶
Parâmetros:
  • $test_mode (bool) – Test mode flag
Returns:

Verdadeiro sucesso, FALSE em falha
Return type:

bool

Start a transaction.

trans_complete()¶
Returns:Verdadeiro sucesso, FALSE em falha
Return type:bool

Transação Completa.

trans_status()¶
Returns:TRUE if the transaction succeeded, FALSE if it failed
Return type:bool

Lets you retrieve the transaction status flag to determinar se falhou.

compile_binds($sql, $binds)¶
Parâmetros:
  • $sql (string) – The SQL statement
  • $binds (array) – An array of binding data
Returns:

A instrução SQL atualizada
Return type:

string

Compiles an SQL query with the bind values passed for it.

is_write_type($sql)¶
Parâmetros:
  • $sql (string) – The SQL statement
Returns:

TRUE if the SQL statement is of “write type”, FALSE if not
Return type:

bool

Determines if a query is of a “write” type (such as INSERT, UPDATE, DELETE) or “read” type (i.e. SELECT).

elapsed_time([$decimals = 6])¶
Parâmetros:
  • $decimals (int) - O número de casas decimais
Returns:

The aggregate query elapsed time, in microseconds
Return type:

string

Calculate the aggregate query elapsed time.

total_queries()¶
Returns:The total number of queries executed
Return type:int

Retorna o número total de consultas que foram executed so far.

last_query()¶
Returns:A última consulta executada
Return type:string

Returns the last query that was executed.

escape($str)¶
Parâmetros:
  • $str (mixed) – The value to escape, or an array of multiple ones
Returns:

The escaped value(s)
Return type:

mixed

Escapes input data based on type, including boolean and NULLs

escape_str($str[, $like = FALSE])¶
Parâmetros:
  • $str (mixed) – A string value or array of multiple ones
  • $like (bool) - Se a string será usada ou não em uma condição LIKE
Returns:

A string (s) de escape
Return type:

mixed

Escapes string values.

Aviso

The returned strings do NOT include quotes around them.

escape_like_str($str)¶
Parâmetros:
  • $str (mixed) – A string value or array of multiple ones
Returns:

A string (s) de escape
Return type:

mixed

Escape LIKE strings.

Similar to escape_str(), but will also escape the % and _ wildcard characters, so that they don’t cause falsos positivos em condições LIKE.

Importante

The escape_like_str() method uses ‘!’ (exclamation mark) to escape special characters for LIKE conditions. Because this method escapes partial strings that you would wrap in quotes você mesmo, ele não pode adicionar automaticamente ESCAPE '!' condição para você, e então você terá que fazer isso manualmente.

primary($table)¶
Parâmetros:
  • $table (string) - Nome da tabela
Returns:

O nome da chave primária, FALSE se nenhum
Return type:

string

Retrieves the primary key of a table.

Nota

If the database platform does not support primary key detection, the first column name may be assumed as the primary key.

count_all([$table = ''])¶
Parâmetros:
  • $table (string) - Nome da tabela
Returns:

Row count for the specified table
Return type:

int

Returns the total number of rows in a table, or 0 if no table was provided.

list_tables([$constrain_by_prefix = FALSE])¶
Parâmetros:
  • $constrain_by_prefix (bool) - TRUE para corresponder nomes de tabelas pelo dbprefix configurado
Returns:

Matriz de nomes de tabela ou FALSE em falha
Return type:

array

Obtém uma lista das tabelas no database atual.

table_exists($table_name)¶
Parâmetros:
  • $table_name (string) - O nome da tabela
Returns:

TRUE if that table exists, FALSE if not
Return type:

bool

Determine se uma tabela específica existe.

list_fields($table)¶

  • $table (string) - O nome da tabela
Returns:

Matriz de nomes de campos ou FALSE em falha
Return type:

array

Gets a list of the field names in a table.

field_exists($field_name, $table_name)¶
Parâmetros:
  • $table_name (string) - O nome da tabela
  • $field_name (string) - O nome do campo
Returns:

TRUE if that field exists in that table, FALSE if not
Return type:

bool

Determine se um determinado campo existe.

field_data($table)¶

  • $table (string) - O nome da tabela

Matriz de itens de dados de campo ou FALSE em falha
Return type:

array

Gets a list containing field data about a table.

Parâmetros:

O item de entrada (s), escapou
Return type:

mixed

Escape SQL identifiers, such as column, table and names.

insert_string($table, $data)¶
Parâmetros:

  • $data (array) - Uma matriz associativa de pares chave/valor
Returns:

The SQL INSERT statement, as a string
Return type:

string

Gere uma string de instrução INSERT.

update_string($table, $data, $where)¶
Parâmetros:
  • $table (string) – The target table
  • $data (array) - Uma matriz associativa de pares chave/valor
  • $where (mixed) – The WHERE statement conditions
Returns:

The SQL UPDATE statement, as a string
Return type:

string

Generate an UPDATE statement string.

call_function($function)¶
Parâmetros:
  • $function (string) – Function name
Returns:

The function result
Return type:

string

Executa uma função PHP nativa, usando uma plataforma agnóstica wrapper.

cache_set_path([$path = ''])¶
Parâmetros:
  • $path (string) – Path to the cache directory
Return type:

void

Sets the directory path to use for caching storage.

cache_on()¶
Returns:TRUE se o cache estiver ativado, FALSE se não estiver
Return type:bool

Enable database results caching.

cache_off()¶
Returns:TRUE se o cache estiver ativado, FALSE se não estiver
Return type:bool

Disable database results caching.

cache_delete([$segment_one = ''[, $segment_two = '']])¶
Parâmetros:
  • $segment_one (string) – First URI segment
  • $segment_two (string) – Second URI segment
Returns:

Verdadeiro sucesso, FALSE em falha
Return type:

bool

Exclua os arquivos de cache associados a um URI específico.

cache_delete_all()¶
Returns:Verdadeiro sucesso, FALSE em falha
Return type:bool

Exclua todos os arquivos de cache.

close()¶
Return type:void

Feche a conexão do database.

display_error([$error = ''[, $swap = ''[, $native = FALSE]]])¶
Parâmetros:
  • $error (string) - a mensagem de erro
  • $swap (string) - Qualquer valor "swap"
  • $native (bool) - Se deseja localizar a mensagem
Return type:

void
Returns:

Exibe o erro do database que representa o modelo application/views/errors/error_db.php
Return type:

string

Exibe uma mensagem de erro e interrompe a execução do script.

The message is displayed using the application/views/errors/error_db.php template.

protect_identifiers($item[, $prefix_single = FALSE[, $protect_identifiers = NULL[, $field_exists = TRUE]]])¶
Parâmetros:
  • $item (string) – The item to work with
  • $prefix_single (bool) - Se deve aplicar o dbprefix mesmo se o item de entrada for um identificador único
  • $protect_identifiers (bool) – Whether to quote identifiers
  • $field_exists (bool) - Se o item fornecido contém um nome de campo ou não
Returns:

The modified item
Return type:

string

Takes a column or table name (optionally with an alias) and applies the configured dbprefix to it.

Alguma lógica é necessária para lidar com nomes de colunas que incluem o caminho.

Consider a query like this:

SELECT * FROM hostname.database.table.column AS c FROM hostname.database.table

Ou uma consulta com aliasing:

SELECT m.member_id, m.member_name FROM members AS m

Since the column name can include up to four segments (host, DB, table, column) or also have an alias prefix, we need to do a bit of work to figure this out and insira o prefixo da tabela (se existir) no position, and escape only the correct identifiers.

This method is used extensively by the Query Builder class.

Referência do Driver DB¶

Essa é a classe de implementação do Database base independente de plataforma. Esta classe não será chamada diretamente. Em vez disso, o adaptador classe para o Database específico irá estender e instanciá-lo.

O material de instruções para isso foi dividido em vários artigos. Este artigo pretende ser uma referência para eles.

Importante

Nem todos os métodos são suportados por todos os drivers de database, alguns deles podem falhar (e retornar FALSE) se o O driver não os suporta.

protect_identifiers($item[, $prefix_single = FALSE[, $protect_identifiers = NULL[, $field_exists = TRUE]]])¶
Parâmetros:
  • $item (string) - O item para trabalhar
  • $prefix_single (bool) - Se deve aplicar o dbprefix mesmo se o item de entrada for um identificador único

string

Obtém um nome de coluna ou tabela (opcionalmente com um alias) and applies the configured dbprefix to it.

Alguma lógica é necessária para lidar com nomes de colunas que incluem o caminho.

Considere uma Query como esta:

SELECT * FROM hostname.database.table.column AS c FROM hostname.database.table

Ou uma Query com aliasing:

SELECT m.member_id, m.member_name FROM members AS m

Como o nome da coluna pode incluir até quatro segmentos (host, Database, tabela, coluna) ou também tem um prefixo de alias, precisamos fazer um pouco de trabalho para descobrir isso e insira o prefixo da tabela (se existir) no posição e escape apenas os identificadores corretos.

Esse método é usado extensivamente pela classe Query Builder.

escape_like_str($str)¶
Parâmetros:
  • $str (mixed) - Um valor de string ou array de vários
Retorna:

A string (s) de escape
Tipo de retorno:

mixed

Escape como cordas.

Similar to escape_str(), but will also escape the % and _ wildcard characters, so that they don’t cause falsos positivos em condições LIKE.

Important

The escape_like_str() method uses ‘!’ (exclamation mark) to escape special characters for LIKE conditions. Because this método escapa seqüências parciais que você iria envolver entre aspas você mesmo, ele não pode adicionar automaticamente ESCAPE '!' condição para você, e então você terá que fazer isso manualmente.

protect_identifiers($item[, $prefix_single = FALSE[, $protect_identifiers = NULL[, $field_exists = TRUE]]])¶
Parâmetros:
  • $item (string) - O item para trabalhar
  • $prefix_single (bool) - Se deve aplicar o dbprefix mesmo se o item de entrada for um identificador único
  • $protect_identifiers (bool) - citar identificadores
  • $field_exists (bool) - Se o item fornecido contém um nome de campo ou não
Retorna:

O item modificado
Tipo de retorno:

string

Obtém um nome de coluna ou tabela (opcionalmente com um alias) and applies the configured dbprefix to it.

Alguma lógica é necessária para lidar com nomes de colunas que incluem o caminho.

Considere uma Query como esta:

SELECT * FROM hostname.database.table.column AS c FROM hostname.database.table

Ou uma Query com aliasing:

SELECT m.member_id, m.member_name FROM members AS m

Como o nome da coluna pode incluir até quatro segmentos (host, Database, tabela, coluna) ou também tem um prefixo de alias, precisamos fazer um pouco de trabalho para descobrir isso e insira o prefixo da tabela (se existir) no posição e escape apenas os identificadores corretos.

Esse método é usado extensivamente pela classe Query Builder.
