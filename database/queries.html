Queries¶

Query Basics¶

Regular Queries¶

To submit a query, use the query function:

$this->db->query('YOUR QUERY HERE');

o query() function returns a database result object when “read” consultas tipo são executadas, que você pode usar para show your results. Quando as consultas do tipo "write" são executadas, simplesmente retorna TRUE ou FALSE dependendo do sucesso ou falha. Ao recuperar dados você normalmente irá atribuir a consulta à sua própria variável, como this:

$query = $this->db->query('YOUR QUERY HERE');

Consultas simplificadas¶

o simple_query método é uma versão simplificada do $this->db->query() method. It DOES NOT return a database result set, nor does it set the query timer, or compile bind data, or store your query for debugging. It simply lets you submit a query. Most users will rarely use this function.

It returns whatever the database drivers’ “execute” function returns. That typically is TRUE/FALSE on success or failure for write type queries such as INSERT, DELETE or UPDATE statements (which is what it really should be used for) and a resource/object on success for queries with fetchable results.

if ($this->db->simple_query('YOUR QUERY'))
{
        echo "Success!";
}
else
{
        echo "Query failed!";
}

Nota

PostgreSQL’s pg_exec() function (for example) always returns a resource on success, even for write type queries. Então, tenha isso em mente se você estiver procurando por um valor booleano.

Working with Database prefixes manually¶

Se você configurou um prefixo de database e gostaria de pré-formatá-lo para a table name for use in a native SQL query for example, then you can use Os seguintes:

$this->db->dbprefix('tablename'); // outputs prefix_tablename

If for any reason you would like to change the prefix programatically sem precisar criar uma nova conexão, você pode usar este método:

$this->db->set_dbprefix('newprefix_');
$this->db->dbprefix('tablename'); // outputs newprefix_tablename

Protecting identifiers¶

Em muitos databases, é aconselhável proteger nomes de tabelas e campos - exemplo com backticks no MySQL. Consultas do Construtor de Consultas são protegido automaticamente, no entanto, se você precisar proteger manualmente identificador que você pode usar:

$this->db->protect_identifiers('table_name');

Importante

Although the Query Builder will try its best to properly cite qualquer nome de campo e tabela que você alimenta, note que is NOT designed to work with arbitrary user input. DO NOT feed it com dados do usuário não-analizados.

This function will also add a table prefix to your table, assuming you tenha um prefixo especificado em seu arquivo de configuração do database. Para habilitar o prefixing set TRUE (booleano) pelo segundo parâmetro:

$this->db->protect_identifiers('table_name', TRUE);

Escaping Queries¶

It’s a very good security practice to escape your data before submitting it into your database. CodeIgniter has three methods that help you do this:

  1. $this->db->escape() Esta função determina o tipo de dados that it can escape only string data. It also automatically adds aspas simples ao redor dos dados para que você não precise:

    $sql = "INSERT INTO table (title) VALUES(".$this->db->escape($title).")";

  2. $this->db->escape_str() This function escapes the data passed to it, regardless of type. Most of the time you’ll use the above function rather than this one. Use the function like this:

    $sql = "INSERT INTO table (title) VALUES('".$this->db->escape_str($title)."')";

  3. $this->db->escape_like_str() This method should be used when strings are to be used in LIKE conditions so that LIKE wildcards (‘%’, ‘_’) in the string are also properly escaped.

$search = '20% raise';
$sql = "SELECT id FROM table WHERE column LIKE '%" .
    $this->db->escape_like_str($search)."%' ESCAPE '!'";

Importante

o escape_like_str() method uses ‘!’ (exclamation mark) to escape special characters for LIKE conditions. Because this method escapes partial strings that you would wrap in quotes você mesmo, ele não pode adicionar automaticamente ESCAPE '!' condição para você, e então você terá que fazer isso manualmente.

Query Bindings¶

Bindings enable you to simplify your query syntax by letting the system put the queries together for you. Consider the following example:

$sql = "SELECT * FROM some_table WHERE id = ? AND status = ? AND author = ?";
$this->db->query($sql, array(3, 'live', 'Rick'));

o question marks in the query are automatically replaced with the valores na matriz no segundo parâmetro da função de consulta.

Binding also work with arrays, which will be transformed to IN sets:

$sql = "SELECT * FROM some_table WHERE id IN ? AND status = ? AND author = ?";
$this->db->query($sql, array(array(3, 6), 'live', 'Rick'));

o resulting query will be:

SELECT * FROM some_table WHERE id IN (3,6) AND status = 'live' AND author = 'Rick'

o secondary benefit of using binds is that the values are automatically escaped, producing safer queries. You don’t have to lembre-se de escapar dados manualmente; o motor faz isso automaticamente para you.

Manipulando Erros¶

$this->db->error();

If you need to get the last error that has occurred, the error() method will return an array containing its code and message. Here’s a quick example:

if ( ! $this->db->simple_query('SELECT `example_field` FROM `example_table`'))
{
        $error = $this->db->error(); // Has keys 'code' and 'message'
}

Querys¶

Noções básicas de Query¶

Regular Querys¶

Para enviar uma Query, use o query function:

$this->db->query('YOUR QUERY HERE');

A função query() retorna um resultado do Database object quando “leia” Querys tipo são executadas, que você pode usar para mostre seu results. Quando as Querys do tipo "write" são executadas, simplesmente retorna TRUE ou FALSE dependendo do sucesso ou falha. Ao recuperar dados você normalmente irá atribuir a Query à sua própria variável, como this:

$query = $this->db->query('YOUR QUERY HERE');

Simplified Querys¶

o simple_query método é uma versão simplificada do $this->db->query() method. It DOES NÃO retorna um conjunto de resultados de Database, nem define o temporizador de Query ou compilar dados de ligação ou armazenar sua Query para depuração. Ele simplesmente permite que você Envie uma Query. A maioria dos usuários raramente usa essa função.

Ele retorna o que a função “execute” dos drivers de Database retorna. Normalmente, isso é TRUE/FALSE em caso de sucesso ou falha para Querys do tipo de gravação como instruções INSERT, DELETE ou UPDATE (que é o que realmente deve ser usado para) e um recurso/objeto de sucesso para Querys com resultados pesquisáveis.

if ($this->db->simple_query('YOUR QUERY'))
{
        echo "Success!";
}
else
{
        echo "Query failed!";
}

Nota

PostgreSQL’s pg_exec() function (for example) always retorna um recurso em sucesso, mesmo para Querys de tipo de gravação. Então, tenha isso em mente se você estiver procurando por um valor booleano.

Trabalhando com prefixos de Database manualmente¶

Se você configurou um prefixo de database e gostaria de pré-formatá-lo para um nome de tabela para uso em uma Query SQL nativa por exemplo, então você pode usar Os seguintes:

$this->db->dbprefix('tablename'); // outputs prefix_tablename

Se por algum motivo você quiser alterar o prefixo programaticamente sem precisar criar uma nova conexão, você pode usar este método:

$this->db->set_dbprefix('newprefix_');
$this->db->dbprefix('tablename'); // outputs newprefix_tablename

Protegendo identificadores¶

Em muitos databases, é aconselhável proteger nomes de tabelas e campos - exemplo com backticks no MySQL. Querys do Construtor de Querys são protegido automaticamente, no entanto, se você precisar proteger manualmente identificador que você pode usar:

$this->db->protect_identifiers('table_name');

Importante

Embora o Construtor de Querys tente o seu melhor para cite qualquer nome de campo e tabela que você alimenta, note que NÃO é projetado para funcionar com entradas arbitrárias do usuário. NÃO ALIMENTE com dados do usuário não-analizados.

Esta função também adicionará um prefixo de tabela à sua tabela, supondo que você tenha um prefixo especificado em seu arquivo de configuração do database. Para habilitar o prefixing set TRUE (booleano) pelo segundo parâmetro:

$this->db->protect_identifiers('table_name', TRUE);

Escaping Querys¶

É uma prática de segurança muito boa para escapar de seus dados antes de enviar no seu Database. O CodeIgniter tem três métodos que ajudam você a fazer this:

  1. $this->db->escape() Esta função determina o tipo de dados que pode escapar apenas dados de string. Também adiciona automaticamente aspas simples ao redor dos dados para que você não precise:

    $sql = "INSERT INTO table (title) VALUES(".$this->db->escape($title).")";

  2. $this->db->escape_str() This function escapes the data passed to isso, independentemente do tipo. Na maioria das vezes você vai usar o acima função em vez de um presente. Use a função assim:

    $sql = "INSERT INTO table (title) VALUES('".$this->db->escape_str($title)."')";

  3. $this->db->escape_like_str() Este método deve ser usado quando seqüências de caracteres devem ser usadas em condições LIKE para que curinges LIKE ('%', '_') na string também são escapados corretamente.

$search = '20% raise';
$sql = "SELECT id FROM table WHERE column LIKE '%" .
    $this->db->escape_like_str($search)."%' ESCAPE '!'";

Importante

o escape_like_str() method uses ‘!’ (exclamation mark) to escape special characters for LIKE conditions. Because this método escapa seqüências parciais que você iria envolver entre aspas você mesmo, ele não pode adicionar automaticamente ESCAPE '!' condição para você, e então você terá que fazer isso manualmente.

Ligações de Query¶

As ligações permitem simplificar a sintaxe da Query, permitindo que o sistema Coloque as Querys juntas para você. Considere o seguinte exemplo:

$sql = "SELECT * FROM some_table WHERE id = ? AND status = ? AND author = ?";
$this->db->query($sql, array(3, 'live', 'Rick'));

o question marks in the query are automatically replaced with the valores na matriz no segundo parâmetro da função de Query.

A vinculação também funciona com arrays, que serão transformados em conjuntos IN:

$sql = "SELECT * FROM some_table WHERE id IN ? AND status = ? AND author = ?";
$this->db->query($sql, array(array(3, 6), 'live', 'Rick'));

o resulting query will be:

SELECT * FROM some_table WHERE id IN (3,6) AND status = 'live' AND author = 'Rick'

o secondary benefit of using binds is that the values are escapou automaticamente, produzindo Querys mais seguras. Você não precisa lembre-se de escapar dados manualmente; o motor faz isso automaticamente para you.

Manipulando Erros¶

$this->db->error();

Se você precisar obter o último erro que ocorreu, o método error() retornará uma matriz contendo seu código e mensagem. Aqui está um rápido example:

if ( ! $this->db->simple_query('SELECT `example_field` FROM `example_table`'))
{
        $error = $this->db->error(); // Has keys 'code' and 'message'
}

Querys¶

Noções básicas de Query¶

Regular Querys¶

Para enviar uma Query, use o query function:

$this->db->query('YOUR QUERY HERE');

A função query() retorna um resultado do Database object quando "leia" Querys tipo são executadas, que você pode usar para show your results. Quando as Querys do tipo "write" são executadas, simplesmente retorna TRUE ou FALSE dependendo do sucesso ou falha. Ao recuperar dados você normalmente irá atribuir a Query à sua própria variável, como this:

$query = $this->db->query('YOUR QUERY HERE');

Simplified Querys¶

o simple_query método é uma versão simplificada do $this->db->query() method. It DOES NÃO retorna um conjunto de resultados de Database, nem define o temporizador de Query ou compilar dados de ligação ou armazenar sua Query para depuração. Ele simplesmente permite que você Envie uma Query. A maioria dos usuários raramente usa essa função.

Ele retorna o que a função "execute" dos drivers de Database retorna. Normalmente, isso é TRUE/FALSE em caso de sucesso ou falha para Querys do tipo de gravação como instruções INSERT, DELETE ou UPDATE (que é o que realmente deve ser usado para) e um recurso/objeto de sucesso para Querys com resultados pesquisáveis.

if ($this->db->simple_query('YOUR QUERY'))
{
        echo "Success!";
}
else
{
        echo "Query failed!";
}

Nota

PostgreSQL’s pg_exec() function (for example) always retorna um recurso em sucesso, mesmo para Querys de tipo de gravação. Então, tenha isso em mente se você estiver procurando por um valor booleano.

Trabalhando com prefixos de Database manualmente¶

Se você configurou um prefixo de database e gostaria de pré-formatá-lo para um nome de tabela para uso em uma Query SQL nativa por exemplo, então você pode usar Os seguintes:

$this->db->dbprefix('tablename'); // outputs prefix_tablename

Se por algum motivo você quiser alterar o prefixo programaticamente sem precisar criar uma nova conexão, você pode usar este método:

$this->db->set_dbprefix('newprefix_');
$this->db->dbprefix('tablename'); // outputs newprefix_tablename

Protegendo identificadores¶

Em muitos databases, é aconselhável proteger nomes de tabelas e campos - exemplo com backticks no MySQL. Querys do Construtor de Querys são protegido automaticamente, no entanto, se você precisar proteger manualmente identificador que você pode usar:

$this->db->protect_identifiers('table_name');

Important

Embora o Construtor de Querys tente o seu melhor para cite qualquer nome de campo e tabela que você alimenta, note que NÃO é projetado para funcionar com entradas arbitrárias do usuário. NÃO ALIMENTE com dados do usuário não-analizados.

Esta função também adicionará um prefixo de tabela à sua tabela, supondo que você tenha um prefixo especificado em seu arquivo de configuração do database. Para habilitar o prefixing set TRUE (booleano) pelo segundo parâmetro:

$this->db->protect_identifiers('table_name', TRUE);

Escaping Querys¶

É uma prática de segurança muito boa para escapar de seus dados antes de enviar no seu Database. O CodeIgniter tem três métodos que ajudam você a fazer this:

  1. $this->db->escape() Esta função determina o tipo de dados que pode escapar apenas dados de string. Também adiciona automaticamente aspas simples ao redor dos dados para que você não precise:

    $sql = "INSERT INTO table (title) VALUES(".$this->db->escape($title).")";

  2. $this->db->escape_str() Esta função escapa dos dados passados para isso, independentemente do tipo. Na maioria das vezes você vai usar o acima função em vez de um presente. Use a função assim:

    $sql = "INSERT INTO table (title) VALUES('".$this->db->escape_str($title)."')";

  3. $this->db->escape_like_str() Este método deve ser usado quando seqüências de caracteres devem ser usadas em condições LIKE para que curinges LIKE ('%', '_') na string também são escapados corretamente.

$search = '20% raise';
$sql = "SELECT id FROM table WHERE column LIKE '%" .
    $this->db->escape_like_str($search)."%' ESCAPE '!'";

Important

o escape_like_str() method uses ‘!’ (exclamation mark) to escape special characters for LIKE conditions. Because this método escapa seqüências parciais que você iria envolver entre aspas você mesmo, ele não pode adicionar automaticamente ESCAPE '!' condição para você, e então você terá que fazer isso manualmente.

Ligações de Query¶

As ligações permitem simplificar a sintaxe da Query, permitindo que o sistema Coloque as Querys juntas para você. Considere o seguinte exemplo:

$sql = "SELECT * FROM some_table WHERE id = ? AND status = ? AND author = ?";
$this->db->query($sql, array(3, 'live', 'Rick'));

o question marks in the query are automatically replaced with the valores na matriz no segundo parâmetro da função de Query.

A vinculação também funciona com arrays, que serão transformados em conjuntos IN:

$sql = "SELECT * FROM some_table WHERE id IN ? AND status = ? AND author = ?";
$this->db->query($sql, array(array(3, 6), 'live', 'Rick'));

o resulting query will be:

SELECT * FROM some_table WHERE id IN (3,6) AND status = 'live' AND author = 'Rick'

o secondary benefit of using binds is that the values are escapou automaticamente, produzindo Querys mais seguras. Você não precisa lembre-se de escapar dados manualmente; o motor faz isso automaticamente para you.

Manipulando Erros¶

$this->db->error();

Se você precisar obter o último erro que ocorreu, o método error() retornará uma matriz contendo seu código e mensagem. Aqui está um rápido example:

if ( ! $this->db->simple_query('SELECT `example_field` FROM `example_table`'))
{
        $error = $this->db->error(); // Has keys 'code' and 'message'
}
