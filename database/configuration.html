Configuração do database¶

CodeIgniter has a config file that lets you store your database valores de conexão (nome de usuário, senha, nome do database, etc.). A configuração file is located at application/config/database.php. You can also set database connection values for specific environments colocando database.php in the respective environment config folder.

o config settings are stored in a multi-dimensional array with this prototype:

$db['default'] = array(
        'dsn'   => '',
        'hostname' => 'localhost',
        'username' => 'root',
        'password' => '',
        'database' => 'database_name',
        'dbdriver' => 'mysqli',
        'dbprefix' => '',
        'pconnect' => TRUE,
        'db_debug' => TRUE,
        'cache_on' => FALSE,
        'cachedir' => '',
        'char_set' => 'utf8',
        'dbcollat' => 'utf8_general_ci',
        'swap_pre' => '',
        'encrypt' => FALSE,
        'compress' => FALSE,
        'stricton' => FALSE,
        'failover' => array()
);

Some database drivers (such as PDO, PostgreSQL, Oracle, ODBC) might require a full DSN string to be provided. If that is the case, you should use the ‘dsn’ configuration setting, as if you’re using the driver’s underlying native PHP extension, like this:

// PDO
$db['default']['dsn'] = 'pgsql:host=localhost;port=5432;dbname=database_name';

// Oracle
$db['default']['dsn'] = '//localhost/XE';

Nota

Se você não especificar uma cadeia DSN para um driver que o exija, o CodeIgniter will try to build it with the rest of the provided settings.

Nota

Se você fornecer uma string DSN e não tiver algumas configurações válidas (por exemplo, database character set), which are present in the rest of the configuration campos, o CodeIgniter irá anexá-los.

Você também pode especificar failovers para a situação em que a conexão principal não pode se conectar por algum motivo. Esses failovers podem ser especificados configurando o failover para uma conexão como esta:

$db['default']['failover'] = array(
                array(
                        'hostname' => 'localhost1',
                        'username' => '',
                        'password' => '',
                        'database' => '',
                        'dbdriver' => 'mysqli',
                        'dbprefix' => '',
                        'pconnect' => TRUE,
                        'db_debug' => TRUE,
                        'cache_on' => FALSE,
                        'cachedir' => '',
                        'char_set' => 'utf8',
                        'dbcollat' => 'utf8_general_ci',
                        'swap_pre' => '',
                        'encrypt' => FALSE,
                        'compress' => FALSE,
                        'stricton' => FALSE
                ),
                array(
                        'hostname' => 'localhost2',
                        'username' => '',
                        'password' => '',
                        'database' => '',
                        'dbdriver' => 'mysqli',
                        'dbprefix' => '',
                        'pconnect' => TRUE,
                        'db_debug' => TRUE,
                        'cache_on' => FALSE,
                        'cachedir' => '',
                        'char_set' => 'utf8',
                        'dbcollat' => 'utf8_general_ci',
                        'swap_pre' => '',
                        'encrypt' => FALSE,
                        'compress' => FALSE,
                        'stricton' => FALSE
                )
        );

You can specify as many failovers as you like.

o reason we use a multi-dimensional array rather than a more simple one is to permit you to optionally store multiple sets of connection values. If, for example, you run multiple environments (development, production, test, etc.) under a single installation, you can set up a connection group for each, then switch between groups as needed. For Por exemplo, para configurar um ambiente de “teste”, você faria isso:

$db['test'] = array(
        'dsn'   => '',
        'hostname' => 'localhost',
        'username' => 'root',
        'password' => '',
        'database' => 'database_name',
        'dbdriver' => 'mysqli',
        'dbprefix' => '',
        'pconnect' => TRUE,
        'db_debug' => TRUE,
        'cache_on' => FALSE,
        'cachedir' => '',
        'char_set' => 'utf8',
        'dbcollat' => 'utf8_general_ci',
        'swap_pre' => '',
        'compress' => FALSE,
        'encrypt' => FALSE,
        'stricton' => FALSE,
        'failover' => array()
);

on, to globally tell the system to use that group you would set this variable located in the config file:

$active_group = 'test';

Nota

o name ‘test’ is arbitrary. It can be anything you want. By padrão, usamos a palavra "padrão" para a conexão principal, but it too can be renamed to something more relevant to your project.

Query Builder¶

o Classe do Construtor de Consultas está habilitado globalmente ou disabled by setting the $query_builder variable in the database configuration file to TRUE/FALSE (boolean). o default setting is TRUE. If you are not using the classe construtor de consulta, definindo-o como FALSE utilizará menos recursos when the database classes are initialized.

$query_builder = TRUE;

Nota

that some CodeIgniter classes such as Sessions require Query Construtor para ser ativado para acessar certas funcionalidades.

Explanation of Values:¶

Name Config Descrição
dsn o DSN connect string (an all-in-one configuration sequence).
hostname o hostname of your database server. Often this is ‘localhost’.
username o username used to connect to the database.
password o password used to connect to the database.
database o name of the database you want to connect to.
dbdriver o database type. ie: mysqli, postgre, odbc, etc. Must be specified in lower case.
dbprefix Um prefixo de tabela opcional que será adicionado ao nome da tabela ao executar Query Builder queries. This permits multiple CodeIgniter instalações para compartilhar um database.
pconnect TRUE/FALSE (booleano) - Se deve usar uma conexão persistente.
db_debug TRUE/FALSE (booleano) - Se os erros do Database devem ser exibidos.
cache_on TRUE/FALSE (booleano) - Se o cache de consulta do database está ativado, see also Database Cache Class.
cachedir o absolute server path to your database query cache directory.
char_set o character set used in communicating with the database.
dbcollat

o character collation used in communicating with the database

Nota

Only used in the ‘mysql’ and ‘mysqli’ drivers.
swap_pre Um prefixo de tabela padrão que deve ser trocado pelo dbprefix. Isso é útil para distribuído applications where you might run manually written queries, and need the prefix to still be customizable by the end user.
schema o database schema, defaults to ‘public’. Used by PostgreSQL and ODBC drivers.
encrypt

Whether or not to use an encrypted connection.

  • ‘mysql’ (deprecated), ‘sqlsrv’ and ‘pdo/sqlsrv’ drivers accept TRUE/FALSE
  • Os drivers 'mysqli' e 'pdo/mysql' aceitam um array com as seguintes opções:
    • ‘ssl_key’ - Path to the private key file
    • 'ssl_cert' - Caminho para o arquivo de certificado de chave pública
    • ‘ssl_ca’ - Path to the certificate authority file
    • ‘ssl_capath’ - Path to a directory containing trusted CA certificates in PEM format
    • ‘ssl_cipher’ - List of allowed ciphers to be used for the encryption, separated by colons (‘:’)
    • ‘ssl_verify’ - TRUE/FALSE; Whether to verify the server certificate or not (‘mysqli’ only)
compress Se deve ou não usar compactação de cliente (somente MySQL).
stricton TRUE/FALSE (boolean) - Whether to force “Strict Mode” connections, good for ensuring strict SQL enquanto desenvolve uma aplicação.
port

o database port number. To use this value you have to add a line to the database config array.

$db['default']['port'] = 5432;

Nota

Depending on what database platform you are using (MySQL, PostgreSQL, etc.) not all values will be needed. For example, when using SQLite you will not need to supply a username or password, and the database name will be the path to your database file. o information above assumes you are using MySQL.

Configuração do database¶

O CodeIgniter tem um arquivo de configuração que permite armazenar seu Database valores de conexão (nome de usuário, senha, nome do database, etc.). A configuração O arquivo está localizado em application/config/database.php. Você também pode definir valores de conexão de Database para environments colocando database.php na respectiva pasta de configuração do ambiente.

As configurações são armazenadas em um array multidimensional com este prototype:

$db['default'] = array(
        'dsn'   => '',
        'hostname' => 'localhost',
        'username' => 'root',
        'password' => '',
        'database' => 'database_name',
        'dbdriver' => 'mysqli',
        'dbprefix' => '',
        'pconnect' => TRUE,
        'db_debug' => TRUE,
        'cache_on' => FALSE,
        'cachedir' => '',
        'char_set' => 'utf8',
        'dbcollat' => 'utf8_general_ci',
        'swap_pre' => '',
        'encrypt' => FALSE,
        'compress' => FALSE,
        'stricton' => FALSE,
        'failover' => array()
);

Alguns drivers de Database (como PDO, PostgreSQL, Oracle, ODBC) requerer uma cadeia DSN completa a ser fornecida. Se for esse o caso, você deve usar a configuração de 'dsn', como se você estivesse usando o extensão PHP nativa subjacente do driver, assim:

// PDO
$db['default']['dsn'] = 'pgsql:host=localhost;port=5432;dbname=database_name';

// Oracle
$db['default']['dsn'] = '//localhost/XE';

Nota

Se você não especificar uma cadeia DSN para um driver que o exija, o CodeIgniter tentará construí-lo com o resto das configurações fornecidas.

Nota

Se você fornecer uma string DSN e não tiver algumas configurações válidas (por exemplo, conjunto de caracteres do Database), que estão presentes no resto da configuração campos, o CodeIgniter irá anexá-los.

Você também pode especificar failovers para a situação em que a conexão principal não pode se conectar por algum motivo. Esses failovers podem ser especificados configurando o failover para uma conexão como esta:

$db['default']['failover'] = array(

                        'password' => '',
                        'database' => '',

                        'dbprefix' => '',
                        'pconnect' => TRUE,
                        'db_debug' => TRUE,
                        'cache_on' => FALSE,
                        'cachedir' => '',
                        'char_set' => 'utf8',
                        'dbcollat' => 'utf8_general_ci',
                        'swap_pre' => '',
                        'encrypt' => FALSE,
                        'compress' => FALSE,
                        'stricton' => FALSE
                ),
                array(
                        'hostname' => 'localhost2',
                        'username' => '',
                        'password' => '',
                        'database' => '',
                        'dbdriver' => 'mysqli',
                        'dbprefix' => '',
                        'pconnect' => TRUE,
                        'db_debug' => TRUE,
                        'cache_on' => FALSE,
                        'cachedir' => '',
                        'char_set' => 'utf8',
                        'dbcollat' => 'utf8_general_ci',
                        'swap_pre' => '',
                        'encrypt' => FALSE,
                        'compress' => FALSE,
                        'stricton' => FALSE
                )
        );

        'dbprefix' => '',
        'pconnect' => TRUE,
        'db_debug' => TRUE,
        'cache_on' => FALSE,
        'cachedir' => '',
        'char_set' => 'utf8',
        'dbcollat' => 'utf8_general_ci',

        'compress' => FALSE,

Configuração do database¶

O CodeIgniter tem um arquivo de configuração que permite armazenar seu Database valores de conexão (nome de usuário, senha, nome do database, etc.). A configuração O arquivo está localizado em application/config/database.php. Você também pode definir valores de conexão de Database para environments by placing database.php na respectiva pasta de configuração do ambiente.

As configurações são armazenadas em um array multidimensional com este prototype:

$db['default'] = array(
        'dsn'   => '',
        'hostname' => 'localhost',
        'username' => 'root',
        'password' => '',
        'database' => 'database_name',
        'dbdriver' => 'mysqli',
        'dbprefix' => '',
        'pconnect' => TRUE,
        'db_debug' => TRUE,
        'cache_on' => FALSE,
        'cachedir' => '',
        'char_set' => 'utf8',
        'dbcollat' => 'utf8_general_ci',
        'swap_pre' => '',
        'encrypt' => FALSE,
        'compress' => FALSE,
        'stricton' => FALSE,
        'failover' => array()
);

Alguns drivers de Database (como PDO, PostgreSQL, Oracle, ODBC) requerer uma cadeia DSN completa a ser fornecida. Se for esse o caso, você deve usar a configuração de 'dsn', como se você estivesse usando o extensão PHP nativa subjacente do driver, assim:

// PDO
$db['default']['dsn'] = 'pgsql:host=localhost;port=5432;dbname=database_name';

// Oracle
$db['default']['dsn'] = '//localhost/XE';

Nota

Se você não especificar uma cadeia DSN para um driver que o exija, o CodeIgniter tentará construí-lo com o resto das configurações fornecidas.

Nota

Se você fornecer uma string DSN e não tiver algumas configurações válidas (por exemplo, conjunto de caracteres do Database), que estão presentes no resto da configuração campos, o CodeIgniter irá anexá-los.

Você também pode especificar failovers para a situação em que a conexão principal não pode se conectar por algum motivo. Esses failovers podem ser especificados configurando o failover para uma conexão como esta:

$db['default']['failover'] = array(
                array(
                        'hostname' => 'localhost1',
                        'username' => '',
                        'password' => '',
                        'database' => '',
                        'dbdriver' => 'mysqli',
                        'dbprefix' => '',
                        'pconnect' => TRUE,
                        'db_debug' => TRUE,
                        'cache_on' => FALSE,
                        'cachedir' => '',
                        'char_set' => 'utf8',
                        'dbcollat' => 'utf8_general_ci',
                        'swap_pre' => '',
                        'encrypt' => FALSE,
                        'compress' => FALSE,
                        'stricton' => FALSE
                ),
                array(
                        'hostname' => 'localhost2',
                        'username' => '',
                        'password' => '',
                        'database' => '',
                        'dbdriver' => 'mysqli',
                        'dbprefix' => '',
                        'pconnect' => TRUE,
                        'db_debug' => TRUE,
                        'cache_on' => FALSE,
                        'cachedir' => '',
                        'char_set' => 'utf8',
                        'dbcollat' => 'utf8_general_ci',
                        'swap_pre' => '',
                        'encrypt' => FALSE,
                        'compress' => FALSE,
                        'stricton' => FALSE
                )
        );

Você pode especificar quantos failovers desejar.

A razão pela qual usamos uma matriz multidimensional em vez de uma mais simples uma é permitir que você armazene opcionalmente vários conjuntos de conexão valores. Se, por exemplo, você executar vários ambientes (desenvolvimento, produção, teste, etc.) sob uma única instalação, você pode configurar grupo de conexão para cada um, depois alterne entre os grupos conforme necessário. Para Por exemplo, para configurar um ambiente de "teste", você faria isso:

$db['test'] = array(
        'dsn'   => '',
        'hostname' => 'localhost',
        'username' => 'root',
        'password' => '',
        'database' => 'database_name',
        'dbdriver' => 'mysqli',
        'dbprefix' => '',
        'pconnect' => TRUE,
        'db_debug' => TRUE,
        'cache_on' => FALSE,
        'cachedir' => '',
        'char_set' => 'utf8',
        'dbcollat' => 'utf8_general_ci',
        'swap_pre' => '',
        'compress' => FALSE,
        'encrypt' => FALSE,
        'stricton' => FALSE,
        'failover' => array()
);

Então, para dizer globalmente ao sistema para usar esse grupo, você configuraria variável localizada no arquivo de configuração:

$active_group = 'test';

Nota

O nome 'teste' é arbitrário. Pode ser o que você quiser. Por padrão, usamos a palavra "padrão" para a conexão principal, mas também pode ser renomeado para algo mais relevante para o seu projeto.
