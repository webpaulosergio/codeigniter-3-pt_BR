Database Forge Class¶

The Database Forge Class contains methods that help you manage your database.

Initializing the Forge Class¶

Importante

In order to initialize the Forge class, your database O driver já deve estar em execução, já que a classe forja depende dele.

Carregue a classe Forge da seguinte maneira:

$this->load->dbforge()

You can also pass another database object to the DB Forge loader, in case the database you want to manage isn’t the default one:

$this->myforge = $this->load->dbforge($this->other_db, TRUE);

In the above example, we’re passing a custom database object as the first parâmetro e, em seguida, dizer-lhe para retornar o objeto dbforge, em vez de assigning it directly to $this->dbforge.

Nota

Both of the parameters can be used individually, just pass an empty value as the first one if you wish to skip it.

Uma vez inicializado, você acessará os métodos usando o $this->dbforge object:

$this->dbforge->some_method();

Criando e descartando Databases¶

$this->dbforge->create_database(‘db_name’)

Permits you to create the database specified in the first parameter. Retorna TRUE/FALSE com base no sucesso ou falha:

if ($this->dbforge->create_database('my_db'))
{
        echo 'Database created!';
}

$this->dbforge->drop_database(‘db_name’)

Permits you to drop the database specified in the first parameter. Retorna TRUE/FALSE com base no sucesso ou falha:

if ($this->dbforge->drop_database('my_db'))
{
        echo 'Database deleted!';
}

Creating and Dropping Tables¶

Há várias coisas que você pode querer fazer ao criar tabelas. Adicionar fields, add keys to the table, alter columns. CodeIgniter provides a mechanism for this.

Adicionando campos¶

Os campos são criados por meio de um array associativo. Dentro da matriz você deve inclua uma chave de "tipo" relacionada ao tipo de dados do campo. Para exemplo, INT, VARCHAR, TEXT, etc. Muitos tipos de dados (por exemplo VARCHAR) também requer uma chave de 'restrição'.

$fields = array(
        'users' => array(
                'type' => 'VARCHAR',
                'constraint' => '100',
        ),
);
// will translate to "users VARCHAR(100)" when the field is added.

Além disso, as seguintes chaves/valores podem ser usados:

  • unsigned/true: para gerar “UNSIGNED” na definição do campo.
  • default/value : to generate a default value in the field definition.
  • null/true : to generate “NULL” in the field definition. Without this, the field will default to “NOT NULL”.
  • auto_increment/true: gera um sinalizador auto_increment no field. Nota that the field type must be a type that supports this, como inteiro.
  • unique/true: para gerar uma chave exclusiva para a definição de campo.
$fields = array(
        'blog_id' => array(
                'type' => 'INT',
                'constraint' => 5,
                'unsigned' => TRUE,
                'auto_increment' => TRUE
        ),
        'blog_title' => array(
                'type' => 'VARCHAR',
                'constraint' => '100',
                'unique' => TRUE,
        ),
        'blog_author' => array(
                'type' =>'VARCHAR',
                'constraint' => '100',
                'default' => 'King of Town',
        ),
        'blog_description' => array(
                'type' => 'TEXT',
                'null' => TRUE,
        ),
);

Após os campos terem sido definidos, eles podem ser adicionados usando $this->dbforge->add_field($fields); followed by a call to the create_table() method.

$this->dbforge->add_field()

O método add fields aceita o array acima.

Passando strings como campos¶

If you know exactly how you want a field to be created, you can pass the string into the field definitions with add_field()

$this->dbforge->add_field("label varchar(100) NOT NULL DEFAULT 'default label'");

Nota

Passing raw strings as fields cannot be followed by add_key() calls on those fields.

Nota

Multiple calls to add_field() are cumulative.

Creating an id field¶

There is a special exception for creating id fields. A field with type id will automatically be assigned as an INT(9) auto_incrementing Primary Key.

$this->dbforge->add_field('id');
// gives id INT(9) NOT NULL AUTO_INCREMENT

Adding Keys¶

Generally speaking, you’ll want your table to have Keys. This is accomplished with $this->dbforge->add_key(‘field’). An optional second parameter set to TRUE will make it a primary key. Nota that add_key() must be followed by a call to create_table().

Chaves não primárias de múltiplas colunas devem ser enviadas como uma matriz. Amostra de saída abaixo é para o MySQL.

$this->dbforge->add_key('blog_id', TRUE);
// gives PRIMARY KEY `blog_id` (`blog_id`)

$this->dbforge->add_key('blog_id', TRUE);
$this->dbforge->add_key('site_id', TRUE);
// gives PRIMARY KEY `blog_id_site_id` (`blog_id`, `site_id`)

$this->dbforge->add_key('blog_name');
// gives KEY `blog_name` (`blog_name`)

$this->dbforge->add_key(array('blog_name', 'blog_label'));
// gives KEY `blog_name_blog_label` (`blog_name`, `blog_label`)

Criando uma tabela¶

After fields and keys have been declared, you can create a new table with

$this->dbforge->create_table('table_name');
// gives CREATE TABLE table_name

An optional second parameter set to TRUE adds an “IF NOT EXISTS” clause into the definition

$this->dbforge->create_table('table_name', TRUE);
// gives CREATE TABLE IF NOT EXISTS table_name

You could also pass optional table attributes, such as MySQL’s ENGINE:

$attributes = array('ENGINE' => 'InnoDB');
$this->dbforge->create_table('table_name', FALSE, $attributes);
// produces: CREATE TABLE `table_name` (...) ENGINE = InnoDB DEFAULT CHARACTER SET utf8 COLLATE utf8_general_ci

Nota

Unless you specify the CHARACTER SET and/or COLLATE attributes, create_table() will always add them with your configured char_set and dbcollat values, as long as they are not empty (MySQL only).

Dropping a table¶

Execute a DROP TABLE statement and optionally add an IF EXISTS clause.

// Produces: DROP TABLE table_name
$this->dbforge->drop_table('table_name');

// Produces: DROP TABLE IF EXISTS table_name
$this->dbforge->drop_table('table_name',TRUE);

Renaming a table¶

Executes a TABLE rename

$this->dbforge->rename_table('old_table_name', 'new_table_name');
// gives ALTER TABLE old_table_name RENAME TO new_table_name

Modificando Tabelas¶

Adicionando uma coluna a uma tabela¶

$this->dbforge->add_column()

The add_column() method is used to modify an existing table. It accepts the same field array as above, and can be used for an unlimited número de campos adicionais.

$fields = array(
        'preferences' => array('type' => 'TEXT')
);
$this->dbforge->add_column('table_name', $fields);
// Executes: ALTER TABLE table_name ADD preferences TEXT

If you are using MySQL or CUBIRD, then you can take advantage of their APÓS e PRIMEIRAS cláusulas para posicionar a nova coluna.

Exemplos:

// Will place the new column after the `another_field` column:
$fields = array(
        'preferences' => array('type' => 'TEXT', 'after' => 'another_field')
);

// Will place the new column at the start of the table definition:
$fields = array(
        'preferences' => array('type' => 'TEXT', 'first' => TRUE)
);

Soltando uma coluna de uma tabela¶

$this->dbforge->drop_column()

Usado para remover uma coluna de uma tabela.

$this->dbforge->drop_column('table_name', 'column_to_drop');

Modifying a Column in a Table¶

$this->dbforge->modify_column()

The usage of this method is identical to add_column(), except it altera uma coluna existente em vez de adicionar uma nova. A fim de change the name you can add a “name” key into the field defining array.

$fields = array(
        'old_name' => array(
                'name' => 'new_name',
                'type' => 'TEXT',
        ),
);
$this->dbforge->modify_column('table_name', $fields);
// gives ALTER TABLE table_name CHANGE old_name new_name TEXT

Inicializando a classe Forge¶

Importante

Para inicializar a classe Forge, seu Database O driver já deve estar em execução, já que a classe forja depende dele.

Carregue a classe Forge da seguinte maneira:

$this->load->dbforge()

Você também pode passar outro objeto de Database para o carregador do DB Forge, caso o Database que você deseja gerenciar não é o padrão:

$this->myforge = $this->load->dbforge($this->other_db, TRUE);

No exemplo acima, estamos passando um objeto de Database personalizado como o primeiro parâmetro e, em seguida, dizer-lhe para retornar o objeto dbforge, em vez de assigning it directly to $this->dbforge.

Nota

Both of the parameters can be used individually, just pass an empty valor como o primeiro se você quiser pular.

Uma vez inicializado, você acessará os métodos usando o $this->dbforge object:

$this->dbforge->some_method();

Criando e descartando Databases¶

$this->dbforge->create_database(‘db_name’)

Permite que você crie o Database especificado no primeiro parâmetro. Retorna TRUE/FALSE com base no sucesso ou falha:

if ($this->dbforge->create_database('my_db'))
{
        echo 'Database created!';
}

$this->dbforge->drop_database(‘db_name’)

Permite que você elimine o Database especificado no primeiro parâmetro. Retorna TRUE/FALSE com base no sucesso ou falha:

if ($this->dbforge->drop_database('my_db'))
{
        echo 'Database deleted!';
}

Criando e Soltando Tabelas¶

Há várias coisas que você pode querer fazer ao criar tabelas. Adicionar campos, adicionar chaves à tabela, alterar colunas. O CodeIgniter fornece um mecanismo para isso.

Adicionando campos¶

Os campos são criados por meio de um array associativo. Dentro da matriz você deve inclua uma chave de "tipo" relacionada ao tipo de dados do campo. Para exemplo, INT, VARCHAR, TEXT, etc. Muitos tipos de dados (por exemplo VARCHAR) também requer uma chave de 'restrição'.

$fields = array(
        'users' => array(
                'type' => 'VARCHAR',
                'constraint' => '100',
        ),
);
// will translate to "users VARCHAR(100)" when the field is added.

Além disso, as seguintes chaves/valores podem ser usados:

  • unsigned/true: para gerar “UNSIGNED” na definição do campo.
  • padrão/valor: para gerar um valor padrão na definição do campo.
  • null/true: para gerar “NULL” na definição do campo. Sem isso, o campo será padronizado como "NOT NULL".
  • auto_increment/true: gera um sinalizador auto_increment no campo. Observe que o tipo de campo deve ser um tipo que suporte isso, como inteiro.
  • unique/true: para gerar uma chave exclusiva para a definição de campo.
$fields = array(
        'blog_id' => array(
                'type' => 'INT',
                'constraint' => 5,
                'unsigned' => TRUE,
                'auto_increment' => TRUE
        ),
        'blog_title' => array(
                'type' => 'VARCHAR',
                'constraint' => '100',
                'unique' => TRUE,
        ),
        'blog_author' => array(
                'type' =>'VARCHAR',
                'constraint' => '100',
                'default' => 'King of Town',
        ),
        'blog_description' => array(
                'type' => 'TEXT',
                'null' => TRUE,
        ),
);

Após os campos terem sido definidos, eles podem ser adicionados usando $this->dbforge->add_field($fields); seguido de uma chamada para o create_table() method.

$this->dbforge->add_field()

O método add fields aceita o array acima.

Passando strings como campos¶

Se você sabe exatamente como deseja que um campo seja criado, pode passar string nas definições de campo com add_field()

$this->dbforge->add_field("label varchar(100) NOT NULL DEFAULT 'default label'");

Nota

Passing raw strings as fields cannot be followed by add_key() calls on those fields.

Nota

Múltiplas chamadas para add_field() são cumulativas.

Criando um campo de id¶

Há uma exceção especial para criar campos de ID. Um campo com tipo id será automaticamente atribuído como INT (9) auto_incrementing Chave primária.

$this->dbforge->add_field('id');
// gives id INT(9) NOT NULL AUTO_INCREMENT

Adicionando Chaves¶

De um modo geral, você quer que sua tabela tenha Chaves. Isto é accomplished with $this->dbforge->add_key(‘field’). An optional second O parâmetro definido como TRUE fará com que seja uma chave primária. Nota que add_key() deve ser seguido por uma chamada para create_table().

Chaves não primárias de múltiplas colunas devem ser enviadas como uma matriz. Amostra de saída abaixo é para o MySQL.

$this->dbforge->add_key('blog_id', TRUE);
// gives PRIMARY KEY `blog_id` (`blog_id`)

$this->dbforge->add_key('blog_id', TRUE);
$this->dbforge->add_key('site_id', TRUE);
// gives PRIMARY KEY `blog_id_site_id` (`blog_id`, `site_id`)

$this->dbforge->add_key('blog_name');
// gives KEY `blog_name` (`blog_name`)

$this->dbforge->add_key(array('blog_name', 'blog_label'));
// gives KEY `blog_name_blog_label` (`blog_name`, `blog_label`)

Criando uma tabela¶

Após campos e chaves terem sido declarados, você pode criar uma nova tabela with

$this->dbforge->create_table('table_name');
// gives CREATE TABLE table_name

Um segundo parâmetro opcional definido como TRUE adiciona uma cláusula “IF NOT EXISTS” na definição

$this->dbforge->create_table('table_name', TRUE);
// gives CREATE TABLE IF NOT EXISTS table_name

You could also pass optional table attributes, such as MySQL’s ENGINE:

$attributes = array('ENGINE' => 'InnoDB');
$this->dbforge->create_table('table_name', FALSE, $attributes);
// produces: CREATE TABLE `table_name` (...) ENGINE = InnoDB DEFAULT CHARACTER SET utf8 COLLATE utf8_general_ci

Nota

Unless you specify the CHARACTER SET and/or COLLATE attributes, create_table() irá sempre adicioná-los com o seu configurado char_set and dbcollat values, as long as they are not empty (MySQL only).

Soltando uma Tabela¶

Execute uma instrução DROP TABLE e, opcionalmente, inclua uma cláusula IF EXISTS.

// Produces: DROP TABLE table_name
$this->dbforge->drop_table('table_name');

// Produces: DROP TABLE IF EXISTS table_name
$this->dbforge->drop_table('table_name',TRUE);

Renomeando uma tabela¶

Executa uma renomeação TABLE

$this->dbforge->rename_table('old_table_name', 'new_table_name');
// gives ALTER TABLE old_table_name RENAME TO new_table_name

Modificando Tabelas¶

Adicionando uma coluna a uma tabela¶

$this->dbforge->add_column()

The add_column() method is used to modify an existing table. It aceita o mesmo array de campo como acima, e pode ser usado para um ilimitado número de campos adicionais.

$fields = array(
        'preferences' => array('type' => 'TEXT')
);
$this->dbforge->add_column('table_name', $fields);
// Executes: ALTER TABLE table_name ADD preferences TEXT

Se você está usando MySQL ou CUBIRD, então você pode tirar proveito de suas APÓS e PRIMEIRAS cláusulas para posicionar a nova coluna.

Exemplos:

// Will place the new column after the `another_field` column:
$fields = array(
        'preferences' => array('type' => 'TEXT', 'after' => 'another_field')
);

// Will place the new column at the start of the table definition:
$fields = array(
        'preferences' => array('type' => 'TEXT', 'first' => TRUE)
);

Soltando uma coluna de uma tabela¶

$this->dbforge->drop_column()

Usado para remover uma coluna de uma tabela.

$this->dbforge->drop_column('table_name', 'column_to_drop');

Modificando uma coluna em uma tabela¶

$this->dbforge->modify_column()

The usage of this method is identical to add_column(), except it altera uma coluna existente em vez de adicionar uma nova. A fim de Para alterar o nome, você pode adicionar uma chave de "nome" ao campo que define a matriz.

$fields = array(
        'old_name' => array(
                'name' => 'new_name',
                'type' => 'TEXT',
        ),
);
$this->dbforge->modify_column('table_name', $fields);
// gives ALTER TABLE table_name CHANGE old_name new_name TEXT

        'blog_description' => array(
                'type' => 'TEXT',
                'null' => TRUE,
        ),
);

Após os campos terem sido definidos, eles podem ser adicionados usando $this->dbforge->add_field($fields); seguido de uma chamada para o create_table() method.

$this->dbforge->add_field()

O método add fields aceita o array acima.

Passando strings como campos¶

Se você sabe exatamente como deseja que um campo seja criado, pode passar string nas definições de campo com add_field()

$this->dbforge->add_field("label varchar(100) NOT NULL DEFAULT 'default label'");

Nota

Passing raw strings as fields cannot be followed by add_key() calls on those fields.

Nota

Múltiplas chamadas para add_field() são cumulativas.

Criando um campo de id¶

Há uma exceção especial para criar campos de ID. Um campo com tipo id será automaticamente atribuído como INT (9) auto_incrementing Chave primária.

$this->dbforge->add_field('id');
// gives id INT(9) NOT NULL AUTO_INCREMENT

Adicionando Chaves¶

De um modo geral, você quer que sua tabela tenha Chaves. Isto é realizado com $this->dbforge->add_key(field). Um segundo opcional O parâmetro definido como TRUE fará com que seja uma chave primária. Nota que add_key() deve ser seguido por uma chamada para create_table().

Chaves não primárias de múltiplas colunas devem ser enviadas como uma matriz. Amostra de saída abaixo é para o MySQL.

$this->dbforge->add_key('blog_id', TRUE);
// gives PRIMARY KEY `blog_id` (`blog_id`)

$this->dbforge->add_key('blog_id', TRUE);
$this->dbforge->add_key('site_id', TRUE);
// gives PRIMARY KEY `blog_id_site_id` (`blog_id`, `site_id`)

$this->dbforge->add_key('blog_name');
// gives KEY `blog_name` (`blog_name`)

$this->dbforge->add_key(array('blog_name', 'blog_label'));
// gives KEY `blog_name_blog_label` (`blog_name`, `blog_label`)

Criando uma tabela¶

Após campos e chaves terem sido declarados, você pode criar uma nova tabela with

$this->dbforge->create_table('table_name');
// gives CREATE TABLE table_name

Um segundo parâmetro opcional definido como TRUE adiciona uma cláusula "IF NOT EXISTS" na definição

$this->dbforge->create_table('table_name', TRUE);
// gives CREATE TABLE IF NOT EXISTS table_name

You could also pass optional table attributes, such as MySQL’s ENGINE:

$attributes = array('ENGINE' => 'InnoDB');
$this->dbforge->create_table('table_name', FALSE, $attributes);
// produces: CREATE TABLE `table_name` (...) ENGINE = InnoDB DEFAULT CHARACTER SET utf8 COLLATE utf8_general_ci

Nota

Unless you specify the CHARACTER SET and/or COLLATE attributes, create_table() irá sempre adicioná-los com o seu configurado char_set and dbcollat values, as long as they are not empty (MySQL only).

Soltando uma Tabela¶

Execute uma instrução DROP TABLE e, opcionalmente, inclua uma cláusula IF EXISTS.

// Produces: DROP TABLE table_name
$this->dbforge->drop_table('table_name');

// Produces: DROP TABLE IF EXISTS table_name
$this->dbforge->drop_table('table_name',TRUE);

Renomeando uma tabela¶

Executa uma renomeação TABLE

$this->dbforge->rename_table('old_table_name', 'new_table_name');
// gives ALTER TABLE old_table_name RENAME TO new_table_name

Modificando Tabelas¶

Adicionando uma coluna a uma tabela¶

$this->dbforge->add_column()

The add_column() method is used to modify an existing table. It aceita o mesmo array de campo como acima, e pode ser usado para um ilimitado número de campos adicionais.

$fields = array(
        'preferences' => array('type' => 'TEXT')
);
$this->dbforge->add_column('table_name', $fields);
// Executes: ALTER TABLE table_name ADD preferences TEXT

Se você está usando MySQL ou CUBIRD, então você pode tirar proveito de suas APÓS e PRIMEIRAS cláusulas para posicionar a nova coluna.

Examples:

// Will place the new column after the `another_field` column:
$fields = array(
        'preferences' => array('type' => 'TEXT', 'after' => 'another_field')
);

// Will place the new column at the start of the table definition:
$fields = array(
        'preferences' => array('type' => 'TEXT', 'first' => TRUE)
);

Soltando uma coluna de uma tabela¶

$this->dbforge->drop_column()

Usado para remover uma coluna de uma tabela.

$this->dbforge->drop_column('table_name', 'column_to_drop');

Modificando uma coluna em uma tabela¶

$this->dbforge->modify_column()

The usage of this method is identical to add_column(), except it altera uma coluna existente em vez de adicionar uma nova. A fim de Para alterar o nome, você pode adicionar uma chave de "nome" ao campo que define a matriz.

$fields = array(
        'old_name' => array(
                'name' => 'new_name',
                'type' => 'TEXT',
        ),
);
$this->dbforge->modify_column('table_name', $fields);
// gives ALTER TABLE table_name CHANGE old_name new_name TEXT
