Escrevendo Documentação do CodeIgniter¶

CodeIgniter uses Sphinx to generate its documentation in a variety of formats, using reStructuredText to handle the formatting. If you are familiar with Markdown ou Textile, você vai rapidamente entender reStructuredText. O foco é na legibilidade e facilidade de uso. While they can be quite technical, we always write for humans!

A local table of contents should always be included, like the one below. É criado automaticamente, inserindo o seguinte:

.. contents::
        :local:

.. raw:: html

<div class="custom-index container"></div>
  • Ferramentas necessárias
  • Cabeçalhos e subtítulos de página e seção
  • Method Documentation

o <div> that is inserted as raw HTML is a hook for the documentation’s JavaScript para adicionar links dinamicamente a qualquer função e definição de método contido na página atual.

Ferramentas necessárias¶

Para ver o HTML renderizado, ePub, PDF, etc., você precisará instalar o Sphinx along with the PHP domain extension for Sphinx. o underlying requirement é ter o Python instalado. Por fim, você instalará o CI Lexer para Pygments, para que blocos de código possam ser devidamente destacados.

easy_install "sphinx==1.2.3"
easy_install "sphinxcontrib-phpdomain==0.1.3.post1"

on follow the directions in the README file in the cilexer folder inside the documentation repository to install the CI Lexer.

Cabeçalhos e subtítulos de página e seção¶

Os cabeçalhos não apenas fornecem pedidos e seções dentro de uma página, mas também are used to automatically build both the page and document table of contents. Os títulos são formados usando certos caracteres como sublinhados para um pouco de text. Major headings, like page titles and section headings also use overlines. Outros títulos apenas usam sublinhados, com a seguinte hierarquia:

# with overline for page titles
* with overline for major sections
= for subsections
- for subsubsections
^ for subsubsubsections
" for subsubsubsubsections (!)

o TextMate ELDocs Bundle pode te ajudar crie estes com os seguintes gatilhos de tabulação:

title->

        ##########
        Page Title
        ##########

sec->

        *************
        Major Section
        *************

sub->

        Subsection
        ==========

sss->

        SubSubSection
        -------------

ssss->

        SubSubSubSection
        ^^^^^^^^^^^^^^^^

sssss->

        SubSubSubSubSection (!)
        """""""""""""""""""""""

Method Documentation¶

When documenting class methods for third party developers, Sphinx provides directives to assist and keep things simple. Por exemplo, considere o seguinte ReST:

.. php:class:: Some_class

        .. php:method:: some_method ( $foo [, $bar [, $bat]])

                This function will perform some action. o ``$bar`` matriz deve conter
                a something and something else, and along with ``$bat`` is an optional
                parameter.

                :param int $foo: o foo id para fazer algo em
                :param mixed $bar: A data array that must contain a something and something else
                :param bool $bat: fazer ou não fazer algo
                :returns: FALSE on failure, TRUE if successful
                :rtype: bool

                ::

                        $this->load->library('some_class');

                        $bar = array (
                                'something'             => 'Here is this parameter!',
                                'something_else'        => 42
                        );

                        $bat = $this->some_class->should_do_something();

                        if ($this->some_class->some_method(4, $bar, $bat) === FALSE)
                        {
                                show_error('An Error Occurred Doing Some Method');
                        }

                .. note:: Aqui está algo que você deve estar ciente ao usar some_method().
                                For real.

                See also :meth:`Some_class::should_do_something`


        .. php:method:: should_do_something()

                :returns: Whether or not something should be done
                :rtype: bool

It creates the following display:

Escrevendo Documentação do CodeIgniter¶

O CodeIgniter usa o Sphinx para gerar sua documentação em uma variedade de formatos, usando reStructuredText para lidar com a formatação. Se você estiver familiarizado com Markdown ou Textile, você vai rapidamente entender reStructuredText. O foco é na legibilidade e facilidade de uso. Enquanto eles podem ser bastante técnicos, nós sempre escrevemos para humanos!

Um índice local deve sempre ser incluído, como o abaixo. É criado automaticamente, inserindo o seguinte:

.. contents::
        :local:

.. raw:: html

<div class="custom-index container"></div>
  • Ferramentas necessárias
  • Cabeçalhos e subtítulos de página e seção
  • Documentação do Método

O <div> que é inserido como HTML bruto é um gancho para a documentação JavaScript para adicionar links dinamicamente a qualquer função e definição de método contido na página atual.

Ferramentas necessárias¶

Para ver o HTML renderizado, ePub, PDF, etc., você precisará instalar o Sphinx junto com a extensão de domínio PHP para o Sphinx. O requisito subjacente é ter o Python instalado. Por fim, você instalará o CI Lexer para Pygments, para que blocos de código possam ser devidamente destacados.

easy_install "sphinx==1.2.3"
easy_install "sphinxcontrib-phpdomain==0.1.3.post1"

on follow the directions in the README file in the cilexer folder dentro do repositório de documentação para instalar o CI Lexer.

Cabeçalhos e subtítulos de página e seção¶

Os cabeçalhos não apenas fornecem pedidos e seções dentro de uma página, mas também são usados para construir automaticamente a página e o índice do documento. Os títulos são formados usando certos caracteres como sublinhados para um pouco de texto. Títulos importantes, como títulos de páginas e cabeçalhos de seção também usam overlines. Outros títulos apenas usam sublinhados, com a seguinte hierarquia:

# with overline for page titles
* with overline for major sections
= for subsections
- for subsubsections
^ for subsubsubsections
" for subsubsubsubsections (!)

o TextMate ELDocs Bundle pode te ajudar crie estes com os seguintes gatilhos de tabulação:

title->

        ##########
        Page Title
        ##########

sec->

        *************
        Major Section
        *************

sub->

        Subsection
        ==========

sss->

        SubSubSection
        -------------

ssss->

        SubSubSubSection
        ^^^^^^^^^^^^^^^^

sssss->

        SubSubSubSubSection (!)
        """""""""""""""""""""""

Documentação do Método¶

Ao documentar métodos de classe para desenvolvedores de terceiros, o Sphinx fornece diretivas para ajudar e manter as coisas simples. Por exemplo, considere o seguinte ReST:

.. php:class:: Some_class

        .. php:method:: some_method ( $foo [, $bar [, $bat]])

                This function will perform some action. o ``$bar`` matriz deve conter
                uma coisa e outra coisa, e junto com ``$bat`` é um opcional
                parameter.

                :param int $foo: o foo id para fazer algo em
                :param mixado $bar: Um array de dados que deve conter algo e algo mais
                :param bool $bat: fazer ou não fazer algo
                :returns: FALSE na falha, TRUE se bem sucedido
                :rtype: bool

                ::

                        $this->load->library('some_class');

                        $bar = array (
                                'something'             => 'Here is this parameter!',
                                'something_else'        => 42
                        );

                        $bat = $this->some_class->should_do_something();

                        if ($this->some_class->some_method(4, $bar, $bat) === FALSE)
                        {
                                show_error('An Error Occurred Doing Some Method');
                        }

                .. note:: Aqui está algo que você deve estar ciente ao usar some_method().
                                For real.

                See also :meth:`Some_class :: should_do_something`


        .. php:method:: should_do_something()

                :returns: Se algo deve ou não ser feito
                :rtype: bool

Cria a seguinte exibição:

class Some_class¶
some_method($foo[, $bar[, $bat]])¶

This function will perform some action. o $bar matriz deve conter uma coisa e outra coisa, e junto com $bat é um opcional parameter.

Parâmetros:
  • $foo (int) – o foo id para fazer algo em
  • $bar (mixed) – Um array de dados que deve conter algo e algo mais
  • $bat (bool) – fazer ou não fazer algo
Retorna:

FALSE na falha, TRUE se bem sucedido
Tipo de retorno:

bool

 
$this->load->library('some_class');

$bar = array(
        'something'             => 'Here is this parameter!',
        'something_else'        => 42
);

$bat = $this->some_class->should_do_something();

if ($this->some_class->some_method(4, $bar, $bat) === FALSE)
{
        show_error('An Error Occurred Doing Some Method');
}

Nota

Aqui está algo que você deve estar ciente ao usar some_method(). For real.

See also Some_class::should_do_something()
