Criando perfil do seu aplicativo¶

The Profiler Class will display benchmark results, queries you have run, and $_POST data at the bottom of suas páginas. This information can be useful during development in order to help with debugging and optimization.

Inicializando a classe¶

Importante

Esta classe não precisa ser inicializada. É carregado automatically by the Output Library if profiling is enabled as shown below.

Ativando o Profiler¶

To enable the profiler place the following line anywhere within your Controller methods:

$this->output->enable_profiler(TRUE);

Quando ativado, um relatório será gerado e inserido na parte inferior suas páginas.

Para desabilitar o profiler, você usará:

$this->output->enable_profiler(FALSE);

Definindo pontos de referência¶

In order for the Profiler to compile and display your benchmark data you must name your mark points using specific syntax.

Por favor, leia as informações sobre como definir pontos de Referência no Benchmark Library page.

Enabling and Disabling Profiler Sections¶

Cada seção de dados do Profiler pode ser ativada ou desativada configurando corresponding config variable to TRUE or FALSE. This can be done one of two ways. First, you can set application wide defaults with the application/config/profiler.php arquivo de configuração.

Example:

$config['config']          = FALSE;
$config['queries']         = FALSE;

In your controllers, you can override the defaults and config file values by calling the set_profiler_sections() method of the Output Library:

$sections = array(
        'config'  => TRUE,
        'queries' => TRUE
);

$this->output->set_profiler_sections($sections);

Available sections and the array key used to access them are described in the table below.

Key Descrição Padrão
benchmarks Elapsed time of Benchmark points and total execution time TRUE
config CodeIgniter Config variables TRUE
controller_info A classe e o método do controlador solicitados TRUE
get Any GET data passed in the request TRUE
http_headers The HTTP headers for the current request TRUE
memory_usage Amount of memory consumed by the current request, in bytes TRUE
post Any POST data passed in the request TRUE
queries Listing of all database queries executed, including execution time TRUE
uri_string O URI do pedido atual TRUE
session_data Dados armazenados na sessão atual TRUE
query_toggle_count The number of queries after which the query block will default to hidden. 25

Nota

Disabling the save_queries setting in a configuração do seu database também desativará efetivamente o perfil consultas de database e processar a configuração de 'consultas' acima de inútil. Você pode optionally override this setting with $this->db->save_queries = TRUE;. Without this setting you won’t be able to view the queries or the last_query <database/helpers>.

Criando perfil do seu aplicativo¶

A Profiler Class exibirá os resultados do benchmark, as Querys que você executou, and $_POST data at the bottom of suas páginas. This information can be útil durante o desenvolvimento, a fim de ajudar com a depuração e optimization.

Inicializando a classe¶

Importante

Esta classe não precisa ser inicializada. É carregado automaticamente pelo Library de saída se a criação de perfil estiver ativada, conforme mostrado abaixo.

Ativando o Profiler¶

Para habilitar o profiler, coloque a seguinte linha em qualquer lugar dentro do seu Controller methods:

$this->output->enable_profiler(TRUE);

Quando ativado, um relatório será gerado e inserido na parte inferior suas páginas.

Para desabilitar o profiler, você usará:

$this->output->enable_profiler(FALSE);

Definindo pontos de referência¶

Para que o Profiler possa compilar e exibir seus dados de benchmark você deve nomear seus pontos de marcação usando uma sintaxe específica.

Por favor, leia as informações sobre como definir pontos de Referência no Library de referência page.

Ativando e Desativando Seções do Profiler¶

Cada seção de dados do Profiler pode ser ativada ou desativada configurando variável de configuração correspondente para TRUE ou FALSE. Isso pode ser feito dois caminhos. Primeiro, você pode definir padrões de largura de aplicativo com o application/config/profiler.php arquivo de configuração.

Exemplo:

$config['config']          = FALSE;
$config['queries']         = FALSE;

Em seus Controlleres, você pode substituir os padrões e o arquivo de configuração values by calling the set_profiler_sections() method of the Library de saída:

$sections = array(
        'config'  => TRUE,
        'queries' => TRUE
);

$this->output->set_profiler_sections($sections);

As seções disponíveis e a chave de matriz usada para acessá-las são descritas na tabela abaixo.

Key Descrição Padrão
benchmarks Tempo decorrido de pontos de referência e tempo total de execução TRUE
config CodeIgniter Config variables TRUE
controller_info A classe e o método do Controller solicitados TRUE
get Qualquer dado GET passado na solicitação TRUE
http_headers Os cabeçalhos HTTP para o pedido atual TRUE
memory_usage Quantidade de memória consumida pela solicitação atual, em bytes TRUE
post Qualquer dado do POST passado na solicitação TRUE
queries Listagem de todas as Querys do Database executadas, incluindo o tempo de execução TRUE
uri_string O URI do pedido atual TRUE
session_data Dados armazenados na sessão atual TRUE
query_toggle_count O número de Querys após o qual o bloco de Query será padronizado para hidden. 25

Nota

Desativando o save_queries estabelecendo a configuração do seu database também desativará efetivamente o perfil Querys de database e processar a configuração de 'Querys' acima de inútil. Você pode optionally override this setting with $this->db->save_queries = TRUE;. Sem essa configuração, você não poderá visualizar as Querys ou o last_query <database/helpers>.

Criando perfil do seu aplicativo¶

A Profiler Class exibirá os resultados do benchmark, as Querys que você executou, and $_POST data at the bottom of suas páginas. This information can be útil durante o desenvolvimento, a fim de ajudar com a depuração e optimization.

Inicializando a classe¶

Important

Esta classe não precisa ser inicializada. É carregado automaticamente pelo Library de saída se a criação de perfil estiver ativada, conforme mostrado abaixo.

Ativando o Profiler¶

Para habilitar o profiler, coloque a seguinte linha em qualquer lugar dentro do seu Controller methods:

$this->output->enable_profiler(TRUE);

Quando ativado, um relatório será gerado e inserido na parte inferior suas páginas.

Para desabilitar o profiler, você usará:

$this->output->enable_profiler(FALSE);

Definindo pontos de referência¶

Para que o Profiler possa compilar e exibir seus dados de benchmark você deve nomear seus pontos de marcação usando uma sintaxe específica.

Por favor, leia as informações sobre como definir pontos de Referência no Library de referência page.

Ativando e Desativando Seções do Profiler¶

Cada seção de dados do Profiler pode ser ativada ou desativada configurando variável de configuração correspondente para TRUE ou FALSE. Isso pode ser feito dois caminhos. Primeiro, você pode definir padrões de largura de aplicativo com o application/config/profiler.php arquivo de configuração.

Exemplo:

$config['config']          = FALSE;
$config['queries']         = FALSE;

Em seus Controlleres, você pode substituir os padrões e o arquivo de configuração values by calling the set_profiler_sections() method of the Library de saída:

$sections = array(
        'config'  => TRUE,
        'queries' => TRUE
);

$this->output->set_profiler_sections($sections);

As seções disponíveis e a chave de matriz usada para acessá-las são descritas na tabela abaixo.

Key Descrição Default
benchmarks Tempo decorrido de pontos de referência e tempo total de execução TRUE
config Variáveis de configuração do CodeIgniter TRUE
controller_info A classe e o método do Controller solicitados TRUE
get Qualquer dado GET passado na solicitação TRUE
http_headers Os cabeçalhos HTTP para o pedido atual TRUE
memory_usage Quantidade de memória consumida pela solicitação atual, em bytes TRUE
post Qualquer dado do POST passado na solicitação TRUE
queries Listagem de todas as Querys do Database executadas, incluindo o tempo de execução TRUE
uri_string O URI do pedido atual TRUE
session_data Dados armazenados na sessão atual TRUE
query_toggle_count O número de Querys após o qual o bloco de Query será padronizado para hidden. 25

Nota

Desativando o save_queries setting in a configuração do seu database também desativará efetivamente o perfil Querys de database e processar a configuração de 'Querys' acima de inútil. Você pode optionally override this setting with $this->db->save_queries = TRUE;. Sem essa configuração, você não poderá visualizar as Querys ou o last_query <Database/helpers>.
