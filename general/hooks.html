Ganchos - Estendendo o Núcleo da Estrutura¶

CodeIgniter’s Hooks feature provides a means to tap into and modify the inner workings of the framework without hacking the core files. When CodeIgniter runs it follows a specific execution process, diagramed in the Application Flow page. There may be instâncias, no entanto, onde você gostaria de causar alguma ação a ter lugar em um estágio específico do processo de execução. Por exemplo, você pode deseja executar um script antes de seus controladores serem carregados ou after, or you might want to trigger one of your own scripts in some outro local.

Enabling Hooks¶

The hooks feature can be globally enabled/disabled by setting the item seguinte no application/config/config.php file:

$config['enable_hooks'] = TRUE;

Definindo um Gancho¶

Hooks are defined in the application/config/hooks.php file. Each hook is specified as an array with this prototype:

$hook['pre_controller'] = array(
        'class'    => 'MyClass',
        'function' => 'Myfunction',
        'filename' => 'Myclass.php',
        'filepath' => 'hooks',
        'params'   => array('beer', 'wine', 'snacks')
);

Notes:

The array index correlates to the name of the particular hook point you quer usar. No exemplo acima, o ponto de gancho é pre_controller. UMA lista de pontos de gancho é encontrada abaixo. Os itens a seguir devem ser definido em seu array de gancho associativo:

  • class O nome da classe que você deseja invocar. Se você preferir use uma função procedural em vez de uma classe, deixe este item em branco.
  • function O nome da função (ou método) que você deseja chamar.
  • filename O nome do arquivo que contém sua classe/função.
  • filepath The name of the directory containing your script. Nota: Seu script deve estar localizado em um diretório INSIDE your application/ diretório, de modo que o caminho do arquivo é relativo a esse diretório. Por exemplo, if your script is located in application/hooks/, you will simply use ‘hooks’ as your filepath. If your script is located in application/hooks/utilities/ you will use ‘hooks/utilities’ as your filepath. No trailing slash.
  • params Any parameters you wish to pass to your script. This item é opcional.

Você também pode usar funções lambda/anoymous (ou fechamentos) como ganchos, com a simpler syntax:

$hook['post_controller'] = function()
{
        /* do something here */
};

Multiple Calls to the Same Hook¶

If want to use the same hook point with more than one script, simply faça sua declaração de array multi-dimensional, assim:

$hook['pre_controller'][] = array(
        'class'    => 'MyClass',
        'function' => 'MyMethod',
        'filename' => 'Myclass.php',
        'filepath' => 'hooks',
        'params'   => array('beer', 'wine', 'snacks')
);

$hook['pre_controller'][] = array(
        'class'    => 'MyOtherClass',
        'function' => 'MyOtherMethod',
        'filename' => 'Myotherclass.php',
        'filepath' => 'hooks',
        'params'   => array('red', 'yellow', 'blue')
);

Notice the brackets after each array index:

$hook['pre_controller'][]

Isso permite que você tenha o mesmo ponto de gancho com vários scripts. o order you define your array will be the execution order.

Hook Points¶

A seguir, uma lista de pontos de gancho disponíveis.

  • pre_system Called very early during system execution. Only the benchmark and A classe de ganchos foi carregada neste momento. Nenhum roteamento ou outro processes have happened.
  • pre_controller Called immediately prior to any of your controllers being called. All base classes, routing, and security checks have been done.
  • post_controller_constructor Chamado imediatamente após o controlador ser instanciado, mas antes to any method calls happening.
  • post_controller Called immediately after your controller is fully executed.
  • display_override Overrides the _display() method, used to send the finalized page para o navegador da Web no final da execução do sistema. Isso permite que você para usar sua própria metodologia de exibição. Note que você precisará reference the CI superobject with $this->CI =& get_instance() and then the finalized data will be available by calling $this->CI->output->get_output().
  • cache_override Permite que você chame seu próprio método em vez do _display_cache() método no Output Library. This permits você usar seu próprio mecanismo de exibição de cache.
  • post_system Called after the final rendered page is sent to the browser, at the final da execução do sistema após os dados finalizados serem enviados para o browser.

Ganchos - Estendendo o Núcleo da Estrutura¶

O recurso Ganchos do CodeIgniter fornece um meio de acessar e modificar o funcionamento interno do framework sem invadir os arquivos principais. Quando O CodeIgniter é executado seguindo um processo de execução específico, the Fluxo de Aplicação página. Pode ser instâncias, no entanto, onde você gostaria de causar alguma ação a ter lugar em um estágio específico do processo de execução. Por exemplo, você pode deseja executar um script antes de seus Controlleres serem carregados ou depois, ou você pode querer acionar um dos seus próprios scripts em algum outro local.

Habilitando Ganchos¶

O recurso de ganchos pode ser ativado/desativado globalmente item seguinte no application/config/config.php file:

$config['enable_hooks'] = TRUE;

Definindo um Gancho¶

Ganchos são definidos no application/config/hooks.php file. Cada gancho é especificado como um array com este protótipo:

$hook['pre_controller'] = array(
        'class'    => 'MyClass',
        'function' => 'Myfunction',
        'filename' => 'Myclass.php',
        'filepath' => 'hooks',
        'params'   => array('beer', 'wine', 'snacks')
);

Notas:

O índice da matriz está correlacionado ao nome do ponto de gancho específico que você quer usar. No exemplo acima, o ponto de gancho é pre_controller. UMA lista de pontos de gancho é encontrada abaixo. Os itens a seguir devem ser definido em seu array de gancho associativo:

  • class O nome da classe que você deseja invocar. Se você preferir use uma função procedural em vez de uma classe, deixe este item em branco.
  • function O nome da função (ou método) que você deseja chamar.
  • filename O nome do arquivo que contém sua classe/função.
  • filepath O nome do diretório que contém seu script. Note: Seu script deve estar localizado em um diretório INSIDE your application/ diretório, de modo que o caminho do arquivo é relativo a esse diretório. Por exemplo, if your script is located in application/hooks/, you will simply use 'ganchos' como seu filepath. Se o seu script estiver localizado em application/hooks/utilities/ você usará 'hooks/utilities' como seu caminho de arquivo. Nenhuma barra final.
  • params Quaisquer parâmetros que você deseja passar para o seu script. Este item é opcional.

Você também pode usar funções lambda/anoymous (ou fechamentos) como ganchos, com uma sintaxe mais simples:

$hook['post_controller'] = function()
{
        /* do something here */
};

Múltiplas chamadas para o mesmo gancho¶

Se quiser usar o mesmo ponto de gancho com mais de um script, simplesmente faça sua declaração de array multi-dimensional, assim:

$hook['pre_controller'][] = array(
        'class'    => 'MyClass',
        'function' => 'MyMethod',
        'filename' => 'Myclass.php',
        'filepath' => 'hooks',
        'params'   => array('beer', 'wine', 'snacks')
);

$hook['pre_controller'][] = array(
        'class'    => 'MyOtherClass',
        'function' => 'MyOtherMethod',
        'filename' => 'Myotherclass.php',
        'filepath' => 'hooks',
        'params'   => array('red', 'yellow', 'blue')
);

Observe os colchetes após cada índice de matriz:

$hook['pre_controller'][]

Isso permite que você tenha o mesmo ponto de gancho com vários scripts. o ordem que você define seu array será a ordem de execução.

Pontos de Gancho¶

A seguir, uma lista de pontos de gancho disponíveis.

  • pre_system Chamado muito cedo durante a execução do sistema. Apenas o benchmark e A classe de ganchos foi carregada neste momento. Nenhum roteamento ou outro processos aconteceram.
  • pre_controller Chamado imediatamente antes de qualquer dos seus Controlleres serem chamados. Todas as classes base, roteamento e verificações de segurança foram feitas.
  • post_controller_constructor Chamado imediatamente após o Controller ser instanciado, mas antes para qualquer método chama acontecendo.
  • post_controller Chamado imediatamente após o seu Controller ser totalmente executado.
  • display_override Overrides the _display() method, used to send the finalized page para o navegador da Web no final da execução do sistema. Isso permite que você para usar sua própria metodologia de exibição. Note que você precisará reference the CI superobject with $this->CI =& get_instance() and então os dados finalizados estarão disponíveis chamando $this->CI->output->get_output().
  • cache_override Permite que você chame seu próprio método em vez do _display_cache() método no Library de saída. Isso permite você usar seu próprio mecanismo de exibição de cache.
  • post_system Chamado depois que a página processada final é enviada ao navegador, no final da execução do sistema após os dados finalizados serem enviados para o browser.

Ganchos - Estendendo o Núcleo da Estrutura¶

O recurso Ganchos do CodeIgniter fornece um meio de acessar e modificar o funcionamento interno do framework sem invadir os arquivos principais. Quando O CodeIgniter é executado seguindo um processo de execução específico, the Fluxo de Aplicação página. Pode ser instâncias, no entanto, onde você gostaria de causar alguma ação a ter lugar em um estágio específico do processo de execução. Por exemplo, você pode deseja executar um script antes de seus Controlleres serem carregados ou depois, ou você pode querer acionar um dos seus próprios scripts em algum outro local.

Habilitando Ganchos¶

O recurso de ganchos pode ser ativado/desativado globalmente item seguinte no application/config/config.php file:

$config['enable_hooks'] = TRUE;

Definindo um Gancho¶

Ganchos são definidos no application/config/hooks.php file. Cada gancho é especificado como um array com este protótipo:

$hook['pre_controller'] = array(
        'class'    => 'MyClass',
        'function' => 'Myfunction',
        'filename' => 'Myclass.php',
        'filepath' => 'hooks',
        'params'   => array('beer', 'wine', 'snacks')
);

Notes:

O índice da matriz está correlacionado ao nome do ponto de gancho específico que você quer usar. No exemplo acima, o ponto de gancho é pre_controller. UMA lista de pontos de gancho é encontrada abaixo. Os itens a seguir devem ser definido em seu array de gancho associativo:

  • class O nome da classe que você deseja invocar. Se você preferir use uma função procedural em vez de uma classe, deixe este item em branco.
  • function O nome da função (ou método) que você deseja chamar.
  • filename O nome do arquivo que contém sua classe/função.
  • filepath O nome do diretório que contém seu script. Note: Seu script deve estar localizado em um diretório INSIDE your application/ diretório, de modo que o caminho do arquivo é relativo a esse diretório. Por exemplo, if your script is located in application/hooks/, you will simply use 'ganchos' como seu filepath. Se o seu script estiver localizado em application/hooks/utilities/ você usará 'hooks/utilities' como seu caminho de arquivo. Nenhuma barra final.
  • params Quaisquer parâmetros que você deseja passar para o seu script. Este item é opcional.

Você também pode usar funções lambda/anoymous (ou fechamentos) como ganchos, com uma sintaxe mais simples:

$hook['post_controller'] = function()
{
        /* do something here */
};

Múltiplas chamadas para o mesmo gancho¶

Se quiser usar o mesmo ponto de gancho com mais de um script, simplesmente faça sua declaração de array multi-dimensional, assim:

$hook['pre_controller'][] = array(
        'class'    => 'MyClass',
        'function' => 'MyMethod',
        'filename' => 'Myclass.php',
        'filepath' => 'hooks',
        'params'   => array('beer', 'wine', 'snacks')
);

$hook['pre_controller'][] = array(
        'class'    => 'MyOtherClass',
        'function' => 'MyOtherMethod',
        'filename' => 'Myotherclass.php',
        'filepath' => 'hooks',
        'params'   => array('red', 'yellow', 'blue')
);

Observe os colchetes após cada índice de matriz:

$hook['pre_controller'][]

Isso permite que você tenha o mesmo ponto de gancho com vários scripts. o ordem que você define seu array será a ordem de execução.

Pontos de Gancho¶

A seguir, uma lista de pontos de gancho disponíveis.

  • pre_system Chamado muito cedo durante a execução do sistema. Apenas o benchmark e A classe de ganchos foi carregada neste momento. Nenhum roteamento ou outro processos aconteceram.
  • pre_controller Chamado imediatamente antes de qualquer dos seus Controlleres serem chamados. Todas as classes base, roteamento e verificações de segurança foram feitas.
  • post_controller_constructor Chamado imediatamente após o Controller ser instanciado, mas antes para qualquer método chama acontecendo.
  • post_controller Chamado imediatamente após o seu Controller ser totalmente executado.
  • display_override Overrides the _display() method, used to send the finalized page para o navegador da Web no final da execução do sistema. Isso permite que você para usar sua própria metodologia de exibição. Note que você precisará reference the CI superobject with $this->CI =& get_instance() and então os dados finalizados estarão disponíveis chamando $this->CI->output->get_output().
  • cache_override Permite que você chame seu próprio método em vez do _display_cache() método no Library de saída. Isso permite você usar seu próprio mecanismo de exibição de cache.
  • post_system Chamado depois que a página processada final é enviada ao navegador, no final da execução do sistema após os dados finalizados serem enviados para o browser.
