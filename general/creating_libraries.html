Criando Librarys¶

When we use the term “Libraries” we are normally referring to the classes that are located in the libraries directory and described in the Class Reference of this user guide. In this case, however, we will instead describe how you can create your own libraries within your application/libraries directory in order to maintain separation between your local resources and the global framework resources.

As an added bonus, CodeIgniter permits your libraries to extend native classes if you simply need to add some functionality to an existing library. Ou você pode até mesmo substituir librarys nativas apenas colocando identically named versions in your application/libraries directory.

In summary:

  • You can create entirely new libraries.
  • You can extend native libraries.
  • Você pode substituir librarys nativas.

The page below explains these three concepts in detail.

Nota

The Database classes can not be extended or replaced with your own classes. All other classes are able to be replaced/extended.

Armazenamento¶

Your library classes should be placed within your application/libraries diretório, pois é onde o CodeIgniter irá procurá-los quando estiverem initialized.

Convenções de Nomenclatura¶

  • Nomes de arquivos devem ser capitalizados. Por exemplo: Myclass.php
  • Declarações de classe devem ser capitalizadas. Por exemplo: class Myclass
  • Nomes de classes e nomes de arquivos devem corresponder.

The Class File¶

As classes devem ter este protótipo básico:

<?php
defined('BASEPATH') OR exit('No direct script access allowed');

class Someclass {

        public function some_method()
        {
        }
}

Nota

We are using the name Someclass purely as an example.

Usando sua classe¶

De dentro de qualquer um dos seus Controller métodos que você pode inicializar sua classe usando o padrão:

$this->load->library('someclass');

Where someclass is the file name, without the “.php” file extension. You can submit the file name capitalized or lower case. CodeIgniter não se importa.

Once loaded you can access your class using the lower case version:

$this->someclass->some_method();  // Object instances will always be lower case

Passando Parâmetros ao Inicializar sua Classe¶

In the library loading method you can dynamically pass data as an array via the second parameter and it will be passed to your class constructor:

$params = array('type' => 'large', 'color' => 'red');

$this->load->library('someclass', $params);

If you use this feature you must set up your class constructor to expect data:

<?php defined('BASEPATH') OR exit('No direct script access allowed');

class Someclass {

        public function __construct($params)
        {
                // Do something with $params
        }
}

You can also pass parameters stored in a config file. Simply create a arquivo de configuração nomeado de forma idêntica ao nome do arquivo de classe e armazená-lo em your application/config/ directory. Nota that if you dynamically pass parâmetros como descrito acima, a opção de arquivo de configuração não será available.

Utilizing CodeIgniter Resources within Your Library¶

Para acessar os recursos nativos do CodeIgniter em sua library, use o get_instance() method. This method returns the CodeIgniter super object.

Normally from within your controller métodos que você will call any of the available CodeIgniter methods using the $this construct:

$this->load->helper('url');
$this->load->library('session');
$this->config->item('base_url');
// etc.

$this, however, only works directly within your controllers, your modelos ou suas visualizações. Se você gostaria de usar as classes do CodeIgniter from within your own custom classes you can do so as follows:

Primeiro, atribua o objeto CodeIgniter a uma variável:

$CI =& get_instance();

Once you’ve assigned the object to a variable, you’ll use that variable instead of $this:

$CI =& get_instance();

$CI->load->helper('url');
$CI->load->library('session');
$CI->config->item('base_url');
// etc.

Nota

You’ll notice that the above get_instance() function is being passado por referência:

$CI =& get_instance();

This is very important. Assigning by reference allows you to use the original CodeIgniter object rather than creating a copy of it.

However, since a library is a class, it would be better if you aproveitar ao máximo os princípios da POO. Então, para ser capaz de usar o superobjeto do CodeIgniter em toda a classe métodos, você é encorajado a atribuí-lo a uma propriedade em vez disso:

class Example_library {

        protected $CI;

        // We'll use a constructor, as you can't directly call a function
        // from a property definition.
        public function __construct()
        {
                // Assign the CodeIgniter super-object
                $this->CI =& get_instance();
        }

        public function foo()
        {
                $this->CI->load->helper('url');
                redirect();
        }

        public function bar()
        {
                echo $this->CI->config->item('base_url');
        }

}

Substituindo Librarys Nativas por Suas Versões¶

Simply by naming your class files identically to a native library will causar CodeIgniter para usá-lo em vez do nativo. Para usar isso feature you must name the file and the class declaration exactly the same as the native library. For example, to replace the native Email library you’ll create a file named application/libraries/Email.php, e declare sua turma com:

class CI_Email {

}

Nota that most native classes are prefixed with CI_.

Para carregar sua library, você verá o método de carregamento padrão:

$this->load->library('email');

Nota

At this time the Database classes can not be replaced with suas próprias versões.

Extending Native Libraries¶

Se tudo que você precisa fazer é adicionar alguma funcionalidade a uma library existente - perhaps add a method or two - then it’s overkill to replace the entire library com a sua versão. Neste caso, é melhor simplesmente estender o class. Extending a class is nearly identical to replacing a class with a couple exceptions:

  • The class declaration must extend the parent class.
  • Your new class name and filename must be prefixed with MY_ (this item is configurable. See below.).

For example, to extend the native Email class you’ll create a file named application/libraries/MY_Email.php, e declare sua turma com:

class MY_Email extends CI_Email {

}

Se você precisa usar um construtor em sua classe, certifique-se estenda o construtor pai:

class MY_Email extends CI_Email {

        public function __construct($config = array())
        {
                parent::__construct($config);
                // Your own constructor code
        }

}

Nota

Not all of the libraries have the same (or any) parameters in their constructor. Take a look at the library that you’re extending first to see how it should be implemented.

Carregando sua subclasse¶

Para carregar sua subclasse você usará a sintaxe padrão normalmente usada. FAZ NOT include your prefix. For example, to load the example above, which estende a classe Email, você usará:

$this->load->library('email');

Uma vez carregado, você usará a variável de classe como faria normalmente the class you are extending. In the case of the email class all calls will use:

$this->email->some_method();

Setting Your Own Prefix¶

Para definir o seu próprio prefixo de subclasse, abra o seu application/config/config.php arquivo e procure por este item:

$config['subclass_prefix'] = 'MY_';

Please note that all native CodeIgniter libraries are prefixed with CI_ então não use isso como seu prefixo.

        }

Você também pode passar parâmetros armazenados em um arquivo de configuração. Basta criar um arquivo de configuração nomeado de forma idêntica ao nome do arquivo de classe e armazená-lo em your application/config/ directory. Nota that if you dynamically pass parâmetros como descrito acima, a opção de arquivo de configuração não será available.

Depois de atribuir o objeto a uma variável, você usará essa variável instead of $this:

$CI =& get_instance();

$CI->load->helper('url');
$CI->load->library('session');
$CI->config->item('base_url');
// etc.

Nota

You’ll notice that the above get_instance() function is being passado por referência:

$CI =& get_instance();

Isto é muito importante. Atribuir por referência permite usar o objeto CodeIgniter original, em vez de criar uma cópia dele.

No entanto, como uma Library é uma Classe, seria melhor se você aproveitar ao máximo os princípios da POO. Então, para ser capaz de usar o superobjeto do CodeIgniter em toda a classe métodos, você é encorajado a atribuí-lo a uma propriedade em vez disso:

class Example_library {

        protected $CI;

        // We'll use a constructor, as you can't directly call a function
        // from a property definition.
        public function __construct()
        {
                // Assign the CodeIgniter super-object
                $this->CI =& get_instance();
        }

        public function foo()
        {
                $this->CI->load->helper('url');
                redirect();
        }

        public function bar()
        {
                echo $this->CI->config->item('base_url');
        }

}

Substituindo Librarys Nativas por Suas Versões¶

Simplesmente nomeando seus arquivos de classe de forma idêntica a uma Library nativa causar CodeIgniter para usá-lo em vez do nativo. Para usar isso característica que você deve nomear o arquivo e a declaração de classe exatamente o mesmo que a Library nativa. Por exemplo, para substituir o email nativo library you’ll create a file named application/libraries/Email.php, e declare sua turma com:

class CI_Email {

}

Observe que a maioria das classes nativas é prefixada com CI_.

Para carregar sua library, você verá o método de carregamento padrão:

$this->load->library('email');

Nota

Neste momento, as classes de Database não podem ser substituídas por suas próprias versões.

Estendendo Librarys nativas¶

Se tudo que você precisa fazer é adicionar alguma funcionalidade a uma library existente - talvez adicione um método ou dois - então é um exagero substituir todo o library com a sua versão. Neste caso, é melhor simplesmente estender o classe. Estender uma turma é quase idêntico a substituir uma turma por uma algumas exceções:

  • A declaração de classe deve estender a classe pai.
  • Seu novo nome de classe e nome de arquivo devem ser prefixados com MY_ item é configurável. Ver abaixo.).

Por exemplo, para estender a classe Email nativa, você criará um arquivo chamado application/libraries/MY_Email.php, e declare sua turma com:

class MY_Email extends CI_Email {

}

Se você precisa usar um construtor em sua classe, certifique-se estenda o construtor pai:

class MY_Email extends CI_Email {

        public function __construct($config = array())
        {
                parent::__construct($config);
                // Your own constructor code
        }

}

Nota

Carregando sua subclasse¶

Para carregar sua subclasse você usará a sintaxe padrão normalmente usada. FAZ NÃO inclua seu prefixo. Por exemplo, para carregar o exemplo acima, que estende a classe Email, você usará:

$this->load->library('email');

Uma vez carregado, você usará a variável de classe como faria normalmente a classe que você está estendendo. No caso da classe de email, todas as chamadas usará:

$this->email->some_method();

Definindo seu próprio prefixo¶

Para definir o seu próprio prefixo de subclasse, abra o seu application/config/config.php arquivo e procure por este item:

$config['subclass_prefix'] = 'MY_';

Criando Librarys¶

Quando usamos o termo "Librarys" normalmente estamos nos referindo ao classes que estão localizadas no diretório de Librarys e descritas no Referência de classe deste guia do usuário. Neste caso, no entanto, nós iremos em vez descrever como você pode criar suas próprias Librarys dentro do seu diretório de aplicativos/Librarys, a fim de manter a separação entre seus recursos locais e os recursos globais do framework.

Como um bônus adicional, o CodeIgniter permite que suas Librarys estendam classes, se você simplesmente precisa adicionar alguma funcionalidade a um existente library. Ou você pode até mesmo substituir librarys nativas apenas colocando identically named versions in your application/libraries directory.

Em suma:

  • Você pode criar Librarys totalmente novas.
  • Você pode estender Librarys nativas.
  • Você pode substituir librarys nativas.

A página abaixo explica esses três conceitos detalhadamente.

Nota

As classes do Database não podem ser estendidas ou substituídas próprias Classes. Todas as outras classes podem ser substituídas/estendidas.

Storage¶

Suas Classes da Library devem ser colocadas dentro do seu application/libraries diretório, pois é onde o CodeIgniter irá procurá-los quando estiverem initialized.

Convenções de Nomenclatura¶

  • Nomes de arquivos devem ser capitalizados. Por exemplo: Myclass.php
  • Declarações de classe devem ser capitalizadas. Por exemplo: class Myclass
  • Nomes de classes e nomes de arquivos devem corresponder.

O arquivo de classe¶

As classes devem ter este protótipo básico:

<?php
defined('BASEPATH') OR exit('No direct script access allowed');

class Someclass {

        public function some_method()
        {
        }
}

Nota

Estamos usando o nome Someclass apenas como exemplo.

Usando sua classe¶

De dentro de qualquer um dos seus Controller métodos que você pode inicializar sua classe usando o padrão:

$this->load->library('someclass');

Where someclass is the file name, without the “.php” file extension. Você pode enviar o nome do arquivo em letras maiúsculas ou minúsculas. CodeIgniter não se importa.

Uma vez carregado, você pode acessar sua turma usando a versão minúscula:

$this->someclass->some_method();  // Object instances will always be lower case

Passando Parâmetros ao Inicializar sua Classe¶

No método de carregamento da Library, você pode passar dinamicamente dados como um matriz através do segundo parâmetro e será passado para sua classe constructor:

$params = array('type' => 'large', 'color' => 'red');

$this->load->library('someclass', $params);

Se você usar esse recurso, deverá configurar seu construtor de classe para esperar data:

<?php defined('BASEPATH') OR exit('No direct script access allowed');

class Someclass {

        public function __construct($params)
        {
                // Do something with $params
        }
}

Você também pode passar parâmetros armazenados em um arquivo de configuração. Basta criar um arquivo de configuração nomeado de forma idêntica ao nome do arquivo de classe e armazená-lo em your application/config/ directory. Nota that if you dynamically pass parâmetros como descrito acima, a opção de arquivo de configuração não será available.

Utilizando os Recursos do CodeIgniter em sua Library¶

Para acessar os recursos nativos do CodeIgniter em sua library, use o get_instance() método. Este método retorna o super do CodeIgniter object.

Normally from within your controller métodos que você will call any of the available CodeIgniter methods using the $this construct:

$this->load->helper('url');
$this->load->library('session');
$this->config->item('base_url');
// etc.

$this, no entanto, só funciona diretamente dentro de seus Controlleres, Models ou suas visualizações. Se você gostaria de usar as classes do CodeIgniter de dentro de suas próprias classes personalizadas, você pode fazer isso da seguinte maneira:

Primeiro, atribua o objeto CodeIgniter a uma variável:

$CI =& get_instance();

Depois de atribuir o objeto a uma variável, você usará essa variável instead of $this:

$CI =& get_instance();

$CI->load->helper('url');
$CI->load->library('session');
$CI->config->item('base_url');
// etc.

Nota

You’ll notice that the above get_instance() function is being passado por referência:

$CI =& get_instance();

Isto é muito importante. Atribuir por referência permite usar o objeto CodeIgniter original, em vez de criar uma cópia dele.

No entanto, como uma Library é uma Classe, seria melhor se você aproveitar ao máximo os princípios da POO. Então, para ser capaz de usar o superobjeto do CodeIgniter em toda a classe métodos, você é encorajado a atribuí-lo a uma propriedade em vez disso:

class Example_library {

        protected $CI;

        // We'll use a constructor, as you can't directly call a function
        // from a property definition.
        public function __construct()
        {
                // Assign the CodeIgniter super-object
                $this->CI =& get_instance();
        }

        public function foo()
        {
                $this->CI->load->helper('url');
                redirect();
        }

        public function bar()
        {
                echo $this->CI->config->item('base_url');
        }

}
