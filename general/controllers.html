Controllers¶

Controllers are the heart of your application, as they determine how HTTP requests should be handled.

What is a Controller?¶

Um Controller é simplesmente um arquivo de classe que é nomeado de uma forma que pode ser associated with a URI.

Considere este URI:

example.com/index.php/blog/

In the above example, CodeIgniter would attempt to find a controller chamado Blog.php e carregá-lo.

When a controller’s name matches the first segment of a URI, it will be loaded.

Vamos tentar: Olá Mundo!¶

Vamos criar um controlador simples para que você possa vê-lo em ação. Usando seu editor de texto, crie um arquivo chamado Blog.php e coloque o seguinte código in it:

<?php
class Blog extends CI_Controller {

        public function index()
        {
                echo 'Olá Mundo!';
        }
}

Then save the file to your application/controllers/ directory.

Importante

The file must be called ‘Blog.php’, with a capital ‘B’.

Agora visite o seu site usando um URL semelhante a este:

example.com/index.php/blog/

If you did it right, you should see:

Olá Mundo!

Importante

Nomes de classes devem começar com uma letra maiúscula.

Isso é válido:

<?php
class Blog extends CI_Controller {

}

This is not valid:

<?php
class blog extends CI_Controller {

}

Além disso, certifique-se sempre de que seu controlador estende o controlador pai class para que ele possa herdar todos os seus métodos.

Methods¶

In the above example the method name is index(). The “index” method é sempre carregado por padrão se o second segment of the URI is empty. Another way to show your “Hello World” message would be this:

example.com/index.php/blog/index/

The second segment of the URI determines which method no controller gets called.

Let’s try it. Add a new method to your controller:

<?php
class Blog extends CI_Controller {

        public function index()
        {
                echo 'Olá Mundo!';
        }

        public function comments()
        {
                echo 'Look at this!';
        }
}

Agora, carregue o seguinte URL para ver o método de comentários:

example.com/index.php/blog/comments/

You should see your new message.

Passing URI Segments to your methods¶

If your URI contains more than two segments they will be passed to your method as parameters.

For example, let’s say you have a URI like this:

example.com/index.php/products/shoes/sandals/123

Your method will be passed URI segments 3 and 4 (“sandals” and “123”):

<?php
class Products extends CI_Controller {

        public function shoes($sandals, $id)
        {
                echo $sandals;
                echo $id;
        }
}

Importante

Se você estiver usando o Roteamento de URI feature, the segments passed to your method will be the re-routed ones.

Definindo um Controlador Padrão¶

O CodeIgniter pode ser instruído a carregar um controlador padrão quando um URI não é present, as will be the case when only your site root URL is requested. Para especificar um controlador padrão, abra o application/config/routes.php file and set this variable:

$route['default_controller'] = 'blog';

Where ‘blog’ is the name of the controller class you want used. If you now carregue seu arquivo index.php principal sem especificar nenhum segmento URI see your “Hello World” message by default.

For more information, please refer to the “Reserved Routes” section of the Roteamento de URI documentation.

Remapping Method Calls¶

Como mencionado acima, o segundo segmento da URI normalmente determina quais method no controller gets called. CodeIgniter permits you to override this behavior through the use of the _remap() method:

public function _remap()
{
        // Some code here...
}

Importante

If your controller contains a method named _remap(), it will always ser chamado independentemente do que o seu URI contém. isto overrides the normal behavior in which the URI determines which method is called, allowing you to define your own method routing rules.

The overridden method call (typically the second segment of the URI) will be passed as a parameter to the _remap() method:

public function _remap($method)
{
        if ($method === 'some_method')
        {
                $this->$method();
        }
        else
        {
                $this->default_method();
        }
}

Any extra segments after the method name are passed into _remap() as an segundo parâmetro opcional. Esta matriz pode ser usada em combinação com PHP’s call_user_func_array() para emular o comportamento padrão do CodeIgniter.

Example:

public function _remap($method, $params = array())
{
        $method = 'process_'.$method;
        if (method_exists($this, $method))
        {
                return call_user_func_array(array($this, $method), $params);
        }
        show_404();
}

Processing Output¶

CodeIgniter has an output class that takes care of sending your final dados renderizados para o navegador da web automaticamente. Mais informações sobre isso can be found no Views and Classe de saída Páginas. Em alguns casos, no entanto, você pode querer post-process the finalized data in some way and send it to the browser yourself. CodeIgniter permits you to add a method named _output() ao seu controlador que receberá os dados de saída finalizados.

Importante

If your controller contains a method named _output(), it will always ser chamado pela classe de saída em vez de echoing the finalized data directly. The first parameter of the method will contano finalized output.

Aqui está um exemplo:

public function _output($output)
{
        echo $output;
}

Nota

Please note that your _output() method will receive the data in its finalized state. Benchmark and memory usage data será processado, arquivos de cache gravados (se você tiver cache enabled), and headers will be sent (if you use that feature) before it is handed off to the _output() method. To have your controller’s output cached properly, its _output() method can use:

if ($this->output->cache_expiration > 0)
{
        $this->output->_write_cache($output);
}

If you are using this feature the page execution timer and memory usage stats might not be perfectly accurate since they não levará em consideração qualquer outro processamento que você fizer. For an alternate way to control output before any of the final processing is done, please see the available methods no Output Library.

Private methods¶

Em alguns casos, você pode querer alguns métodos ocultos do acesso público. In order to achieve this, simply declare the method as being private ou protegido e não será veiculado por meio de uma solicitação de URL. Por exemplo, if you were to have a method like this:

private function _utility()
{
        // some code
}

Tentar acessá-lo por meio do URL, assim, não funcionará:

example.com/index.php/blog/_utility/

Nota

Prefixing method names with an underscore will also prevent them from being called. This is a legacy feature that is left para compatibilidade com versões anteriores.

Organizando seus controladores em subdiretórios¶

Se você estiver criando um aplicativo grande, talvez queira hierarquicamente organize ou estruture seus controladores em subdiretórios. CodeIgniter permite que você faça isso.

Simply create sub-directories under the main application/controllers/ one and place your controller classes withnom.

Nota

Ao usar esse recurso, o primeiro segmento do seu URI deve especifique a pasta. Por exemplo, digamos que você tenha um controlador localizado here:

application/controllers/products/Shoes.php

To call the above controller your URI will look something like this:

example.com/index.php/products/shoes/show/123

Each of your sub-directories may contain a default controller which will be called if the URL contains only the sub-directory. Simply put a controller nore that matches the name of your ‘default_controller’ as specified in your application/config/routes.php file.

O CodeIgniter também permite que você remapeie seus URIs usando URI Roteamento feature.

Construtores de Classe¶

If you intend to use a constructor in any of your Controllers, you DEVO coloque a seguinte linha de código:

parent::__construct();

A razão pela qual esta linha é necessária é porque o seu construtor local be overriding the one no parent controller class so we need to manually call it.

Example:

<?php
class Blog extends CI_Controller {

        public function __construct()
        {
                parent::__construct();
                // Your own constructor code
        }
}

Constructors are useful if you need to set some default values, or run a default process when your class is instantiated. Constructors can’t Retornar um valor, mas eles podem fazer algum trabalho padrão.

Nomes de métodos reservados¶

Since your controller classes will extend the main application controlador você deve ter cuidado para não nomear seus métodos de forma idêntica the ones used by that class, otherwise your local functions will override them. See Nomes Reservados for a full list.

Importante

Você também nunca deve ter um método chamado identicamente to its class name. If you do, and there is no __construct() method no same class, then your e.g. Index::index() method will be executed as a class constructor! This is a PHP4 backwards-compatibility feature.

That’s it!¶

Isso, em poucas palavras, é tudo o que há para saber sobre controladores.

class blog extends CI_Controller {

}

Além disso, certifique-se sempre de que seu Controller estende o Controller pai class para que ele possa herdar todos os seus métodos.

Métodos¶

In the above example the method name is index(). The “index” method é sempre carregado por padrão se o segundo segmento da URI é esvaziar. Outra maneira de mostrar sua mensagem "Hello World" seria esta:

example.com/index.php/blog/index/

The segundo segmento of the URI determines which method no Controller é chamado.

Vamos tentar. Adicione um novo método ao seu Controller:

<?php
class Blog extends CI_Controller {

        public function index()
        {
                echo 'Olá Mundo!';
        }

        public function comments()
        {
                echo 'Look at this!';
        }
}

Agora, carregue o seguinte URL para ver o método de comentários:

example.com/index.php/blog/comments/

Você deveria ver sua nova mensagem.

Passando Segmentos URI para seus métodos¶

If your URI contains more than two segments they will be passed to your método como parâmetros.

Por exemplo, digamos que você tenha um URI assim:

example.com/index.php/products/shoes/sandals/123

Seu método será passado nos segmentos URI 3 e 4 (“sandálias” e “123”):

<?php
class Products extends CI_Controller {

        public function shoes($sandals, $id)
        {
                echo $sandals;
                echo $id;
        }
}

Importante

Se você estiver usando o Roteamento de URI feature, the segments passed to your method will be the re-routed ones.

Definindo um Controller Padrão¶

O CodeIgniter pode ser instruído a carregar um Controller padrão quando um URI não é presente, como será o caso quando apenas o URL raiz do seu site é solicitado. Para especificar um Controller padrão, abra o application/config/routes.php arquivo e definir essa variável:

$route['default_controller'] = 'blog';

Onde 'blog' é o nome da classe do Controller que você deseja usar. Se você agora carregue seu arquivo index.php principal sem especificar nenhum segmento URI veja sua mensagem "Hello World" por padrão.

Para mais informações, por favor consulte a seção “Rotas reservadas” do Roteamento de URI documentation.

Remapeamento de chamadas do método¶

As noted above, the segundo segmento of the URI typically determines which method no Controller é chamado. CodeIgniter permits you to override this behavior through the use of the _remap() method:

public function _remap()
{
        // Some code here...
}

Importante

Se o seu Controller contiver um método chamado _remap(), será always ser chamado independentemente do que o seu URI contém. isto substitui o comportamento normal em que o URI determina qual método é chamado, permitindo que você defina suas próprias regras de roteamento de método.

The overridden method call (typically the segundo segmento of the URI) will be passed as a parameter to the _remap() method:

public function _remap($method)
{
        if ($method === 'some_method')
        {
                $this->$method();
        }
        else
        {
                $this->default_method();
        }
}

Any extra segments after the method name are passed into _remap() as an segundo parâmetro opcional. Esta matriz pode ser usada em combinação com PHP's call_user_func_array() para emular o comportamento padrão do CodeIgniter.

Exemplo:

public function _remap($method, $params = array())
{
        $method = 'process_'.$method;
        if (method_exists($this, $method))
        {
                return call_user_func_array(array($this, $method), $params);
        }
        show_404();
}

Saída de Processamento¶

O CodeIgniter tem uma classe de saída que cuida do envio do seu final dados renderizados para o navegador da web automaticamente. Mais informações sobre isso pode ser encontrado no Views and Classe de saída Páginas. Em alguns casos, no entanto, você pode querer pós-processar os dados finalizados de alguma forma e enviá-los para o navegador você mesmo. O CodeIgniter permite que você adicione um método chamado _output() ao seu Controller que receberá os dados de saída finalizados.

Importante

If your controller contains a method named _output(), será always ser chamado pela classe de saída em vez de ecoando os dados finalizados diretamente. O primeiro parâmetro do método conterá a saída finalizada.

Aqui está um exemplo:

public function _output($output)
{
        echo $output;
}

Nota

Please note that your _output() method will receive the dados em seu estado finalizado. Dados de referência e uso de memória será processado, arquivos de cache gravados (se você tiver cache ativado), e os cabeçalhos serão enviados (se você usar feature) antes de ser entregue to the _output() method. Para que a saída do seu Controller seja armazenada corretamente, _output() método pode usar:

if ($this->output->cache_expiration > 0)
{
        $this->output->_write_cache($output);
}

Se você estiver usando este recurso, o temporizador de execução da página e as estatísticas de uso de memória podem não ser perfeitamente precisas, pois não levará em consideração qualquer outro processamento que você fizer. For an alternate way to control output before any of the processamento final é feito, por favor veja os métodos disponíveis no Library de saída.

Métodos privados¶

Em alguns casos, você pode querer alguns métodos ocultos do acesso público. Para conseguir isso, simplesmente declare o método como sendo privado or protected and será not be served via a URL request. For example, se você tivesse um método como este:

private function _utility()
{
        // some code
}

Tentar acessá-lo por meio do URL, assim, não funcionará:

example.com/index.php/blog/_utility/

Nota

Prefixar nomes de métodos com um sublinhado também evitará them from being called. Isto é a legacy feature that is left para compatibilidade com versões anteriores.

Organizando seus Controlleres em subdiretórios¶

Se você estiver criando um aplicativo grande, talvez queira hierarquicamente organize ou estruture seus Controlleres em subdiretórios. CodeIgniter permite que você faça isso.

Basta criar sub-diretórios sob o principal application/controllers/ one and place your controller classes withnom.

Nota

Ao usar esse recurso, o primeiro segmento do seu URI deve especifique a pasta. Por exemplo, digamos que você tenha um Controller localizado here:

application/controllers/products/Shoes.php

Para chamar o Controller acima, seu URI será parecido com isto:

example.com/index.php/products/shoes/show/123

Cada um dos seus subdiretórios pode conter um Controller padrão que será called if the URL contains only the sub-directory. Simply put a controller nore that matches the name of your ‘default_controller’ as specified in your application/config/routes.php file.

O CodeIgniter também permite que você remapeie seus URIs usando URI Roteamento feature.

Construtores de Classe¶

Se você pretende usar um construtor em qualquer um dos seus Controllers, MUST place the following line of code nisso:

parent::__construct();

A razão pela qual esta linha é necessária é porque o seu construtor local be overriding the one no parent controller class so we need to manualmente chamá-lo.

Exemplo:

<?php
class Blog extends CI_Controller {

        public function __construct()
        {
                parent::__construct();
                // Your own constructor code
        }
}

Os construtores são úteis se você precisar definir alguns valores padrão ou executar um processo padrão quando sua classe é instanciada. Construtores não podem Retornar um valor, mas eles podem fazer algum trabalho padrão.

Nomes de métodos reservados¶

Já que suas classes de Controller estenderão o aplicativo principal Controller você deve ter cuidado para não nomear seus métodos de forma idêntica the ones used by that class, otherwise your local functions will substituí-los. Vejo Nomes Reservados para um completo list.

Importante

Você também nunca deve ter um método chamado identicamente ao seu nome de classe. Se você faz, e não há __construct() method no same class, then your e.g. Index::index() method will be executed as a class constructor! Isto é a PHP4 recurso de compatibilidade com versões anteriores.

É isso aí!¶

Isso, em poucas palavras, é tudo o que há para saber sobre Controlleres.

Os Controlleres são o coração da sua aplicação, pois determinam como Solicitações HTTP devem ser tratadas.

O que é um Controller?¶

Um Controller é simplesmente um arquivo de classe que é nomeado de uma forma que pode ser associado a um URI.

Considere este URI:

example.com/index.php/blog/

No exemplo acima, o CodeIgniter tentaria encontrar um Controller chamado Blog.php e carregá-lo.

Quando o nome de um Controller corresponde ao primeiro segmento de um URI, ele be loaded.

Vamos tentar: Olá Mundo!¶

Vamos criar um Controller simples para que você possa vê-lo em ação. Usando seu editor de texto, crie um arquivo chamado Blog.php e coloque o seguinte código in it:

<?php
class Blog extends CI_Controller {

        public function index()
        {
                echo 'Olá Mundo!';
        }
}

Then save the file to your application/controllers/ directory.

Important

O arquivo deve ser chamado de 'Blog.php', com um 'B' maiúsculo.

Agora visite o seu site usando um URL semelhante a este:

example.com/index.php/blog/

Se você fez certo, você deve ver:

Olá Mundo!

Important

Nomes de classes devem começar com uma letra maiúscula.

Isso é válido:

<?php
class Blog extends CI_Controller {

}

Isto é not valid:

<?php
class blog extends CI_Controller {

}

Além disso, certifique-se sempre de que seu Controller estende o Controller pai class para que ele possa herdar todos os seus métodos.

Methods¶

In the above example the method name is index(). The “index” method é sempre carregado por padrão se o segundo segmento da URI é esvaziar. Outra maneira de mostrar sua mensagem "Hello World" seria esta:

example.com/index.php/blog/index/

The segundo segmento of the URI determines which method in the Controller é chamado.

Vamos tentar. Adicione um novo método ao seu Controller:

<?php
class Blog extends CI_Controller {

        public function index()
        {
                echo 'Olá Mundo!';
        }

        public function comments()
        {
                echo 'Look at this!';
        }
}

Agora, carregue o seguinte URL para ver o método de comentários:

example.com/index.php/blog/comments/

Você deveria ver sua nova mensagem.

Passando Segmentos URI para seus métodos¶

Se o seu URI contiver mais de dois segmentos, eles serão passados para o seu método como parâmetros.

Por exemplo, digamos que você tenha um URI assim:

example.com/index.php/products/shoes/sandals/123

Seu método será passado nos segmentos URI 3 e 4 ("sandálias" e "123"):

<?php
class Products extends CI_Controller {

        public function shoes($sandals, $id)
        {
                echo $sandals;
                echo $id;
        }
}

Important

Se você estiver usando o Roteamento de URI característica, os segmentos passados para o seu método será o reencaminhado ones.

Definindo um Controller Padrão¶

O CodeIgniter pode ser instruído a carregar um Controller padrão quando um URI não é presente, como será o caso quando apenas o URL raiz do seu site é solicitado. Para especificar um Controller padrão, abra o application/config/routes.php arquivo e definir essa variável:

$route['default_controller'] = 'blog';

Onde 'blog' é o nome da classe do Controller que você deseja usar. Se você agora carregue seu arquivo index.php principal sem especificar nenhum segmento URI veja sua mensagem "Hello World" por padrão.

Para mais informações, por favor consulte a seção "Rotas reservadas" do Roteamento de URI documentation.

Remapeamento de chamadas do método¶

As noted above, the segundo segmento of the URI typically determines which method in the Controller é chamado. CodeIgniter permits you to override this behavior through the use of the _remap() method:

public function _remap()
{
        // Some code here...
}

Important

Se o seu Controller contiver um método chamado _remap(), it will always ser chamado independentemente do que o seu URI contém. isto substitui o comportamento normal em que o URI determina qual método é chamado, permitindo que você defina suas próprias regras de roteamento de método.

The overridden method call (typically the segundo segmento of the URI) will be passed as a parameter to the _remap() method:

public function _remap($method)
{
        if ($method === 'some_method')
        {
                $this->$method();
        }
        else
        {
                $this->default_method();
        }
}

Any extra segments after the method name are passed into _remap() as an segundo parâmetro opcional. Esta matriz pode ser usada em combinação com PHP’s call_user_func_array() para emular o comportamento padrão do CodeIgniter.

Exemplo:

public function _remap($method, $params = array())
{
        $method = 'process_'.$method;
        if (method_exists($this, $method))
        {
                return call_user_func_array(array($this, $method), $params);
        }
        show_404();
}

O CodeIgniter permite que você adicione um método chamado _output() ao seu Controller que receberá os dados de saída finalizados.

Important

If your controller contains a method named _output(), it will always ser chamado pela classe de saída em vez de ecoando os dados finalizados diretamente. O primeiro parâmetro do método conterá a saída finalizada.

Aqui está um exemplo:

public function _output($output)
{
        echo $output;
}

Dados de referência e uso de memória será processado, arquivos de cache gravados (se você tiver cache ativado), e os cabeçalhos serão enviados (se você usar feature) antes de ser entregue to the _output() method. if ($this->output->cache_expiration > 0)

private function _utility()

Se você estiver criando um aplicativo grande, talvez queira hierarquicamente organize ou estruture seus Controlleres em subdiretórios. CodeIgniter permite que você faça isso.

Basta criar sub-diretórios sob o principal application/controllers/ um e coloque suas classes de Controller dentro delas.

parent::__construct();

A razão pela qual esta linha é necessária é porque o seu construtor local estar substituindo o da classe Controller pai por isso precisamos manualmente chamá-lo.

Exemplo:

<?php
class Blog extends CI_Controller {

        public function __construct()
        {
                parent::__construct();
                // Your own constructor code
        }
}

Os construtores são úteis se você precisar definir alguns valores padrão ou executar um processo padrão quando sua classe é instanciada. Construtores não podem Retornar um valor, mas eles podem fazer algum trabalho padrão.
