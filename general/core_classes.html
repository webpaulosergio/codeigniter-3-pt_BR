Criando Classes do Sistema Principal¶

Every time CodeIgniter runs there are several base classes that are inicializado automaticamente como parte do framework principal. É possível, however, to swap any of the core system classes with your own versions ou até mesmo estender as versões principais.

Most users will never have any need to do this, but the option to replace or extend them does exist for those who would like to significantly alter the CodeIgniter core.

Nota

Messing with a core system class has a lot of implications, so verifique se você sabe o que está fazendo antes de tentar.

System Class List¶

The following is a list of the core system files that are invoked every tempo CodeIgniter é executado:

  • Referência
  • Config
  • Controller
  • Exceções
  • Hooks
  • Input
  • Language
  • Loader
  • Log
  • Saída
  • Roteador
  • Segurança
  • URI
  • Utf8

Replacing Core Classes¶

Para usar uma das suas próprias classes de sistema em vez de uma padrão simplesmente place your version inside your local application/core/ directory:

application/core/some_class.php

If this directory does not exist you can create it.

Any file named identically to one from the list above will be used em vez do normalmente usado.

Por favor, note que sua classe deve usar o IC como um prefixo. Por exemplo, se your file is named Input.php the class will be named:

class CI_Input {

}

Extending Core Class¶

Se tudo que você precisa fazer é adicionar alguma funcionalidade a uma library existente - perhaps add a method or two - then it’s overkill to replace the entire library com a sua versão. Neste caso, é melhor simplesmente estender o class. Extending a class is nearly identical to replacing a class with a couple exceptions:

  • The class declaration must extend the parent class.
  • Your new class name and filename must be prefixed with MY_ (this item is configurable. See below.).

For example, to extend the native Input class you’ll create a file named application/core/MY_Input.php e declare sua classe com:

class MY_Input extends CI_Input {

}

Nota

Se você precisa usar um construtor em sua classe, certifique-se estenda o construtor pai:

class MY_Input extends CI_Input {

        public function __construct()
        {
                parent::__construct();
                // Your own constructor code
        }
}

Tip: Any functions in your class that are named identically to the methods in the parent class will be used instead of the native ones (this is known as “method overriding”). This allows you to substantially alter the CodeIgniter core.

If you are extending the Controller core class, then be sure to extend sua nova classe nos construtores do seu controlador de aplicativo.

class Welcome extends MY_Controller {

        public function index()
        {
                $this->load->view('welcome_message');
        }
}

Setting Your Own Prefix¶

Para definir o seu próprio prefixo de subclasse, abra o seu application/config/config.php arquivo e procure por este item:

$config['subclass_prefix'] = 'MY_';

Please note that all native CodeIgniter libraries are prefixed with CI_ so DO NOT use that as your prefix.

Criando Classes do Sistema Principal¶

Toda vez que o CodeIgniter é executado, existem várias classes base que são inicializado automaticamente como parte do framework principal. É possível, no entanto, para trocar qualquer uma das classes principais do sistema com suas próprias versões ou até mesmo estender as versões principais.

A maioria dos usuários nunca terá necessidade de fazer isso, mas a opção de substituir ou estendê-los existe para aqueles que gostariam de alterar significativamente o núcleo do CodeIgniter.

Nota

Mexer com uma classe de sistema central tem muitas implicações, então verifique se você sabe o que está fazendo antes de tentar.

Lista de Classes do Sistema¶

A seguir, uma lista dos principais arquivos de sistema que são chamados a cada tempo CodeIgniter é executado:

  • Referência
  • Config
  • Controller
  • Exceções
  • Hooks
  • Input
  • Língua
  • Carregador
  • Log
  • Saída
  • Roteador
  • Segurança
  • URI
  • Utf8

Substituindo Classes Principais¶

Para usar uma das suas próprias classes de sistema em vez de uma padrão simplesmente place your version inside your local application/core/ directory:

application/core/some_class.php

Se este diretório não existir, você poderá criá-lo.

Qualquer arquivo com nome idêntico a um da lista acima será usado em vez do normalmente usado.

Por favor, note que sua classe deve usar o IC como um prefixo. Por exemplo, se seu arquivo é chamado Input.php a classe será nomeada:

class CI_Input {

}

Estendendo Classe Principal¶

Se tudo que você precisa fazer é adicionar alguma funcionalidade a uma library existente - talvez adicione um método ou dois - então é um exagero substituir todo o library com a sua versão. Neste caso, é melhor simplesmente estender o classe. Estender uma turma é quase idêntico a substituir uma turma por uma algumas exceções:

  • A declaração de classe deve estender a classe pai.
  • Seu novo nome de classe e nome de arquivo devem ser prefixados com MY_ item é configurável. Ver abaixo.).

Por exemplo, para estender a classe Input nativo, você criará um arquivo chamado application/core/MY_Input.php e declare sua classe com:

class MY_Input extends CI_Input {

}

Nota

Se você precisa usar um construtor em sua classe, certifique-se estenda o construtor pai:

class MY_Input extends CI_Input {

        public function __construct()
        {
                parent::__construct();
                // Your own constructor code
        }
}

Tip: Todas as funções da sua classe que são nomeadas de forma idêntica à methods in the parent class will be used instead of the native ones (isso é conhecido como “método de substituição”). Isso permite que você substancialmente altere o núcleo do CodeIgniter.

Se você estiver estendendo a classe principal do Controller, certifique-se de estender sua nova classe nos construtores do seu Controller de aplicativo.

class Welcome extends MY_Controller {

        public function index()
        {
                $this->load->view('welcome_message');
        }
}

Definindo seu próprio prefixo¶

Para definir o seu próprio prefixo de subclasse, abra o seu application/config/config.php arquivo e procure por este item:

$config['subclass_prefix'] = 'MY_';

Por favor, note que todas as Librarys nativas do CodeIgniter são prefixadas com CI_, então não use isso como seu prefixo.

Criando Classes do Sistema Principal¶

Toda vez que o CodeIgniter é executado, existem várias classes base que são inicializado automaticamente como parte do framework principal. É possível, no entanto, para trocar qualquer uma das classes principais do sistema com suas próprias versões ou até mesmo estender as versões principais.

A maioria dos usuários nunca terá necessidade de fazer isso, mas a opção de substituir ou estendê-los existe para aqueles que gostariam de alterar significativamente o núcleo do CodeIgniter.

Nota

Mexer com uma classe de sistema central tem muitas implicações, então verifique se você sabe o que está fazendo antes de tentar.

Lista de Classes do Sistema¶

A seguir, uma lista dos principais arquivos de sistema que são chamados a cada tempo CodeIgniter é executado:

  • Benchmark
  • Config
  • Controller
  • Exceptions
  • Hooks
  • Input
  • Língua
  • Loader
  • Log
  • Output
  • Router
  • Segurança
  • URI
  • Utf8

Substituindo Classes Principais¶

Para usar uma das suas próprias classes de sistema em vez de uma padrão simplesmente place your version inside your local application/core/ directory:

application/core/some_class.php

Se este diretório não existir, você poderá criá-lo.

Qualquer arquivo com nome idêntico a um da lista acima será usado em vez do normalmente usado.

Por favor, note que sua classe deve usar o IC como um prefixo. Por exemplo, se seu arquivo é chamado Input.php a classe será nomeada:

class CI_Input {

}

Estendendo Classe Principal¶

Se tudo que você precisa fazer é adicionar alguma funcionalidade a uma library existente - talvez adicione um método ou dois - então é um exagero substituir todo o library com a sua versão. Neste caso, é melhor simplesmente estender o classe. Estender uma turma é quase idêntico a substituir uma turma por uma algumas exceções:

  • A declaração de classe deve estender a classe pai.
  • Seu novo nome de classe e nome de arquivo devem ser prefixados com MY_ item é configurável. Ver abaixo.).

Por exemplo, para estender a classe Input nativo, você criará um arquivo chamado application/core/MY_Input.php e declare sua classe com:

class MY_Input extends CI_Input {

}

Nota

Se você precisa usar um construtor em sua classe, certifique-se estenda o construtor pai:

class MY_Input extends CI_Input {

        public function __construct()
        {
                parent::__construct();
                // Your own constructor code
        }
}

Tip: Todas as funções da sua classe que são nomeadas de forma idêntica à métodos na classe pai serão usados em vez dos nativos (isso é conhecido como "método de substituição"). Isso permite que você substancialmente altere o núcleo do CodeIgniter.

Se você estiver estendendo a classe principal do Controller, certifique-se de estender sua nova classe nos construtores do seu Controller de aplicativo.

class Welcome extends MY_Controller {

        public function index()
        {
                $this->load->view('welcome_message');
        }
}

Definindo seu próprio prefixo¶

Para definir o seu próprio prefixo de subclasse, abra o seu application/config/config.php arquivo e procure por este item:

$config['subclass_prefix'] = 'MY_';

Por favor, note que todas as Librarys nativas do CodeIgniter são prefixadas com CI_, então não use isso como seu prefixo.
