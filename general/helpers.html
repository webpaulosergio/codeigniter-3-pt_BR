Helper Functions¶

Os Helpers, como o nome sugere, ajudam você nas tarefas. Cada arquivo auxiliar é simply a collection of functions in a particular category. There are URL Helpers, that assist in creating links, there are Helper de Formulários that help você cria elementos de formulário, Auxiliar de Textos realizar várias formatações de texto routines, Helpers de Cookie set and read cookies, Arquivo auxiliars help you lidar com arquivos, etc.

Unlike most other systems in CodeIgniter, Helpers are not written in an Object Oriented format. They are simple, procedural functions. Each função auxiliar executa uma tarefa específica, sem dependência de outras functions.

CodeIgniter does not load Helper Files by default, so the first step in using a Helper is to load it. Once loaded, it becomes globally available in your controller and views.

Os helpers são normalmente armazenados em seu system/helpers, or application/helpers directory. O CodeIgniter irá procurar primeiro no seu application/helpers directory. If the directory does not exist or the specified helper is not located there CI will instead look in your global system/helpers/ directory.

Loading a Helper¶

Carregar um arquivo auxiliar é bastante simples usando o seguinte método:

$this->load->helper('name');

Where name is the file name of the helper, without the .php file extension or the “helper” part.

Por exemplo, para carregar o Helper de URL arquivo, que é nomeado url_helper.php, você faria isso:

$this->load->helper('url');

Um Helper pode ser carregado em qualquer lugar dentro de seus métodos de controle (ou even within your View files, although that’s not a good practice), as contanto que você carregue antes de usá-lo. Você pode carregar seus Helpers em seu controller constructor so that they become available automatically in qualquer função, ou você pode carregar um auxiliar em uma função específica que precisa it.

Nota

O método de carregamento auxiliar acima não retorna um valor, portanto não tente atribuí-lo a uma variável. Apenas use como mostrado.

Loading Multiple Helpers¶

If you need to load more than one helper you can specify them in an array, assim:

$this->load->helper(
        array('helper1', 'helper2', 'helper3')
);

Auto-loading Helpers¶

If you find that you need a particular helper globally throughout your aplicação, você pode dizer ao CodeIgniter para auto-carregá-lo durante o initialization. This is done by opening the application/config/autoload.php file and adding the helper to the autoload array.

Using a Helper¶

Once you’ve loaded the Helper File containing the function you intend to use, você vai chamá-lo da maneira que você faria uma função PHP padrão.

For example, to create a link using the anchor() function in one of your view files you would do this:

<?php echo anchor('blog/comments', 'Click Here');?>

Onde “Click Here” é o nome do link, e “blog/comments” é o URI para o controlador/método que você deseja vincular.

“Extending” Helpers¶

Para “estender” os Helpers, crie um arquivo em seu application/helpers/ folder with an identical name to the existing Helper, but prefixed with MY_ (this item is configurable. See below.).

Se tudo que você precisa fazer é adicionar alguma funcionalidade a um helper existente - perhaps add a function or two, or change how a particular helper função funciona - então é um exagero para substituir todo o helper com Sua versão. Neste caso, é melhor simplesmente "estender" o Helper.

Nota

O termo "estender" é usado livremente, já que as funções auxiliares são processuais e discretos e não podem ser estendidos no tradicional programmatic sense. Under the hood, this gives you the ability to add to or or to replace the functions a Helper provides.

For example, to extend the native Helper de Matriz você vai criar um arquivo named application/helpers/MY_array_helper.phpe adicione ou anule functions:

// any_in_array() is not in the Helper de Matriz, so it defines a new function
function any_in_array($needle, $haystack)
{
        $needle = is_array($needle) ? $needle : array($needle);

        foreach ($needle as $item)
        {
                if (in_array($item, $haystack))
                {
                        return TRUE;
                }
        }

        return FALSE;
}

// random_element() is included in Helper de Matriz, so it overrides the native function
function random_element($array)
{
        shuffle($array);
        return array_pop($array);
}

Setting Your Own Prefix¶

O prefixo do nome do arquivo para “estender” os Helpers é o mesmo usado para estender librarys e classes principais. Para definir o seu próprio prefixo, abra o seu application/config/config.php arquivo e procure por este item:

$config['subclass_prefix'] = 'MY_';

Por favor, note que todas as librarys nativas do CodeIgniter são prefixadas com CI_ então não use isso como seu prefixo.

Funções Helperes¶

Os Helpers, como o nome sugere, ajudam você nas tarefas. Cada arquivo Helper é simplesmente uma coleção de funções em uma categoria específica. tem URL Helpers, that assist in creating links, there are Helper de Formulários that help você cria elementos de formulário, Helper de Textos realizar várias formatações de texto routines, Helper Cookies set and read cookies, Arquivo Helpers help you lidar com arquivos, etc.

Ao contrário da maioria dos outros sistemas no CodeIgniter, os Helpers não são escritos Formato orientado a objetos. São funções simples e procedurais. Cada função Helper executa uma tarefa específica, sem dependência de outras functions.

O CodeIgniter não carrega arquivos Helperes por padrão, então o primeiro passo usar um Helper é carregá-lo. Uma vez carregado, torna-se globalmente disponível na tua controller and views.

Helpers are typically stored na tua system/helpers, or diretório de aplicativos/Helpers. CodeIgniter will look first na tua diretório de aplicativos/Helpers. Se o diretório não existir ou o specified helper is not located there CI will instead look na tua global system/helpers/ directory.

Carregando um Helper¶

Carregar um arquivo Helper é bastante simples usando o seguinte método:

$this->load->helper('name');

Where name é o nome do arquivo do Helper, sem o arquivo .php extensão ou a parte “Helper”.

Por exemplo, para carregar o Helper de URL arquivo, que é nomeado url_helper.php, você faria isso:

$this->load->helper('url');

A helper can be loaded anywhere withna tua controller methods (or even withna tua View files, although that’s not a good practice), as long as you load it before you use it. You can load your helpers na tua construtor Controller para que eles se tornem disponíveis automaticamente em qualquer função, ou você pode carregar um Helper em uma função específica que precisa it.

Nota

O método de carregamento Helper acima não retorna um valor, portanto não tente atribuí-lo a uma variável. Apenas use como mostrado.

Carregando vários Helperes¶

Se você precisar carregar mais de um Helper, poderá especificá-los em um array, assim:

$this->load->helper(
        array('helper1', 'helper2', 'helper3')
);

Helperes de carregamento automático¶

Se você achar que precisa de um Helper em particular em todo o seu aplicação, você pode dizer ao CodeIgniter para auto-carregá-lo durante o inicialização. Isso é feito abrindo a application/config/autoload.php arquivo e adicionando o Helper ao array autoload.

Usando um Helper¶

Depois de ter carregado o arquivo Helper contendo a função que você pretende use, você vai chamá-lo da maneira que você faria uma função PHP padrão.

For example, to create a link using the anchor() function in one of seus arquivos de visualização você faria isso:

<?php echo anchor('blog/comments', 'Click Here');?>

Onde “Click Here” é o nome do link, e “blog/comments” é o URI para o Controller/método que você deseja vincular.

Helperes "Estendendo"¶

To “extend” Helpers, create a file na tua application/helpers/ folder com um nome idêntico ao Helper existente, mas prefixado com MY_ (este item é configurável. Veja abaixo).

Se tudo que você precisa fazer é adicionar alguma funcionalidade a um helper existente - talvez adicione uma função ou duas, ou mude como um Helper em particular função funciona - então é um exagero para substituir todo o helper com Sua versão. Neste caso, é melhor simplesmente "estender" o Helper.

Nota

O termo "estender" é usado livremente, já que as funções Helperes são processuais e discretos e não podem ser estendidos no tradicional sentido programático. Sob o capô, isso lhe dá a capacidade de adicione ou substitua as funções fornecidas por um Helper.

Por exemplo, para estender o nativo Helper de Matriz você vai criar um arquivo named aplicação/Helpers/MY_array_helper.phpe adicione ou anule functions:

// any_in_array() is not in the Helper de Matriz, so it defines a new function
function any_in_array($needle, $haystack)
{
        $needle = is_array($needle) ? $needle : array($needle);

        foreach ($needle as $item)
        {
                if (in_array($item, $haystack))
                {
                        return TRUE;
                }
        }

        return FALSE;
}

// random_element() is included in Helper de Matriz, so it overrides the native function
function random_element($array)
{
        shuffle($array);
        return array_pop($array);
}

Definindo seu próprio prefixo¶

O prefixo do nome do arquivo para “estender” os Helpers é o mesmo usado para estender librarys e classes principais. Para definir o seu próprio prefixo, abra o seu application/config/config.php arquivo e procure por este item:

$config['subclass_prefix'] = 'MY_';

Por favor, note que todas as librarys nativas do CodeIgniter são prefixadas com CI_ então não use isso como seu prefixo.

Funções Helperes¶

Os Helpers, como o nome sugere, ajudam você nas tarefas. Cada arquivo Helper é simplesmente uma coleção de funções em uma categoria específica. tem URL Helpers, that assist in creating links, there are Helper de Formulários that help você cria elementos de formulário, Helper de Textos realizar várias formatações de texto routines, Helper Cookies set and read cookies, Arquivo Helpers help you lidar com arquivos, etc.

Ao contrário da maioria dos outros sistemas no CodeIgniter, os Helpers não são escritos Formato orientado a objetos. São funções simples e procedurais. Cada função Helper executa uma tarefa específica, sem dependência de outras functions.

O CodeIgniter não carrega arquivos Helperes por padrão, então o primeiro passo usar um Helper é carregá-lo. Uma vez carregado, torna-se globalmente disponível in your controller and views.

Os helpers são normalmente armazenados em seu system/helpers, or diretório de aplicativos/Helpers. O CodeIgniter irá procurar primeiro no seu diretório de aplicativos/Helpers. Se o diretório não existir ou o Helper especificado não está localizado lá CI, em vez disso, olhar em seu global system/helpers/ directory.

Carregando um Helper¶

Carregar um arquivo Helper é bastante simples usando o seguinte método:

$this->load->helper('name');

Where name é o nome do arquivo do Helper, sem o arquivo .php extensão ou a parte "Helper".

Por exemplo, para carregar o Helper de URL arquivo, que é nomeado url_helper.php, você faria isso:

$this->load->helper('url');

Um Helper pode ser carregado em qualquer lugar dentro de seus métodos de controle (ou mesmo dentro dos arquivos do View, embora isso não seja uma boa prática), contanto que você carregue antes de usá-lo. Você pode carregar seus Helpers em seu construtor Controller para que eles se tornem disponíveis automaticamente em qualquer função, ou você pode carregar um Helper em uma função específica que precisa it.

Nota

O método de carregamento Helper acima não retorna um valor, portanto não tente atribuí-lo a uma variável. Apenas use como mostrado.

Carregando vários Helperes¶

Se você precisar carregar mais de um Helper, poderá especificá-los em um array, assim:

$this->load->helper(
        array('helper1', 'helper2', 'helper3')
);

Helperes de carregamento automático¶

Se você achar que precisa de um Helper em particular em todo o seu aplicação, você pode dizer ao CodeIgniter para auto-carregá-lo durante o inicialização. Isso é feito abrindo a application/config/autoload.php arquivo e adicionando o Helper ao array autoload.

Usando um Helper¶

Depois de ter carregado o arquivo Helper contendo a função que você pretende use, você vai chamá-lo da maneira que você faria uma função PHP padrão.

For example, to create a link using the anchor() function in one of seus arquivos de visualização você faria isso:

<?php echo anchor('blog/comments', 'Click Here');?>

Onde "Click Here" é o nome do link, e "blog/comments" é o URI para o Controller/método que você deseja vincular.

Helperes "Estendendo"¶

Para "estender" os Helpers, crie um arquivo em seu application/helpers/ folder com um nome idêntico ao Helper existente, mas prefixado com MY_ (este item é configurável. Veja abaixo).

Se tudo que você precisa fazer é adicionar alguma funcionalidade a um helper existente - talvez adicione uma função ou duas, ou mude como um Helper em particular função funciona - então é um exagero para substituir todo o helper com Sua versão. Neste caso, é melhor simplesmente "estender" o Helper.

Nota

O termo "estender" é usado livremente, já que as funções Helperes são processuais e discretos e não podem ser estendidos no tradicional sentido programático. Sob o capô, isso lhe dá a capacidade de adicione ou substitua as funções fornecidas por um Helper.

Por exemplo, para estender o nativo Helper de Matriz você vai criar um arquivo named aplicação/Helpers/MY_array_helper.phpe adicione ou anule functions:

// any_in_array() is not in the Helper de Matriz, so it defines a new function
function any_in_array($needle, $haystack)
{
        $needle = is_array($needle) ? $needle : array($needle);

        foreach ($needle as $item)
        {
                if (in_array($item, $haystack))
                {
                        return TRUE;
                }
        }

        return FALSE;
}

// random_element() is included in Helper de Matriz, so it overrides the native function
function random_element($array)
{
        shuffle($array);
        return array_pop($array);
}

Definindo seu próprio prefixo¶

O prefixo do nome do arquivo para "estender" os Helpers é o mesmo usado para estender librarys e classes principais. Para definir o seu próprio prefixo, abra o seu application/config/config.php arquivo e procure por este item:

$config['subclass_prefix'] = 'MY_';

Por favor, note que todas as librarys nativas do CodeIgniter são prefixadas com CI_ então não use isso como seu prefixo.
