Tratamento de erros¶

O CodeIgniter permite que você crie relatórios de erro em seus aplicativos usando as funções descritas abaixo. Além disso, tem um log de erros class that permits error and debugging messages to be saved as text files.

Nota

Por padrão, o CodeIgniter exibe todos os erros do PHP. Você pode Deseja mudar esse comportamento assim que seu desenvolvimento estiver completo. Você vai encontre a função error_reporting() localizada no topo da sua arquivo index.php. Desativar o relatório de erros NÃO impedirá os arquivos de log de ser escrito se houver erros.

Ao contrário da maioria dos sistemas no CodeIgniter, as funções de erro são simples procedural interfaces that are available globally throughout the application. This approach permits error messages to get triggered without having to worry about class/function scoping.

CodeIgniter also returns a status code whenever a portion of the core calls exit(). This exit status code is separate from the HTTP status code, and serves as a notice to other processes that may be watching of whether the script completed successfully, or if not, what kind of problem it encountered that caused it to abort. These values are defined in application/config/constants.php. While exit status codes são mais úteis em configurações de CLI, retornando o código apropriado ajuda o servidor software keep track of your scripts and the health of your application.

The following functions let you generate errors:

show_error($message, $status_code, $heading = 'An Error Was Encountered')¶
Parâmetros:
  • $message (mixed) – Error message
  • $status_code (int) – HTTP Response status code
  • $heading (string) – Error page heading
Return type:

void

Esta função exibirá a mensagem de erro fornecida a ela usando o modelo de erro apropriado para sua execução:

application/views/errors/html/error_general.php

or:

application/views/errors/cli/error_general.php

The optional parameter $status_code determines what HTTP status code should be sent with the error. If $status_code is less than 100, the HTTP status code will be set to 500, and the exit status code will be set to $status_code + EXIT__AUTO_MIN. If that value is larger than EXIT__AUTO_MAX, or if $status_code is 100 or higher, the exit status code will be set to EXIT_ERROR. You can check in application/config/constants.php for more detail.

show_404($page = '', $log_error = TRUE)¶
Parâmetros:
  • $page (string) - string URI
  • $log_error (bool) - Se registrar o erro
Return type:

void

This function will display the 404 error message supplied to it using o modelo de erro apropriado para sua execução:

application/views/errors/html/error_404.php

or:

application/views/errors/cli/error_404.php

The function expects the string passed to it to be the file path to a página que não foi encontrada. O código de status de saída será definido como EXIT_UNKNOWN_FILE. Nota that CodeIgniter automatically shows 404 messages if controllers are not found.

CodeIgniter automatically logs any show_404() calls. Setting the O segundo parâmetro opcional para FALSE irá ignorar o registro.

log_message($level, $message)¶
Parâmetros:
  • $level (string) - Nível de log: 'error', 'debug' ou 'info'
  • $message (string) – Message to log
Return type:

void

This function lets you write messages to your log files. You must fornecer um dos três "níveis" no primeiro parâmetro, indicando o que tipo de mensagem é (debug, error, info), com a própria mensagem no segundo parâmetro.

Example:

if ($some_var == '')
{
        log_message('error', 'Some variable did not contain a value.');
}
else
{
        log_message('debug', 'Some variable was correctly set');
}

log_message('info', 'The purpose of some variable is to provide some value.');

Existem três tipos de mensagens:

  1. Error Messages. These are actual errors, such as PHP errors or user errors.
  2. Mensagens de depuração. Estas são mensagens que auxiliam na depuração. Para Por exemplo, se uma classe foi inicializada, você pode registrá-la como debugging info.
  3. Informational Messages. These are the lowest priority messages, simply giving information regarding some process.

Nota

Para que o arquivo de log seja realmente gravado, logs/ directory must be writable. In addition, you must set the “threshold” for logging in application/config/config.php. You might, for example, only want error messages to be logged, and not the other two types. If you set it to zero logging will be disabled.

  • $heading (string) - cabeçalho da página de erro

void

log_message($level, $message)¶

void

Tratamento de erros¶

O CodeIgniter permite que você crie relatórios de erro em seus aplicativos usando as funções descritas abaixo. Além disso, tem um log de erros classe que permite que mensagens de erro e depuração sejam salvas como texto files.

Nota

Por padrão, o CodeIgniter exibe todos os erros do PHP. Você pode Deseja mudar esse comportamento assim que seu desenvolvimento estiver completo. Você vai encontre a função error_reporting() localizada no topo da sua arquivo index.php. Desativar o relatório de erros NÃO impedirá os arquivos de log de ser escrito se houver erros.

Ao contrário da maioria dos sistemas no CodeIgniter, as funções de erro são simples interfaces processuais que estão disponíveis globalmente em todo o aplicação. Essa abordagem permite que mensagens de erro sejam acionadas sem ter que se preocupar com o escopo de classe/função.

O CodeIgniter também retorna um código de status sempre que uma parte do núcleo calls exit(). This exit status code is separate from the HTTP status código, e serve como um aviso para outros processos que podem estar assistindo se o script foi concluído com sucesso ou, se não, que tipo de problema que encontrou que causou o aborto. Esses valores são defined in application/config/constants.php. While exit status codes são mais úteis em configurações de CLI, retornando o código apropriado ajuda o servidor o software monitora seus scripts e a integridade do seu aplicativo.

As seguintes funções permitem gerar erros:

show_error($message, $status_code, $heading = 'An Error Was Encountered')¶
Parâmetros:
  • $message (mixed) - Mensagem de erro
  • $status_code (int) - Código de status da resposta HTTP
  • $heading (string) - cabeçalho da página de erro
Tipo de retorno:

void

Esta função exibirá a mensagem de erro fornecida a ela usando o Model de erro apropriado para sua execução:

application/views/errors/html/error_general.php

or:

application/views/errors/cli/error_general.php

The optional parameter $status_code determines what HTTP status code should be sent with the error. If $status_code is less de 100, o código de status HTTP será definido como 500 e a saída status code will be set to $status_code + EXIT__AUTO_MIN. If that value is larger than EXIT__AUTO_MAX, or if $status_code é 100 ou superior, o código de status de saída será definido to EXIT_ERROR. You can check in application/config/constants.php for more detail.

show_404($page = '', $log_error = TRUE)¶
Parâmetros:
  • $page (string) - string URI
  • $log_error (bool) - Se registrar o erro
Tipo de retorno:

void

Esta função exibirá a mensagem de erro 404 fornecida a ela using o Model de erro apropriado para sua execução:

application/views/errors/html/error_404.php

or:

application/views/errors/cli/error_404.php

A função espera que a string passada para ele seja o caminho do arquivo para a página que não foi encontrada. O código de status de saída será definido como EXIT_UNKNOWN_FILE. Nota que o CodeIgniter mostra automaticamente 404 mensagens se Controlleres não são encontrados.

CodeIgniter automatically logs any show_404() calls. Setting the O segundo parâmetro opcional para FALSE irá ignorar o registro.

log_message($level, $message)¶
Parâmetros:
  • $level (string) - Nível de log: 'error', 'debug' ou 'info'
  • $message (string) - Mensagem para log
Tipo de retorno:

void

Essa função permite gravar mensagens em seus arquivos de log. Você deve fornecer um dos três "níveis" no primeiro parâmetro, indicando o que tipo de mensagem é (debug, error, info), com a própria mensagem no segundo parâmetro.

Exemplo:

if ($some_var == '')
{
        log_message('error', 'Some variable did not contain a value.');
}
else
{
        log_message('debug', 'Some variable was correctly set');
}

log_message('info', 'The purpose of some variable is to provide some value.');

Existem três tipos de mensagens:

  1. Mensagens de erro. Estes são erros reais, como erros PHP ou erros do usuário.
  2. Mensagens de depuração. Estas são mensagens que auxiliam na depuração. Para Por exemplo, se uma classe foi inicializada, você pode registrá-la como informações de depuração.
  3. Mensagens informativas. Estas são as mensagens de prioridade mais baixa, simplesmente dando informações sobre algum processo.

Nota

Para que o arquivo de log seja realmente gravado, logs/ diretório deve ser gravável. Além disso, você deve definir o "limite" para o login application/config/config.php. Você pode, por exemplo, apenas deseja que mensagens de erro sejam registradas, e não as outras dois tipos. Se você configurá-lo para zero, o registro será desativado.
