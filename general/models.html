Models¶

Models are optionally disponível para aqueles que querem usar um abordagem MVC tradicional.

O que é um modelo?¶

Modelos são classes PHP que são projetadas para trabalhar com informações seu Database. Por exemplo, digamos que você use o CodeIgniter para gerenciar blog. Você pode ter uma classe de modelo que contenha funções para inserir, update, and retrieve your blog data. Here is an example of what such a A classe de modelo pode se parecer com:

class Blog_model extends CI_Model {

        public $title;
        public $content;
        public $date;

        public function get_last_ten_entries()
        {
                $query = $this->db->get('entries', 10);
                return $query->result();
        }

        public function insert_entry()
        {
                $this->title    = $_POST['title']; // please read the below note
                $this->content  = $_POST['content'];
                $this->date     = time();

                $this->db->insert('entries', $this);
        }

        public function update_entry()
        {
                $this->title    = $_POST['title'];
                $this->content  = $_POST['content'];
                $this->date     = time();

                $this->db->update('entries', $this, array('id' => $_POST['id']));
        }

}

Nota

The methods in the above example use the Query Builder database methods.

Nota

For the sake of simplicity in this example we’re using $_POST directly. This is generally bad practice, and a more common approach would be to use the Input Library $this->input->post('title').

Anatomy of a Model¶

Model classes are stored in your application/models/ directory. Eles podem ser aninhados em subdiretórios se você quiser esse tipo de organization.

The basic prototype for a model class is this:

class Model_name extends CI_Model {

}

Where Model_name is the name of your class. Class names must have the first letter capitalized with the rest of the name lowercase. Make sure your class extends the base Model class.

O nome do arquivo deve corresponder ao nome da classe. Por exemplo, se esta é sua classe:

class User_model extends CI_Model {

}

Your file will be this:

application/models/User_model.php

Loading a Model¶

Seus modelos serão tipicamente carregados e chamados de dentro do seu controller methods. To load a model you will use the following method:

$this->load->model('model_name');

If your model is located in a sub-directory, include the relative path from your models directory. For example, if you have a model located at application/models/blog/Queries.php you’ll load it using:

$this->load->model('blog/queries');

Once loaded, you will access your model methods using an object with the same name as your class:

$this->load->model('model_name');

$this->model_name->method();

Se você quiser que seu modelo seja atribuído a um nome de objeto diferente, você pode especificá-lo através do segundo parâmetro do método de carregamento:

$this->load->model('model_name', 'foobar');

$this->foobar->method();

Aqui está um exemplo de um controlador, que carrega um modelo e, em seguida, serve um view:

class Blog_controller extends CI_Controller {

        public function blog()
        {
                $this->load->model('blog');

                $data['query'] = $this->blog->get_last_ten_entries();

                $this->load->view('blog', $data);
        }
}

Modelos de carregamento automático¶

If you find that you need a particular model globally throughout your aplicação, você pode dizer ao CodeIgniter para auto-carregá-lo durante o initialization. This is done by opening the application/config/autoload.php file and adding the model to the autoload array.

Conectando-se ao seu database¶

When a model is loaded it does NÃO conectar-se automaticamente ao seu base de dados. As seguintes opções para conexão estão disponíveis para você:

  • Você pode se conectar usando os métodos de Database padrão described here, seja de dentro do seu Controller class or your Model class.

  • Você pode dizer ao método de carregamento do modelo para se conectar automaticamente passando TRUE (boolean) via the third parameter, and connectivity settings, conforme definido no seu arquivo de configuração do database será usado:

    $this->load->model('model_name', '', TRUE);

  • Você pode passar manualmente as configurações de conectividade do Database através do terceiro parameter:

    $config['hostname'] = 'localhost';
$config['username'] = 'myusername';
$config['password'] = 'mypassword';
$config['database'] = 'mydatabase';
$config['dbdriver'] = 'mysqli';
$config['dbprefix'] = '';
$config['pconnect'] = FALSE;
$config['db_debug'] = TRUE;

$this->load->model('model_name', '', $config);

O que é um Model?¶

Models são PHP classes that are designed to work with information in seu Database. Por exemplo, digamos que você use o CodeIgniter para gerenciar blog. Você pode ter uma classe de Model que contenha funções para inserir, atualizar e recuperar os dados do seu blog. Aqui está um exemplo do que tal A classe de Model pode se parecer com:

class Blog_model extends CI_Model {

        public $title;
        public $content;
        public $date;

        public function get_last_ten_entries()
        {
                $query = $this->db->get('entries', 10);
                return $query->result();
        }

        public function insert_entry()
        {
                $this->title    = $_POST['title']; // please read the below note
                $this->content  = $_POST['content'];
                $this->date     = time();

                $this->db->insert('entries', $this);
        }

        public function update_entry()
        {
                $this->title    = $_POST['title'];
                $this->content  = $_POST['content'];
                $this->date     = time();

                $this->db->update('entries', $this, array('id' => $_POST['id']));
        }

}

Nota

Os métodos no exemplo acima usam o Construtor de Querys métodos de Database.

Nota

Por uma questão de simplicidade, neste exemplo, estamos usando $_POST diretamente. Esta é geralmente uma prática ruim e uma abordagem mais comum seria usar o Library de entrada $this->input->post('title').

Anatomia de um Model¶

As classes de Model são armazenadas em seu application/models/ directory. Eles podem ser aninhados em subdiretórios se você quiser esse tipo de organization.

O protótipo básico de uma classe de Model é o seguinte:

class Model_name extends CI_Model {

}

Where Model_name é o nome da sua turma. Nomes de classe must have a primeira letra maiúscula com o resto do nome em minúsculas. Faço Certifique-se de que sua classe estenda a classe de Model base.

O nome do arquivo deve corresponder ao nome da classe. Por exemplo, se esta é sua classe:

class User_model extends CI_Model {

}

Seu arquivo será este:

application/models/User_model.php

Carregando um Model¶

Seus Models serão tipicamente carregados e chamados de dentro do seu controller métodos. Para carregar um Model, você usará o seguinte método:

$this->load->model('model_name');

Se o seu Model estiver localizado em um subdiretório, inclua o caminho relativo do seu diretório de Models. Por exemplo, se você tem um Model localizado em application/models/blog/Queries.php você carregará usando:

$this->load->model('blog/queries');

Uma vez carregado, você acessará seus métodos de Model usando um objeto com o mesmo nome da sua turma:

$this->load->model('model_name');

$this->model_name->method();

Se você quiser que seu Model seja atribuído a um nome de objeto diferente, você pode especificá-lo através do segundo parâmetro do método de carregamento:

$this->load->model('model_name', 'foobar');

$this->foobar->method();

Aqui está um exemplo de um Controller, que carrega um Model e, em seguida, serve um view:

class Blog_controller extends CI_Controller {

        public function blog()
        {
                $this->load->model('blog');

                $data['query'] = $this->blog->get_last_ten_entries();

                $this->load->view('blog', $data);
        }
}

Models de carregamento automático¶

Se você achar que precisa de um Model específico globalmente em toda a sua aplicação, você pode dizer ao CodeIgniter para auto-carregá-lo durante o inicialização. Isso é feito abrindo a application/config/autoload.php arquivo e adicionando o Model para o matriz de carregamento automático.

Conectando-se ao seu database¶

Quando um Model é carregado, ele NOT conectar-se automaticamente ao seu base de dados. As seguintes opções para conexão estão disponíveis para você:

  • Você pode se conectar usando os métodos de Database padrão described here, seja de dentro do seu Classe de Controller ou sua classe de Model.

  • Você pode dizer ao método de carregamento do Model para se conectar automaticamente passando TRUE (booleano) através do terceiro parâmetro, e configurações de conectividade, conforme definido no seu arquivo de configuração do database será usado:

    $this->load->model('model_name', '', TRUE);

  • Você pode passar manualmente as configurações de conectividade do Database através do terceiro parameter:

    $config['hostname'] = 'localhost';
$config['username'] = 'myusername';
$config['password'] = 'mypassword';
$config['database'] = 'mydatabase';
$config['dbdriver'] = 'mysqli';
$config['dbprefix'] = '';
$config['pconnect'] = FALSE;
$config['db_debug'] = TRUE;

$this->load->model('model_name', '', $config);

Models¶

Models are optionally disponível para aqueles que querem usar um abordagem MVC tradicional.

O que é um Model?¶

Models são classes PHP que são projetadas para trabalhar com informações seu Database. Por exemplo, digamos que você use o CodeIgniter para gerenciar blog. Você pode ter uma classe de Model que contenha funções para inserir, atualizar e recuperar os dados do seu blog. Aqui está um exemplo do que tal A classe de Model pode se parecer com:

class Blog_model extends CI_Model {

        public $title;
        public $content;
        public $date;

        public function get_last_ten_entries()
        {
                $query = $this->db->get('entries', 10);
                return $query->result();
        }

        public function insert_entry()
        {
                $this->title    = $_POST['title']; // please read the below note
                $this->content  = $_POST['content'];
                $this->date     = time();

                $this->db->insert('entries', $this);
        }

        public function update_entry()
        {
                $this->title    = $_POST['title'];
                $this->content  = $_POST['content'];
                $this->date     = time();

                $this->db->update('entries', $this, array('id' => $_POST['id']));
        }

}

Nota

Os métodos no exemplo acima usam o Construtor de Querys métodos de Database.

Nota

Por uma questão de simplicidade, neste exemplo, estamos usando $_POST diretamente. Esta é geralmente uma prática ruim e uma abordagem mais comum seria usar o Library de entrada $this->input->post('title').

Anatomia de um Model¶

As classes de Model são armazenadas em seu application/models/ directory. Eles podem ser aninhados em subdiretórios se você quiser esse tipo de organization.

O protótipo básico de uma classe de Model é o seguinte:

class Model_name extends CI_Model {

}

Where Model_name é o nome da sua turma. Nomes de classe must have a primeira letra maiúscula com o resto do nome em minúsculas. Faço Certifique-se de que sua classe estenda a classe de Model base.

O nome do arquivo deve corresponder ao nome da classe. Por exemplo, se esta é sua classe:

class User_model extends CI_Model {

}

Seu arquivo será este:

application/models/User_model.php

Carregando um Model¶

Seus Models serão tipicamente carregados e chamados de dentro do seu controller métodos. Para carregar um Model, você usará o seguinte método:

$this->load->model('model_name');

Se o seu Model estiver localizado em um subdiretório, inclua o caminho relativo do seu diretório de Models. Por exemplo, se você tem um Model localizado em application/models/blog/Queries.php você carregará usando:

$this->load->model('blog/queries');

Uma vez carregado, você acessará seus métodos de Model usando um objeto com o mesmo nome da sua turma:

$this->load->model('model_name');

$this->model_name->method();

Se você quiser que seu Model seja atribuído a um nome de objeto diferente, você pode especificá-lo através do segundo parâmetro do método de carregamento:

$this->load->model('model_name', 'foobar');

$this->foobar->method();

Aqui está um exemplo de um Controller, que carrega um Model e, em seguida, serve um view:

class Blog_controller extends CI_Controller {

        public function blog()
        {
                $this->load->model('blog');

                $data['query'] = $this->blog->get_last_ten_entries();

                $this->load->view('blog', $data);
        }
}

Models de carregamento automático¶

Se você achar que precisa de um Model específico globalmente em toda a sua aplicação, você pode dizer ao CodeIgniter para auto-carregá-lo durante o inicialização. Isso é feito abrindo a application/config/autoload.php arquivo e adicionando o Model para o matriz de carregamento automático.

Conectando-se ao seu database¶

Quando um Model é carregado, ele NOT conectar-se automaticamente ao seu base de dados. As seguintes opções para conexão estão disponíveis para você:

  • Você pode se conectar usando os métodos de Database padrão described here, seja de dentro do seu Classe de Controller ou sua classe de Model.

  • Você pode dizer ao método de carregamento do Model para se conectar automaticamente passando TRUE (booleano) através do terceiro parâmetro, e configurações de conectividade, conforme definido no seu arquivo de configuração do database será usado:

    $this->load->model('model_name', '', TRUE);

  • Você pode passar manualmente as configurações de conectividade do Database através do terceiro parameter:

    $config['hostname'] = 'localhost';
$config['username'] = 'myusername';
$config['password'] = 'mypassword';
$config['database'] = 'mydatabase';
$config['dbdriver'] = 'mysqli';
$config['dbprefix'] = '';
$config['pconnect'] = FALSE;
$config['db_debug'] = TRUE;

$this->load->model('model_name', '', $config);
