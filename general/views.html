Views¶

A view is simply a web page, or a page fragment, like a header, footer, barra lateral, etc. Na verdade, as visualizações podem ser incorporadas de maneira flexível a outras visualizações (within other views, etc., etc.) if you need this type of hierarchy.

As visualizações nunca são chamadas diretamente, elas devem ser carregadas por um controller. Lembre-se que em um framework MVC, o Controller acts as the traffic cop, so it is responsible for fetching a visão particular. Se você não leu o Controllers page you should do so before continuing.

Usando o controlador de exemplo que você criou no controller página, vamos adicionar uma visão para isso.

Criando uma Vista¶

Using your text editor, create a file called blogview.php, and put this in it:

<html>
<head>
        <title>My Blog</title>
</head>
<body>
        <h1>Welcome to my Blog!</h1>
</body>
</html>

Then save the file in your application/views/ directory.

Carregando uma visão¶

To load a particular view file you will use the following method:

$this->load->view('name');

Onde nome é o nome do seu arquivo de visão.

Nota

The .php file extension does not need to be specified a menos que você use algo diferente de .php.

Now, open the controller file you made earlier called Blog.php, and replace the echo statement with the view loading method:

<?php
class Blog extends CI_Controller {

        public function index()
        {
                $this->load->view('blogview');
        }
}

If you visit your site using the URL you did earlier you should see your new view. The URL was similar to this:

example.com/index.php/blog/

Carregando várias visualizações¶

CodeIgniter will intelligently handle multiple calls to $this->load->view() de dentro de um controlador. Se mais de uma chamada acontece que eles serão anexados juntos. Por exemplo, você pode querer tem uma vista de cabeçalho, uma vista de menu, uma vista de conteúdo e uma vista de rodapé. que might look something like this:

<?php

class Page extends CI_Controller {

        public function index()
        {
                $data['page_title'] = 'Your title';
                $this->load->view('header');
                $this->load->view('menu');
                $this->load->view('content', $data);
                $this->load->view('footer');
        }

}

No exemplo acima, estamos usando "dados adicionados dinamicamente", que você vai ver abaixo.

Armazenando exibições dentro de subdiretórios¶

Your view files can also be stored within sub-directories if you prefer esse tipo de organização. Ao fazer isso, você precisará incluir o directory name loading the view. Example:

$this->load->view('directory_name/file_name');

Adding Dynamic Data to the View¶

Data is passed from the controller to the view by way of an array or an object in the second parameter of the view loading method. Here is an example using an array:

$data = array(
        'title' => 'My Title',
        'heading' => 'My Heading',
        'message' => 'My Message'
);

$this->load->view('blogview', $data);

E aqui está um exemplo usando um objeto:

$data = new Someclass();
$this->load->view('blogview', $data);

Nota

If you use an object, the class variables will be turned em elementos de matriz.

Vamos tentar com o seu arquivo de controle. Abra-o adicione este código:

<?php
class Blog extends CI_Controller {

        public function index()
        {
                $data['title'] = "My Real Title";
                $data['heading'] = "My Real Heading";

                $this->load->view('blogview', $data);
        }
}

Now open your view file and change the text to variables that correspond para as chaves do array em seus dados:

<html>
<head>
        <title><?php echo $title;?></title>
</head>
<body>
        <h1><?php echo $heading;?></h1>
</body>
</html>

Em seguida, carregue a página no URL que você está usando e você verá a variables replaced.

Creating Loops¶

O array de dados que você passa para seus arquivos de visualização não se limita a simples variables. You can pass multi dimensional arrays, which can be looped to gerar várias linhas. Por exemplo, se você extrair dados do seu Database it will typically be in the form of a multi-dimensional array.

Aqui está um exemplo simples. Adicione isto ao seu controlador:

<?php
class Blog extends CI_Controller {

        public function index()
        {
                $data['todo_list'] = array('Clean House', 'Call Mom', 'Run Errands');

                $data['title'] = "My Real Title";
                $data['heading'] = "My Real Heading";

                $this->load->view('blogview', $data);
        }
}

Now open your view file and create a loop:

<html>
<head>
        <title><?php echo $title;?></title>
</head>
<body>
        <h1><?php echo $heading;?></h1>

        <h3>My Todo List</h3>

        <ul>
        <?php foreach ($todo_list as $item):?>

                <li><?php echo $item;?></li>

        <?php endforeach;?>
        </ul>

</body>
</html>

Nota

You’ll notice that in the example above we are using PHP’s sintaxe alternativa. Se você não estiver familiarizado com isso, pode ler sobre it here.

Returning views as data¶

There is a third optional parâmetro permite alterar o comportamento de the method so that it returns data as a string rather than sending it para o seu navegador. Isso pode ser útil se você quiser processar os dados em de alguma maneira. Se você definir o parâmetro como TRUE (booleano), ele retornará data. The default behavior is false, which sends it to your browser. Lembre-se de atribuí-lo a uma variável se você quiser que os dados sejam retornados:

$string = $this->load->view('myfile', '', TRUE);

Views¶

Uma visualização é simplesmente uma página da Web ou um fragmento de página, como um cabeçalho, um rodapé barra lateral, etc. Na verdade, as visualizações podem ser incorporadas de maneira flexível a outras visualizações (dentro de outras visões, etc., etc.) se você precisar desse tipo de hierarquia.

As visualizações nunca são chamadas diretamente, elas devem ser carregadas por um controller. Lembre-se que em um framework MVC, o O Controller atua como o policial de trânsito, por isso é responsável por buscar um View particular. Se você não leu o Controllers página você deve fazer isso antes continuing.

Usando o Controller de exemplo que você criou no controller página, vamos adicionar uma View para isso.

Criando uma Vista¶

Usando seu editor de texto, crie um arquivo chamado blogview.php e coloque nisso:

<html>
<head>
        <title>My Blog</title>
</head>
<body>
        <h1>Welcome to my Blog!</h1>
</body>
</html>

Then save the file in your application/views/ directory.

Carregando uma View¶

Para carregar um determinado arquivo de View, você utilizará o seguinte método:

$this->load->view('name');

Onde nome é o nome do seu arquivo de View.

Nota

A extensão de arquivo .php não precisa ser especificada a menos que você use algo diferente de .php.

Agora, abra o arquivo do Controller que você criou anteriormente chamado Blog.php, e substitua a instrução echo pelo método de carregamento de visualização:

<?php
class Blog extends CI_Controller {

        public function index()
        {
                $this->load->view('blogview');
        }
}

Se você visitar seu site usando o URL que você criou antes, verá sua nova View. O URL foi semelhante a este:

example.com/index.php/blog/

Carregando várias visualizações¶

O CodeIgniter manipulará de forma inteligente várias chamadas para $this->load->view() de dentro de um Controller. Se mais de uma chamada acontece que eles serão anexados juntos. Por exemplo, você pode querer tem uma vista de cabeçalho, uma vista de menu, uma vista de conteúdo e uma vista de rodapé. que pode ser algo como isto:

No exemplo acima, estamos usando "dados adicionados dinamicamente", que você vai ver abaixo.

Armazenando exibições dentro de subdiretórios¶

$this->load->view('directory_name/file_name');

Adicionando dados dinâmicos à vista¶

$data = array(
        'title' => 'My Title',
        'heading' => 'My Heading',
        'message' => 'My Message'
);

$this->load->view('blogview', $data);

$data = new Someclass();
$this->load->view('blogview', $data);

Vamos tentar com o seu arquivo de controle. Abra-o adicione este código:

<?php

        public function index()
        {
                $data['title'] = "My Real Title";
                $data['heading'] = "My Real Heading";

<html>

</body>

Criando Loops¶

O array de dados que você passa para seus arquivos de visualização não se limita a simples variáveis. Você pode passar matrizes multidimensionais, que podem ser colocadas em loop para gerar várias linhas. Por exemplo, se você extrair dados do seu Database normalmente estará na forma de um array multidimensional.

Aqui está um exemplo simples. Adicione isto ao seu Controller:

<?php
class Blog extends CI_Controller {

        public function index()
        {
                $data['todo_list'] = array('Clean House', 'Call Mom', 'Run Errands');

                $data['title'] = "My Real Title";
                $data['heading'] = "My Real Heading";

                $this->load->view('blogview', $data);
        }
}

Agora abra seu arquivo de View e crie um loop:

<html>
<head>
        <title><?php echo $title;?></title>
</head>
<body>
        <h1><?php echo $heading;?></h1>

        <h3>My Todo List</h3>

        <ul>

                <li><?php echo $item;?></li>

Views¶

Uma visualização é simplesmente uma página da Web ou um fragmento de página, como um cabeçalho, um rodapé barra lateral, etc. Na verdade, as visualizações podem ser incorporadas de maneira flexível a outras visualizações (dentro de outras visões, etc., etc.) se você precisar desse tipo de hierarquia.

As visualizações nunca são chamadas diretamente, elas devem ser carregadas por um controller. Lembre-se que em um framework MVC, o O Controller atua como o policial de trânsito, por isso é responsável por buscar um View particular. Se você não leu o Controllers página você deve fazer isso antes continuing.

Usando o Controller de exemplo que você criou no controller página, vamos adicionar uma View para isso.

Criando uma Vista¶

Usando seu editor de texto, crie um arquivo chamado blogview.php e coloque in it:

<html>
<head>
        <title>My Blog</title>
</head>
<body>
        <h1>Welcome to my Blog!</h1>
</body>
</html>

Then save the file in your application/views/ directory.

Carregando uma View¶

Para carregar um determinado arquivo de View, você utilizará o seguinte método:

$this->load->view('name');

Onde nome é o nome do seu arquivo de View.

Nota

A extensão de arquivo .php não precisa ser especificada a menos que você use algo diferente de .php.

Agora, abra o arquivo do Controller que você criou anteriormente chamado Blog.php, e substitua a instrução echo pelo método de carregamento de visualização:

<?php
class Blog extends CI_Controller {

        public function index()
        {
                $this->load->view('blogview');
        }
}

Se você visitar seu site usando o URL que você criou antes, verá sua nova View. O URL foi semelhante a este:

example.com/index.php/blog/

Carregando várias visualizações¶

O CodeIgniter manipulará de forma inteligente várias chamadas para $this->load->view() de dentro de um Controller. Se mais de uma chamada acontece que eles serão anexados juntos. Por exemplo, você pode querer tem uma vista de cabeçalho, uma vista de menu, uma vista de conteúdo e uma vista de rodapé. que pode ser algo como isto:

<?php

class Page extends CI_Controller {

        public function index()
        {
                $data['page_title'] = 'Your title';
                $this->load->view('header');
                $this->load->view('menu');
                $this->load->view('content', $data);
                $this->load->view('footer');
        }

}

No exemplo acima, estamos usando "dados adicionados dinamicamente", que você vai ver abaixo.

Armazenando exibições dentro de subdiretórios¶

Seus arquivos de View também podem ser armazenados em subdiretórios se você preferir esse tipo de organização. Ao fazer isso, você precisará incluir o nome do diretório carregando a exibição. Exemplo:

$this->load->view('directory_name/file_name');

Adicionando dados dinâmicos à vista¶

Os dados são passados do Controller para a View por meio de array or an object no segundo parâmetro do método de carregamento da visualização. Aqui é um exemplo usando uma matriz:

$data = array(
        'title' => 'My Title',
        'heading' => 'My Heading',
        'message' => 'My Message'
);

$this->load->view('blogview', $data);

E aqui está um exemplo usando um objeto:

$data = new Someclass();
$this->load->view('blogview', $data);

Nota

Se você usar um objeto, as variáveis de classe serão ativadas em elementos de matriz.

Vamos tentar com o seu arquivo de controle. Abra-o adicione este código:

<?php
class Blog extends CI_Controller {

        public function index()
        {
                $data['title'] = "My Real Title";
                $data['heading'] = "My Real Heading";

                $this->load->view('blogview', $data);
        }
}

Agora abra seu arquivo de View e mude o texto para variáveis que correspondam para as chaves do array em seus dados:

<html>
<head>
        <title><?php echo $title;?></title>
</head>
<body>
        <h1><?php echo $heading;?></h1>
</body>
</html>

Em seguida, carregue a página no URL que você está usando e você verá a variáveis substituídas.

Criando Loops¶

O array de dados que você passa para seus arquivos de visualização não se limita a simples variáveis. Você pode passar matrizes multidimensionais, que podem ser colocadas em loop para gerar várias linhas. Por exemplo, se você extrair dados do seu Database normalmente estará na forma de um array multidimensional.

Aqui está um exemplo simples. Adicione isto ao seu Controller:

<?php
class Blog extends CI_Controller {

        public function index()
        {
                $data['todo_list'] = array('Clean House', 'Call Mom', 'Run Errands');

                $data['title'] = "My Real Title";
                $data['heading'] = "My Real Heading";

                $this->load->view('blogview', $data);
        }
}

Agora abra seu arquivo de View e crie um loop:

<html>
<head>
        <title><?php echo $title;?></title>
</head>
<body>
        <h1><?php echo $heading;?></h1>

        <h3>My Todo List</h3>

        <ul>
        <?php foreach ($todo_list as $item):?>

                <li><?php echo $item;?></li>

        <?php endforeach;?>
        </ul>

</body>
</html>

Nota

Você notará que no exemplo acima estamos usando PHP's sintaxe alternativa. Se você não estiver familiarizado com isso, pode ler sobre it here.
