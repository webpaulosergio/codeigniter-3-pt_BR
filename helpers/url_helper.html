Helper de URL¶

o Helper de URL file contains functions that assist in working with URLs.

  • Loading this Helper
  • Funções Disponíveis

Loading this Helper¶

Este helper é carregado usando o seguinte código:

$this->load->helper('url');

Funções Disponíveis¶

As seguintes funções estão disponíveis:

site_url([$uri = ''[, $protocol = NULL]])¶
Parâmetros:
  • $uri (string) - string URI
  • $protocol (string) – Protocol, e.g. ‘http’ or ‘https’
Returns:

Site URL
Return type:

string

Returns your site URL, as specified in your config file. o index.php arquivo (ou o que você definiu como seu site index_page in your config arquivo) será adicionado ao URL, assim como qualquer segmento URI que você passar para o function, plus the url_suffix como definido no seu arquivo de configuração.

Você é encorajado a usar esta função sempre que precisar gerar um local URL so that your pages become more portable in the event your URL changes.

Segments can be optionally passed to the function as a string or an array. Aqui está um exemplo de string:

echo site_url('news/local/123');

o above example would return something like: http://example.com/index.php/news/local/123

Here is an example of segments passed as an array:

$segments = array('news', 'local', '123');
echo site_url($segments);

This function is an alias for CI_Config::site_url(). For more info, please see the Config Library documentation.

base_url($uri = '', $protocol = NULL)¶
Parâmetros:
  • $uri (string) - string URI
  • $protocol (string) – Protocol, e.g. ‘http’ or ‘https’
Returns:

URL base
Return type:

string

Retorna o URL base do seu site, conforme especificado no seu arquivo de configuração. Exemplo:

echo base_url();

Esta função retorna a mesma coisa que site_url(), without the index_page or url_suffix being appended.

Also like site_url(), you can supply segments as a string or an array. Aqui está um exemplo de string:

echo base_url("blog/post/123");

o above example would return something like: http://example.com/blog/post/123

This is useful because unlike site_url(), você pode fornecer uma string para um arquivo, como uma imagem ou folha de estilo. Por exemplo:

echo base_url("images/icons/edit.png");

Isso lhe daria algo como: http://example.com/images/icons/edit.png

This function is an alias for CI_Config::base_url(). For more info, please see the Config Library documentation.

current_url()¶
Returns:o current URL
Return type:string

Retorna o URL completo (incluindo segmentos) da página que está sendo atualmente viewed.

Nota

Calling this function is the same as doing this: | | site_url(uri_string());

uri_string()¶
Returns:An URI string
Return type:string

Retorna os segmentos de URI de qualquer página que contenha essa função. For example, if your URL was this:

http://some-site.com/blog/comments/123

A função retornaria:

blog/comments/123

This function is an alias for CI_Config::uri_string(). For more info, please see the Config Library documentation.

index_page()¶
Returns:valor 'index_page'
Return type:mixed

Returns your site index_page, as specified in your config file. Example:

echo index_page();
anchor($uri = '', $title = '', $attributes = '')¶
Parâmetros:
  • $uri (string) - string URI
  • $title (string) - título da âncora
  • $attributes (mixed) - atributos HTML
Returns:

HTML hyperlink (anchor tag)
Return type:

string

Cria um link de âncora HTML padrão com base na URL do seu site local.

O primeiro parâmetro pode conter todos os segmentos que você deseja anexar ao URL. As with the site_url() função acima, os segmentos podem seja uma string ou um array.

Nota

If you are building links that are internal to your application não inclua o URL base (http: //…). Isso será adicionado automaticamente a partir das informações especificadas no seu arquivo de configuração. Inclua apenas os segmentos de URI que você deseja anexar ao URL.

O segundo segmento é o texto que você gostaria que o link dissesse. Se vocês deixe em branco, o URL será usado.

o third parameter can contain a list of attributes you would like adicionado ao link. Os atributos podem ser uma string simples ou um associative array.

Here are some examples:

echo anchor('news/local/123', 'My News', 'title="News title"');
// Prints: <a href="http://example.com/index.php/news/local/123" title="News title">My News</a>

echo anchor('news/local/123', 'My News', array('title' => 'o best news!'));
// Prints: <a href="http://example.com/index.php/news/local/123" title="o best news!">My News</a>

echo anchor('', 'Click here');
// Prints: <a href="http://example.com">Click Here</a>
anchor_popup($uri = '', $title = '', $attributes = FALSE)¶
Parâmetros:
  • $uri (string) - string URI
  • $title (string) - título da âncora
  • $attributes (mixed) - atributos HTML
Returns:

Pop-up hyperlink
Return type:

string

Quase idêntico ao anchor() function except that it abre o URL em uma nova janela. Você pode especificar a janela JavaScript attributes in the third parameter to control how the window is opened. Se o terceiro parâmetro não for definido, ele simplesmente abrirá uma nova janela com your own browser settings.

Here is an example with attributes:

$atts = array(
        'width'       => 800,
        'height'      => 600,
        'scrollbars'  => 'yes',
        'status'      => 'yes',
        'resizable'   => 'yes',
        'screenx'     => 0,
        'screeny'     => 0,
        'window_name' => '_blank'
);

echo anchor_popup('news/local/123', 'Click Me!', $atts);

Nota

o above attributes are the function defaults so you only need to defina os que são diferentes do que você precisa. Se você quiser o function to use all of its defaults simply pass an empty array in the third parameter: | | echo anchor_popup(‘news/local/123’, ‘Click Me!’, array());

Nota

o window_name não é realmente um atributo, mas um argumento para the JavaScript window.open() <http://www.w3schools.com/jsref/met_win_open.asp> method, which accepts either a window name or a window target.

Nota

Any other attribute than the listed above will be parsed as an HTML attribute to the anchor tag.

mailto($email, $title = '', $attributes = '')¶
Parâmetros:
  • $email (string) - Endereço de e-mail
  • $title (string) - título da âncora
  • $attributes (mixed) - atributos HTML
Returns:

A “mail to” hyperlink
Return type:

string

Creates a standard HTML e-mail link. Usage example:

echo mailto('me@my-site.com', 'Click Here to Contact Me');

As with the anchor() tab above, you can set attributes using the third parameter:

$attributes = array('title' => 'Mail me');
echo mailto('me@my-site.com', 'Contact Me', $attributes);
safe_mailto($email, $title = '', $attributes = '')¶
Parâmetros:
  • $email (string) - Endereço de e-mail
  • $title (string) - título da âncora
  • $attributes (mixed) - atributos HTML
Returns:

A spam-safe “mail to” hyperlink
Return type:

string

Identical to the mailto() function except it writes an obfuscated version of the mailto tag using ordinal numbers written with JavaScript to help prevent the e-mail address from being harvested by spam bots.

auto_link($str, $type = 'both', $popup = FALSE)¶
Parâmetros:
  • $str (string) – Input string
  • $type (string) – Link type (‘email’, ‘url’ or ‘both’)
  • $popup (bool) – Whether to create popup links
Returns:

Linkified string
Return type:

string

Automatically turns URLs and e-mail addresses contained in a string into links. Example:

$string = auto_link($string);

o second parameter determines whether URLs and e-mails are converted or just one or the other. Default behavior is both if the parameter is not specified. E-mail links are encoded as safe_mailto() as shown above.

Converte apenas URLs:

$string = auto_link($string, 'url');

Converts only e-mail addresses:

$string = auto_link($string, 'email');

o third parameter determines whether links are shown in a new window. o value can be TRUE or FALSE (boolean):

$string = auto_link($string, 'both', TRUE);
url_title($str, $separator = '-', $lowercase = FALSE)¶
Parâmetros:
  • $str (string) – Input string
  • $separator (string) – Word separator
  • $lowercase (bool) - Se deseja transformar a cadeia de saída em minúscula
Returns:

URL-formatted string
Return type:

string

Takes a string as input and creates a human-friendly URL string. This is useful if, for example, you have a blog in which you’d like to use o title of your entries in the URL. Example:

$title = "What's wrong with CSS?";
$url_title = url_title($title);
// Produces: Whats-wrong-with-CSS

o second parameter determines the word delimiter. By default dashes are used. Preferred options are: - (traço) ou _ (underscore)

Example:

$title = "What's wrong with CSS?";
$url_title = url_title($title, 'underscore');
// Produces: Whats_wrong_with_CSS

Nota

Uso antigo de 'traço' e 'sublinhado' como o segundo parâmetro is DEPRECATED.

o third parameter determines whether or not lowercase characters are forçado. Por padrão, eles não são. As opções são booleanas TRUE/FALSE.

Example:

$title = "What's wrong with CSS?";
$url_title = url_title($title, 'underscore', TRUE);
// Produces: whats_wrong_with_css
prep_url($str = '')¶
Parâmetros:
  • $str (string) – URL string
Returns:

Protocol-prefixed URL string
Return type:

string

Esta função irá adicionar http: // no caso de um prefixo de protocolo is missing from a URL.

Passe a string de URL para a função assim:

$url = prep_url('example.com');
redirect($uri = '', $method = 'auto', $code = NULL)¶
Parâmetros:
  • $uri (string) - string URI
  • $method (string) – Redirect method (‘auto’, ‘location’ or ‘refresh’)
  • $code (string) – HTTP Response code (usually 302 or 303)
Return type:

void

Does a “header redirect” to the URI specified. If you specify the full site URL that link will be built, but for local links simply providing os segmentos URI para o controlador que você deseja direcionar irá criar o link. o function will build the URL based on your config file values.

o optional second parameter allows you to force a particular redirection method. o available methods are auto, location and refresh, com a localização sendo mais rápida, mas menos confiável em servidores IIS. o default is auto, which will attempt to intelligently choose the method based on the server environment.

o optional third parameter allows you to send a specific HTTP Response Código - isso pode ser usado, por exemplo, para criar redirecionamentos 301 para pesquisa engine purposes. o default Response Code is 302. o third parameter is only disponível com location redirects, and not refresh. Examples:

if ($logged_in == FALSE)
{
        redirect('/login/form/');
}

// with 301 redirect
redirect('/article/13', 'location', 301);

Nota

Para que esta função funcione, ela deve ser usada antes de qualquer coisa é enviado para o navegador, uma vez que utiliza cabeçalhos de servidor.

Nota

Para um controle muito fino sobre cabeçalhos, você deve usar o Output Library set_header() method.

Nota

To IIS users: if you hide the Server HTTP header, the auto method won’t detect IIS, in that case it is advised you explicitly use o refresh method.

Nota

When the location method is used, an HTTP status code of 303 will automatically be selected when the page is currently accessed via POST e HTTP/1.1 é usado.

Importante

This function will terminate script execution.

Helper de URL¶

o Helper de URL file contains functions that assist in working with URLs.

  • Carregando este Helper
  • Funções Disponíveis

Carregando este Helper¶

Este helper é carregado usando o seguinte código:

$this->load->helper('url');

Funções Disponíveis¶

As seguintes funções estão disponíveis:

site_url([$uri = ''[, $protocol = NULL]])¶
Parâmetros:
  • $uri (string) - string URI
  • $protocol (string) - Protocolo, por exemplo, 'http' ou 'https'
Retorna:

URL do site
Tipo de retorno:

string

Retorna o URL do seu site, conforme especificado no seu arquivo de configuração. O index.php arquivo (ou o que você definiu como seu site index_page na sua configuração arquivo) será adicionado ao URL, assim como qualquer segmento URI que você passar para o função, mais o url_suffix as set na sua configuração file.

Você é encorajado a usar esta função sempre que precisar gerar um URL local para que suas páginas se tornem mais portáteis no caso de seu URL changes.

Segmentos podem ser passados opcionalmente para a função como uma string ou array. Aqui está um exemplo de string:

echo site_url('news/local/123');

O exemplo acima retornaria algo como: http://example.com/index.php/news/local/123

Aqui está um exemplo de segmentos passados como uma matriz:

$segments = array('news', 'local', '123');
echo site_url($segments);

This function is an alias for CI_Config::site_url(). For more info, por favor veja o Library de configuração documentation.

base_url($uri = '', $protocol = NULL)¶
Parâmetros:
  • $uri (string) - string URI
  • $protocol (string) - Protocolo, por exemplo, 'http' ou 'https'
Retorna:

URL base
Tipo de retorno:

string

Retorna seu site base URL, as specified na sua configuração file. Exemplo:

echo base_url();

Esta função retorna a mesma coisa que site_url()sem the index_page or url_suffix being appended.

Também gosto site_url(), você pode fornecer segmentos como uma string ou an array. Aqui está um exemplo de string:

echo base_url("blog/post/123");

O exemplo acima retornaria algo como: http://example.com/blog/post/123

Isso é útil porque ao contrário site_url(), você pode fornecer uma string para um arquivo, como uma imagem ou folha de estilo. Por exemplo:

echo base_url("images/icons/edit.png");

Isso lhe daria algo como: http://example.com/images/icons/edit.png

This function is an alias for CI_Config::base_url(). For more info, por favor veja o Library de configuração documentation.

current_url()¶
Retorna:O URL atual
Tipo de retorno:string

Retorna o URL completo (incluindo segmentos) da página que está sendo atualmente viewed.

Nota

Chamar essa função é o mesmo que fazer isso: | | site_url (uri_string());

uri_string()¶
Retorna:Uma string de URI
Tipo de retorno:string

Retorna os segmentos de URI de qualquer página que contenha essa função. Por exemplo, se o seu URL fosse este:

http://some-site.com/blog/comments/123

A função retornaria:

blog/comments/123

This function is an alias for CI_Config::uri_string(). For more info, por favor veja o Library de configuração documentation.

index_page()¶
Retorna:valor 'index_page'
Tipo de retorno:mixed

Retorna seu site index_page, as specified na sua configuração file. Exemplo:

echo index_page();
anchor($uri = '', $title = '', $attributes = '')¶
Parâmetros:
  • $uri (string) - string URI
  • $title (string) - título da âncora
  • $attributes (mixed) - atributos HTML
Retorna:

Hyperlink HTML (tag âncora)
Tipo de retorno:

string

Cria um link de âncora HTML padrão com base na URL do seu site local.

O primeiro parâmetro pode conter todos os segmentos que você deseja anexar ao URL. Como com o site_url() função acima, os segmentos podem seja uma string ou um array.

Nota

Se você estiver criando links internos à sua aplicação não inclua o URL base (http: //). Isso será adicionado automatically from the information specified na sua configuração file. Inclua apenas os segmentos de URI que você deseja anexar ao URL.

O segundo segmento é o texto que você gostaria que o link dissesse. Se vocês deixe em branco, o URL será usado.

O terceiro parâmetro pode conter uma lista de atributos que você gostaria adicionado ao link. Os atributos podem ser uma string simples ou um matriz associativa.

aqui estão alguns exemplos:

echo anchor('news/local/123', 'My News', 'title="News title"');
// Prints: <a href="http://example.com/index.php/news/local/123" title="News title">My News</a>

echo anchor('news/local/123', 'My News', array('title' => 'o best news!'));
// Prints: <a href="http://example.com/index.php/news/local/123" title="o best news!">My News</a>

echo anchor('', 'Click here');
// Prints: <a href="http://example.com">Click Here</a>
anchor_popup($uri = '', $title = '', $attributes = FALSE)¶
Parâmetros:
  • $uri (string) - string URI
  • $title (string) - título da âncora
  • $attributes (mixed) - atributos HTML
Retorna:

Hiperlink pop-up
Tipo de retorno:

string

Quase idêntico ao anchor() função exceto que abre o URL em uma nova janela. Você pode especificar a janela JavaScript atributos no terceiro parâmetro para controlar como a janela é aberta. Se o terceiro parâmetro não for definido, ele simplesmente abrirá uma nova janela com suas próprias configurações do navegador.

Aqui está um exemplo com atributos:

$atts = array(
        'width'       => 800,
        'height'      => 600,
        'scrollbars'  => 'yes',
        'status'      => 'yes',
        'resizable'   => 'yes',
        'screenx'     => 0,
        'screeny'     => 0,
        'window_name' => '_blank'
);

echo anchor_popup('news/local/123', 'Click Me!', $atts);

Nota

Os atributos acima são os padrões da função, então você só precisa defina os que são diferentes do que você precisa. Se você quiser o função de usar todos os seus padrões simplesmente passar um array vazio no terceiro parâmetro: | | echo anchor_popup ('news/local/123', 'Clique em mim!', array());

Nota

o window_name não é realmente um atributo, mas um argumento para o JavaScript window.open() <http://www.w3schools.com/jsref/met_win_open.asp> método, que aceita um nome de janela ou um destino de janela.

Nota

Qualquer outro atributo além dos listados acima será analisado como um Atributo HTML para a tag de âncora.

mailto($email, $title = '', $attributes = '')¶
Parâmetros:
  • $email (string) - Endereço de e-mail
  • $title (string) - título da âncora
  • $attributes (mixed) - atributos HTML
Retorna:

Um hiperlink "mail to"
Tipo de retorno:

string

Cria um link de email HTML padrão. Exemplo de uso:

echo mailto('me@my-site.com', 'Click Here to Contact Me');

Como com o anchor() aba acima, você pode definir atributos usando o terceiro parâmetro:

$attributes = array('title' => 'Mail me');
echo mailto('me@my-site.com', 'Contact Me', $attributes);
safe_mailto($email, $title = '', $attributes = '')¶
Parâmetros:
  • $email (string) - Endereço de e-mail
  • $title (string) - título da âncora
  • $attributes (mixed) - atributos HTML
Retorna:

Um hiperlink de "email para" com spam
Tipo de retorno:

string

Idêntico ao mailto() função, exceto que escreve um ofuscado version of the mailto tag using ordinal numbers written with JavaScript to Ajudar a impedir que o endereço de e-mail seja coletado por robôs de spam.

auto_link($str, $type = 'both', $popup = FALSE)¶
Parâmetros:
  • $str (string) - string de entrada
  • $type (string) - Tipo de link ('email', 'url' ou 'both')
  • $popup (bool) - Criar links pop-up
Retorna:

Corda Linkificada
Tipo de retorno:

string

Automaticamente transforma URLs e endereços de e-mail contidos em uma string em links. Exemplo:

$string = auto_link($string);

o second parameter determines whether URLs and e-mails are converted or apenas um ou outro. O comportamento padrão é ambos se o parâmetro não for Especificadas. Links de e-mail são codificados como safe_mailto() como mostrado above.

Converte apenas URLs:

$string = auto_link($string, 'url');

Converte apenas endereços de e-mail:

$string = auto_link($string, 'email');

o third parameter determines whether links are shown in a new window. o value can be TRUE or FALSE (boolean):

$string = auto_link($string, 'both', TRUE);
url_title($str, $separator = '-', $lowercase = FALSE)¶
Parâmetros:
  • $str (string) - string de entrada
  • $separator (string) - separador de palavras
  • $lowercase (bool) - Se deseja transformar a cadeia de saída em minúscula
Retorna:

String formatada por URL
Tipo de retorno:

string

Toma uma string como entrada e cria uma string de URL amigável para o ser humano. Isto é útil se, por exemplo, você tiver um blog no qual gostaria de usar o título de suas entradas no URL. Exemplo:

$title = "What's wrong with CSS?";
$url_title = url_title($title);
// Produces: Whats-wrong-with-CSS

o second parameter determines the word delimiter. By default dashes são usados. As opções preferidas são: - (traço) ou _ (underscore)

Exemplo:

$title = "What's wrong with CSS?";
$url_title = url_title($title, 'underscore');
// Produces: Whats_wrong_with_CSS

Nota

Uso antigo de 'traço' e 'sublinhado' como o segundo parâmetro é DEPRECADO.

o third parameter determines whether or not lowercase characters are forçado. Por padrão, eles não são. As opções são booleanas TRUE/FALSE.

Exemplo:

$title = "What's wrong with CSS?";
$url_title = url_title($title, 'underscore', TRUE);
// Produces: whats_wrong_with_css
prep_url($str = '')¶
Parâmetros:
  • $str (string) - cadeia de URL
Retorna:

Sequência de URL com prefixo de protocolo
Tipo de retorno:

string

Esta função irá adicionar http: // no caso de um prefixo de protocolo está faltando em um URL.

Passe a string de URL para a função assim:

$url = prep_url('example.com');
redirect($uri = '', $method = 'auto', $code = NULL)¶
Parâmetros:
  • $uri (string) - string URI
  • $method (string) - Método de redirecionamento ('auto', 'location' ou 'refresh')
  • $code (string) - Código de resposta HTTP (geralmente 302 ou 303)
Tipo de retorno:

void

Faz um "redirecionamento de cabeçalho" para o URI especificado. Se você especificar o URL do site que o link será construído, mas para links locais simplesmente fornecendo os segmentos URI para o Controller que você deseja direcionar irá criar o link. o function will build the URL based on your config file values.

o optional second parameter allows you to force a particular redirection method. o available methods are auto, location and refresh, com a localização sendo mais rápida, mas menos confiável em servidores IIS. o default is auto, que tentará escolher inteligentemente o método baseado no ambiente do servidor.

o optional third parameter allows you to send a specific HTTP Response Código - isso pode ser usado, por exemplo, para criar redirecionamentos 301 para pesquisa engine purposes. o default Response Code is 302. o third parameter is only disponível com location redirects, and not refresh. Examples:

if ($logged_in == FALSE)
{
        redirect('/login/form/');
}

// with 301 redirect
redirect('/article/13', 'location', 301);

Nota

Para que esta função funcione, ela deve ser usada antes de qualquer coisa é enviado para o navegador, uma vez que utiliza cabeçalhos de servidor.

Nota

Para um controle muito fino sobre cabeçalhos, você deve usar o Library de saída set_header() method.

Nota

Para usuários do IIS: se você ocultar o Servidor Cabeçalho HTTP, o auto método não irá detectar o IIS, nesse caso, é aconselhável explicitamente use o refresh method.

Nota

Quando o location método é usado, um código de status HTTP de 303 will automatically be selected when the page is currently accessed via POST e HTTP/1.1 é usado.

Importante

Esta função terminará a execução do script.

Quase idêntico ao anchor() função exceto que abre o URL em uma nova janela. Você pode especificar a janela JavaScript atributos no terceiro parâmetro para controlar como a janela é aberta. Se o terceiro parâmetro não for definido, ele simplesmente abrirá uma nova janela com suas próprias configurações do navegador.

Aqui está um exemplo com atributos:

$atts = array(
        'width'       => 800,
        'height'      => 600,
        'scrollbars'  => 'yes',
        'status'      => 'yes',
        'resizable'   => 'yes',
        'screenx'     => 0,
        'screeny'     => 0,
        'window_name' => '_blank'
);

echo anchor_popup('news/local/123', 'Click Me!', $atts);

Nota

Os atributos acima são os padrões da função, então você só precisa defina os que são diferentes do que você precisa. Se você quiser o função de usar todos os seus padrões simplesmente passar um array vazio no terceiro parâmetro: | | echo anchor_popup ('news/local/123', 'Clique em mim!', array());

Nota

o window_name não é realmente um atributo, mas um argumento para o JavaScript window.open() <http://www.w3schools.com/jsref/met_win_open.asp> método, que aceita um nome de janela ou um destino de janela.

Nota

Qualquer outro atributo além dos listados acima será analisado como um Atributo HTML para a tag de âncora.

auto_link($str, $type = 'both', $popup = FALSE)¶
Parâmetros:
  • $str (string) - string de entrada
  • $type (string) - Tipo de link ('email', 'url' ou 'both')
  • $popup (bool) - Criar links pop-up
Retorna:

Corda Linkificada
Tipo de retorno:

string

Automaticamente transforma URLs e endereços de e-mail contidos em uma string em links. Exemplo:

$string = auto_link($string);

o second parameter determines whether URLs and e-mails are converted or apenas um ou outro. O comportamento padrão é ambos se o parâmetro não for Especificadas. Links de e-mail são codificados como safe_mailto() as shown above.

Converte apenas URLs:

$string = auto_link($string, 'url');

Converte apenas endereços de e-mail:

$string = auto_link($string, 'email');

o third parameter determines whether links are shown in a new window. o value can be TRUE or FALSE (boolean):

$string = auto_link($string, 'both', TRUE);
url_title($str, $separator = '-', $lowercase = FALSE)¶
Parâmetros:
  • $str (string) - string de entrada
  • $separator (string) - separador de palavras
  • $lowercase (bool) - Se deseja transformar a cadeia de saída em minúscula
Retorna:

String formatada por URL
Tipo de retorno:

string

Toma uma string como entrada e cria uma string de URL amigável para o ser humano. Isto é útil se, por exemplo, você tiver um blog no qual gostaria de usar o título de suas entradas no URL. Exemplo:

$title = "What's wrong with CSS?";
$url_title = url_title($title);
// Produces: Whats-wrong-with-CSS

o second parameter determines the word delimiter. By default dashes são usados. As opções preferidas são: - (dash) or _ (underscore)

Exemplo:

$title = "What's wrong with CSS?";
$url_title = url_title($title, 'underscore');
// Produces: Whats_wrong_with_CSS

Nota

Uso antigo de 'traço' e 'sublinhado' como o segundo parâmetro é DEPRECADO.

o third parameter determines whether or not lowercase characters are forçado. Por padrão, eles não são. As opções são booleanas TRUE/FALSE.

Exemplo:

$title = "What's wrong with CSS?";
$url_title = url_title($title, 'underscore', TRUE);
// Produces: whats_wrong_with_css

redirect($uri = '', $method = 'auto', $code = NULL)¶
Parâmetros:
  • $uri (string) - string URI
  • $method (string) - Método de redirecionamento ('auto', 'location' ou 'refresh')
  • $code (string) - Código de resposta HTTP (geralmente 302 ou 303)
Tipo de retorno:

void

Faz um "redirecionamento de cabeçalho" para o URI especificado. Se você especificar o URL do site que o link será construído, mas para links locais simplesmente fornecendo os segmentos URI para o Controller que você deseja direcionar irá criar o link. o function will build the URL based on your config file values.

o optional second parameter allows you to force a particular redirection method. o available methods are auto, location and refresh, com a localização sendo mais rápida, mas menos confiável em servidores IIS. o default is auto, que tentará escolher inteligentemente o método baseado no ambiente do servidor.

o optional third parameter allows you to send a specific HTTP Response Código - isso pode ser usado, por exemplo, para criar redirecionamentos 301 para pesquisa engine purposes. o default Response Code is 302. o third parameter is only disponível com location redirects, and not refresh. Examples:

if ($logged_in == FALSE)
{
        redirect('/login/form/');
}

// with 301 redirect
redirect('/article/13', 'location', 301);

Nota

Para que esta função funcione, ela deve ser usada antes de qualquer coisa é enviado para o navegador, uma vez que utiliza cabeçalhos de servidor.

Nota

Para um controle muito fino sobre cabeçalhos, você deve usar o Library de saída set_header() method.

Nota

Para usuários do IIS: se você ocultar o Server Cabeçalho HTTP, o auto método não irá detectar o IIS, nesse caso, é aconselhável explicitamente use the refresh method.

Nota

When the location método é usado, um código de status HTTP de 303 will automatically be selected when the page is currently accessed via POST e HTTP/1.1 é usado.

Important

Esta função terminará a execução do script.
