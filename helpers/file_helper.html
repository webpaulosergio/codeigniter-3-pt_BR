Arquivo auxiliar¶

The Arquivo auxiliar file contains functions that assist in working with files.

  • Loading this Helper
  • Funções Disponíveis

Loading this Helper¶

Este helper é carregado usando o seguinte código:

$this->load->helper('file');

Funções Disponíveis¶

As seguintes funções estão disponíveis:

read_file($file)¶
Parâmetros:
  • $file (string) - Caminho de arquivo
Returns:

File contents or FALSE on failure
Return type:

string

Retorna os dados contidos no arquivo especificado no caminho.

Example:

$string = read_file('./path/to/file.php');

The path can be a relative or full server path. Returns FALSE (boolean) on failure.

Nota

O caminho é relativo ao seu arquivo index.php do site principal, NÃO ao seu controller or view files. CodeIgniter uses a front controller so paths are always relative to the main site index.

Nota

Esta função é DEPRECADA. Use o nativo file_get_contents() instead.

Importante

Se o seu servidor estiver executando open_basedir restriction this function might not work if you are trying to access a file above the calling script.

write_file($path, $data[, $mode = 'wb'])¶
Parâmetros:
  • $path (string) - Caminho de arquivo
  • $data (string) - Dados para gravar no arquivo
  • $mode (string) – fopen() mode
Returns:

TRUE if the write was successful, FALSE in case of an error
Return type:

bool

Grava dados no arquivo especificado no caminho. Se o arquivo não existir, o função irá criá-lo.

Example:

$data = 'Some file data';
if ( ! write_file('./path/to/file.php', $data))
{
        echo 'Unable to write the file';
}
else
{
        echo 'File written!';
}

You can optionally set the write mode via the third parameter:

write_file('./path/to/file.php', $data, 'r+');

The default mode is ‘wb’. Please see the Guia do usuário PHP para as opções de modo.

Nota

O caminho é relativo ao seu arquivo index.php do site principal, NÃO ao seu controller or view files. CodeIgniter uses a front controller so paths are always relative to the main site index.

Nota

This function acquires an exclusive lock on the file while writing to it.

delete_files($path[, $del_dir = FALSE[, $htdocs = FALSE]])¶
Parâmetros:
  • $path (string) – Directory path
  • $del_dir (bool) - Se também excluir diretórios
  • $htdocs (bool) - Ignorar a exclusão de arquivos .htaccess e de páginas de índice
Returns:

TRUE on success, FALSE in case of an error
Return type:

bool

Deletes ALL files contained in the supplied path.

Example:

delete_files('./path/to/directory/');

If the second parameter is set to TRUE, any directories contained within the supplied root path will be deleted as well.

Example:

delete_files('./path/to/directory/', TRUE);

Nota

The files must be writable or owned by the system in order to be deleted.

get_filenames($source_dir[, $include_path = FALSE])¶
Parâmetros:
  • $source_dir (string) – Directory path
  • $include_path (bool) – Whether to include the path as part of the filenames
Returns:

An array of file names
Return type:

array

Utiliza um caminho do servidor como entrada e retorna uma matriz contendo os nomes de todos os arquivos contido dentro dele. O caminho do arquivo pode, opcionalmente, ser adicionado aos nomes dos arquivos, definindo the second parameter to TRUE.

Example:

$controllers = get_filenames(APPPATH.'controllers/');
get_dir_file_info($source_dir, $top_level_only)¶
Parâmetros:
  • $source_dir (string) – Directory path
  • $top_level_only (bool) - Se procurar apenas no diretório especificado (excluindo subdiretórios)
Returns:

An array containing info on the supplied directory’s contents
Return type:

array

Reads the specified directory and builds an array containing the filenames, filesize, dates, and permissions. Sub-folders contained within the specified path are only read if forced by sending the second parameter to FALSE, as this can be an intensive operation.

Example:

$models_info = get_dir_file_info(APPPATH.'models/');
get_file_info($file[, $returned_values = array('name', 'server_path', 'size', 'date')])¶
Parâmetros:
  • $file (string) - Caminho de arquivo
  • $returned_values (array) – What type of info to return
Returns:

An array containing info on the specified file or FALSE on failure
Return type:

array

Given a file and path, returns (optionally) the name, path, size and date modified information attributes for a file. Second parameter allows you to explicitly declare what informações que você deseja retornar.

Valid $returned_values options are: name, size, date, readable, writeable, executable and fileperms.

get_mime_by_extension($filename)¶
Parâmetros:
  • $filename (string) - Nome do arquivo
Returns:

MIME type string or FALSE on failure
Return type:

string

Translates a filename extension into a MIME type based on config/mimes.php. Returns FALSE if it can’t determine the type, or read the MIME config file.

$file = 'somefile.png';
echo $file.' is has a mime type of '.get_mime_by_extension($file);

Nota

This is not an accurate way of determining file MIME types, and está aqui estritamente por conveniência. Não deve ser usado para segurança purposes.

symbolic_permissions($perms)¶
Parâmetros:
  • $perms (int) – Permissions
Returns:

String de permissões simbólicas
Return type:

string

Takes numeric permissions (such as is returned by fileperms()) and returns standard symbolic notation of file permissions.

echo symbolic_permissions(fileperms('./index.php'));  // -rw-r--r--
octal_permissions($perms)¶
Parâmetros:
  • $perms (int) – Permissions
Returns:

Octal permissions string
Return type:

string

Takes numeric permissions (such as is returned by fileperms()) and returns a three character octal notation of file permissions.

echo octal_permissions(fileperms('./index.php')); // 644

$this->load->helper('file');

As seguintes funções estão disponíveis:

Parâmetros:
  • $file (string) - Caminho de arquivo
Retorna:

Conteúdo do arquivo ou FALSE em falha
Tipo de retorno:

string

$string = read_file('./path/to/file.php');

Se o seu servidor estiver executando open_basedir restringir isso função pode não funcionar se você está tentando acessar um arquivo acima do chamando script.

write_file($path, $data[, $mode = 'wb'])¶
Parâmetros:
  • $path (string) - Caminho de arquivo
  • $data (string) - Dados para gravar no arquivo
  • $mode (string) – fopen() mode
Retorna:

TRUE se a gravação foi bem sucedida, FALSE em caso de erro
Tipo de retorno:

bool

Grava dados no arquivo especificado no caminho. Se o arquivo não existir, o função irá criá-lo.

Exemplo:

$data = 'Some file data';

Nota

Esta função adquire um bloqueio exclusivo no arquivo durante a gravação.

delete_files($path[, $del_dir = FALSE[, $htdocs = FALSE]])¶
Parâmetros:
  • $path (string) - Caminho de diretório
  • $del_dir (bool) - Se também excluir diretórios
  • $htdocs (bool) - Ignorar a exclusão de arquivos .htaccess e de páginas de índice
Retorna:

VERDADEIRA sobre sucesso, FALSE em caso de erro
Tipo de retorno:

bool

Exclui todos os arquivos contidos no caminho fornecido.

Exemplo:

delete_files('./path/to/directory/');

Se o segundo parâmetro estiver configurado para TRUE, quaisquer diretórios contidos no O caminho da raiz também será excluído.

Exemplo:

delete_files('./path/to/directory/', TRUE);

Nota

The files must be writable or owned by the system in order to be deleted.

get_filenames($source_dir[, $include_path = FALSE])¶
Parâmetros:
  • $source_dir (string) - Caminho de diretório
  • $include_path (bool) - Incluir o caminho como parte dos nomes de arquivos
Retorna:

Uma matriz de nomes de arquivos
Tipo de retorno:

array

Utiliza um caminho do servidor como entrada e retorna uma matriz contendo os nomes de todos os arquivos contido dentro dele. O caminho do arquivo pode, opcionalmente, ser adicionado aos nomes dos arquivos, definindo o segundo parâmetro para TRUE.

Exemplo:

$controllers = get_filenames(APPPATH.'controllers/');
get_dir_file_info($source_dir, $top_level_only)¶
Parâmetros:
  • $source_dir (string) - Caminho de diretório
  • $top_level_only (bool) - Se procurar apenas no diretório especificado (excluindo subdiretórios)
Retorna:

Uma matriz contendo informações no conteúdo do diretório fornecido
Tipo de retorno:

array

Lê o diretório especificado e constrói uma matriz contendo os nomes dos arquivos, tamanho do arquivo, datas e permissões. As subpastas contidas no caminho especificado são lidas apenas se for forçado enviando o segundo parâmetro para FALSE, pois isso pode ser um operation.

Exemplo:

$models_info = get_dir_file_info(APPPATH.'models/');
get_file_info($file[, $returned_values = array('name', 'server_path', 'size', 'date')])¶

Arquivo Helper¶

The Arquivo Helper file contains functions that assist in working with files.

  • Carregando este Helper
  • Funções Disponíveis

Carregando este Helper¶

Este helper é carregado usando o seguinte código:

$this->load->helper('file');

Funções Disponíveis¶

As seguintes funções estão disponíveis:

read_file($file)¶
Parâmetros:
  • $file (string) - Caminho de arquivo
Retorna:

Conteúdo do arquivo ou FALSE em falha
Tipo de retorno:

string

Retorna os dados contidos no arquivo especificado no caminho.

Exemplo:

$string = read_file('./path/to/file.php');

O caminho pode ser um caminho relativo ou completo do servidor. Retorna FALSE (booleano) na falha.

Nota

O caminho é relativo ao seu arquivo index.php do site principal, NÃO ao seu Controller ou visualizar arquivos. O CodeIgniter usa um front controller para caminhos são sempre relativos ao índice do site principal.

Nota

Esta função é DEPRECADA. Use o nativo file_get_contents() instead.

Important

Se o seu servidor estiver executando open_basedir restringir isso função pode não funcionar se você está tentando acessar um arquivo acima do chamando script.

write_file($path, $data[, $mode = 'wb'])¶
Parâmetros:
  • $path (string) - Caminho de arquivo
  • $data (string) - Dados para gravar no arquivo
  • $mode (string) – fopen() mode
Retorna:

TRUE se a gravação foi bem sucedida, FALSE em caso de erro
Tipo de retorno:

bool

Grava dados no arquivo especificado no caminho. Se o arquivo não existir, o função irá criá-lo.

Exemplo:

$data = 'Some file data';
if ( ! write_file('./path/to/file.php', $data))
{
        echo 'Unable to write the file';
}
else
{
        echo 'File written!';
}

Você pode, opcionalmente, definir o modo de gravação através do terceiro parâmetro:

write_file('./path/to/file.php', $data, 'r+');

O modo padrão é 'wb'. por favor veja o Guia do usuário PHP para as opções de modo.

Nota

O caminho é relativo ao seu arquivo index.php do site principal, NÃO ao seu Controller ou visualizar arquivos. O CodeIgniter usa um front controller para caminhos são sempre relativos ao índice do site principal.

Nota

Esta função adquire um bloqueio exclusivo no arquivo durante a gravação.

delete_files($path[, $del_dir = FALSE[, $htdocs = FALSE]])¶
Parâmetros:
  • $path (string) - Caminho de diretório
  • $del_dir (bool) - Se também excluir diretórios
  • $htdocs (bool) - Ignorar a exclusão de arquivos .htaccess e de páginas de índice
Retorna:

VERDADEIRA sobre sucesso, FALSE em caso de erro
Tipo de retorno:

bool

Exclui todos os arquivos contidos no caminho fornecido.

Exemplo:

delete_files('./path/to/directory/');

Se o segundo parâmetro estiver configurado para TRUE, quaisquer diretórios contidos no O caminho da raiz também será excluído.

Exemplo:

delete_files('./path/to/directory/', TRUE);

Nota

Os arquivos devem ser graváveis ou de propriedade do sistema para serem excluídos.

get_filenames($source_dir[, $include_path = FALSE])¶
Parâmetros:
  • $source_dir (string) - Caminho de diretório
  • $include_path (bool) - Incluir o caminho como parte dos nomes de arquivos
Retorna:

Uma matriz de nomes de arquivos
Tipo de retorno:

array

Utiliza um caminho do servidor como entrada e retorna uma matriz contendo os nomes de todos os arquivos contido dentro dele. O caminho do arquivo pode, opcionalmente, ser adicionado aos nomes dos arquivos, definindo o segundo parâmetro para TRUE.

Exemplo:

$controllers = get_filenames(APPPATH.'controllers/');
get_dir_file_info($source_dir, $top_level_only)¶
Parâmetros:
  • $source_dir (string) - Caminho de diretório
  • $top_level_only (bool) - Se procurar apenas no diretório especificado (excluindo subdiretórios)
Retorna:

Uma matriz contendo informações no conteúdo do diretório fornecido
Tipo de retorno:

array

Lê o diretório especificado e constrói uma matriz contendo os nomes dos arquivos, tamanho do arquivo, datas e permissões. As subpastas contidas no caminho especificado são lidas apenas se for forçado enviando o segundo parâmetro para FALSE, pois isso pode ser um operation.

Exemplo:

$models_info = get_dir_file_info(APPPATH.'models/');
