Classe de teste unitário¶

O teste unitário é uma abordagem ao desenvolvimento de software em que os testes são written for each function in your application. If you are not familiar with the concept you might do a little googling on the subject.

CodeIgniter’s Unit Test class is quite simple, consisting of an função de avaliação e duas funções de resultado. Não se destina a ser um suite de testes completa, mas sim um mecanismo simples para avaliar o seu código para determinar se está produzindo o tipo de dados e resultado corretos.

Using the Unit Testing Library¶

Inicializando a classe¶

Like most other classes in CodeIgniter, the Unit Test class is initialized in your controller using the $this->load->library function:

$this->load->library('unit_test');

Uma vez carregado, o objeto Unit Test estará disponível usando $this->unit

Running Tests¶

A execução de um teste envolve o fornecimento de um teste e um resultado esperado no teste. following way:

$this->unit->run(‘test’, ‘expected result’, ‘test name’, ‘notes’);

Onde teste é o resultado do código que você deseja testar, resultado esperado é o tipo de dados que você espera, o nome do teste é um nome opcional que você pode dar your test, and notes are optional notes. Example:

$test = 1 + 1;

$expected_result = 2;

$test_name = 'Adds one plus one';

$this->unit->run($test, $expected_result, $test_name);

The expected result you supply can either be a literal match, or a data correspondência de tipo. Aqui está um exemplo de um literal:

$this->unit->run('Foo', 'Foo');

Here is an example of a data type match:

$this->unit->run('Foo', 'is_string');

Notice the use of “is_string” in the second parameter? This tells the função para avaliar se o seu teste está produzindo uma string como o result. Here is a list of allowed comparison types:

  • is_object
  • is_string
  • is_bool
  • is_true
  • is_false
  • is_int
  • is_numeric
  • is_float
  • is_double
  • is_array
  • is_null
  • is_resource

Generating Reports¶

Você pode exibir os resultados após cada teste ou pode executar vários tests and generate a report at the end. To show a report directly simply echo or return the run function:

echo $this->unit->run($test, $expected_result);

To run a full report of all tests, use this:

echo $this->unit->report();

O relatório será formatado em uma tabela HTML para visualização. Se você preferir the raw data you can retrieve an array using:

echo $this->unit->result();

modo estrito¶

Por padrão, a classe de teste unitário avalia as correspondências literais de forma imprecisa. Considere este exemplo:

$this->unit->run(1, TRUE);

The test is evaluating an integer, but the expected result is a boolean. PHP, no entanto, devido à sua digitação de dados solta irá avaliar o código acima as TRUE using a normal equality test:

if (1 == TRUE) echo 'This evaluates as true';

Se preferir, você pode colocar a classe de teste de unidade no modo estrito, que will compare the data type as well as the value:

if (1 === TRUE) echo 'This evaluates as FALSE';

To enable strict mode use this:

$this->unit->use_strict(TRUE);

Enabling/Disabling Unit Testing¶

If you would like to leave some testing in place in your scripts, but não executá-lo a menos que seja necessário, você pode desabilitar o teste de unidade usando:

$this->unit->active(FALSE);

Unit Test Display¶

When your unit test results display, the following items show by default:

  • Test Name (test_name)
  • Test Datatype (test_datatype)
  • Tipo de dados esperado (res_datatype)
  • Resultado (resultado)
  • Nome do arquivo (arquivo)
  • Número da linha (linha)
  • Any notes you entered for the test (notes)

Você pode personalizar quais desses itens são exibidos usando $this->unit->set_test_items(). For example, if you only wanted the test name e o resultado exibido:

Customizing displayed tests¶

$this->unit->set_test_items(array('test_name', 'result'));

Criando um modelo¶

Se você gostaria que os resultados do seu teste fossem formatados de forma diferente, então o default you can set your own template. Here is an example of a simple template. Nota the required pseudo-variables:

$str = '
<table border="0" cellpadding="4" cellspacing="1">
{rows}
        <tr>
                <td>{item}</td>
                <td>{result}</td>
        </tr>
{/rows}
</table>';

$this->unit->set_template($str);

Nota

Seu modelo deve ser declarado before executando a unidade processo de teste.

echo $this->unit->result();

Classe de teste unitário¶

O teste unitário é uma abordagem ao desenvolvimento de software em que os testes são escrito para cada função em sua aplicação. Se você não estiver familiarizado com o conceito você pode fazer um pouco de googling sobre o assunto.

A classe de Teste de Unidade do CodeIgniter é bastante simples, consistindo de uma função de avaliação e duas funções de resultado. Não se destina a ser um suite de testes completa, mas sim um mecanismo simples para avaliar o seu código para determinar se está produzindo o tipo de dados e resultado corretos.

Usando a Library de Testes Unitários¶

Inicializando a classe¶

Como a maioria das outras classes no CodeIgniter, a classe Unit Test é inicializado em seu Controller usando a função $this-> load->library function:

$this->load->library('unit_test');

Uma vez carregado, o objeto Unit Test estará disponível usando $this->unit

Testes em Execução¶

A execução de um teste envolve o fornecimento de um teste e um resultado esperado no teste. seguinte maneira:

$this->unit->run(‘test’, ‘expected result’, ‘test name’, ‘notes’);

Onde teste é o resultado do código que você deseja testar, resultado esperado é o tipo de dados que você espera, o nome do teste é um nome opcional que você pode dar seu teste e notas são notas opcionais. Exemplo:

$test = 1 + 1;

$expected_result = 2;

$test_name = 'Adds one plus one';

$this->unit->run($test, $expected_result, $test_name);

O resultado esperado fornecido pode ser uma correspondência literal ou um dado correspondência de tipo. Aqui está um exemplo de um literal:

$this->unit->run('Foo', 'Foo');

Aqui está um exemplo de uma correspondência de tipo de dados:

$this->unit->run('Foo', 'is_string');

Observe o uso de "is_string" no segundo parâmetro? Isso diz ao função para avaliar se o seu teste está produzindo uma string como o resultado. Aqui está uma lista de tipos de comparação permitidos:

  • is_object
  • is_string
  • is_bool
  • is_true
  • is_false
  • is_int
  • is_numeric
  • is_float
  • is_double
  • is_array
  • is_null
  • is_resource

Gerando Relatórios¶

Você pode exibir os resultados após cada teste ou pode executar vários testes e gerar um relatório no final. Para mostrar um relatório diretamente de forma simples eco ou retorne a função de execução:

echo $this->unit->run($test, $expected_result);

Para executar um relatório completo de todos os testes, use isto:

echo $this->unit->report();

O relatório será formatado em uma tabela HTML para visualização. Se você preferir os dados brutos você pode recuperar uma matriz usando:

echo $this->unit->result();

modo estrito¶

Por padrão, a classe de teste unitário avalia as correspondências literais de forma imprecisa. Considere este exemplo:

$this->unit->run(1, TRUE);

O teste está avaliando um inteiro, mas o resultado esperado é um booleano. PHP, no entanto, devido à sua digitação de dados solta irá avaliar o código acima como TRUE usando um teste de igualdade normal:

if (1 == TRUE) echo 'This evaluates as true';

Se preferir, você pode colocar a classe de teste de unidade no modo estrito, que irá comparar o tipo de dados, bem como o valor:

if (1 === TRUE) echo 'This evaluates as FALSE';

Para ativar o modo estrito, use isto:

$this->unit->use_strict(TRUE);

Ativando/Desativando o Teste Unitário¶

Se você gostaria de deixar alguns testes em seus scripts, mas não executá-lo a menos que seja necessário, você pode desabilitar o teste de unidade usando:

$this->unit->active(FALSE);

Display de Teste de Unidade¶

Quando os resultados do seu teste de unidade são exibidos, os seguintes itens mostram default:

  • Nome do Teste (test_name)
  • Tipo de dados de teste (test_datatype)
  • Tipo de dados esperado (res_datatype)
  • Resultado (resultado)
  • Nome do arquivo (arquivo)
  • Número da linha (linha)
  • Quaisquer notas que você inseriu para o teste (notas)

Você pode personalizar quais desses itens são exibidos usando $this->unit->set_test_items(). For example, if you only wanted the test name e o resultado exibido:
