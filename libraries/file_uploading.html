Classe de Upload de Arquivo¶

CodeIgniter’s Classe de Upload de Arquivo permits files to be uploaded. You can defina várias preferências, restringindo o tipo e tamanho dos arquivos.

The Process¶

Uploading a file involves the following general process:

  • An upload form is displayed, allowing a user to select a file and upload it.
  • When the form is submitted, the file is uploaded to the destination você especifica.
  • Ao longo do caminho, o arquivo é validado para garantir que seja permitido uploaded based on the preferences you set.
  • Once uploaded, the user will be shown a success message.

To demonstrate this process here is brief tutorial. Afterward you’ll encontrar informações de referência.

Creating the Upload Form¶

Usando um editor de texto, crie um formulário chamado upload_form.php. Nele, coloque este código e salve-o no seu application/views/ directory:

<html>
<head>
<title>Upload Form</title>
</head>
<body>

<?php echo $error;?>

<?php echo form_open_multipart('upload/do_upload');?>

<input type="file" name="userfile" size="20" />

<br /><br />

<input type="submit" value="upload" />

</form>

</body>
</html>

Você perceberá que estamos usando um auxiliar de formulário para criar a tag de formulário de abertura. Uploads de arquivos exigem um formulário de várias partes, para que o helper crie o syntax for you. You’ll also notice we have an $error variable. This is para que possamos mostrar mensagens de erro caso o usuário faça algo wrong.

The Success Page¶

Usando um editor de texto, crie um formulário chamado upload_success.php. Nisso, coloque este código e salve-o no seu application/views/ directory:

<html>
<head>
<title>Upload Form</title>
</head>
<body>

<h3>Your file was successfully uploaded!</h3>

<ul>
<?php foreach ($upload_data as $item => $value):?>
<li><?php echo $item;?>: <?php echo $value;?></li>
<?php endforeach; ?>
</ul>

<p><?php echo anchor('upload', 'Upload Another File!'); ?></p>

</body>
</html>

The Controller¶

Using a text editor, create a controller called Upload.php. In it, place este código e salve-o no seu application/controllers/ directory:

<?php

class Upload extends CI_Controller {

        public function __construct()
        {
                parent::__construct();
                $this->load->helper(array('form', 'url'));
        }

        public function index()
        {
                $this->load->view('upload_form', array('error' => ' ' ));
        }

        public function do_upload()
        {
                $config['upload_path']          = './uploads/';
                $config['allowed_types']        = 'gif|jpg|png';
                $config['max_size']             = 100;
                $config['max_width']            = 1024;
                $config['max_height']           = 768;

                $this->load->library('upload', $config);

                if ( ! $this->upload->do_upload('userfile'))
                {
                        $error = array('error' => $this->upload->display_errors());

                        $this->load->view('upload_form', $error);
                }
                else
                {
                        $data = array('upload_data' => $this->upload->data());

                        $this->load->view('upload_success', $data);
                }
        }
}
?>

O diretório de upload¶

Você precisará de um diretório de destino para suas imagens enviadas. Criar uma directory at the root of your CodeIgniter installation called uploads e defina suas permissões de arquivo para 777.

Try it!¶

To try your form, visit your site using a URL similar to this one:

example.com/index.php/upload/

Você deve ver um formulário de upload. Tente fazer o upload de um arquivo de imagem jpg, gif, or png). If the path in your controller is correct it should work.

Guia de referência¶

Inicializando a classe de upload¶

Like most other classes in CodeIgniter, the Upload class is initialized in your controller using the $this->load->library() method:

$this->load->library('upload');

Once the Upload class is loaded, the object will be available using: $this->upload

Definindo Preferências¶

Similar to other libraries, you’ll control what is allowed to be upload based on your preferences. In the controller you built above you set the following preferences:

$config['upload_path'] = './uploads/';
$config['allowed_types'] = 'gif|jpg|png';
$config['max_size']     = '100';
$config['max_width'] = '1024';
$config['max_height'] = '768';

$this->load->library('upload', $config);

// Alternately you can set preferences by calling the ``initialize()`` method. Useful if you auto-load the class:
$this->upload->initialize($config);

The above preferences should be fairly self-explanatory. Below is a table describing all available preferences.

Preferências¶

The following preferences are available. The default value indicates what will be used if you do not specify that preference.

Preferência Default Value Opções Descrição
upload_path None None The path to the directory where the upload should be placed. The directory must be writable and the path can be absolute or relative.
allowed_types None None Os tipos mime correspondentes aos tipos de arquivos que você permite uploaded. Usually the file extension can be used as the mime type. Can be either an array or a pipe-separated string.
file_name None Desired file name If set CodeIgniter will rename the uploaded file to this name. The extension provided in the file name must also be an allowed file type. If no extension is provided in the original file_name will be used.
file_ext_tolower FALSE TRUE/FALSE (boolean) Se definido como TRUE, a extensão do arquivo será forçada a minúscula
overwrite FALSE TRUE/FALSE (boolean) Se definido como true, se um arquivo com o mesmo nome daquele que você é uploading exists, it will be overwritten. If set to false, a number will be appended to the filename if another with the same name exists.
max_size 0 None O tamanho máximo (em kilobytes) que o arquivo pode ser. Definido como zero para não limit. Nota: Most PHP installations have their own limit, as specified no arquivo php.ini. Geralmente 2 MB (ou 2048 KB) por padrão.
max_width 0 None A largura máxima (em pixels) que a imagem pode ser. Definido como zero para não limit.
max_height 0 None The maximum height (in pixels) that the image can be. Set to zero for no limit.
min_width 0 None The minimum width (in pixels) that the image can be. Set to zero for no limit.
min_height 0 None The minimum height (in pixels) that the image can be. Set to zero for no limit.
max_filename 0 None The maximum length that a file name can be. Set to zero for no limit.
max_filename_increment 100 None Quando overwrite for definido como FALSE, use-o para definir o nome de arquivo máximo increment for CodeIgniter to append to the filename.
encrypt_name FALSE TRUE/FALSE (boolean) If set to TRUE the file name will be converted to a random encrypted string. This can be useful if you would like the file saved with a name que não pode ser discernido pela pessoa que faz o upload.
remove_spaces TRUE TRUE/FALSE (boolean) If set to TRUE, any spaces in the file name will be converted to underscores. This is recommended.
detect_mime TRUE TRUE/FALSE (boolean) If set to TRUE, a server side detection of the file type will be realizada para evitar ataques de injeção de código. NÃO desative esta opção a menos que você não tenha outra opção, isso causaria um risco de segurança.
mod_mime_fix TRUE TRUE/FALSE (boolean) Se definido como TRUE, várias extensões de nome de arquivo serão sufixadas com um underscore in order to avoid triggering Apache mod_mime. DO NOT turn off this option if your upload directory is public, as this é um risco de segurança.

Configurando Preferências em um Arquivo de Configuração¶

If you prefer not to set preferences using the above method, you can em vez disso, coloque-os em um arquivo de configuração. Basta criar um novo arquivo chamado upload.php, add the $config array in that file. Then save the file in: config/upload.php e será usado automaticamente. Você não vai need to use the $this->upload->initialize() method if you save your preferências em um arquivo de configuração.

Class Reference¶

class CI_Upload¶
initialize([array $config = array()[, $reset = TRUE]])¶
Parâmetros:
  • $config (array) – Preferências
  • $reset (bool) - Se deseja redefinir as preferências (que não são fornecidas no $config) para seus padrões
Returns:

CI_Upload instance (encadeamento de métodos)
Return type:

CI_Upload
do_upload([$field = 'userfile'])¶
Parâmetros:
  • $field (string) – Name of the form field
Returns:

Verdadeiro sucesso, FALSE em falha
Return type:

bool

Performs the upload based on the preferences you’ve set.

Nota

By default the upload routine expects the file to come from um campo de formulário chamado userfile e o formulário deve ser do tipo “multipart”.

<form method="post" action="some_action" enctype="multipart/form-data" />

If you would like to set your own field name simply pass its value to the do_upload() method:

$field_name = "some_field_name";
$this->upload->do_upload($field_name);
display_errors([$open = '<p>'[, $close = '</p>']])¶
Parâmetros:
  • $open (string) – Opening markup
  • $close (string) – Closing markup
Returns:

Formatted error message(s)
Return type:

string

Retrieves any error messages if the do_upload() method returned false. The method does not echo automatically, it returns the data so you can assign it however you need.

Formatting Errors

By default the above method wraps any errors within <p> tags. You can set your own delimiters like this:

$this->upload->display_errors('<p>', '</p>');
data([$index = NULL])¶
Parâmetros:
  • $data (string) – Element to return instead of the full array
Returns:

Information about the uploaded file
Return type:

mixed

This is a helper method that returns an array containing all of the data related to the file you uploaded. Here is the array prototype:

Array
(
        [file_name]     => mypic.jpg
        [file_type]     => image/jpeg
        [file_path]     => /path/to/your/upload/
        [full_path]     => /path/to/your/upload/jpg.jpg
        [raw_name]      => mypic
        [orig_name]     => mypic.jpg
        [client_name]   => mypic.jpg
        [file_ext]      => .jpg
        [file_size]     => 22.2
        [is_image]      => 1
        [image_width]   => 800
        [image_height]  => 600
        [image_type]    => jpeg
        [image_size_str] => width="800" height="200"
)

To return one element from the array:

$this->upload->data('file_name');       // Returns: mypic.jpg

Aqui está uma tabela explicando os itens da matriz acima exibidos:

Item Descrição
file_name Name of the file that was uploaded, including the filename extension
file_type Identificador de tipo de arquivo MIME
file_path Absolute server path to the file
full_path Absolute server path, including the file name
raw_name Nome do arquivo, sem a extensão
orig_name Nome do arquivo original. Isso é útil apenas se você usar a opção de nome criptografado.
client_name File name supplied by the client user agent, but possibly sanitized
file_ext Extensão de nome de arquivo, período incluído
file_size Tamanho do arquivo em kilobytes
is_image Whether the file is an image or not. 1 = image. 0 = not.
image_width Image width
image_height Altura da imagem
image_type Image type (usually the file name extension without the period)
image_size_str Uma string contendo a largura e a altura (útil para colocar em uma tag de imagem)

Classe de Upload de Arquivo¶

CodeIgniter’s Classe de Upload de Arquivo permits files to be uploaded. You can defina várias preferências, restringindo o tipo e tamanho dos arquivos.

O processo¶

O upload de um arquivo envolve o seguinte processo geral:

  • Um formulário de upload é exibido, permitindo que um usuário selecione um arquivo e faça o upload.
  • Quando o formulário é enviado, o arquivo é enviado para o destino você especifica.
  • Ao longo do caminho, o arquivo é validado para garantir que seja permitido carregado com base nas preferências definidas por você.
  • Uma vez carregado, o usuário será mostrado uma mensagem de sucesso.

Para demonstrar este processo aqui é um breve tutorial. Depois você vai encontrar informações de referência.

Criando o formulário de upload¶

Usando um editor de texto, crie um formulário chamado upload_form.php. Nele, coloque este código e salve-o no seu application/views/ directory:

<html>
<head>
<title>Upload Form</title>
</head>
<body>

<?php echo $error;?>

<?php echo form_open_multipart('upload/do_upload');?>

<input type="file" name="userfile" size="20" />

<br /><br />

<input type="submit" value="upload" />

</form>

</body>
</html>

Você perceberá que estamos usando um Helper de formulário para criar a tag de formulário de abertura. Uploads de arquivos exigem um formulário de várias partes, para que o helper crie o sintaxe para você. Você também notará que temos uma variável de $error. Isto é para que possamos mostrar mensagens de erro caso o usuário faça algo wrong.

A página de sucesso¶

Usando um editor de texto, crie um formulário chamado upload_success.php. Nisso, coloque este código e salve-o no seu application/views/ directory:

<html>
<head>
<title>Upload Form</title>
</head>
<body>

<h3>Your file was successfully uploaded!</h3>

<ul>
<?php foreach ($upload_data as $item => $value):?>
<li><?php echo $item;?>: <?php echo $value;?></li>
<?php endforeach; ?>
</ul>

<p><?php echo anchor('upload', 'Upload Another File!'); ?></p>

</body>
</html>

O Controller¶

Usando um editor de texto, crie um Controller chamado Upload.php. Nele, coloque este código e salve-o no seu application/controllers/ directory:

<?php

class Upload extends CI_Controller {

        public function __construct()
        {
                parent::__construct();
                $this->load->helper(array('form', 'url'));
        }

        public function index()
        {
                $this->load->view('upload_form', array('error' => ' ' ));
        }

        public function do_upload()
        {
                $config['upload_path']          = './uploads/';
                $config['allowed_types']        = 'gif|jpg|png';
                $config['max_size']             = 100;
                $config['max_width']            = 1024;
                $config['max_height']           = 768;

                $this->load->library('upload', $config);

                if ( ! $this->upload->do_upload('userfile'))
                {
                        $error = array('error' => $this->upload->display_errors());

                        $this->load->view('upload_form', $error);
                }
                else
                {
                        $data = array('upload_data' => $this->upload->data());

                        $this->load->view('upload_success', $data);
                }
        }
}
?>

O diretório de upload¶

Você precisará de um diretório de destino para suas imagens enviadas. Criar uma diretório na raiz da sua instalação do CodeIgniter chamada uploads e defina suas permissões de arquivo para 777.

Tente!¶

Para testar seu formulário, visite seu site usando um URL semelhante a este:

example.com/index.php/upload/

Você deve ver um formulário de upload. Tente fazer o upload de um arquivo de imagem jpg, gif ou png). Se o caminho no seu Controller estiver correto, ele deve work.

Guia de referência¶

Inicializando a classe de upload¶

Como a maioria das outras classes no Code Igniter, a classe Upload é inicializada in your controller using the $this->load->library() method:

$this->load->library('upload');

Depois que a classe Upload for carregada, o objeto estará disponível usando: $this->upload

Definindo Preferências¶

Semelhante a outras Librarys, você controla o que é permitido fazer upload com base nas suas preferências. No Controller que você criou acima, você define o seguintes preferências:

$config['upload_path'] = './uploads/';
$config['allowed_types'] = 'gif|jpg|png';
$config['max_size']     = '100';
$config['max_width'] = '1024';
$config['max_height'] = '768';

$this->load->library('upload', $config);

// Alternately you can set preferences by calling the ``initialize()`` method. Useful if you auto-load the class:
$this->upload->initialize($config);

As preferências acima devem ser bastante auto-explicativas. Abaixo está um tabela descrevendo todas as preferências disponíveis.

Preferências¶

As seguintes preferências estão disponíveis. O valor padrão indica o que será usado se você não especificar essa preferência.

Preferência Valor padrão Opções Descrição
upload_path None None O caminho para o diretório em que o upload deve ser colocado. o O diretório deve ser gravável e o caminho pode ser absoluto ou relativo.
allowed_types None None Os tipos mime correspondentes aos tipos de arquivos que você permite carregado. Normalmente, a extensão do arquivo pode ser usada como o tipo mime. Pode ser uma matriz ou uma string separada por pipe.
file_name None Nome do arquivo desejado Se configurado, o CodeIgniter irá renomear o arquivo carregado para este nome. o A extensão fornecida no nome do arquivo também deve ser um tipo de arquivo permitido. Se nenhuma extensão for fornecida no original, o nome do arquivo será usado.
file_ext_tolower FALSE TRUE/FALSE (booleano) Se definido como TRUE, a extensão do arquivo será forçada a minúscula
overwrite FALSE TRUE/FALSE (booleano) Se definido como true, se um arquivo com o mesmo nome daquele que você é o upload existe, ele será sobrescrito. Se definido como falso, um número será ser acrescentado ao nome do arquivo se existir outro com o mesmo nome.
max_size 0 None O tamanho máximo (em kilobytes) que o arquivo pode ser. Definido como zero para não limite. Nota: A maioria das instalações do PHP tem seu próprio limite, conforme especificado no arquivo php.ini. Geralmente 2 MB (ou 2048 KB) por padrão.
max_width 0 None A largura máxima (em pixels) que a imagem pode ser. Definido como zero para não limit.
max_height 0 None A altura máxima (em pixels) que a imagem pode ser. Definido como zero para não limit.
min_width 0 None A largura mínima (em pixels) que a imagem pode ser. Definido como zero para não limit.
min_height 0 None A altura mínima (em pixels) que a imagem pode ser. Definido como zero para não limit.
max_filename 0 None O comprimento máximo que um nome de arquivo pode ser. Definido como zero para nenhum limite.
max_filename_increment 100 None Quando overwrite for definido como FALSE, use-o para definir o nome de arquivo máximo incremento para CodeIgniter para acrescentar ao nome do arquivo.
encrypt_name FALSE TRUE/FALSE (booleano) Se definido como TRUE, o nome do arquivo será convertido em um arquivo criptografado aleatório. corda. Isso pode ser útil se você quiser que o arquivo seja salvo com um nome que não pode ser discernido pela pessoa que faz o upload.
remove_spaces TRUE TRUE/FALSE (booleano) Se definido como TRUE, quaisquer espaços no nome do arquivo serão convertidos para sublinhados. Isso é recomendado.
detect_mime TRUE TRUE/FALSE (booleano) Se definido como TRUE, uma detecção do lado do servidor do tipo de arquivo será realizada para evitar ataques de injeção de código. NÃO desative esta opção a menos que você não tenha outra opção, isso causaria um risco de segurança.
mod_mime_fix TRUE TRUE/FALSE (booleano) Se definido como TRUE, várias extensões de nome de arquivo serão sufixadas com um sublinhado, a fim de evitar o desencadeamento Apache mod_mime. NÃO desligue esta opção se o seu diretório de upload for público, já que é um risco de segurança.

Configurando Preferências em um Arquivo de Configuração¶

Se preferir não definir preferências usando o método acima, você pode em vez disso, coloque-os em um arquivo de configuração. Basta criar um novo arquivo chamado upload.php, adicione o array $config nesse arquivo. Em seguida, salve o arquivo em: config/upload.php e será usado automaticamente. Você não vai need to use the $this->upload->initialize() method if you save your preferências em um arquivo de configuração.

$field_name = "some_field_name";

Array

Guia de referência¶

Inicializando a classe de upload¶

Como a maioria das outras classes no Code Igniter, a classe Upload é inicializada in your controller using the $this->load->library() method:

$this->load->library('upload');

Depois que a classe Upload for carregada, o objeto estará disponível usando: $this->upload

Definindo Preferências¶

Semelhante a outras Librarys, você controla o que é permitido fazer upload com base nas suas preferências. No Controller que você criou acima, você define o seguintes preferências:

$config['upload_path'] = './uploads/';
$config['allowed_types'] = 'gif|jpg|png';
$config['max_size']     = '100';
$config['max_width'] = '1024';
$config['max_height'] = '768';

$this->load->library('upload', $config);

// Alternately you can set preferences by calling the ``initialize()`` method. Useful if you auto-load the class:
$this->upload->initialize($config);

As preferências acima devem ser bastante auto-explicativas. Abaixo está um tabela descrevendo todas as preferências disponíveis.

Preferências¶

As seguintes preferências estão disponíveis. O valor padrão indica o que será usado se você não especificar essa preferência.

Preferência Valor padrão Opções Descrição
upload_path None None O caminho para o diretório em que o upload deve ser colocado. o O diretório deve ser gravável e o caminho pode ser absoluto ou relativo.
allowed_types None None Os tipos mime correspondentes aos tipos de arquivos que você permite carregado. Normalmente, a extensão do arquivo pode ser usada como o tipo mime. Pode ser uma matriz ou uma string separada por pipe.
file_name None Nome do arquivo desejado Se configurado, o CodeIgniter irá renomear o arquivo carregado para este nome. o A extensão fornecida no nome do arquivo também deve ser um tipo de arquivo permitido. Se nenhuma extensão for fornecida no original, o nome do arquivo será usado.
file_ext_tolower FALSE TRUE/FALSE (booleano) Se definido como TRUE, a extensão do arquivo será forçada a minúscula
overwrite FALSE TRUE/FALSE (booleano) Se definido como true, se um arquivo com o mesmo nome daquele que você é o upload existe, ele será sobrescrito. Se definido como falso, um número será ser acrescentado ao nome do arquivo se existir outro com o mesmo nome.
max_size 0 None O tamanho máximo (em kilobytes) que o arquivo pode ser. Definido como zero para não limite. Nota: A maioria das instalações do PHP tem seu próprio limite, conforme especificado no arquivo php.ini. Geralmente 2 MB (ou 2048 KB) por padrão.
max_width 0 None A largura máxima (em pixels) que a imagem pode ser. Definido como zero para não limit.
max_height 0 None A altura máxima (em pixels) que a imagem pode ser. Definido como zero para não limit.
min_width 0 None A largura mínima (em pixels) que a imagem pode ser. Definido como zero para não limit.
min_height 0 None A altura mínima (em pixels) que a imagem pode ser. Definido como zero para não limit.
max_filename 0 None O comprimento máximo que um nome de arquivo pode ser. Definido como zero para nenhum limite.
max_filename_increment 100 None Quando overwrite for definido como FALSE, use-o para definir o nome de arquivo máximo incremento para CodeIgniter para acrescentar ao nome do arquivo.
encrypt_name FALSE TRUE/FALSE (booleano) Se definido como TRUE, o nome do arquivo será convertido em um arquivo criptografado aleatório. corda. Isso pode ser útil se você quiser que o arquivo seja salvo com um nome que não pode ser discernido pela pessoa que faz o upload.
remove_spaces TRUE TRUE/FALSE (booleano) Se definido como TRUE, quaisquer espaços no nome do arquivo serão convertidos para sublinhados. Isso é recomendado.
detect_mime TRUE TRUE/FALSE (booleano) Se definido como TRUE, uma detecção do lado do servidor do tipo de arquivo será realizada para evitar ataques de injeção de código. NÃO desative esta opção a menos que você não tenha outra opção, isso causaria um risco de segurança.
mod_mime_fix TRUE TRUE/FALSE (booleano) Se definido como TRUE, várias extensões de nome de arquivo serão sufixadas com um sublinhado, a fim de evitar o desencadeamento Apache mod_mime. NÃO desligue esta opção se o seu diretório de upload for público, já que é um risco de segurança.

Configurando Preferências em um Arquivo de Configuração¶

Se preferir não definir preferências usando o método acima, você pode em vez disso, coloque-os em um arquivo de configuração. Basta criar um novo arquivo chamado upload.php, adicione o array $config nesse arquivo. Em seguida, salve o arquivo em: config/upload.php e será usado automaticamente. Você não vai need to use the $this->upload->initialize() method if you save your preferências em um arquivo de configuração.

do_upload([$field = 'userfile'])¶
Parâmetros:
  • $field (string) - Nome do campo de formulário
Retorna:

Verdadeiro sucesso, FALSE em falha
Tipo de retorno:

bool

Executa o upload com base nas preferências que você definiu.

Nota

Por padrão, a rotina de upload espera que o arquivo venha um campo de formulário chamado userfile e o formulário deve ser do tipo “multipart”.

<form method="post" action="some_action" enctype="multipart/form-data" />

Se você quiser definir seu próprio nome de campo, simplesmente passe seu valor para the do_upload() method:

$field_name = "some_field_name";
$this->upload->do_upload($field_name);

Retrieves any error messages if the do_upload() method returned falso. O método não ecoa automaticamente, retorna os dados assim você pode atribuí-lo como quiser.

Erros de formatação

Por padrão, o método acima envolve qualquer erro dentro das tags <p>. Você pode defina seus próprios delimitadores assim:

$this->upload->display_errors('<p>', '</p>');
data([$index = NULL])¶
Parâmetros:
  • $data (string) - Elemento para retornar em vez do array completo
Retorna:

Informações sobre o arquivo enviado
Tipo de retorno:

mixed

Este é um método Helper que retorna uma matriz contendo todos os dados relacionados ao arquivo que você enviou. Aqui está o protótipo da matriz:

Array
(
        [file_name]     => mypic.jpg
        [file_type]     => image/jpeg
        [file_path]     => /path/to/your/upload/
        [full_path]     => /path/to/your/upload/jpg.jpg
        [raw_name]      => mypic
        [orig_name]     => mypic.jpg
        [client_name]   => mypic.jpg
        [file_ext]      => .jpg
        [file_size]     => 22.2
        [is_image]      => 1
        [image_width]   => 800
        [image_height]  => 600
        [image_type]    => jpeg
        [image_size_str] => width="800" height="200"
)

Para retornar um elemento da matriz:

$this->upload->data('file_name');       // Retorna: mypic.jpg

Aqui está uma tabela explicando os itens da matriz acima exibidos:

Item Descrição
file_name Nome do arquivo que foi enviado, incluindo a extensão do nome do arquivo
file_type Identificador de tipo de arquivo MIME
file_path Caminho absoluto do servidor para o arquivo
full_path Caminho absoluto do servidor, incluindo o nome do arquivo
raw_name Nome do arquivo, sem a extensão
orig_name Nome do arquivo original. Isso é útil apenas se você usar a opção de nome criptografado.
client_name Nome do arquivo fornecido pelo agente do usuário do cliente, mas possivelmente sanitizado
file_ext Extensão de nome de arquivo, período incluído
file_size Tamanho do arquivo em kilobytes
is_image Se o arquivo é uma imagem ou não. 1 = imagem. 0 = não.
image_width Largura da imagem
image_height Altura da imagem
image_type Tipo de imagem (geralmente a extensão do nome do arquivo sem o período)
image_size_str Uma string contendo a largura e a altura (útil para colocar em uma tag de imagem)
