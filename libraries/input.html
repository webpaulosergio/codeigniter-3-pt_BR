Classe de Entrada¶

The Classe de Entrada serves two purposes:

  1. Ele pré-processa dados de entrada globais para segurança.
  2. It provides some helper methods for fetching input data and pre-processing it.

Nota

Esta classe é inicializada automaticamente pelo sistema para que is no need to do it manually.

  • Input Filtering
    • Segurança Filtering
    • XSS Filtering
  • Acessando dados de formulário
    • Usando dados POST, GET, COOKIE ou SERVER
    • Usando o fluxo de entrada php: //

Input Filtering¶

Segurança Filtering¶

O método de filtragem de segurança é chamado automaticamente quando um novo controller is invoked. It does the following:

  • If $config['allow_get_array'] is FALSE (default is TRUE), destroys o array GET global.
  • Destroys all global variables in the event register_globals is turned on.
  • Filtra as chaves do array GET/POST/COOKIE, permitindo apenas o número alfa-numérico (and a few other) characters.
  • Provides XSS (Cross-site Scripting Hacks) filtering. This can be enabled globally, or upon request.
  • Standardizes newline characters to PHP_EOL (\n in UNIX-based OSes, \r\n under Windows). This is configurable.

XSS Filtering¶

A classe Input tem a capacidade de filtrar a entrada automaticamente para evitar ataques de script entre sites. Se você quiser que o filtro seja executado automatically every time it encounters POST or COOKIE data you can enable it by opening your application/config/config.php file and setting this:

$config['global_xss_filtering'] = TRUE;

Please refer to the Segurança class documentation for informações sobre como usar a Filtragem XSS em seu aplicativo.

Importante

The ‘global_xss_filtering’ setting is DEPRECATED and kept solely for backwards-compatibility purposes. XSS escaping should be performed on output, not input!

Acessando dados de formulário¶

Usando dados POST, GET, COOKIE ou SERVER¶

CodeIgniter comes with helper methods that let you fetch POST, GET, COOKIE or SERVER items. The main advantage of using the provided methods rather than fetching an item directly ($_POST['something']) é que os métodos verificarão se o item está definido e retornará NULL se não. Isso permite usar convenientemente os dados sem having to test whether an item exists first. In other words, normally you might do something like this:

$something = isset($_POST['something']) ? $_POST['something'] : NULL;

Com os métodos embutidos do CodeIgniter, você pode simplesmente fazer isso:

$something = $this->input->post('something');

The main methods are:

  • $this->input->post()
  • $this->input->get()
  • $this->input->cookie()
  • $this->input->server()

Usando o fluxo de entrada php: //¶

Se você quiser utilizar o PUT, DELETE, PATCH ou outro pedido exótico methods, they can only be accessed via a special input stream, that can only be read once. This isn’t as easy as just reading from e.g. the $_POST array, because it will always exist and you can try and access multiple variables without caring that you might only have Um tiro em todos os dados do POST.

CodeIgniter will take care of that for you, and you can read the data from the php://input stream at any time, just by using the $raw_input_stream property:

$this->input->raw_input_stream;

Additionally if the input stream is form-encoded like $_POST you can access its values by calling the input_stream() method:

$this->input->input_stream('key');

Similar to other methods such as get() and post(), if the dados solicitados não são encontrados, ele retornará NULL e você também pode decide whether to run the data through xss_clean() by passing a boolean value as the second parameter:

$this->input->input_stream('key', TRUE); // XSS Clean
$this->input->input_stream('key', FALSE); // No XSS filter

Nota

You can utilize method() in order to know if you’re reading PUT, DELETE or PATCH data.

Class Reference¶

class CI_Input¶
$raw_input_stream

Read only property that will return php://input data as is.

The property can be read multiple times.

post([$index = NULL[, $xss_clean = NULL]])¶
Parâmetros:
  • $index (mixed) – POST parameter name
  • $xss_clean (bool) – Whether to apply XSS filtering
Returns:

$_POST if no parameters supplied, otherwise the POST value if found or NULL if not
Return type:

mixed

O primeiro parâmetro conterá o nome do item POST que você está procurando por:

$this->input->post('some_data');

O método retorna NULL se o item que você está tentando recuperar does not exist.

The second optional parameter lets you run the data through the XSS filter. It’s enabled by setting the second parameter to boolean TRUE or by setting your $config['global_xss_filtering'] to TRUE.

$this->input->post('some_data', TRUE);

To return an array of all POST items call without any parameters.

To return all POST items and pass them through the XSS filter set the first parameter NULL while setting the second parameter to boolean TRUE.

$this->input->post(NULL, TRUE); // returns all POST items with XSS filter
$this->input->post(NULL, FALSE); // returns all POST items without XSS filter

To return an array of multiple POST parameters, pass all the required keys como uma matriz.

$this->input->post(array('field1', 'field2'));

Mesma regra aplicada aqui, para recuperar os parâmetros com a filtragem XSS habilitada, defina second parameter to boolean TRUE.

$this->input->post(array('field1', 'field2'), TRUE);
get([$index = NULL[, $xss_clean = NULL]])¶
Parâmetros:
  • $index (mixed) – GET parameter name
  • $xss_clean (bool) – Whether to apply XSS filtering
Returns:

$_GET se nenhum parâmetro for fornecido, caso contrário, o valor GET se encontrado ou NULL se não for
Return type:

mixed

This method is identical to post(), only it fetches GET data.

$this->input->get('some_data', TRUE);

To return an array of all GET items call without any parameters.

To return all GET items and pass them through the XSS filter set the first parameter NULL while setting the second parameter to boolean TRUE.

$this->input->get(NULL, TRUE); // returns all GET items with XSS filter
$this->input->get(NULL, FALSE); // returns all GET items without XSS filtering

Para retornar uma matriz de múltiplos parâmetros GET, passe todas as chaves necessárias como uma matriz.

$this->input->get(array('field1', 'field2'));

Mesma regra aplicada aqui, para recuperar os parâmetros com a filtragem XSS habilitada, defina second parameter to boolean TRUE.

$this->input->get(array('field1', 'field2'), TRUE);
post_get($index[, $xss_clean = NULL])¶
Parâmetros:
  • $index (string) - nome do parâmetro POST/GET
  • $xss_clean (bool) – Whether to apply XSS filtering
Returns:

POST/GET value if found, NULL if not
Return type:

mixed

This method works pretty much the same way as post() and get(), only combined. It will search through both POST and GET streams for data, procurando no POST primeiro e, em seguida, no GET:

$this->input->post_get('some_data', TRUE);
get_post($index[, $xss_clean = NULL])¶
Parâmetros:
  • $index (string) – GET/POST parameter name
  • $xss_clean (bool) – Whether to apply XSS filtering
Returns:

GET/POST value if found, NULL if not
Return type:

mixed

This method works the same way as post_get() only it looks for GET dados primeiro.

$this->input->get_post(‘some_data’, TRUE);

Nota

This method used to act EXACTLY like post_get(), but it’s behavior has changed in CodeIgniter 3.0.

cookie([$index = NULL[, $xss_clean = NULL]])¶
Parâmetros:
  • $index (mixed) - nome do cookie
  • $xss_clean (bool) – Whether to apply XSS filtering
Returns:

$_COOKIE if no parameters supplied, otherwise the COOKIE value if found or NULL if not
Return type:

mixed

This method is identical to post() and get(), only it fetches cookie data:

$this->input->cookie('some_cookie');
$this->input->cookie('some_cookie', TRUE); // with XSS filter

To return an array of multiple cookie values, pass all the required keys como uma matriz.

$this->input->cookie(array('some_cookie', 'some_cookie2'));

Nota

Unlike the Cookie Helper function get_cookie(), this method does NOT prepend your configured $config['cookie_prefix'] value.

server($index[, $xss_clean = NULL])¶
Parâmetros:
  • $index (mixed) – Value name
  • $xss_clean (bool) – Whether to apply XSS filtering
Returns:

$_SERVER item value if found, NULL if not
Return type:

mixed

This method is identical to the post(), get() and cookie() methods, only it fetches server data ($_SERVER):

$this->input->server('some_data');

To return an array of multiple $_SERVER values, pass all the required keys como uma matriz.

$this->input->server(array('SERVER_PROTOCOL', 'REQUEST_URI'));
input_stream([$index = NULL[, $xss_clean = NULL]])¶
Parâmetros:
  • $index (mixed) – Key name
  • $xss_clean (bool) – Whether to apply XSS filtering
Returns:

Matriz de fluxo de entrada se nenhum parâmetro for fornecido, caso contrário, o valor especificado, se encontrado, ou NULL, se não for
Return type:

mixed

This method is identical to get(), post() and cookie(), only it fetches the php://input stream data.

set_cookie($name = ''[, $value = ''[, $expire = ''[, $domain = ''[, $path = '/'[, $prefix = ''[, $secure = NULL[, $httponly = NULL]]]]]]])¶
Parâmetros:
  • $name (mixed) – Cookie name or an array of parameters
  • $value (string) – Cookie value
  • $expire (int) - Tempo de expiração do cookie em segundos
  • $domain (string) – Cookie domain
  • $path (string) – Cookie path
  • $prefix (string) - prefixo do nome do cookie
  • $secure (bool) - Se apenas transferir o cookie por meio de HTTPS
  • $httponly (bool) – Whether to only make the cookie accessible for HTTP requests (no JavaScript)
Return type:

void

Define um cookie contendo os valores que você especifica. Existem duas maneiras de passar informações para esse método para que um cookie possa ser definido: Array Method, and Discrete Parâmetros:

Método de Matriz

Using this method, an associative array is passed to the first parameter:

$cookie = array(
        'name'   => 'The Cookie Name',
        'value'  => 'The Value',
        'expire' => '86500',
        'domain' => '.some-domain.com',
        'path'   => '/',
        'prefix' => 'myprefix_',
        'secure' => TRUE
);

$this->input->set_cookie($cookie);

Notas

Apenas o nome e o valor são obrigatórios. Para excluir um cookie, defina-o com o expiração em branco.

The expiration is set in seconds, que será adicionado ao atual Tempo. Não inclua o tempo, mas apenas o número de segundos from now that you wish the cookie to be valid. If the expiration is definido como zero, o cookie durará apenas enquanto o navegador estiver aberto.

For site-wide cookies regardless of how your site is requested, add your URL to the domain starting with a period, like this: .your-domain.com

The path is usually not needed since the method sets a root path.

The prefix is only needed if you need to avoid name collisions with other identically named cookies for your server.

The httponly and secure flags, when omitted, will default to your $config['cookie_httponly'] and $config['cookie_secure'] settings.

Discrete Parameters

If you prefer, you can set the cookie by passing data using individual parameters:

$this->input->set_cookie($name, $value, $expire, $domain, $path, $prefix, $secure);
ip_address()¶
Returns:Visitor’s IP address or ‘0.0.0.0’ if not valid
Return type:string

Returns the IP address for the current user. If the IP address is not valid, the method will return ‘0.0.0.0’:

echo $this->input->ip_address();

Importante

This method takes into account the $config['proxy_ips'] setting and will return the reported HTTP_X_FORWARDED_FOR, HTTP_CLIENT_IP, HTTP_X_CLIENT_IP ou HTTP_X_CLUSTER_CLIENT_IP endereço para os endereços IP permitidos.

valid_ip($ip[, $which = ''])¶
Parâmetros:
  • $ip (string) - Endereço de IP
  • $which (string) – IP protocol (‘ipv4’ or ‘ipv6’)
Returns:

TRUE if the address is valid, FALSE if not
Return type:

bool

Toma um endereço IP como entrada e retorna TRUE ou FALSE (booleano) dependendo on whether it is valid or not.

Nota

The $this->input->ip_address() method above automatically validates the IP address.

if ( ! $this->input->valid_ip($ip))
{
        echo 'Not Valid';
}
else
{
        echo 'Valid';
}

Aceita um segundo parâmetro de string opcional de 'ipv4' ou 'ipv6' para especificar an IP format. The default checks for both formats.

user_agent([$xss_clean = NULL])¶
Returns:

User agent string or NULL if not set
Parâmetros:
  • $xss_clean (bool) – Whether to apply XSS filtering
Return type:

mixed

Returns the user agent string (web browser) being used by the current user, or NULL if it’s not available.

echo $this->input->user_agent();

Veja o Classe de agente do usuário for methods which extract informações da cadeia do agente do usuário.

request_headers([$xss_clean = FALSE])¶
Parâmetros:
  • $xss_clean (bool) – Whether to apply XSS filtering
Returns:

Uma matriz de cabeçalhos de solicitação HTTP
Return type:

array

Returns an array of HTTP request headers. Useful if running in a non-Apache environment where apache_request_headers() não será suportado.

$headers = $this->input->request_headers();
get_request_header($index[, $xss_clean = FALSE])¶
Parâmetros:
  • $index (string) - nome do cabeçalho da solicitação HTTP
  • $xss_clean (bool) – Whether to apply XSS filtering
Returns:

Um cabeçalho de solicitação HTTP ou NULL, se não encontrado
Return type:

string

Retorna um único membro da matriz de cabeçalhos de solicitação ou NULL se o cabeçalho procurado não for encontrado.

$this->input->get_request_header('some-header', TRUE);
is_ajax_request()¶
Returns:Verdadeiro se for um pedido de Ajax, FALSE se não for
Return type:bool

Verifica se o cabeçalho do servidor HTTP_X_REQUESTED_WITH foi set, and returns boolean TRUE if it is or FALSE if not.

is_cli_request()¶
Returns:TRUE if it is a CLI request, FALSE if not
Return type:bool

Verifica se o aplicativo foi executado a partir da linha de comando interface.

Nota

This method checks both the PHP SAPI name currently in use and if the STDIN constant is defined, which is usually a failsafe way to see if PHP is being run via the command line.

$this->input->is_cli_request()

Nota

This method is DEPRECATED and is now just an alias for the is_cli() function.

method([$upper = FALSE])¶
Parâmetros:
  • $upper (bool) - Se deve devolver o nome do método de pedido em maiúsculas ou minúsculas
Returns:

Método de solicitação HTTP
Return type:

string

Returns the $_SERVER['REQUEST_METHOD'], with the option to set it in uppercase or lowercase.

echo $this->input->method(TRUE); // Outputs: POST
echo $this->input->method(FALSE); // Outputs: post
echo $this->input->method(); // Outputs: post

Classe de Entrada¶

The Classe de Entrada serves two purposes:

  1. Ele pré-processa dados de entrada globais para segurança.
  2. Ele fornece alguns métodos Helperes para buscar dados de entrada e pré-processá-los.

Nota

Esta classe é inicializada automaticamente pelo sistema para que Não é necessário fazer isso manualmente.

  • Filtragem de entrada
    • Segurança Filtering
    • Filtragem XSS
  • Acessando dados de formulário
    • Usando dados POST, GET, COOKIE ou SERVER
    • Usando o fluxo de entrada php: //

Filtragem de entrada¶

Segurança Filtering¶

O método de filtragem de segurança é chamado automaticamente quando um novo controller é invocado. Ele faz o following:

  • If $config['allow_get_array'] is FALSE (default is TRUE), destroys o array GET global.
  • Destrói todas as variáveis globais no evento register_globals é ligadas.
  • Filtra as chaves do array GET/POST/COOKIE, permitindo apenas o número alfa-numérico (e alguns outros) caracteres.
  • Fornece filtragem XSS (Cross-site Scripting Hacks). Isso pode ser ativada globalmente ou mediante solicitação.
  • Standardizes newline characters to PHP_EOL (\n in UNIX-based OSes, \ r \ n no Windows). Isso é configurável.

Filtragem XSS¶

A classe Input tem a capacidade de filtrar a entrada automaticamente para evitar ataques de script entre sites. Se você quiser que o filtro seja executado automaticamente toda vez que encontrar dados POST ou COOKIE que você puder enable it by opening your application/config/config.php file and setting this:

$config['global_xss_filtering'] = TRUE;

Por favor, consulte o Segurança class documentação para information on using Filtragem XSS in your application.

Importante

A configuração 'global_xss_filtering' é DEPRECADA e mantida apenas para fins de compatibilidade com versões anteriores. XSS escapando deve be performed on output, not input!

Acessando dados de formulário¶

Usando dados POST, GET, COOKIE ou SERVER¶

CodeIgniter vem com métodos Helperes que permitem buscar POST, GET, Itens de cookie ou servidor. A principal vantagem de usar o métodos em vez de buscar um item diretamente ($_POST['something']) é que os métodos verificarão se o item está definido e retornará NULL se não. Isso permite usar convenientemente os dados sem ter que testar se um item existe primeiro. Em outras palavras, normalmente você pode fazer algo assim:

$something = isset($_POST['something']) ? $_POST['something'] : NULL;

Com os métodos embutidos do CodeIgniter, você pode simplesmente fazer isso:

$something = $this->input->post('something');

Os principais métodos são:

  • $this->input->post()
  • $this->input->get()
  • $this->input->cookie()
  • $this->input->server()

Usando o fluxo de entrada php: //¶

Se você quiser utilizar o PUT, DELETE, PATCH ou outro pedido exótico métodos, eles só podem ser acessados através de um fluxo de entrada especial, que só pode ser lido uma vez. Isso não é tão fácil quanto apenas ler, por exemplo, the $_POST array, because it will always exist and you can try e acessar múltiplas variáveis sem se importar que você possa ter apenas Um tiro em todos os dados do POST.

O CodeIgniter vai cuidar disso para você, e você pode ler os dados de php://input transmitir a qualquer momento, apenas usando o $raw_input_stream property:

$this->input->raw_input_stream;

Além disso, se o fluxo de entrada for codificado por formulário, como $_POST, você pode acessar seus valores chamando o input_stream() method:

$this->input->input_stream('key');

Similar to other methods such as get() and post(), if the dados solicitados não são encontrados, ele retornará NULL e você também pode decide whether to run the data through xss_clean() by passing um valor booleano como o segundo parâmetro:

$this->input->input_stream('key', TRUE); // XSS Clean
$this->input->input_stream('key', FALSE); // No XSS filter

Nota

You can utilize method() in order to know if you’re reading PUT, DELETE ou PATCH dados.

Referência de Classe¶

class CI_Input¶
$raw_input_stream

Propriedade somente leitura que retornará php: // input data as is.

A propriedade pode ser lida várias vezes.

post([$index = NULL[, $xss_clean = NULL]])¶
Parâmetros:
  • $index (mixed) - nome do parâmetro POST
  • $xss_clean (bool) - Aplicar ou não a filtragem de XSS
Retorna:

$_POST se nenhum parâmetro for fornecido, caso contrário, o valor POST será encontrado ou NULL se não for
Tipo de retorno:

mixed

O primeiro parâmetro conterá o nome do item POST que você está procurando por:

$this->input->post('some_data');

O método retorna NULL se o item que você está tentando recuperar não existe.

O segundo parâmetro opcional permite executar os dados por meio do XSS filtro. Está habilitado, configurando o segundo parâmetro para booleano TRUE or by setting your $config['global_xss_filtering'] to TRUE.

$this->input->post('some_data', TRUE);

Para retornar uma matriz de todos os itens POST, chame sem nenhum parâmetro.

Para retornar todos os itens do POST e passá-los pelo filtro XSS, defina primeiro parâmetro NULL ao definir o segundo parâmetro como booleano TRUE.

$this->input->post(NULL, TRUE); // returns all POST items with XSS filter
$this->input->post(NULL, FALSE); // returns all POST items without XSS filter

Para retornar uma matriz de vários parâmetros POST, passe todas as chaves necessárias como uma matriz.

$this->input->post(array('field1', 'field2'));

Mesma regra aplicada aqui, para recuperar os parâmetros com a filtragem XSS habilitada, defina segundo parâmetro para booleano TRUE.

$this->input->post(array('field1', 'field2'), TRUE);
get([$index = NULL[, $xss_clean = NULL]])¶
Parâmetros:
  • $index (mixed) - nome do parâmetro GET
  • $xss_clean (bool) - Aplicar ou não a filtragem de XSS
Retorna:

$_GET se nenhum parâmetro for fornecido, caso contrário, o valor GET se encontrado ou NULL se não for
Tipo de retorno:

mixed

This method is identical to post(), only it fetches GET data.

$this->input->get('some_data', TRUE);

Para retornar uma matriz de todos os itens GET, chame sem nenhum parâmetro.

Para retornar todos os itens GET e passá-los pelo filtro XSS, defina primeiro parâmetro NULL ao definir o segundo parâmetro como booleano TRUE.

$this->input->get(NULL, TRUE); // returns all GET items with XSS filter
$this->input->get(NULL, FALSE); // returns all GET items without XSS filtering

Para retornar uma matriz de múltiplos parâmetros GET, passe todas as chaves necessárias como uma matriz.

$this->input->get(array('field1', 'field2'));

Mesma regra aplicada aqui, para recuperar os parâmetros com a filtragem XSS habilitada, defina segundo parâmetro para booleano TRUE.

$this->input->get(array('field1', 'field2'), TRUE);
post_get($index[, $xss_clean = NULL])¶
Parâmetros:
  • $index (string) - nome do parâmetro POST/GET
  • $xss_clean (bool) - Aplicar ou não a filtragem de XSS
Retorna:

Valor POST/GET se encontrado, NULL se não for
Tipo de retorno:

mixed

This method works pretty much the same way as post() and get(), apenas combinado. Ele pesquisará por meio de fluxos POST e GET para dados, procurando no POST primeiro e, em seguida, no GET:

$this->input->post_get('some_data', TRUE);
get_post($index[, $xss_clean = NULL])¶
Parâmetros:
  • $index (string) - nome do parâmetro GET/POST
  • $xss_clean (bool) - Aplicar ou não a filtragem de XSS
Retorna:

Valor GET/POST se encontrado, NULL se não for
Tipo de retorno:

mixed

This method works the same way as post_get() only it looks for GET dados primeiro.

$this-> input-> get_post ('some_data', TRUE);

Nota

This method used to act EXACTLY like post_get(), but it’s comportamento mudou no CodeIgniter 3.0.

cookie([$index = NULL[, $xss_clean = NULL]])¶
Parâmetros:
  • $index (mixed) - nome do cookie
  • $xss_clean (bool) - Aplicar ou não a filtragem de XSS
Retorna:

$_COOKIE se nenhum parâmetro for fornecido, caso contrário, o valor de COOKIE se encontrado ou NULL se não for
Tipo de retorno:

mixed

This method is identical to post() and get(), only it fetches cookie data:

$this->input->cookie('some_cookie');
$this->input->cookie('some_cookie', TRUE); // with XSS filter

Para retornar uma matriz de vários valores de cookie, passe todas as chaves necessárias como uma matriz.

$this->input->cookie(array('some_cookie', 'some_cookie2'));

Nota

Ao contrário do Helper Cookie function get_cookie(), este método NÃO prefixa your configured $config['cookie_prefix'] value.

server($index[, $xss_clean = NULL])¶
Parâmetros:
  • $index (mixed) - nome do valor
  • $xss_clean (bool) - Aplicar ou não a filtragem de XSS
Retorna:

Valor do item $_SERVER se encontrado, NULL se não for
Tipo de retorno:

mixed

This method is identical to the post(), get() and cookie() methods, only it fetches server data ($_SERVER):

$this->input->server('some_data');

To return an array of multiple $_SERVER values, pass all the required keys como uma matriz.

$this->input->server(array('SERVER_PROTOCOL', 'REQUEST_URI'));
input_stream([$index = NULL[, $xss_clean = NULL]])¶
Parâmetros:
  • $index (mixed) - nome da chave
  • $xss_clean (bool) - Aplicar ou não a filtragem de XSS
Retorna:

Matriz de fluxo de entrada se nenhum parâmetro for fornecido, caso contrário, o valor especificado, se encontrado, ou NULL, se não for
Tipo de retorno:

mixed

This method is identical to get(), post() and cookie(), only it fetches the php://input stream data.

set_cookie($name = ''[, $value = ''[, $expire = ''[, $domain = ''[, $path = '/'[, $prefix = ''[, $secure = NULL[, $httponly = NULL]]]]]]])¶
Parâmetros:
  • $name (mixed) - Nome do cookie ou uma matriz de parâmetros
  • $value (string) - valor do cookie
  • $expire (int) - Tempo de expiração do cookie em segundos
  • $domain (string) - Domínio de cookie
  • $path (string) - caminho do cookie
  • $prefix (string) - prefixo do nome do cookie
  • $secure (bool) - Se apenas transferir o cookie por meio de HTTPS
  • $httponly (bool) - Se apenas tornar o cookie acessível para solicitações HTTP (sem JavaScript)
Tipo de retorno:

void

Define um cookie contendo os valores que você especifica. Existem duas maneiras de passar informações para esse método para que um cookie possa ser definido: Array Método e Parâmetros Discretos:

Método de Matriz

Usando este método, um array associativo é passado para o primeiro parameter:

$cookie = array(
        'name'   => 'The Cookie Name',
        'value'  => 'The Value',
        'expire' => '86500',
        'domain' => '.some-domain.com',
        'path'   => '/',
        'prefix' => 'myprefix_',
        'secure' => TRUE
);

$this->input->set_cookie($cookie);

Notas

Apenas o nome e o valor são obrigatórios. Para excluir um cookie, defina-o com o expiração em branco.

A expiração é definida em seconds, que será adicionado ao atual Tempo. Não inclua o tempo, mas apenas o número de segundos from now that you wish the cookie to be valid. If the expiration is definido como zero, o cookie durará apenas enquanto o navegador estiver aberto.

Para cookies em todo o site, independentemente da forma como o site é solicitado, adicione o seu URL para o domain começando com um período, assim: .your-domain.com

O caminho geralmente não é necessário, pois o método define um caminho raiz.

O prefixo só é necessário se você precisar evitar colisões de nomes com outros cookies com nomes idênticos para o seu servidor.

The httponly and secure flags, when omitted, will default to your $config['cookie_httponly'] and $config['cookie_secure'] settings.

Parâmetros Discretos

Se preferir, você pode definir o cookie passando dados usando parameters:

$this->input->set_cookie($name, $value, $expire, $domain, $path, $prefix, $secure);
ip_address()¶
Retorna:Endereço IP do visitante ou '0.0.0.0' se não for válido
Tipo de retorno:string

Retorna o endereço IP para o usuário atual. Se o endereço IP não for válido, o método retornará '0.0.0.0':

echo $this->input->ip_address();

Importante

Este método leva em consideração $config['proxy_ips'] configuração e retornará o HTTP_X_FORWARDED_FOR, HTTP_CLIENT_IP, HTTP_X_CLIENT_IP ou HTTP_X_CLUSTER_CLIENT_IP endereço para os endereços IP permitidos.

valid_ip($ip[, $which = ''])¶
Parâmetros:
  • $ip (string) - Endereço de IP
  • $which (string) - protocolo IP ('ipv4' ou 'ipv6')
Retorna:

TRUE se o endereço for válido, FALSE se não for
Tipo de retorno:

bool

Toma um endereço IP como entrada e retorna TRUE ou FALSE (booleano) dependendo sobre se é válido ou não.

Nota

The $this->input->ip_address() method above automatically valida o endereço IP.

if ( ! $this->input->valid_ip($ip))
{
        echo 'Not Valid';
}
else
{
        echo 'Valid';
}

Aceita um segundo parâmetro de string opcional de 'ipv4' ou 'ipv6' para especificar um formato IP. O padrão verifica os dois formatos.

user_agent([$xss_clean = NULL])¶
Retorna:

Seqüência do agente do usuário ou NULL, se não estiver definido
Parâmetros:
  • $xss_clean (bool) - Aplicar ou não a filtragem de XSS
Tipo de retorno:

mixed

Retorna a string do agente do usuário (navegador da web) sendo usada pelo usuário atual, ou NULL se não estiver disponível.

echo $this->input->user_agent();

Veja o Classe de agente do usuário para métodos que extraem information de user agent string.

request_headers([$xss_clean = FALSE])¶
Parâmetros:
  • $xss_clean (bool) - Aplicar ou não a filtragem de XSS
Retorna:

Uma matriz de cabeçalhos de solicitação HTTP
Tipo de retorno:

array

Retorna uma matriz de cabeçalhos de solicitação HTTP. Útil se estiver executando em um ambiente não-Apache onde apache_request_headers() não será suportado.

$headers = $this->input->request_headers();
get_request_header($index[, $xss_clean = FALSE])¶
Parâmetros:
  • $index (string) - nome do cabeçalho da solicitação HTTP
  • $xss_clean (bool) - Aplicar ou não a filtragem de XSS
Retorna:

Um cabeçalho de solicitação HTTP ou NULL, se não encontrado
Tipo de retorno:

string

Retorna um único membro da matriz de cabeçalhos de solicitação ou NULL se o cabeçalho procurado não for encontrado.

$this->input->get_request_header('some-header', TRUE);
is_ajax_request()¶
Retorna:Verdadeiro se for um pedido de Ajax, FALSE se não for
Tipo de retorno:bool

Verifica se o cabeçalho do servidor HTTP_X_REQUESTED_WITH foi set e retorna booleano TRUE se for ou FALSE se não for.

is_cli_request()¶
Retorna:TRUE se for uma solicitação CLI, FALSE se não for
Tipo de retorno:bool

Checks to see if the application was run de command-line interface.

Nota

Este método verifica o nome PHP SAPI atualmente em uso and if the STDIN constant is defined, which is usually a Uma maneira segura de ver se o PHP está sendo executado através da linha de comando.

$this->input->is_cli_request()

Nota

Esse método é DEPRECADO e agora é apenas um alias para o is_cli() function.

method([$upper = FALSE])¶
Parâmetros:
  • $upper (bool) - Se deve devolver o nome do método de pedido em maiúsculas ou minúsculas
Retorna:

Método de solicitação HTTP
Tipo de retorno:

string

Returns the $_SERVER['REQUEST_METHOD'], with the option to set it em maiúsculas ou minúsculas.

echo $this->input->method(TRUE); // Outputs: POST
echo $this->input->method(FALSE); // Outputs: post
echo $this->input->method(); // Outputs: post

Classe de Entrada¶

The Classe de Entrada serves two purposes:

  1. Ele pré-processa dados de entrada globais para segurança.
  2. Ele fornece alguns métodos Helperes para buscar dados de entrada e pré-processá-los.

Nota

Esta classe é inicializada automaticamente pelo sistema para que Não é necessário fazer isso manualmente.

  • Filtragem de entrada
    • Segurança Filtering
    • Filtragem XSS
  • Acessando dados de formulário
    • Usando dados POST, GET, COOKIE ou SERVER
    • Usando o fluxo de entrada php: //
  • Referência de Classe

Filtragem de entrada¶

Segurança Filtering¶

O método de filtragem de segurança é chamado automaticamente quando um novo controller é invocado. Ele faz o following:

  • If $config['allow_get_array'] is FALSE (default is TRUE), destroys o array GET global.
  • Destrói todas as variáveis globais no evento register_globals é turned on.
  • Filtra as chaves do array GET/POST/COOKIE, permitindo apenas o número alfa-numérico (e alguns outros) caracteres.
  • Fornece filtragem XSS (Cross-site Scripting Hacks). Isso pode ser ativada globalmente ou mediante solicitação.
  • Standardizes newline characters to PHP_EOL (\n in UNIX-based OSes, \ r \ n no Windows). Isso é configurável.

Filtragem XSS¶

A classe Input tem a capacidade de filtrar a entrada automaticamente para evitar ataques de script entre sites. Se você quiser que o filtro seja executado automaticamente toda vez que encontrar dados POST ou COOKIE que você puder enable it by opening your application/config/config.php file and setting this:

$config['global_xss_filtering'] = TRUE;

Por favor, consulte o Segurança class documentação para information on using Filtragem XSS in your application.

Important

A configuração 'global_xss_filtering' é DEPRECADA e mantida apenas para fins de compatibilidade com versões anteriores. XSS escapando deve be performed on output, not input!

Acessando dados de formulário¶

Usando dados POST, GET, COOKIE ou SERVER¶

CodeIgniter vem com métodos Helperes que permitem buscar POST, GET, Itens de cookie ou servidor. A principal vantagem de usar o métodos em vez de buscar um item diretamente ($_POST['something']) é que os métodos verificarão se o item está definido e retornará NULL se não. Isso permite usar convenientemente os dados sem ter que testar se um item existe primeiro. Em outras palavras, normalmente você pode fazer algo assim:

$something = isset($_POST['something']) ? $_POST['something'] : NULL;

Com os métodos embutidos do CodeIgniter, você pode simplesmente fazer isso:

$something = $this->input->post('something');

Os principais métodos são:

  • $this->input->post()
  • $this->input->get()
  • $this->input->cookie()
  • $this->input->server()

Usando o fluxo de entrada php: //¶

Se você quiser utilizar o PUT, DELETE, PATCH ou outro pedido exótico métodos, eles só podem ser acessados através de um fluxo de entrada especial, que só pode ser lido uma vez. Isso não é tão fácil quanto apenas ler, por exemplo, the $_POST array, because it will always exist and you can try e acessar múltiplas variáveis sem se importar que você possa ter apenas Um tiro em todos os dados do POST.

O CodeIgniter vai cuidar disso para você, e você pode ler os dados from the php://input transmitir a qualquer momento, apenas usando o $raw_input_stream property:

$this->input->raw_input_stream;

Além disso, se o fluxo de entrada for codificado por formulário, como $_POST, você pode acessar seus valores chamando o input_stream() method:

$this->input->input_stream('key');

Similar to other methods such as get() and post(), if the dados solicitados não são encontrados, ele retornará NULL e você também pode decide whether to run the data through xss_clean() by passing um valor booleano como o segundo parâmetro:

$this->input->input_stream('key', TRUE); // XSS Clean
$this->input->input_stream('key', FALSE); // No XSS filter

Nota

You can utilize method() in order to know if you’re reading PUT, DELETE ou PATCH dados.

class CI_Input¶
$raw_input_stream

Propriedade somente leitura que retornará php: // input data as is.

A propriedade pode ser lida várias vezes.

post([$index = NULL[, $xss_clean = NULL]])¶
Parâmetros:
  • $index (mixed) - nome do parâmetro POST
  • $xss_clean (bool) - Aplicar ou não a filtragem de XSS
Retorna:

$_POST se nenhum parâmetro for fornecido, caso contrário, o valor POST será encontrado ou NULL se não for
Tipo de retorno:

mixed

O primeiro parâmetro conterá o nome do item POST que você está procurando por:

$this->input->post('some_data');

O método retorna NULL se o item que você está tentando recuperar não existe.

O segundo parâmetro opcional permite executar os dados por meio do XSS filtro. Está habilitado, configurando o segundo parâmetro para booleano TRUE or by setting your $config['global_xss_filtering'] to TRUE.

$this->input->post('some_data', TRUE);

Para retornar uma matriz de todos os itens POST, chame sem nenhum parâmetro.

Para retornar todos os itens do POST e passá-los pelo filtro XSS, defina primeiro parâmetro NULL ao definir o segundo parâmetro como booleano TRUE.

$this->input->post(NULL, TRUE); // returns all POST items with XSS filter
$this->input->post(NULL, FALSE); // returns all POST items without XSS filter

Para retornar uma matriz de vários parâmetros POST, passe todas as chaves necessárias como uma matriz.

$this->input->post(array('field1', 'field2'));

Mesma regra aplicada aqui, para recuperar os parâmetros com a filtragem XSS habilitada, defina segundo parâmetro para booleano TRUE.

$this->input->post(array('field1', 'field2'), TRUE);
get([$index = NULL[, $xss_clean = NULL]])¶
Parâmetros:
  • $index (mixed) - nome do parâmetro GET
  • $xss_clean (bool) - Aplicar ou não a filtragem de XSS
Retorna:

$_GET se nenhum parâmetro for fornecido, caso contrário, o valor GET se encontrado ou NULL se não for
Tipo de retorno:

mixed

This method is identical to post(), only it fetches GET data.

$this->input->get('some_data', TRUE);

Para retornar uma matriz de todos os itens GET, chame sem nenhum parâmetro.

Para retornar todos os itens GET e passá-los pelo filtro XSS, defina primeiro parâmetro NULL ao definir o segundo parâmetro como booleano TRUE.

$this->input->get(NULL, TRUE); // returns all GET items with XSS filter
$this->input->get(NULL, FALSE); // returns all GET items without XSS filtering

Para retornar uma matriz de múltiplos parâmetros GET, passe todas as chaves necessárias como uma matriz.

$this->input->get(array('field1', 'field2'));

Mesma regra aplicada aqui, para recuperar os parâmetros com a filtragem XSS habilitada, defina segundo parâmetro para booleano TRUE.

$this->input->get(array('field1', 'field2'), TRUE);
post_get($index[, $xss_clean = NULL])¶

Valor do item $_SERVER se encontrado, NULL se não for

set_cookie($name = ''[, $value = ''[, $expire = ''[, $domain = ''[, $path = '/'[, $prefix = ''[, $secure = NULL[, $httponly = NULL]]]]]]])¶
Parâmetros:
  • $name (mixed) - Nome do cookie ou uma matriz de parâmetros
  • $value (string) - valor do cookie
  • $expire (int) - Tempo de expiração do cookie em segundos
  • $domain (string) - Domínio de cookie
  • $path (string) - caminho do cookie
  • $prefix (string) - prefixo do nome do cookie
  • $secure (bool) - Se apenas transferir o cookie por meio de HTTPS
  • $httponly (bool) - Se apenas tornar o cookie acessível para solicitações HTTP (sem JavaScript)
Tipo de retorno:

void

Define um cookie contendo os valores que você especifica. Existem duas maneiras de passar informações para esse método para que um cookie possa ser definido: Array Método e Parâmetros Discretos:

Método de Matriz

Usando este método, um array associativo é passado para o primeiro parameter:

$cookie = array(
        'name'   => 'The Cookie Name',
        'value'  => 'The Value',
        'expire' => '86500',
        'domain' => '.some-domain.com',
        'path'   => '/',
        'prefix' => 'myprefix_',
        'secure' => TRUE
);

$this->input->set_cookie($cookie);

Notas

Apenas o nome e o valor são obrigatórios. Para excluir um cookie, defina-o com o expiração em branco.

A expiração é definida em seconds, que será adicionado ao atual Tempo. Não inclua o tempo, mas apenas o número de segundos from now that you wish the cookie to be valid. If the expiration is definido como zero, o cookie durará apenas enquanto o navegador estiver aberto.

Para cookies em todo o site, independentemente da forma como o site é solicitado, adicione o seu URL to the domain começando com um período, assim: .your-domain.com

O caminho geralmente não é necessário, pois o método define um caminho raiz.

O prefixo só é necessário se você precisar evitar colisões de nomes com outros cookies com nomes idênticos para o seu servidor.

The httponly and secure flags, when omitted, will default to your $config['cookie_httponly'] and $config['cookie_secure'] settings.

Parâmetros Discretos

Se preferir, você pode definir o cookie passando dados usando parameters:

$this->input->set_cookie($name, $value, $expire, $domain, $path, $prefix, $secure);
ip_address()¶
Retorna:Endereço IP do visitante ou '0.0.0.0' se não for válido
Tipo de retorno:string

Retorna o endereço IP para o usuário atual. Se o endereço IP não for válido, o método retornará '0.0.0.0':

echo $this->input->ip_address();

Important

Este método leva em consideração $config['proxy_ips'] configuração e retornará o HTTP_X_FORWARDED_FOR, HTTP_CLIENT_IP, HTTP_X_CLIENT_IP ou HTTP_X_CLUSTER_CLIENT_IP endereço para os endereços IP permitidos.

valid_ip($ip[, $which = ''])¶
Parâmetros:
  • $ip (string) - Endereço de IP
  • $which (string) - protocolo IP ('ipv4' ou 'ipv6')
Retorna:

TRUE se o endereço for válido, FALSE se não for
Tipo de retorno:

bool

Toma um endereço IP como entrada e retorna TRUE ou FALSE (booleano) dependendo sobre se é válido ou não.

Nota

The $this->input->ip_address() method above automatically valida o endereço IP.

if ( ! $this->input->valid_ip($ip))
{
        echo 'Not Valid';
}
else
{
        echo 'Valid';
}

Aceita um segundo parâmetro de string opcional de 'ipv4' ou 'ipv6' para especificar um formato IP. O padrão verifica os dois formatos.
