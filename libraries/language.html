Aula de idioma¶

o Aula de idioma provides functions to retrieve language files and linhas de texto para fins de internacionalização.

In your CodeIgniter system pasta, você encontrará um language sub-directory containing a set of language files for the english idiom. Os arquivos neste diretório (system/language/english/) define the regular messages, error messages, and other generally output terms or expressions, for the different parts do framework CodeIgniter.

You can create or incorporate your own language files, as needed, in order to provide application-specific error and other messages, or to provide translations of the core messages into other languages. ose translations or additional messages would go inside your application/language/ diretório, com subdiretórios separados para cada idioma (for instance, ‘french’ or ‘german’).

o CodeIgniter framework comes with a set of language files for the “english” idiom. Additional approved translations for different idioms may be found in the CodeIgniter 3 Translations repositories. Each repository deals with a single idiom.

When CodeIgniter loads language files, it will load the one in system/language/ first and will then look for an override in your application/language/ directory.

Nota

Cada idioma deve ser armazenado em sua própria pasta. Por exemplo, the English files are located at: system/language/english

Handling Multiple Languages¶

Se você quiser oferecer suporte a vários idiomas em seu aplicativo, forneça pastas dentro your application/language/ directory for each of them, and you would specify the default idioma em seu application/config/config.php.

o application/language/english/ diretório conteria quaisquer arquivos de idioma adicionais needed by your application, for instance for error messages.

Each of the other idiom-specific directories would contain the core language files that you obtained from the translations repositories, or that you translated yourself, as well as any additional ones needed by your application.

Você armazenaria o idioma que está usando atualmente, por exemplo, em uma variável de sessão.

Amostra de arquivos de idioma¶

system/
        language/
                english/
                        ...
                        email_lang.php
                        form_validation_lang.php
                        ...

application/
        language/
                english/
                        error_messages_lang.php
                french/
                        ...
                        email_lang.php
                        error_messages_lang.php
                        form_validation_lang.php
                        ...

Example of switching languages¶

$idiom = $this->session->get_userdata('language');
$this->lang->load('error_messages', $idiom);
$oops = $this->lang->line('message_key');

Internationalization¶

o Language class in CodeIgniter is meant to provide an easy and lightweight way to support multiplelanguages in your application. It is not meant to be a implementação completa do que é comumente chamado internacionalização e localização.

Usamos o termo "idioma" para nos referir a um idioma usando seu nome comum, rather than using any of the international standards, such as “en”, “en-US”, or “en-CA-x-ca” for English and some of its variants.

Nota

ore is nothing to prevent you from using those abbreviations in your application!

Using the Aula de idioma¶

Criando arquivos de idioma¶

Language files must be named with _lang.php como a extensão do nome do arquivo. For example, let’s say you want to create a file containing error messages. You might name it: error_lang.php

Within the file you will assign each line of text to an array called $lang com este protótipo:

$lang['language_key'] = 'o actual message to be shown';

Nota

É uma boa prática usar um prefixo comum para todas as mensagens in a given file to avoid collisions with similarly named items in other arquivos. Por exemplo, se você estiver criando mensagens de erro, talvez prefira eles com erro_

$lang['error_email_missing'] = 'You must submit an email address';
$lang['error_url_missing'] = 'You must submit a URL';
$lang['error_username_missing'] = 'You must submit a username';

Loading A Language File¶

Para buscar uma linha de um determinado arquivo, você deve carregar o arquivo primeiro. Carregar um arquivo de idioma é feito com o seguinte código:

$this->lang->load('filename', 'language');

Onde filename é o nome do arquivo que você deseja carregar (sem o nome extensão de arquivo), e language é o conjunto de idiomas que o contém (por Inglês). Se o segundo parâmetro estiver ausente, o conjunto de idiomas padrão in your application/config/config.php file will be used.

Você também pode carregar vários arquivos de idioma ao mesmo tempo passando uma matriz de arquivos de idioma como primeiro parâmetro.

$this->lang->load(array('filename1', 'filename2'));

Nota

o language parameter can only consist of letters.

Fetching a Line of Text¶

Once your desired language file is loaded you can access any line of texto usando esta função:

$this->lang->line('language_key');

Where language_key is the array key corresponding to the line you wish to show.

You can optionally pass FALSE as the second argument of that method to disable error logging, in case you’re not sure if the line exists:

$this->lang->line('misc_key', FALSE);

Nota

This method simply returns the line. It does not echo it.

Using language lines as form labels¶

This feature has been deprecated from the language library and moved to the lang() função do Auxiliar de Idioma.

Auto-loading Languages¶

If you find that you need a particular language globally throughout your application, you can tell CodeIgniter to auto-load durante a inicialização do sistema. Isso está feito by opening the application/config/autoload.php file and adding the language(s) to the autoload array.

Carregando um arquivo de idioma¶

Para buscar uma linha de um determinado arquivo, você deve carregar o arquivo primeiro. Carregar um arquivo de idioma é feito com o seguinte código:

$this->lang->load('filename', 'language');

Onde filename é o nome do arquivo que você deseja carregar (sem o nome extensão de arquivo), e language é o conjunto de idiomas que o contém (por Inglês). Se o segundo parâmetro estiver ausente, o conjunto de idiomas padrão na tua application/config/config.php arquivo será usado.

Você também pode carregar vários arquivos de idioma ao mesmo tempo passando uma matriz de arquivos de idioma como primeiro parâmetro.

$this->lang->load(array('filename1', 'filename2'));

Nota

o language parameter can only consist of letters.

Buscando uma linha de texto¶

Quando o arquivo de idioma desejado for carregado, você poderá acessar qualquer linha de texto usando esta função:

$this->lang->line('language_key');

Where language_key is the array key corresponding to the line you wish mostrar.

Você pode, opcionalmente, passar FALSE como o segundo argumento desse método para desabilite o log de erros, caso você não tenha certeza se a linha existe:

$this->lang->line('misc_key', FALSE);

Nota

Esse método simplesmente retorna a linha. Não faz eco.

Classe de idioma¶

o Classe de idioma provides functions to retrieve language files and linhas de texto para fins de internacionalização.

Em seu CodeIgniter system pasta, você encontrará um language sub-directory contendo um conjunto de arquivos de idioma para o english idiom. Os arquivos neste diretório (system/language/english/) definir as mensagens regulares, mensagens de erro e outros termos ou expressões geralmente de saída, para as diferentes partes do framework CodeIgniter.

Você pode criar ou incorporar seus próprios arquivos de idioma, conforme necessário, para fornecer erro específico da aplicação e outras mensagens, ou para fornecer traduções do núcleo mensagens para outros idiomas. Essas traduções ou mensagens adicionais iriam para dentro your application/language/ diretório, com subdiretórios separados para cada idioma (por exemplo, 'francês' ou 'alemão').

O framework do CodeIgniter vem com um conjunto de arquivos de idioma para o idioma "inglês". Traduções aprovadas adicionais para diferentes idiomas podem ser encontradas no Repositórios de Tradução do CodeIgniter 3. Cada repositório lida com uma única expressão idiomática.

Quando o CodeIgniter carrega arquivos de idioma, ele carrega aquele em system/language/ primeiro e, em seguida, procure uma substituição em seu application/language/ directory.

Nota

Cada idioma deve ser armazenado em sua própria pasta. Por exemplo, os arquivos em inglês estão localizados em: system/language/english

Manipulando vários idiomas¶

Se você quiser oferecer suporte a vários idiomas em seu aplicativo, forneça pastas dentro your application/language/ diretório para cada um deles, e você especificaria o padrão idioma em seu application/config/config.php.

o application/language/english/ diretório conteria quaisquer arquivos de idioma adicionais necessário pelo seu aplicativo, por exemplo, para mensagens de erro.

Cada um dos outros diretórios específicos do idioma conterá os arquivos de idioma principais que você obtidos dos repositórios de traduções, ou que você se traduziu, bem como quaisquer adicionais necessários ao seu aplicativo.

Você armazenaria o idioma que está usando atualmente, por exemplo, em uma variável de sessão.

Amostra de arquivos de idioma¶

system/
        language/
                english/
                        ...
                        email_lang.php
                        form_validation_lang.php
                        ...

application/
        language/
                english/
                        error_messages_lang.php
                french/
                        ...
                        email_lang.php
                        error_messages_lang.php
                        form_validation_lang.php
                        ...

Exemplo de troca de idiomas¶

$idiom = $this->session->get_userdata('language');
$this->lang->load('error_messages', $idiom);
$oops = $this->lang->line('message_key');

Internacionalização¶

o Language class in CodeIgniter is meant to provide an easy and lightweight maneira de suportar multiplolinguagens em sua aplicação. Não é para ser um implementação completa do que é comumente chamado internacionalização e localização.

Usamos o termo "idioma" para nos referir a um idioma usando seu nome comum, em vez de usar qualquer um dos padrões internacionais, como "en", "en-US", ou "en-CA-x-ca" para o inglês e algumas de suas variantes.

Nota

ore is nothing to prevent you from using those abbreviations in your application!

Criando arquivos de idioma¶

Os arquivos de idioma devem ser nomeados com _lang.php como a extensão do nome do arquivo. Por exemplo, digamos que você queira criar um arquivo contendo mensagens de erro. Você pode nomear: error_lang.php

Dentro do arquivo você irá atribuir cada linha de texto a um array chamado $lang com este protótipo:

$lang['language_key'] = 'o actual message to be shown';

Nota

É uma boa prática usar um prefixo comum para todas as mensagens em um determinado arquivo para evitar colisões com itens com nomes semelhantes em outros arquivos. Por exemplo, se você estiver criando mensagens de erro, talvez prefira eles com erro_

$lang['error_email_missing'] = 'You must submit an email address';

$lang['error_username_missing'] = 'You must submit a username';

$this->lang->load('filename', 'language');

Você também pode carregar vários arquivos de idioma ao mesmo tempo passando uma matriz de arquivos de idioma como primeiro parâmetro.

$this->lang->load(array('filename1', 'filename2'));

o language parameter can only consist of letters.

$this->lang->line('language_key');
