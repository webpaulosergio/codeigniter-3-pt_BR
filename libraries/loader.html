Classe Loader¶

Loader, as the name suggests, is used to load elements. These elements can be libraries (classes) ver arquivos, Drivers, Helpers, Models, or your own files.

Nota

Esta classe é inicializada automaticamente pelo sistema para que is no need to do it manually.

  • Aplicação "Pacotes"
    • Arquivos de visualização de pacote
  • Class Reference

Aplicação "Pacotes"¶

Um pacote de aplicativos permite a fácil distribuição de conjuntos completos de recursos em um único diretório, completo com suas próprias librarys, models, helpers, config, and language files. It is recommended that these packages be placed in the application/third_party directory. Below is a sample map of an package directory.

The following is an example of a directory for an application package chamado "Foo Bar".

/application/third_party/foo_bar

config/
helpers/
language/
libraries/
models/

Whatever the purpose of the “Foo Bar” application package, it has its próprios arquivos de configuração, Helpers, arquivos de idiomas, Librarys e modelos. Usar esses recursos em seus controladores, você primeiro precisa informar ao Loader that you are going to be loading resources from a package, by adding the package path via the add_package_path() method.

Arquivos de visualização de pacote¶

By Default, package view files paths are set when add_package_path() é chamado. Os caminhos de visualização são percorridos e uma vez que uma correspondência é encountered that view is loaded.

Neste caso, é possível visualizar colisões de nomes dentro packages to occur, and possibly the incorrect package being loaded. To ensure against this, set an optional second parameter of FALSE when calling add_package_path().

$this->load->add_package_path(APPPATH.'my_app', FALSE);
$this->load->view('my_app_index'); // Loads
$this->load->view('welcome_message'); // Will not load the default welcome_message b/c the second param to add_package_path is FALSE

// Reset things
$this->load->remove_package_path(APPPATH.'my_app');

// Again without the second parameter:
$this->load->add_package_path(APPPATH.'my_app');
$this->load->view('my_app_index'); // Loads
$this->load->view('welcome_message'); // Loads

Class Reference¶

class CI_Loader¶
library($library[, $params = NULL[, $object_name = NULL]])¶
Parâmetros:
  • $library (mixed) – Library name as a string or an array with multiple libraries
  • $params (array) – Optional array of parameters to pass to the loaded library’s constructor
  • $object_name (string) – Optional object name to assign the library to
Returns:

CI_Loader instance (method chaining)
Return type:

CI_Loader

This method is used to load core classes.

Nota

Usamos os termos "classe" e "library" de maneira intercambiável.

For example, if you would like to send email with CodeIgniter, the first passo é carregar a classe de email dentro do seu controlador:

$this->load->library('email');

Once loaded, the library will be ready for use, using $this->email.

Library files can be stored in subdirectories within the main “libraries” directory, or within your personal application/libraries directory. To load a file located in a subdirectory, simply include the path, relative to the “libraries” directory. For example, if you have file located at:

libraries/flavors/Chocolate.php

Você irá carregá-lo usando:

$this->load->library('flavors/chocolate');

You may nest the file in as many subdirectories as you want.

Além disso, várias librarys podem ser carregadas ao mesmo tempo passando uma matriz de librarys para o método load.

$this->load->library(array('email', 'table'));

Opções de configuração

The second (optional) parameter allows you to optionally pass configuração. Você normalmente passará isso como uma matriz:

$config = array (
        'mailtype' => 'html',
        'charset'  => 'utf-8',
        'priority' => '1'
);

$this->load->library('email', $config);

Config options can usually also be set via a config file. Each library é explicado em detalhes em sua própria página, então por favor leia as informações regarding each one you would like to use.

Please take note, when multiple libraries are supplied in an array for the first parameter, each will receive the same parameter information.

Assigning a Library to a different object name

If the third (optional) parameter is blank, the library will usually be atribuído a um objeto com o mesmo nome da library. Por exemplo, se the library is named Calendar, it will be assigned to a variable named $this->calendar.

If you prefer to set your own class names you can pass its value to the third parameter:

$this->load->library('calendar', NULL, 'my_calendar');

// Calendar class is now accessed using:
$this->my_calendar

Please take note, when multiple libraries are supplied in an array for the first parameter, this parameter is discarded.

driver($library[, $params = NULL[, $object_name]])¶
Parâmetros:
  • $library (mixed) – Library name as a string or an array with multiple libraries
  • $params (array) – Optional array of parameters to pass to the loaded library’s constructor
  • $object_name (string) – Optional object name to assign the library to
Returns:

CI_Loader instance (method chaining)
Return type:

CI_Loader

This method is used to load driver libraries, acts very much like the library() method.

As an example, if you would like to use sessions with CodeIgniter, the first step is to load the session driver within your controller:

$this->load->driver('session');

Once loaded, the library will be ready for use, using $this->session.

Driver files must be stored in a subdirectory within the main “libraries” directory, or within your personal application/libraries diretório. O subdiretório deve corresponder ao nome da classe pai. Leia o Drivers description for details.

Additionally, multiple driver libraries can be loaded at the same time by passing an array of drivers to the load method.

$this->load->driver(array('session', 'cache'));

Opções de configuração

The second (optional) parameter allows you to optionally pass configuration settings. You will typically pass these as an array:

$config = array(
        'sess_driver' => 'cookie',
        'sess_encrypt_cookie'  => true,
        'encryption_key' => 'mysecretkey'
);

$this->load->driver('session', $config);

Config options can usually also be set via a config file. Each library é explicado em detalhes em sua própria página, então por favor leia as informações regarding each one you would like to use.

Assigning a Driver to a different object name

If the third (optional) parameter is blank, the library will be assigned to an object with the same name as the parent class. For example, if a library é denominada Session, será atribuída a uma variável chamada $this->session.

If you prefer to set your own class names you can pass its value to the third parameter:

$this->load->library('session', '', 'my_session');

// Session class is now accessed using:
$this->my_session
view($view[, $vars = array()[, return = FALSE]])¶
Parâmetros:
  • $view (string) – View name
  • $vars (array) – An associative array of variables
  • $return (bool) - Se deve devolver a vista carregada
Returns:

Exibir a sequência de conteúdo se $return estiver definido como TRUE, caso contrário, a ocorrência de CI_Loader (encadeamento do método)
Return type:

mixed

This method is used to load your ver arquivos. If you haven’t read the Views seção do guia do usuário é recommended that you do since it shows you how this method is normalmente usado.

The first parameter is required. It is the name of the view file you would like to load.

Nota

A extensão de arquivo .php não precisa ser especificada a menos que you use something other than .php.

The second optional parâmetro pode ter uma matriz associativa ou um objeto como entrada, que é executado através do PHP extract() função para converter em variáveis that can be used in your view files. Again, read the Views page to learn how this might be useful.

O terceiro optional parameter lets you change the behavior of the method so that it returns data as a string rather than sending it to your browser. This can be useful if you want to process the data in some way. If you set the parameter to TRUE (boolean) it will return data. The o comportamento padrão é FALSE, que envia para o seu navegador. Lembrar de atribuí-lo a uma variável se você quiser que os dados retornados:

$string = $this->load->view('myfile', '', TRUE);
vars($vars[, $val = ''])¶
Parâmetros:
  • $vars (mixed) – An array of variables or a single variable name
  • $val (mixed) - Valor da variável opcional
Returns:

CI_Loader instance (method chaining)
Return type:

CI_Loader

Esse método usa um array associativo como entrada e gera variables using the PHP extract() função. Esse método produz o mesmo resultado que usar o segundo parameter of the $this->load->view() method above. The reason you might want to use this method independently is if you would like to set some global variables in the constructor of your controller and have eles ficam disponíveis em qualquer arquivo de visualização carregado de qualquer método. Você pode tem várias chamadas para este método. Os dados são armazenados em cache e mesclados into one array for conversion to variables.

is_loaded($class)¶
Parâmetros:
  • $class (string) – Class name
Returns:

Singleton property name if found, FALSE if not
Return type:

mixed

Allows you to check if a class has already been loaded or not.

Nota

The word “class” here refers to libraries and drivers.

Se a classe solicitada foi carregada, o método retorna sua atribuição nome no Superobjeto de CI e FALSE se não for:

$this->load->library('form_validation');
$this->load->is_loaded('Form_validation');      // returns 'form_validation'

$this->load->is_loaded('Nonexistent_library');  // returns FALSE

Importante

If you have more than one instance of a class (assigned to propriedades diferentes), então o primeiro será retornado.

$this->load->library('form_validation', $config, 'fv');
$this->load->library('form_validation');

$this->load->is_loaded('Form_validation');      // returns 'fv'
add_package_path($path[, $view_cascade = TRUE])¶
Parâmetros:
  • $path (string) – Path to add
  • $view_cascade (bool) - Se usar vistas em cascata
Returns:

CI_Loader instance (method chaining)
Return type:

CI_Loader

Adicionar um caminho de pacote instrui a classe Loader a preceder um determinado caminho para solicitações subseqüentes de recursos. Como exemplo, o “Foo Bar” application package above has a library named Foo_bar.php. In our controller, we’d do the following:

$this->load->add_package_path(APPPATH.'third_party/foo_bar/')
        ->library('foo_bar');
remove_package_path([$path = ''])¶
Parâmetros:
  • $path (string) - Caminho para remover
Returns:

CI_Loader instance (method chaining)
Return type:

CI_Loader

When your controller is finished using resources from an application package, and particularly if you have other application packages you quer trabalhar com, você pode querer remover o caminho do pacote para que o Loader não procura mais nesse diretório por recursos. Para remover o último caminho added, simply call the method with no parameters.

Ou para remover um caminho de pacote específico, especifique o mesmo caminho anteriormente given to add_package_path() for a package.:

$this->load->remove_package_path(APPPATH.'third_party/foo_bar/');

Classe Loader¶

Loader, como o nome sugere, é usado para carregar elementos. Esses elementos podem ser Librarys (classes) ver arquivos, Drivers, Helpers, Modelsou seus próprios arquivos.

Nota

Esta classe é inicializada automaticamente pelo sistema para que Não é necessário fazer isso manualmente.

  • Aplicação "Pacotes"
    • Arquivos de visualização de pacote
  • Referência de Classe

Aplicação "Pacotes"¶

Um pacote de aplicativos permite a fácil distribuição de conjuntos completos de recursos em um único diretório, completo com suas próprias librarys, Models, helpers, config e arquivos de idioma. É recomendado que esses pacotes podem ser colocados no diretório application/third_party. Abaixo é um mapa de amostra de um diretório de pacotes.

A seguir, um exemplo de um diretório para um pacote de aplicativos chamado "Foo Bar".

/application/third_party/foo_bar

config/
helpers/
language/
libraries/
models/

Qualquer que seja a finalidade do pacote de aplicativos "Foo Bar", ele tem próprios arquivos de configuração, Helpers, arquivos de idiomas, Librarys e Models. Usar esses recursos em seus Controlleres, você primeiro precisa informar ao Loader você estará carregando recursos de um pacote, adicionando o package path via the add_package_path() method.

Arquivos de visualização de pacote¶

Por padrão, os caminhos dos arquivos de exibição de pacote são definidos add_package_path() é chamado. Os caminhos de visualização são percorridos e uma vez que uma correspondência é encontrou essa View é carregada.

Neste caso, é possível visualizar colisões de nomes dentro pacotes a ocorrer e possivelmente o pacote incorreto sendo carregado. Para garantir contra isso, defina um segundo parâmetro opcional de FALSE quando calling add_package_path().

$this->load->add_package_path(APPPATH.'my_app', FALSE);
$this->load->view('my_app_index'); // Loads
$this->load->view('welcome_message'); // Will not load the default welcome_message b/c the second param to add_package_path is FALSE

// Reset things
$this->load->remove_package_path(APPPATH.'my_app');

// Again without the second parameter:
$this->load->add_package_path(APPPATH.'my_app');
$this->load->view('my_app_index'); // Loads
$this->load->view('welcome_message'); // Loads

library($library[, $params = NULL[, $object_name = NULL]])¶
Parâmetros:
  • $library (mixed) - Nome da Library como uma string ou uma matriz com várias Librarys
  • $params (array) - Opcional matriz de parâmetros para passar para o construtor da Library carregada
  • $object_name (string) - Nome do objeto opcional para atribuir a Library a
Retorna:

Instância CI_Loader (encadeamento do método)
Tipo de retorno:

CI_Loader

Este método é usado para carregar classes principais.

Nota

Usamos os termos "classe" e "library" de maneira intercambiável.

Por exemplo, se você quiser enviar e-mails com o CodeIgniter, o primeiro passo é carregar a classe de email dentro do seu Controller:

$this->load->library('email');

Once loaded, the library will be ready for use, using $this->email.

Arquivos de Library podem ser armazenados em subdiretórios dentro do Diretório "libraries", ou dentro do seu diretório application/libraries diretório. Para carregar um arquivo localizado em um subdiretório, simplesmente inclua o caminho, relativo ao diretório "libraries". Por exemplo, se você tem arquivo localizado em:

libraries/flavors/Chocolate.php

Você irá carregá-lo usando:

$this->load->library('flavors/chocolate');

Você pode aninhar o arquivo em quantos subdiretórios desejar.

Além disso, várias librarys podem ser carregadas ao mesmo tempo passando uma matriz de librarys para o método load.

$this->load->library(array('email', 'table'));

Opções de configuração

O segundo parâmetro (opcional) permite que você passe opcionalmente configuração. Você normalmente passará isso como uma matriz:

$config = array (
        'mailtype' => 'html',
        'charset'  => 'utf-8',
        'priority' => '1'
);

$this->load->library('email', $config);

As opções de configuração geralmente também podem ser definidas por meio de um arquivo de configuração. Cada Library é explicado em detalhes em sua própria página, então por favor leia as informações em relação a cada um que você gostaria de usar.

Por favor, tome nota, quando várias Librarys são fornecidas em uma matriz para o primeiro parâmetro, cada um receberá a mesma informação de parâmetro.

Atribuindo uma Library a um nome de objeto diferente

Se o terceiro parâmetro (opcional) estiver em branco, a Library geralmente será atribuído a um objeto com o mesmo nome da library. Por exemplo, se a Library é chamada Calendário, ela será atribuída a uma variável chamada $this->calendar.

Se você preferir definir seus próprios nomes de classe, pode passar seu valor para o terceiro parâmetro:

$this->load->library('calendar', NULL, 'my_calendar');

// Calendar class is now accessed using:
$this->my_calendar

Por favor, tome nota, quando várias Librarys são fornecidas em uma matriz para o primeiro parâmetro, este parâmetro é descartado.

driver($library[, $params = NULL[, $object_name]])¶
Parâmetros:
  • $library (mixed) - Nome da Library como uma string ou uma matriz com várias Librarys
  • $params (array) - Opcional matriz de parâmetros para passar para o construtor da Library carregada
  • $object_name (string) - Nome do objeto opcional para atribuir a Library a
Retorna:

Instância CI_Loader (encadeamento do método)
Tipo de retorno:

CI_Loader

Este método é usado para carregar Librarys de drivers, age muito library() method.

Por exemplo, se você gostaria de usar sessões com o CodeIgniter, o primeiro passo é carregar o driver da sessão dentro do seu Controller:

$this->load->driver('session');

Once loaded, the library will be ready for use, using $this->session.

Os arquivos do driver devem ser armazenados em um subdiretório dentro do Diretório "libraries", ou dentro do seu diretório application/libraries diretório. O subdiretório deve corresponder ao nome da classe pai. Leia o Drivers descrição para detalhes.

Além disso, várias Librarys de drivers podem ser carregadas ao mesmo passando uma matriz de drivers para o método load.

$this->load->driver(array('session', 'cache'));

Opções de configuração

O segundo parâmetro (opcional) permite que você passe opcionalmente definições de configuração. Você normalmente passará isso como uma matriz:

$config = array(
        'sess_driver' => 'cookie',
        'sess_encrypt_cookie'  => true,
        'encryption_key' => 'mysecretkey'
);

$this->load->driver('session', $config);

As opções de configuração geralmente também podem ser definidas por meio de um arquivo de configuração. Cada Library é explicado em detalhes em sua própria página, então por favor leia as informações em relação a cada um que você gostaria de usar.

Atribuindo um driver a um nome de objeto diferente

Se o terceiro parâmetro (opcional) estiver em branco, a Library será atribuída para um objeto com o mesmo nome da classe pai. Por exemplo, se a library é denominada Session, será atribuída a uma variável chamada $this->session.

Se você preferir definir seus próprios nomes de classe, pode passar seu valor para o terceiro parâmetro:

$this->load->library('session', '', 'my_session');

// Session class is now accessed using:
$this->my_session
view($view[, $vars = array()[, return = FALSE]])¶
Parâmetros:
  • $view (string) - Ver nome
  • $vars (array) - Uma matriz associativa de variáveis
  • $return (bool) - Se deve devolver a vista carregada
Retorna:

Exibir a sequência de conteúdo se $return estiver definido como TRUE, caso contrário, a ocorrência de CI_Loader (encadeamento do método)
Tipo de retorno:

mixed

This method is used to load your ver arquivos. If you haven’t read the Views seção do guia do usuário é recomendado que você faz, uma vez que mostra como este método é normalmente usado.

O primeiro parâmetro é obrigatório. É o nome do arquivo de View que você gostaria de carregar.

Nota

A extensão de arquivo .php não precisa ser especificada a menos que você usa algo diferente de .php.

O segundo optional parâmetro pode ter uma matriz associativa ou um objeto como entrada, que é executado através do PHP extract() função para converter em variáveis que pode ser usado em seus arquivos de visualização. Mais uma vez, leia o Views página para saber como isso pode ser útil.

O terceiro optional parâmetro permite alterar o comportamento do método para que ele retorna dados como uma string em vez de enviá-lo para seu navegador. Isso pode ser útil se você quiser processar os dados em alguns caminho. Se você definir o parâmetro como TRUE (booleano), ele retornará os dados. o o comportamento padrão é FALSE, que envia para o seu navegador. Lembrar de atribuí-lo a uma variável se você quiser que os dados retornados:

$string = $this->load->view('myfile', '', TRUE);

Parâmetros:

Nome da propriedade singleton se for encontrado, FALSE se não for

mixed

Permite que você verifique se uma classe já foi carregada ou não.

A palavra "classe" aqui se refere a Librarys e drivers.

Se a classe solicitada foi carregada, o método retorna sua atribuição nome no Superobjeto de CI e FALSE se não for:

$this->load->library('form_validation');

$this->load->library('form_validation', $config, 'fv');

Classe Loader¶

Loader, como o nome sugere, é usado para carregar elementos. Esses elementos podem ser Librarys (classes) View files, Drivers, Helpers, Modelsou seus próprios arquivos.

Nota

Esta classe é inicializada automaticamente pelo sistema para que Não é necessário fazer isso manualmente.

  • Aplicação "Pacotes"
    • Arquivos de visualização de pacote
  • Referência de Classe

Aplicação "Pacotes"¶

Um pacote de aplicativos permite a fácil distribuição de conjuntos completos de recursos em um único diretório, completo com suas próprias librarys, Models, helpers, config e arquivos de idioma. É recomendado que esses pacotes podem ser colocados no diretório application/third_party. Abaixo é um mapa de amostra de um diretório de pacotes.

A seguir, um exemplo de um diretório para um pacote de aplicativos chamado "Foo Bar".

/application/third_party/foo_bar

config/
helpers/
language/
libraries/
models/

Qualquer que seja a finalidade do pacote de aplicativos "Foo Bar", ele tem próprios arquivos de configuração, Helpers, arquivos de idiomas, Librarys e Models. Usar esses recursos em seus Controlleres, você primeiro precisa informar ao Loader você estará carregando recursos de um pacote, adicionando o package path via the add_package_path() method.

Arquivos de visualização de pacote¶

Por padrão, os caminhos dos arquivos de exibição de pacote são definidos add_package_path() é chamado. Os caminhos de visualização são percorridos e uma vez que uma correspondência é encontrou essa View é carregada.

Neste caso, é possível visualizar colisões de nomes dentro pacotes a ocorrer e possivelmente o pacote incorreto sendo carregado. Para garantir contra isso, defina um segundo parâmetro opcional de FALSE quando calling add_package_path().

$this->load->add_package_path(APPPATH.'my_app', FALSE);
$this->load->view('my_app_index'); // Loads
$this->load->view('welcome_message'); // Will not load the default welcome_message b/c the second param to add_package_path is FALSE

// Reset things
$this->load->remove_package_path(APPPATH.'my_app');

// Again without the second parameter:
$this->load->add_package_path(APPPATH.'my_app');
$this->load->view('my_app_index'); // Loads
$this->load->view('welcome_message'); // Loads

library($library[, $params = NULL[, $object_name = NULL]])¶
Parâmetros:
  • $library (mixed) - Nome da Library como uma string ou uma matriz com várias Librarys
  • $params (array) - Opcional matriz de parâmetros para passar para o construtor da Library carregada
  • $object_name (string) - Nome do objeto opcional para atribuir a Library a
Retorna:

Instância CI_Loader (encadeamento do método)
Tipo de retorno:

CI_Loader

Este método é usado para carregar classes principais.

Nota

Usamos os termos "classe" e "library" de maneira intercambiável.

Por exemplo, se você quiser enviar e-mails com o CodeIgniter, o primeiro passo é carregar a classe de email dentro do seu Controller:

$this->load->library('email');

Once loaded, the library will be ready for use, using $this->email.

Arquivos de Library podem ser armazenados em subdiretórios dentro do Diretório "libraries", ou dentro do seu diretório application/libraries diretório. Para carregar um arquivo localizado em um subdiretório, simplesmente inclua o caminho, relativo ao diretório "libraries". Por exemplo, se você tem arquivo localizado em:

libraries/flavors/Chocolate.php

Você irá carregá-lo usando:

$this->load->library('flavors/chocolate');

Você pode aninhar o arquivo em quantos subdiretórios desejar.

Além disso, várias librarys podem ser carregadas ao mesmo tempo passando uma matriz de librarys para o método load.

$this->load->library(array('email', 'table'));

Opções de configuração

O segundo parâmetro (opcional) permite que você passe opcionalmente configuração. Você normalmente passará isso como uma matriz:

$config = array (
        'mailtype' => 'html',
        'charset'  => 'utf-8',
        'priority' => '1'
);

$this->load->library('email', $config);

As opções de configuração geralmente também podem ser definidas por meio de um arquivo de configuração. Cada Library é explicado em detalhes em sua própria página, então por favor leia as informações em relação a cada um que você gostaria de usar.

Por favor, tome nota, quando várias Librarys são fornecidas em uma matriz para o primeiro parâmetro, cada um receberá a mesma informação de parâmetro.

Atribuindo uma Library a um nome de objeto diferente

Se o terceiro parâmetro (opcional) estiver em branco, a Library geralmente será atribuído a um objeto com o mesmo nome da library. Por exemplo, se a Library é chamada Calendário, ela será atribuída a uma variável chamada $this->calendar.

Se você preferir definir seus próprios nomes de classe, pode passar seu valor para o terceiro parâmetro:

$this->load->library('calendar', NULL, 'my_calendar');

// Calendar class is now accessed using:
$this->my_calendar

Por favor, tome nota, quando várias Librarys são fornecidas em uma matriz para o primeiro parâmetro, este parâmetro é descartado.

driver($library[, $params = NULL[, $object_name]])¶
Parâmetros:
  • $library (mixed) - Nome da Library como uma string ou uma matriz com várias Librarys
  • $params (array) - Opcional matriz de parâmetros para passar para o construtor da Library carregada
  • $object_name (string) - Nome do objeto opcional para atribuir a Library a
Retorna:

Instância CI_Loader (encadeamento do método)
Tipo de retorno:

CI_Loader

Este método é usado para carregar Librarys de drivers, age muito library() method.

Por exemplo, se você gostaria de usar sessões com o CodeIgniter, o primeiro passo é carregar o driver da sessão dentro do seu Controller:

$this->load->driver('session');

Once loaded, the library will be ready for use, using $this->session.

Os arquivos do driver devem ser armazenados em um subdiretório dentro do Diretório "libraries", ou dentro do seu diretório application/libraries diretório. O subdiretório deve corresponder ao nome da classe pai. Leia o Drivers descrição para detalhes.

Além disso, várias Librarys de drivers podem ser carregadas ao mesmo passando uma matriz de drivers para o método load.

$this->load->driver(array('session', 'cache'));

Opções de configuração

O segundo parâmetro (opcional) permite que você passe opcionalmente definições de configuração. Você normalmente passará isso como uma matriz:

$config = array(
        'sess_driver' => 'cookie',
        'sess_encrypt_cookie'  => true,
        'encryption_key' => 'mysecretkey'
);

$this->load->driver('session', $config);

As opções de configuração geralmente também podem ser definidas por meio de um arquivo de configuração. Cada Library é explicado em detalhes em sua própria página, então por favor leia as informações em relação a cada um que você gostaria de usar.

Atribuindo um driver a um nome de objeto diferente

Se o terceiro parâmetro (opcional) estiver em branco, a Library será atribuída para um objeto com o mesmo nome da classe pai. Por exemplo, se a library é denominada Session, será atribuída a uma variável chamada $this->session.

Se você preferir definir seus próprios nomes de classe, pode passar seu valor para o terceiro parâmetro:

$this->load->library('session', '', 'my_session');

// Session class is now accessed using:
$this->my_session
view($view[, $vars = array()[, return = FALSE]])¶
Parâmetros:
  • $view (string) - Ver nome
  • $vars (array) - Uma matriz associativa de variáveis
  • $return (bool) - Se deve devolver a vista carregada
Retorna:

Exibir a sequência de conteúdo se $return estiver definido como TRUE, caso contrário, a ocorrência de CI_Loader (encadeamento do método)
Tipo de retorno:

mixed

Este método é usado para carregar seus arquivos do View. Se você ainda não leu Views seção do guia do usuário é recomendado que você faz, uma vez que mostra como este método é normalmente usado.

O primeiro parâmetro é obrigatório. É o nome do arquivo de View que você gostaria de carregar.

Nota

A extensão de arquivo .php não precisa ser especificada a menos que você usa algo diferente de .php.

The second optional parâmetro pode ter uma matriz associativa ou um objeto como entrada, que é executado através do PHP extract() função para converter em variáveis que pode ser usado em seus arquivos de visualização. Mais uma vez, leia o Views página para saber como isso pode ser útil.

The third optional parâmetro permite alterar o comportamento do método para que ele retorna dados como uma string em vez de enviá-lo para seu navegador. Isso pode ser útil se você quiser processar os dados em alguns caminho. Se você definir o parâmetro como TRUE (booleano), ele retornará os dados. o o comportamento padrão é FALSE, que envia para o seu navegador. Lembrar de atribuí-lo a uma variável se você quiser que os dados retornados:

$string = $this->load->view('myfile', '', TRUE);

Retorna:

Esse método usa um array associativo como entrada e gera variáveis usando o PHP extract() função. Parâmetros:

Retorna:

Nome da propriedade singleton se for encontrado, FALSE se não for

mixed

Permite que você verifique se uma classe já foi carregada ou não.

A palavra "classe" aqui se refere a Librarys e drivers.
