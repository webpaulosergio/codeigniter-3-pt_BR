Classe de calendários¶

The Calendar class enables you to dynamically create calendars. Your calendars can be formatted through the use of a calendar template, allowing 100% control over every aspect of its design. In addition, you can pass data to your calendar cells.

  • Using the Classe de calendários
    • Inicializando a classe
    • Displaying a Calendar
    • Passando dados para suas células do calendário
    • Setting Display Preferências
    • Showing Next/Previous Month Links
    • Creating a Calendar Template
  • Class Reference

Using the Classe de calendários¶

Inicializando a classe¶

Like most other classes in CodeIgniter, the Calendar class is initialized in your controller using the $this->load->library function:

$this->load->library('calendar');

Depois de carregado, o objeto Calendário estará disponível usando:

$this->calendar

Displaying a Calendar¶

Here is a very simple example showing how you can display a calendar:

$this->load->library('calendar');
echo $this->calendar->generate();

The above code will generate a calendar for the current month/year based no horário do seu servidor. Para mostrar um calendário para um mês e ano específicos you will pass this information to the calendar generating function:

$this->load->library('calendar');
echo $this->calendar->generate(2006, 6);

The above code will generate a calendar showing the month of June in 2006. O primeiro parâmetro especifica o ano, o segundo parâmetro especifica o mês.

Passando dados para suas células do calendário¶

To add data to your calendar cells involves creating an associative array in which the keys correspond to the days you wish to populate and o valor da matriz contém os dados. A matriz é passada para o terceiro parâmetro da função de geração de calendário. Considere este exemplo:

$this->load->library('calendar');

$data = array(
        3  => 'http://example.com/news/article/2006/06/03/',
        7  => 'http://example.com/news/article/2006/06/07/',
        13 => 'http://example.com/news/article/2006/06/13/',
        26 => 'http://example.com/news/article/2006/06/26/'
);

echo $this->calendar->generate(2006, 6, $data);

Usando o exemplo acima, os números dos dias 3, 7, 13 e 26 se tornarão links pointing to the URLs you’ve provided.

Nota

By default it is assumed that your array will contain links. Na seção que explica o modelo de calendário abaixo, você verá como you can customize how data passed to your cells is handled so you can pass different types of information.

Setting Display Preferências¶

There are seven preferences you can set to control various aspects of the calendar. Preferências are set by passing an array of preferences in o segundo parâmetro da função de carregamento. Aqui está um exemplo:

$prefs = array(
        'start_day'    => 'saturday',
        'month_type'   => 'long',
        'day_type'     => 'short'
);

$this->load->library('calendar', $prefs);

echo $this->calendar->generate();

The above code would start the calendar on saturday, use the “long” month heading, and the “short” day names. More information regarding preferences below.

Preferência Padrão Opções Descrição
template None None
A string or array containing your calendar template.
See the template section below.
local_time time() None A Unix timestamp corresponding to the current time.
start_day sunday Qualquer dia da semana (domingo, segunda-feira, terça-feira, etc.) Sets the day of the week the calendar should start on.
month_type long longo, curto
Determina qual versão do nome do mês deve ser usada no cabeçalho.
long = janeiro, short = jan.
day_type abr longo, curto, abr
Determines what version of the weekday names to use in
os cabeçalhos das colunas. long = domingo, short = sol, abr = su.
show_next_prev FALSE TRUE/FALSE (boolean)
Determina se deseja exibir links que permitem alternar
to next/previous months. See information on this feature below.
next_prev_url controller/method A URL Sets the basepath used in the next/previous calendar links.
show_other_days FALSE TRUE/FALSE (boolean)
Determines whether to display days of other months that share the
first or last week of the calendar month.

Showing Next/Previous Month Links¶

Para permitir que seu calendário seja incrementado/decrementado dinamicamente next/previous links requires that you set up your calendar code similar para este exemplo:

$prefs = array(
        'show_next_prev'  => TRUE,
        'next_prev_url'   => 'http://example.com/index.php/calendar/show/'
);

$this->load->library('calendar', $prefs);

echo $this->calendar->generate($this->uri->segment(3), $this->uri->segment(4));

You’ll notice a few things about the above example:

  • You must set the “show_next_prev” to TRUE.
  • You must supply the URL to the controller containing your calendar in a preferência "next_prev_url". Se você não fizer isso, ele será definido para o atual controller/method.
  • You must supply the “year” and “month” to the calendar generating função através dos segmentos URI onde aparecem (Nota: O calendário classe adiciona automaticamente o ano/mês para o URL base você provide.).

Creating a Calendar Template¶

By creating a calendar template you have 100% control over the design of your calendar. Using the string method, each component of your calendar será colocado dentro de um par de pseudo-variáveis, como mostrado aqui:

$prefs['template'] = '

        {table_open}<table border="0" cellpadding="0" cellspacing="0">{/table_open}

        {heading_row_start}<tr>{/heading_row_start}

        {heading_previous_cell}<th><a href="{previous_url}">&lt;&lt;</a></th>{/heading_previous_cell}
        {heading_title_cell}<th colspan="{colspan}">{heading}</th>{/heading_title_cell}
        {heading_next_cell}<th><a href="{next_url}">&gt;&gt;</a></th>{/heading_next_cell}

        {heading_row_end}</tr>{/heading_row_end}

        {week_row_start}<tr>{/week_row_start}
        {week_day_cell}<td>{week_day}</td>{/week_day_cell}
        {week_row_end}</tr>{/week_row_end}

        {cal_row_start}<tr>{/cal_row_start}
        {cal_cell_start}<td>{/cal_cell_start}
        {cal_cell_start_today}<td>{/cal_cell_start_today}
        {cal_cell_start_other}<td class="other-month">{/cal_cell_start_other}

        {cal_cell_content}<a href="{content}">{day}</a>{/cal_cell_content}
        {cal_cell_content_today}<div class="highlight"><a href="{content}">{day}</a></div>{/cal_cell_content_today}

        {cal_cell_no_content}{day}{/cal_cell_no_content}
        {cal_cell_no_content_today}<div class="highlight">{day}</div>{/cal_cell_no_content_today}

        {cal_cell_blank}&nbsp;{/cal_cell_blank}

        {cal_cell_other}{day}{/cal_cel_other}

        {cal_cell_end}</td>{/cal_cell_end}
        {cal_cell_end_today}</td>{/cal_cell_end_today}
        {cal_cell_end_other}</td>{/cal_cell_end_other}
        {cal_row_end}</tr>{/cal_row_end}

        {table_close}</table>{/table_close}
';

$this->load->library('calendar', $prefs);

echo $this->calendar->generate();

Usando o método array, você passará key => value pairs. You can pass as many or as few values as you’d like. Omitted keys will use the default values inherited in the calendar class.

Example:

$prefs['template'] = array(
        'table_open'           => '<table class="calendar">',
        'cal_cell_start'       => '<td class="day">',
        'cal_cell_start_today' => '<td class="today">'
);

$this->load->library('calendar', $prefs);

echo $this->calendar->generate();

Classe de calendários¶

A classe Calendar permite que você crie dinamicamente calendários. Seu os calendários podem ser formatados através do uso de um Model de calendário, permitindo 100% de controle sobre todos os aspectos de seu design. Além disso, você pode passar dados para as células do seu calendário.

Using the Classe de calendários¶

Inicializando a classe¶

Como a maioria das outras classes no CodeIgniter, a classe Calendar é inicializado em seu Controller usando a função $this-> load-> library:

$this->load->library('calendar');

Depois de carregado, o objeto Calendário estará disponível usando:

$this->calendar

Exibindo um calendário¶

Aqui está um exemplo muito simples mostrando como você pode exibir um calendário:

$this->load->library('calendar');
echo $this->calendar->generate();

O código acima irá gerar um calendário para o mês atual/ano com base no horário do seu servidor. Para mostrar um calendário para um mês e ano específicos você passará esta informação para a função de geração de calendário:

Passando dados para suas células do calendário¶

Adicionar dados às células do seu calendário envolve a criação de uma associação matriz em que as chaves correspondem aos dias que você deseja preencher e o valor da matriz contém os dados. A matriz é passada para o terceiro parâmetro da função de geração de calendário. Considere este exemplo:

$this->load->library('calendar');

$data = array(
        3  => 'http://example.com/news/article/2006/06/03/',
        7  => 'http://example.com/news/article/2006/06/07/',
        13 => 'http://example.com/news/article/2006/06/13/',
        26 => 'http://example.com/news/article/2006/06/26/'
);

echo $this->calendar->generate(2006, 6, $data);

Usando o exemplo acima, os números dos dias 3, 7, 13 e 26 se tornarão links apontando para os URLs que você forneceu.

Nota

Por padrão, é assumido que sua matriz conterá links. Na seção que explica o Model de calendário abaixo, você verá como você pode personalizar como os dados passados para suas células são manipulados para que você possa passar diferentes tipos de informação.

Definir preferências de exibição¶

Existem sete preferências que você pode definir para controlar vários aspectos o calendário. As preferências são definidas passando uma matriz de preferências em o segundo parâmetro da função de carregamento. Aqui está um exemplo:

$prefs = array(
        'start_day'    => 'saturday',
        'month_type'   => 'long',
        'day_type'     => 'short'
);

$this->load->library('calendar', $prefs);

echo $this->calendar->generate();

O código acima iniciaria o calendário no sábado, use o "long" cabeçalho do mês e os nomes dos dias "curtos". Mais informações sobre preferências abaixo.

        {heading_row_end}</tr>{/heading_row_end}

        {week_row_start}<tr>{/week_row_start}
        {week_day_cell}<td>{week_day}</td>{/week_day_cell}
        {week_row_end}</tr>{/week_row_end}

        {cal_cell_content_today}<div class="highlight"><a href="{content}">{day}</a></div>{/cal_cell_content_today}

        {cal_cell_no_content}{day}{/cal_cell_no_content}

        {cal_cell_end_other}</td>{/cal_cell_end_other}

Exemplo:

$prefs['template'] = array(
        'table_open'           => '<table class="calendar">',
        'cal_cell_start'       => '<td class="day">',
        'cal_cell_start_today' => '<td class="today">'
);

$this->load->library('calendar', $prefs);

echo $this->calendar->generate();

        {cal_cell_end_other}</td>{/cal_cell_end_other}
        {cal_row_end}</tr>{/cal_row_end}

        {table_close}</table>{/table_close}
';

$this->load->library('calendar', $prefs);

echo $this->calendar->generate();

Usando o método array, você passará key => value pares. Você pode passar como muitos ou quantos valores você desejar. Chaves omitidas usarão os valores padrão herdado na classe do calendário.

Exemplo:

$prefs['template'] = array(
        'table_open'           => '<table class="calendar">',
        'cal_cell_start'       => '<td class="day">',
        'cal_cell_start_today' => '<td class="today">'
);

$this->load->library('calendar', $prefs);

echo $this->calendar->generate();

adjust_date($month, $year)¶
Parâmetros:
  • $month (int) – Month
  • $year (int) – Year
Retorna:

Uma matriz associativa contendo mês e ano
Tipo de retorno:

array

Este método garante que você tenha um mês/ano válido. Por exemplo, se você envia 13 como o mês, o ano vai aumentar e o mês vai tornar-se janeiro:

print_r($this->calendar->adjust_date(13, 2014));

outputs:

Array
(
        [month] => '01'
        [year] => '2015'
)
