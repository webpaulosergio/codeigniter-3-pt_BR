Classe de Trackback¶

The Classe de Trackback provides functions that enable you to send and receive Trackback data.

If you are not familiar with Trackbacks you’ll find more information here.

Using the Classe de Trackback¶

Inicializando a classe¶

Como a maioria das outras classes no CodeIgniter, a classe Trackback é initialized in your controller using the $this->load->library() method:

$this->load->library('trackback');

Once loaded, the Trackback library object will be available using:

$this->trackback

Envio de Trackbacks¶

Um Trackback pode ser enviado de qualquer uma das suas funções de controlador usando código semelhante a este exemplo:

$this->load->library('trackback');

$tb_data = array(
        'ping_url'  => 'http://example.com/trackback/456',
        'url'       => 'http://www.my-example.com/blog/entry/123',
        'title'     => 'The Title of My Entry',
        'excerpt'   => 'The entry content.',
        'blog_name' => 'My Blog Name',
        'charset'   => 'utf-8'
);

if ( ! $this->trackback->send($tb_data))
{
        echo $this->trackback->display_errors();
}
else
{
        echo 'Trackback was sent!';
}

Descrição dos dados da matriz:

  • ping_url - The URL of the site you are sending the Trackback to. Você pode enviar Trackbacks para vários URLs, separando cada URL com uma vírgula.
  • url - O URL para o seu site onde a entrada do blog pode ser vista.
  • title - The title of your weblog entry.
  • excerpt - The content of your weblog entry.
  • blog_name - O nome do seu blog.
  • charset - The character encoding your weblog is written in. If omitted, UTF-8 will be used.

Nota

A classe Trackback enviará automaticamente apenas os primeiros 500 caracteres do seu entrada. Ele também tira todo o HTML.

The Trackback sending method returns TRUE/FALSE (boolean) on success or failure. If it fails, you can retrieve the error message using:

$this->trackback->display_errors();

Recebendo Trackbacks¶

Before you can receive Trackbacks you must create a weblog. If you don’t have a blog yet there’s no point in continuing.

Recebendo Trackbacks is a little more complex than sending them, only because you will need a database table in which to store them, and you will need to validate the incoming trackback data. You are encouraged to implement a thorough validation process to guard against spam and dados duplicados. Você também pode querer limitar o número de Trackbacks permitir a partir de um determinado IP dentro de um determinado período de tempo para mais reduzir o spam. O processo de receber um Trackback é bastante simples; a Validação é o que leva a maior parte do esforço.

Your Ping URL¶

In order to accept Trackbacks you must display a Trackback URL next to cada uma das entradas do seu blog. Este será o URL que as pessoas vão use to send you Trackbacks (we will refer to this as your “Ping URL”).

Your Ping URL must point to a controller function where your Trackback receiving code is located, and the URL must contain the ID number for each particular entry, so that when the Trackback is received you’ll be able to associate it with a particular entry.

Por exemplo, se sua classe de controlador é chamada de Trackback, e receiving function is called receive, your Ping URLs will look something like this:

http://example.com/index.php/trackback/receive/entry_id

Where entry_id represents the individual ID number for each of your entries.

Criando uma Tabela de Trackback¶

Before you can receive Trackbacks you must create a table in which to store them. Here is a basic prototype for such a table:

CREATE TABLE trackbacks (
        tb_id int(10) unsigned NOT NULL auto_increment,
        entry_id int(10) unsigned NOT NULL default 0,
        url varchar(200) NOT NULL,
        title varchar(100) NOT NULL,
        excerpt text NOT NULL,
        blog_name varchar(100) NOT NULL,
        tb_date int(10) NOT NULL,
        ip_address varchar(45) NOT NULL,
        PRIMARY KEY `tb_id` (`tb_id`),
        KEY `entry_id` (`entry_id`)
);

The Trackback specification only requires four pieces of information to be sent in a Trackback (url, title, excerpt, blog_name), but to make os dados mais úteis nós adicionamos mais alguns campos na tabela acima esquema (data, endereço IP, etc.).

Processando um Trackback¶

Here is an example showing how you will receive and process a Trackback. O código a seguir é destinado para uso dentro da função do controlador where you expect to receive Trackbacks.:

$this->load->library('trackback');
$this->load->database();

if ($this->uri->segment(3) == FALSE)
{
        $this->trackback->send_error('Unable to determine the entry ID');
}

if ( ! $this->trackback->receive())
{
        $this->trackback->send_error('The Trackback did not contain valid data');
}

$data = array(
        'tb_id'      => '',
        'entry_id'   => $this->uri->segment(3),
        'url'        => $this->trackback->data('url'),
        'title'      => $this->trackback->data('title'),
        'excerpt'    => $this->trackback->data('excerpt'),
        'blog_name'  => $this->trackback->data('blog_name'),
        'tb_date'    => time(),
        'ip_address' => $this->input->ip_address()
);

$sql = $this->db->insert_string('trackbacks', $data);
$this->db->query($sql);

$this->trackback->send_success();

Notas:¶

The entry ID number is expected in the third segment of your URL. This is based on the URI example we gave earlier:

http://example.com/index.php/trackback/receive/entry_id

Notice the entry_id is in the third URI segment, which you can retrieve using:

$this->uri->segment(3);

In our Trackback receiving code above, if the third segment is missing, nós emitiremos um erro. Sem um ID de entrada válido, não há motivo para continue.

The $this->trackback->receive() function is simply a validation function that looks at the incoming data and makes sure it contains the four pedaços de dados que são necessários (url, título, trecho, blog_name). isto retorna TRUE no sucesso e FALSE na falha. Se falhar, você emitirá uma mensagem de erro.

The incoming Trackback data can be retrieved using this function:

$this->trackback->data('item')

Onde item representa uma dessas quatro informações: url, title, trecho ou blog_name

If the Trackback data is successfully received, you will issue a success message using:

$this->trackback->send_success();

Nota

The above code contains no data validation, which you are encouraged to add.

$this->load->library('trackback');

$tb_data = array(

  • url - O URL para o seu site onde a entrada do blog pode ser vista.
  • title - O título da sua entrada no blog.
  • excerpt - O conteúdo da sua entrada no blog.
  • blog_name - O nome do seu blog.

A classe Trackback enviará automaticamente apenas os primeiros 500 caracteres do seu entrada. Ele também tira todo o HTML.

$this->trackback->display_errors();

Antes de poder receber Trackbacks, você deve criar um weblog. Se você não tenha um blog ainda não há nenhum ponto em continuar.

Recebendo Trackbacks is a little more complex than sending them, only porque você precisará de uma tabela de Database para armazená-los e precisará validar os dados de trackback recebidos. Você é encorajado a implementar um processo de validação completo para proteger contra spam e dados duplicados. Você também pode querer limitar o número de Trackbacks permitir a partir de um determinado IP dentro de um determinado período de tempo para mais reduzir o spam. O processo de receber um Trackback é bastante simples; a Validação é o que leva a maior parte do esforço.

Seu URL de Ping¶

Para aceitar os Trackbacks, você deve exibir um URL de Trackback ao lado de cada uma das entradas do seu blog. Este será o URL que as pessoas vão usar para enviar Trackbacks (vamos nos referir a isso como seu "URL Ping").

Seu URL de Ping must point to a controller function where your Trackback O código de recepção está localizado e o URL deve conter o número de identificação para cada entrada em particular, para que quando o Trackback for recebido, você capaz de associá-lo a uma entrada específica.

Por exemplo, se sua classe de Controller é chamada de Trackback, e função de recebimento é chamada de receber, seus URLs de ping vão procurar algo como isso:

http://example.com/index.php/trackback/receive/entry_id

Onde entry_id representa o número de identificação individual de cada um dos seus entries.

Antes de poder receber Trackbacks, você deve criar uma tabela na qual guarde-os. Aqui está um protótipo básico para essa tabela:

CREATE TABLE trackbacks (
        tb_id int(10) unsigned NOT NULL auto_increment,
        entry_id int(10) unsigned NOT NULL default 0,
        url varchar(200) NOT NULL,
        title varchar(100) NOT NULL,
        excerpt text NOT NULL,
        blog_name varchar(100) NOT NULL,
        tb_date int(10) NOT NULL,
        ip_address varchar(45) NOT NULL,

        KEY `entry_id` (`entry_id`)

A especificação do Trackback requer apenas quatro informações para ser enviado em um Trackback (url, título, trecho, blog_name), mas para fazer os dados mais úteis nós adicionamos mais alguns campos na tabela acima esquema (data, endereço IP, etc.).

Processando um Trackback¶

Aqui está um exemplo mostrando como você receberá e processará um Trackback. O código a seguir é destinado para uso dentro da função do Controller onde você espera receber Trackbacks .:

$this->load->library('trackback');
$this->load->database();

if ($this->uri->segment(3) == FALSE)
{
        $this->trackback->send_error('Unable to determine the entry ID');
}

if ( ! $this->trackback->receive())
{
        $this->trackback->send_error('The Trackback did not contain valid data');
}

$data = array(
        'tb_id'      => '',
        'entry_id'   => $this->uri->segment(3),
        'url'        => $this->trackback->data('url'),
        'title'      => $this->trackback->data('title'),
        'excerpt'    => $this->trackback->data('excerpt'),
        'blog_name'  => $this->trackback->data('blog_name'),
        'tb_date'    => time(),
        'ip_address' => $this->input->ip_address()
);

$sql = $this->db->insert_string('trackbacks', $data);
$this->db->query($sql);

$this->trackback->send_success();

Notas:¶

O número do ID de entrada é esperado no terceiro segmento do seu URL. este é baseado no exemplo de URI que demos anteriormente:

http://example.com/index.php/trackback/receive/entry_id

Observe que o entry_id está no terceiro segmento de URI, que você pode recuperar using:

$this->uri->segment(3);

No nosso código de recepção do Trackback acima, se o terceiro segmento estiver faltando, nós emitiremos um erro. Sem um ID de entrada válido, não há motivo para continue.

The $this->trackback->receive() function is simply a validation function que olha para os dados recebidos e garante que contém os quatro pedaços de dados que são necessários (url, título, trecho, blog_name). isto retorna TRUE no sucesso e FALSE na falha. Se falhar, você emitirá uma mensagem de erro.

Os dados do Trackback de entrada podem ser recuperados usando esta função:

$this->trackback->data('item')

Onde item representa uma dessas quatro informações: url, title, trecho ou blog_name

Se os dados do Trackback forem recebidos com sucesso, você emitirá um sucesso mensagem usando:

$this->trackback->send_success();

Nota

O código acima não contém validação de dados, o que você é encorajado a adicionar.

Retorna:Verdadeiro sucesso, FALSE em falha
Tipo de retorno:bool

Esse método simplesmente valida os dados TB recebidos, retornando TRUE no sucesso e FALSE na falha. If the data is valid it is set to the $this->data array so that it can be inserted into a database.
