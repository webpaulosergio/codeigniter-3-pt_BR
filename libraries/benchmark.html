Classe de benchmarking¶

CodeIgniter has a Benchmarking class that is always active, enabling the time difference between any two marked points to be calculated.

Nota

Esta classe é inicializada automaticamente pelo sistema para que is no need to do it manually.

Além disso, o benchmark é sempre iniciado no momento em que o framework é invoked, and ended by the output class right before sending the final view to the browser, enabling a very accurate timing of the entire system execution to be shown.

  • Using the Benchmark Class
    • Profiling Your Benchmark Points
    • Displaying Total Execution Time
    • Displaying Memory Consumption
  • Class Reference

Using the Benchmark Class¶

A classe Benchmark pode ser usada dentro do seu controllers, viewsou o seu models. The process for usage is this:

  1. Mark a start point
  2. Mark an end point
  3. Execute a função "tempo decorrido" para visualizar os resultados

Aqui está um exemplo usando código real:

$this->benchmark->mark('code_start');

// Some code happens here

$this->benchmark->mark('code_end');

echo $this->benchmark->elapsed_time('code_start', 'code_end');

Nota

The words “code_start” and “code_end” are arbitrary. They are simply words used to set two markers. You can use any words you quer, e você pode definir vários conjuntos de marcadores. Considere este exemplo:

$this->benchmark->mark('dog');

// Some code happens here

$this->benchmark->mark('cat');

// More code happens here

$this->benchmark->mark('bird');

echo $this->benchmark->elapsed_time('dog', 'cat');
echo $this->benchmark->elapsed_time('cat', 'bird');
echo $this->benchmark->elapsed_time('dog', 'bird');

Profiling Your Benchmark Points¶

If you want your benchmark data to be available to the Profiler all of your marked points must be set up in pairs, and each mark point name must end with _start and _end. Each pair of points must otherwise be named identically. Example:

$this->benchmark->mark('my_mark_start');

// Some code happens here...

$this->benchmark->mark('my_mark_end');

$this->benchmark->mark('another_mark_start');

// Some more code happens here...

$this->benchmark->mark('another_mark_end');

Por favor, leia o Página de perfil for more information.

Displaying Total Execution Time¶

Se você gostaria de exibir o tempo total decorrido desde o momento O CodeIgniter começa no momento em que a saída final é enviada ao navegador, basta colocar isso em um dos seus modelos de visualização:

<?php echo $this->benchmark->elapsed_time();?>

You’ll notice that it’s the same function used in the examples above to calcular o tempo entre dois pontos, exceto que você é not using any parameters. When the parameters are absent, CodeIgniter does not stop the benchmark until right before the final output is sent to the browser. It doesn’t matter where you use the function call, the timer will continue to run until the very end.

An alternate way to show your elapsed time in your view files is to use this pseudo-variable, if you prefer not to use the pure PHP:

{elapsed_time}

Nota

If you want to benchmark anything within your controller funções você deve definir seus próprios pontos de início/fim.

Displaying Memory Consumption¶

If your PHP installation is configured with –enable-memory-limit, you pode exibir a quantidade de memória consumida por todo o sistema usando o following code in one of your view file:

<?php echo $this->benchmark->memory_usage();?>

Nota

This function can only be used in your view files. The consumption will reflect the total memory used by the entire app.

An alternate way to show your memory usage in your view files is to use this pseudo-variable, if you prefer not to use the pure PHP:

{memory_usage}

Classe de benchmarking¶

O CodeIgniter possui uma classe de Benchmarking que está sempre ativa, permitindo diferença de tempo entre quaisquer dois pontos marcados a serem calculados.

Nota

Esta classe é inicializada automaticamente pelo sistema para que Não é necessário fazer isso manualmente.

Além disso, o benchmark é sempre iniciado no momento em que o framework é invocado e finalizado pela classe de saída antes de enviar a final vista para o navegador, permitindo um timing muito preciso de todo o execução do sistema a ser mostrada.

  • Usando a Classe de Referência
    • Criação de perfil dos seus pontos de referência
    • Exibindo o tempo total de execução
    • Exibindo Consumo de Memória
  • Referência de Classe

Usando a Classe de Referência¶

A classe Benchmark pode ser usada dentro do seu controllers, viewsou o seu models. O processo para uso é este:

  1. Marque um ponto inicial
  2. Marcar um ponto final
  3. Execute a função "tempo decorrido" para visualizar os resultados

Aqui está um exemplo usando código real:

$this->benchmark->mark('code_start');

// Some code happens here

$this->benchmark->mark('code_end');

echo $this->benchmark->elapsed_time('code_start', 'code_end');

Nota

As palavras "code_start" e "code_end" são arbitrárias. Eles são simplesmente palavras usadas para definir dois marcadores. Você pode usar qualquer palavra que quer, e você pode definir vários conjuntos de marcadores. Considere este exemplo:

$this->benchmark->mark('dog');

// Some code happens here

$this->benchmark->mark('cat');

// More code happens here

$this->benchmark->mark('bird');

echo $this->benchmark->elapsed_time('dog', 'cat');
echo $this->benchmark->elapsed_time('cat', 'bird');
echo $this->benchmark->elapsed_time('dog', 'bird');

Criação de perfil dos seus pontos de referência¶

Se você quiser que seus dados de benchmark estejam disponíveis para o perfil todos os seus pontos marcados devem ser configurado em pares, e cada nome de ponto de marca deve terminar com _start e _fim. Cada par de pontos deve ser nomeado de forma idêntica. Exemplo:

$this->benchmark->mark('my_mark_start');

// Some code happens here...

$this->benchmark->mark('my_mark_end');

$this->benchmark->mark('another_mark_start');

// Some more code happens here...

$this->benchmark->mark('another_mark_end');

Por favor, leia o perfil page para mais information.

Exibindo o tempo total de execução¶

Se você gostaria de exibir o tempo total decorrido desde o momento O CodeIgniter começa no momento em que a saída final é enviada ao navegador, basta colocar isso em um dos seus Models de visualização:

<?php echo $this->benchmark->elapsed_time();?>

Você notará que é a mesma função usada nos exemplos acima para calcular o tempo entre dois pontos, exceto que você é not usando qualquer parâmetros. Quando os parâmetros estão ausentes, o CodeIgniter não para o benchmark até logo antes da saída final é enviada para o navegador. Não importa onde você usa a chamada de função, o temporizador continuará a ser executado até o final.

Uma maneira alternativa de mostrar o tempo decorrido em seus arquivos de visualização é usar essa pseudo-variável, se você preferir não usar o PHP puro:

{elapsed_time}

Nota

Se você quiser avaliar qualquer coisa dentro do seu Controller funções você deve definir seus próprios pontos de início/fim.

Exibindo Consumo de Memória¶

Se a sua instalação do PHP estiver configurada com enable-memory-limit, você pode exibir a quantidade de memória consumida por todo o sistema usando o código a seguir em um dos seus arquivos de visualização:

<?php echo $this->benchmark->memory_usage();?>

Nota

Esta função só pode ser usada em seus arquivos de View. O consumo refletirá a memória total usada por todo o aplicativo.

Uma maneira alternativa de mostrar seu uso de memória em seus arquivos de visualização é usar essa pseudo-variável, se você preferir não usar o PHP puro:

{memory_usage}

Classe de benchmarking¶

O CodeIgniter possui uma classe de Benchmarking que está sempre ativa, permitindo diferença de tempo entre quaisquer dois pontos marcados a serem calculados.

Nota

Esta classe é inicializada automaticamente pelo sistema para que Não é necessário fazer isso manualmente.

Além disso, o benchmark é sempre iniciado no momento em que o framework é invocado e finalizado pela classe de saída antes de enviar a final vista para o navegador, permitindo um timing muito preciso de todo o execução do sistema a ser mostrada.

  • Usando a Classe de Referência
    • Criação de perfil dos seus pontos de referência
    • Exibindo o tempo total de execução
    • Exibindo Consumo de Memória
  • Referência de Classe

Usando a Classe de Referência¶

A classe Benchmark pode ser usada dentro do seu controllers, views, or your models. O processo para uso é este:

  1. Marque um ponto inicial
  2. Marcar um ponto final
  3. Execute a função "tempo decorrido" para visualizar os resultados

Aqui está um exemplo usando código real:

$this->benchmark->mark('code_start');

// Some code happens here

$this->benchmark->mark('code_end');

echo $this->benchmark->elapsed_time('code_start', 'code_end');

Nota

As palavras "code_start" e "code_end" são arbitrárias. Eles são simplesmente palavras usadas para definir dois marcadores. Você pode usar qualquer palavra que quer, e você pode definir vários conjuntos de marcadores. Considere este exemplo:

$this->benchmark->mark('dog');

// Some code happens here

$this->benchmark->mark('cat');

// More code happens here

$this->benchmark->mark('bird');

echo $this->benchmark->elapsed_time('dog', 'cat');
echo $this->benchmark->elapsed_time('cat', 'bird');
echo $this->benchmark->elapsed_time('dog', 'bird');

Criação de perfil dos seus pontos de referência¶

Se você quiser que seus dados de benchmark estejam disponíveis para o Profiler todos os seus pontos marcados devem ser configurado em pares, e cada nome de ponto de marca deve terminar com _start e _fim. Cada par de pontos deve ser nomeado de forma idêntica. Exemplo:

$this->benchmark->mark('my_mark_start');

// Some code happens here...

$this->benchmark->mark('my_mark_end');

$this->benchmark->mark('another_mark_start');

// Some more code happens here...

$this->benchmark->mark('another_mark_end');

Por favor, leia o Página de perfil for more information.

Exibindo o tempo total de execução¶

Se você gostaria de exibir o tempo total decorrido desde o momento O CodeIgniter começa no momento em que a saída final é enviada ao navegador, basta colocar isso em um dos seus Models de visualização:

<?php echo $this->benchmark->elapsed_time();?>

Você notará que é a mesma função usada nos exemplos acima para calcular o tempo entre dois pontos, exceto que você é not using any parâmetros. Quando os parâmetros estão ausentes, o CodeIgniter não para o benchmark até logo antes da saída final é enviada para o navegador. Não importa onde você usa a chamada de função, o temporizador continuará a ser executado até o final.

Uma maneira alternativa de mostrar o tempo decorrido em seus arquivos de visualização é usar essa pseudo-variável, se você preferir não usar o PHP puro:

{elapsed_time}

Nota

Se você quiser avaliar qualquer coisa dentro do seu Controller funções você deve definir seus próprios pontos de início/fim.

Exibindo Consumo de Memória¶

Se a sua instalação do PHP estiver configurada com enable-memory-limit, você pode exibir a quantidade de memória consumida por todo o sistema usando o código a seguir em um dos seus arquivos de visualização:

<?php echo $this->benchmark->memory_usage();?>

Nota

Esta função só pode ser usada em seus arquivos de View. O consumo refletirá a memória total usada por todo o aplicativo.

Uma maneira alternativa de mostrar seu uso de memória em seus arquivos de visualização é usar essa pseudo-variável, se você preferir não usar o PHP puro:

{memory_usage}

Referência de Classe¶

class CI_Benchmark¶
mark($name)¶
Parâmetros:
  • $name (string) - o nome que você deseja atribuir ao seu marcador
Tipo de retorno:

void

Define um marcador de referência.

elapsed_time([$point1 = ''[, $point2 = ''[, $decimals = 4]]])¶
Parâmetros:
  • $point1 (string) - um ponto marcado particular
  • $point2 (string) - um ponto marcado particular
  • $decimals (int) - número de casas decimais para precisão
Retorna:

Tempo gasto
Tipo de retorno:

string

Calcula e retorna a diferença de tempo entre dois pontos marcados.

Se o primeiro parâmetro estiver vazio, esta função retorna o {elapsed_time} pseudo-variável. Isso permite que o sistema completo tempo de execução a ser mostrado em um Model. A classe de saída Troque o valor real por esta variável.

memory_usage()¶
Retorna:Informações de uso da memória
Tipo de retorno:string

Simply returns the {memory_usage} marker.

Isso permite colocá-lo em qualquer lugar em um Model sem a memória sendo calculado até o final. o Classe de saída will Troque o valor real por esta variável.
