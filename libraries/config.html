Classe Config¶

The Config class provides a means to retrieve configuration preferences. Essas preferências podem vir do arquivo de configuração padrão (application/config/config.php) ou de seus próprios arquivos de configuração personalizados.

Nota

Esta classe é inicializada automaticamente pelo sistema para que is no need to do it manually.

Working with the Classe Config¶

Anatomy of a Config File¶

Por padrão, o CodeIgniter tem um arquivo de configuração principal, localizado em application/config/config.php. If you open the file using your text editor você verá que os itens de configuração são armazenados em uma matriz chamada $config.

You can add your own config items to this file, or if you prefer to keep seus itens de configuração separados (supondo que você precise de itens de configuração), simply create your own file and save it in config folder.

Nota

If you do create your own config files use the same format as the primary one, storing your items in an array called $config. O CodeIgniter irá gerenciar inteligentemente esses arquivos para que não haja conflito mesmo que a matriz tenha o mesmo nome (assumindo uma matriz índice não é nomeado o mesmo que outro).

Loading a Config File¶

Nota

O CodeIgniter carrega automaticamente o arquivo de configuração principal (application/config/config.php), so you will only need to load a config arquivo se você criou o seu próprio.

There are two ways to load a config file:

Manual Loading¶

Para carregar um dos seus arquivos de configuração personalizados, você usará o seguinte function within the controller that needs it:

$this->config->load('filename');

Onde filename é o nome do seu arquivo de configuração, sem o arquivo .php extension.

If you need to load multiple config files normally they will be merged into one master config array. Name collisions can occur, no entanto, se você tiver nomeado identicamente índices de matriz em diferentes arquivos de configuração. Para evitar colisões, você pode definir o segundo parâmetro para TRUE and each config file will be stored in an array index corresponding to the name of the config file. Example:

// Stored in an array with this prototype: $this->config['blog_settings'] = $config
$this->config->load('blog_settings', TRUE);

Please see the section entitled Fetching Config Items below to learn como recuperar itens de configuração definidos dessa maneira.

O terceiro parâmetro permite suprimir erros no caso de um config file does not exist:

$this->config->load('blog_settings', FALSE, TRUE);

Carregamento automático¶

Se você achar que precisa de um arquivo de configuração específico globalmente, tê-lo carregado automaticamente pelo sistema. Para fazer isso, abra o autoload.php file, located at application/config/autoload.php, e adicione seu arquivo de configuração conforme indicado no arquivo.

Fetching Config Items¶

Para recuperar um item do seu arquivo de configuração, use a seguinte função:

$this->config->item('item_name');

Where item_name is the $config array index you want to retrieve. For Por exemplo, para buscar sua escolha de idioma, você fará isso:

$lang = $this->config->item('language');

The function returns NULL if the item you are trying to fetch does not exist.

If you are using the second parameter of the $this->config->load function in order to assign your config items to a specific index you can retrieve it by specifying the index name in the second parameter of the $this->config->item() function. Example:

// Loads a config file named blog_settings.php and assigns it to an index named "blog_settings"
$this->config->load('blog_settings', TRUE);

// Retrieve a config item named site_name contained within the blog_settings array
$site_name = $this->config->item('site_name', 'blog_settings');

// An alternate way to specify the same item:
$blog_config = $this->config->item('blog_settings');
$site_name = $blog_config['site_name'];

Definindo um item de configuração¶

If you would like to dynamically set a config item or change an existing one, you can do so using:

$this->config->set_item('item_name', 'item_value');

Where item_name is the $config array index you want to change, and item_value is its value.

Environments¶

You may load different configuration files depending on the current meio Ambiente. A constante ENVIRONMENT é definida em index.php e é described in detail in the Handling Environments section.

Para criar um arquivo de configuração específico do ambiente, crie ou copie um configuration file in application/config/{ENVIRONMENT}/{FILENAME}.php

For example, to create a production-only config.php, you would:

  1. Create the directory application/config/production/
  2. Copy your existing config.php into the above directory
  3. Edit application/config/production/config.php so it contains your configurações de produção

When you set the ENVIRONMENT constant to ‘production’, the settings for seu novo config.php de produção somente será carregado.

You can place the following configuration files in environment-specific folders:

  • Arquivos de configuração padrão do CodeIgniter
  • Seus próprios arquivos de configuração personalizados

Nota

CodeIgniter always loads the global config file first (i.e., the one in application/config/), then tries to load the configuration files for the current environment. Isso significa que você não é obrigado a colocar all de seus arquivos de configuração em um environment folder. Only the files that change per environment. Additionally you don’t have to copy all os itens de configuração no arquivo de configuração do ambiente. Apenas os itens de configuração that you wish to change for your environment. The config items declared in your environment folders always overwrite those in your global config files.

A classe Config fornece um meio para recuperar preferências de configuração. Essas preferências podem vir do arquivo de configuração padrão (application/config/config.php) ou de seus próprios arquivos de configuração personalizados.

Esta classe é inicializada automaticamente pelo sistema para que Não é necessário fazer isso manualmente.

Working with the Classe Config¶

Anatomia de um arquivo de configuração¶

Por padrão, o CodeIgniter tem um arquivo de configuração principal, localizado em application/config/config.php. Se você abrir o arquivo usando seu texto editor você verá que os itens de configuração são armazenados em uma matriz chamada $config.

Você pode adicionar seus próprios itens de configuração a este arquivo, ou se você preferir manter seus itens de configuração separados (supondo que você precise de itens de configuração), basta criar seu próprio arquivo e salvá-lo na pasta de configuração.

Nota

Se você criar seus próprios arquivos de configuração, use o mesmo formato que a principal, armazenando seus itens em uma matriz chamada $config. O CodeIgniter irá gerenciar inteligentemente esses arquivos para que não haja conflito mesmo que a matriz tenha o mesmo nome (assumindo uma matriz índice não é nomeado o mesmo que outro).

Carregando um arquivo de configuração¶

Nota

O CodeIgniter carrega automaticamente o arquivo de configuração principal (application/config/config.php), então você só precisa carregar uma configuração arquivo se você criou o seu próprio.

Existem duas maneiras de carregar um arquivo de configuração:

Carregamento manual¶

Para carregar um dos seus arquivos de configuração personalizados, você usará o seguinte função dentro do controller that precisa disso:

$this->config->load('filename');

Onde filename é o nome do seu arquivo de configuração, sem o arquivo .php extension.

Se você precisar carregar vários arquivos de configuração normalmente eles serão mesclado em uma matriz de configuração principal. Colisões de nomes podem ocorrer no entanto, se você tiver nomeado identicamente índices de matriz em diferentes arquivos de configuração. Para evitar colisões, você pode definir o segundo parâmetro para TRUE e cada arquivo de configuração será armazenado em um índice de array correspondente ao nome do arquivo de configuração. Exemplo:

// Stored in an array with this prototype: $this->config['blog_settings'] = $config
$this->config->load('blog_settings', TRUE);

Por favor, veja a seção intitulada Buscando Itens de Configuração abaixo para aprender como recuperar itens de configuração definidos dessa maneira.

O terceiro parâmetro permite suprimir erros no caso de um arquivo de configuração não existe:

$this->config->load('blog_settings', FALSE, TRUE);

Carregamento automático¶

Se você achar que precisa de um arquivo de configuração específico globalmente, tê-lo carregado automaticamente pelo sistema. Para fazer isso, abra o autoload.php arquivo, localizado em application/config/autoload.php, e adicione seu arquivo de configuração conforme indicado no arquivo.

Buscando itens de configuração¶

Para recuperar um item do seu arquivo de configuração, use a seguinte função:

$this->config->item('item_name');

Onde item_name é o índice da matriz $config que você deseja recuperar. Para Por exemplo, para buscar sua escolha de idioma, você fará isso:

$lang = $this->config->item('language');

A função retorna NULL se o item que você está tentando buscar não existe.

Se você está usando o segundo parâmetro do $this-> config-> load função, a fim de atribuir seus itens de configuração para um índice específico você pode recuperá-lo, especificando o nome do índice no segundo parâmetro de a função $this-> config-> item(). Exemplo:

// Loads a config file named blog_settings.php and assigns it to an index named "blog_settings"
$this->config->load('blog_settings', TRUE);

// Retrieve a config item named site_name contained within the blog_settings array
$site_name = $this->config->item('site_name', 'blog_settings');

// An alternate way to specify the same item:
$blog_config = $this->config->item('blog_settings');
$site_name = $blog_config['site_name'];

Definindo um item de configuração¶

Se você gostaria de definir dinamicamente um item de configuração ou alterar um um, você pode fazer isso usando:

$this->config->set_item('item_name', 'item_value');

Onde item_name é o índice da matriz $config que você deseja alterar e item_value é o seu valor.

Ambientes¶

Você pode carregar diferentes arquivos de configuração dependendo da meio Ambiente. A constante ENVIRONMENT é definida em index.php e é descrito em detalhes no Manipulação Ambientes section.

Para criar um arquivo de configuração específico do ambiente, crie ou copie um arquivo de configuração em application/config/{ENVIRONMENT}/{FILENAME} .php

Por exemplo, para criar um config.php somente de produção, você faria:

  1. Crie o diretório application/config/production /
  2. Copie seu config.php existente no diretório acima
  3. Edite application/config/production/config.php para que ele contenha seu configurações de produção

Quando você define a constante de AMBIENTE para 'produção', as configurações para seu novo config.php de produção somente será carregado.

Você pode colocar os seguintes arquivos de configuração em folders:

  • Arquivos de configuração padrão do CodeIgniter
  • Seus próprios arquivos de configuração personalizados

Nota

O CodeIgniter sempre carrega o arquivo de configuração global primeiro (isto é, aquele em application/config /), em seguida, tenta carregar os arquivos de configuração para o ambiente atual. Isso significa que você não é obrigado a colocar all de seus arquivos de configuração em um pasta do ambiente. Apenas os arquivos que mudam por ambiente. Além disso você não tem que copiar all os itens de configuração no arquivo de configuração do ambiente. Apenas os itens de configuração que você deseja mudar para o seu ambiente. Os itens de configuração declarados em seu ambiente As pastas sempre sobrescrevem aquelas em seus arquivos globais de configuração.

Classe Config¶

A classe Config fornece um meio para recuperar preferências de configuração. Essas preferências podem vir do arquivo de configuração padrão (application/config/config.php) ou de seus próprios arquivos de configuração personalizados.

Nota

Esta classe é inicializada automaticamente pelo sistema para que Não é necessário fazer isso manualmente.

Working with the Classe Config¶

Anatomia de um arquivo de configuração¶

Por padrão, o CodeIgniter tem um arquivo de configuração principal, localizado em application/config/config.php. Se você abrir o arquivo usando seu texto editor você verá que os itens de configuração são armazenados em uma matriz chamada $config.

Você pode adicionar seus próprios itens de configuração a este arquivo, ou se você preferir manter seus itens de configuração separados (supondo que você precise de itens de configuração), basta criar seu próprio arquivo e salvá-lo na pasta de configuração.

Nota

Se você criar seus próprios arquivos de configuração, use o mesmo formato que a principal, armazenando seus itens em uma matriz chamada $config. O CodeIgniter irá gerenciar inteligentemente esses arquivos para que não haja conflito mesmo que a matriz tenha o mesmo nome (assumindo uma matriz índice não é nomeado o mesmo que outro).

Carregando um arquivo de configuração¶

Nota

O CodeIgniter carrega automaticamente o arquivo de configuração principal (application/config/config.php), então você só precisa carregar uma configuração arquivo se você criou o seu próprio.

Existem duas maneiras de carregar um arquivo de configuração:

Carregamento manual¶

Para carregar um dos seus arquivos de configuração personalizados, você usará o seguinte função dentro do controller that needs it:

$this->config->load('filename');

Onde filename é o nome do seu arquivo de configuração, sem o arquivo .php extension.

Se você precisar carregar vários arquivos de configuração normalmente eles serão mesclado em uma matriz de configuração principal. Colisões de nomes podem ocorrer no entanto, se você tiver nomeado identicamente índices de matriz em diferentes arquivos de configuração. Para evitar colisões, você pode definir o segundo parâmetro para TRUE e cada arquivo de configuração será armazenado em um índice de array correspondente ao nome do arquivo de configuração. Exemplo:

// Stored in an array with this prototype: $this->config['blog_settings'] = $config
$this->config->load('blog_settings', TRUE);

Por favor, veja a seção intitulada Buscando Itens de Configuração abaixo para aprender como recuperar itens de configuração definidos dessa maneira.

O terceiro parâmetro permite suprimir erros no caso de um arquivo de configuração não existe:

$this->config->load('blog_settings', FALSE, TRUE);

Carregamento automático¶

Se você achar que precisa de um arquivo de configuração específico globalmente, tê-lo carregado automaticamente pelo sistema. Para fazer isso, abra o autoload.php arquivo, localizado em application/config/autoload.php, e adicione seu arquivo de configuração conforme indicado no arquivo.

Buscando itens de configuração¶

Para recuperar um item do seu arquivo de configuração, use a seguinte função:

$this->config->item('item_name');

Onde item_name é o índice da matriz $config que você deseja recuperar. Para Por exemplo, para buscar sua escolha de idioma, você fará isso:

$lang = $this->config->item('language');

A função retorna NULL se o item que você está tentando buscar não existe.

Se você está usando o segundo parâmetro do $this-> config->load função, a fim de atribuir seus itens de configuração para um índice específico você pode recuperá-lo, especificando o nome do índice no segundo parâmetro de the $this->config->item() function. Example:

// Loads a config file named blog_settings.php and assigns it to an index named "blog_settings"
$this->config->load('blog_settings', TRUE);

// Retrieve a config item named site_name contained within the blog_settings array
$site_name = $this->config->item('site_name', 'blog_settings');

// An alternate way to specify the same item:
$blog_config = $this->config->item('blog_settings');
$site_name = $blog_config['site_name'];

Definindo um item de configuração¶

Se você gostaria de definir dinamicamente um item de configuração ou alterar um um, você pode fazer isso usando:

$this->config->set_item('item_name', 'item_value');

Onde item_name é o índice da matriz $config que você deseja alterar e item_value é o seu valor.
