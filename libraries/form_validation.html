Overview¶

Before explaining CodeIgniter’s approach to data validation, let’s descreva o cenário ideal:

  1. Um formulário é exibido.
  2. You fill it in and submit it.
  3. If you submitted something invalid, or perhaps missed a required item, the form is redisplayed containing your data along with an error message describing the problem.
  4. This process continues until you have submitted a valid form.

On the receiving end, the script must:

  1. Check for required data.
  2. Verify that the data is of the correct type, and meets the correct critério. Por exemplo, se um nome de usuário for enviado, ele deve ser validado para conter apenas caracteres permitidos. Deve ser de um minimum length, and not exceed a maximum length. o username can’t be someone else’s existing username, or perhaps even a reserved word. Etc.
  3. Sanitize the data for security.
  4. Pré-formate os dados, se necessário (os dados precisam ser aparados? HTML codificado? Etc.)
  5. Prep the data for insertion in the database.

UMAlthough there is nothing terribly complex about the above process, it usually requires a significant amount of code, and to display error mensagens, várias estruturas de controle geralmente são colocadas dentro do formulário HTML. Form validation, while simple to create, is generally very messy e tedioso para implementar.

Validação de Formulário Tutorial¶

What follows is a “hands on” tutorial for implementing CodeIgniters Form Validação.

In order to implement form validation you’ll need three things:

  1. UMA Visão file containing a form.
  2. UMA Visão file containing a “success” message to be displayed upon successful submission.
  3. UMA controller método para receber e process the submitted data.

Let’s create those three things, using a member sign-up form as the example.

A forma¶

Usando um editor de texto, crie um formulário chamado myform.php. Nele, coloque isso código e salve-o em seu aplicativo/views/folder:

<html>
<head>
<title>My Form</title>
</head>
<body>

<?php echo validation_errors(); ?>

<?php echo form_open('form'); ?>

<h5>Username</h5>
<input type="text" name="username" value="" size="50" />

<h5>Password</h5>
<input type="text" name="password" value="" size="50" />

<h5>Password Confirm</h5>
<input type="text" name="passconf" value="" size="50" />

<h5>Email UMAddress</h5>
<input type="text" name="email" value="" size="50" />

<div><input type="submit" value="Submit" /></div>

</form>

</body>
</html>

o Success Page¶

Using a text editor, create a form called formsuccess.php. In it, place this código e salve-o em seu aplicativo/views/folder:

<html>
<head>
<title>My Form</title>
</head>
<body>

<h3>Your form was successfully submitted!</h3>

<p><?php echo anchor('form', 'Try it again!'); ?></p>

</body>
</html>

o Controller¶

Using a text editor, create a controller called Form.php. In it, place this code and save it to your application/controllers/ folder:

<?php

class Form extends CI_Controller {

        public function index()
        {
                $this->load->helper(array('form', 'url'));

                $this->load->library('form_validation');

                if ($this->form_validation->run() == FUMALSE)
                {
                        $this->load->view('myform');
                }
                else
                {
                        $this->load->view('formsuccess');
                }
        }
}

Try it!¶

To try your form, visit your site using a URL similar to this one:

example.com/index.php/form/

If you submit the form you should simply see the form reload. That’s porque você ainda não configurou nenhuma regra de validação.

Since you haven’t told the Validação de Formulário class to validate anything yet, it returns FUMALSE (boolean false) by default. ``o run()`` method only returns TRUE if it has successfully applied your rules without any of them failing.

Explicação¶

Você notará várias coisas sobre as páginas acima:

o form (myform.php) is a standard web form with a couple exceptions:

  1. It uses a form helper to create the form opening. Technically, this não é necessário. Você poderia criar o formulário usando o HTML padrão. However, the benefit of usando o helper is that it generates the URL de ação para você, com base no URL no seu arquivo de configuração. Isto faz sua aplicação mais portável no caso de seus URLs mudarem.

  2. UMNo topo do formulário, você notará a seguinte chamada de função:

    <?php echo validation_errors(); ?>

    This function will return any error messages sent back by the validator. If there are no messages it returns an empty string.

o controller (Form.php) has one method: index(). This method initializes the validation class and loads the form helper and URL helper usado pelos arquivos de visualização. Também executa a rotina de validação. Based on whether the validation was successful it either presents the form or the success page.

Definindo regras de validação¶

CodeIgniter lets you set as many validation rules as you need for a given field, cascading them in order, and it even lets you prep and pré-processar os dados de campo ao mesmo tempo. Para definir regras de validação você will use the set_rules() method:

$this->form_validation->set_rules();

o above method takes three parameters as input:

  1. O nome do campo - o nome exato que você deu ao campo do formulário.
  2. UMA “human” name for this field, which will be inserted into the error message. For example, if your field is named “user” you might give it a human name of “Username”.
  3. o validation rules for this form field.
  4. (opcional) Defina mensagens de erro personalizadas em quaisquer regras dadas para o campo atual. Se não for fornecido, usará o padrão.

Nota

If you would like the field name to be stored in a language file, please see Translating Field Names.

Aqui está um exemplo. No seu controlador (Form.php), adicione este código apenas below the validation initialization method:

$this->form_validation->set_rules('username', 'Username', 'required');
$this->form_validation->set_rules('password', 'Password', 'required');
$this->form_validation->set_rules('passconf', 'Password Confirmation', 'required');
$this->form_validation->set_rules('email', 'Email', 'required');

Your controller should now look like this:

<?php

class Form extends CI_Controller {

        public function index()
        {
                $this->load->helper(array('form', 'url'));

                $this->load->library('form_validation');

                $this->form_validation->set_rules('username', 'Username', 'required');
                $this->form_validation->set_rules('password', 'Password', 'required',
                        array('required' => 'You must provide a %s.')
                );
                $this->form_validation->set_rules('passconf', 'Password Confirmation', 'required');
                $this->form_validation->set_rules('email', 'Email', 'required');

                if ($this->form_validation->run() == FUMALSE)
                {
                        $this->load->view('myform');
                }
                else
                {
                        $this->load->view('formsuccess');
                }
        }
}

Now submit the form with the fields blank and you should see the error messages. If you submit the form with all the fields populated you’ll see your success page.

Nota

Os campos de formulário ainda não estão sendo preenchidos com os dados when there is an error. We’ll get to that shortly.

Setting Rules Using an UMArray¶

Before moving on it should be noted that the rule setting method can be passed an array if you prefer to set all your rules in one action. If you use this approach, you must name your array keys as indicated:

$config = array(
        array(
                'field' => 'username',
                'label' => 'Username',
                'rules' => 'required'
        ),
        array(
                'field' => 'password',
                'label' => 'Password',
                'rules' => 'required',
                'errors' => array(
                        'required' => 'You must provide a %s.',
                ),
        ),
        array(
                'field' => 'passconf',
                'label' => 'Password Confirmation',
                'rules' => 'required'
        ),
        array(
                'field' => 'email',
                'label' => 'Email',
                'rules' => 'required'
        )
);

$this->form_validation->set_rules($config);

Cascading Rules¶

O CodeIgniter permite que você canalize várias regras juntas. Vamos tentar. mudança your rules in the third parameter of rule setting method, like this:

$this->form_validation->set_rules(
        'username', 'Username',
        'required|min_length[5]|max_length[12]|is_unique[users.username]',
        array(
                'required'      => 'You have not provided %s.',
                'is_unique'     => 'This %s already exists.'
        )
);
$this->form_validation->set_rules('password', 'Password', 'required');
$this->form_validation->set_rules('passconf', 'Password Confirmation', 'required|matches[password]');
$this->form_validation->set_rules('email', 'Email', 'required|valid_email|is_unique[users.email]');

o above code sets the following rules:

  1. O campo de nome de usuário não deve ter menos de 5 caracteres e não mais que 12.
  2. o password field must match the password confirmation field.
  3. o email field must contain a valid email address.

Give it a try! Submit your form without the proper data and you’ll see new error messages that correspond to your new rules. ore are numerous regras disponíveis sobre as quais você pode ler na referência de validação.

Nota

You can also pass an array of rules to set_rules(), instead of a string. Example:

$this->form_validation->set_rules('username', 'Username', array('required', 'min_length[5]'));

Prepping Data¶

In addition to the validation method like the ones we used above, you também pode preparar seus dados de várias maneiras. Por exemplo, você pode configurar regras como esta:

$this->form_validation->set_rules('username', 'Username', 'trim|required|min_length[5]|max_length[12]');
$this->form_validation->set_rules('password', 'Password', 'trim|required|min_length[8]');
$this->form_validation->set_rules('passconf', 'Password Confirmation', 'trim|required|matches[password]');
$this->form_validation->set_rules('email', 'Email', 'trim|required|valid_email');

In the above example, we are “trimming” the fields, checking for length onde necessário e certificando-se de que ambos os campos de senha coincidam.

UMAny native PHP function that accepts one parameter can be used as a regra, como `` htmlspecialchars() ``, `` trim() ``, etc.

Nota

You will generally want to use the prepping functions after the validation rules so if there is an error, the original data will be shown in the form.

Re-preenchendo o formulário¶

Thus far we have only been dealing with errors. It’s time to repopulate o campo de formulário com os dados enviados. O CodeIgniter oferece vários funções auxiliares que permitem que você faça isso. Aquele que você mais usará commonly is:

set_value('field name')

Open your myform.php view file and update the value in each field usando o set_value() function:

Don’t forget to include each field name in the :php:func:`set_value()` function calls!

<html>
<head>
<title>My Form</title>
</head>
<body>

<?php echo validation_errors(); ?>

<?php echo form_open('form'); ?>

<h5>Username</h5>
<input type="text" name="username" value="<?php echo set_value('username'); ?>" size="50" />

<h5>Password</h5>
<input type="text" name="password" value="<?php echo set_value('password'); ?>" size="50" />

<h5>Password Confirm</h5>
<input type="text" name="passconf" value="<?php echo set_value('passconf'); ?>" size="50" />

<h5>Email UMAddress</h5>
<input type="text" name="email" value="<?php echo set_value('email'); ?>" size="50" />

<div><input type="submit" value="Submit" /></div>

</form>

</body>
</html>

Now reload your page and submit the form so that it triggers an error. Your form fields should now be re-populated

Nota

o Class Reference section below contains methods that permit you to re-populate <select> menus, radio buttons, and checkboxes.

Importante

If you use an array as the name of a form field, you deve fornecê-lo como um array para a função. Exemplo:

<input type="text" name="colors[]" value="<?php echo set_value('colors[]'); ?>" size="50" />

Para mais informações, por favor consulte o Using UMArrays as Field Names section below.

Callbacks: Your own Validation Methods¶

o validation system supports callbacks to your own validation methods. This permits you to extend the validation class to meet your needs. For example, if you need to run a database query to see if the user is choosing a unique username, you can create a callback method that does that. Let’s create an example of this.

No seu controlador, altere a regra "username" para isso:

$this->form_validation->set_rules('username', 'Username', 'callback_username_check');

on add a new method called username_check() to your controller. Here’s how your controller should now look:

<?php

class Form extends CI_Controller {

        public function index()
        {
                $this->load->helper(array('form', 'url'));

                $this->load->library('form_validation');

                $this->form_validation->set_rules('username', 'Username', 'callback_username_check');
                $this->form_validation->set_rules('password', 'Password', 'required');
                $this->form_validation->set_rules('passconf', 'Password Confirmation', 'required');
                $this->form_validation->set_rules('email', 'Email', 'required|is_unique[users.email]');

                if ($this->form_validation->run() == FUMALSE)
                {
                        $this->load->view('myform');
                }
                else
                {
                        $this->load->view('formsuccess');
                }
        }

        public function username_check($str)
        {
                if ($str == 'test')
                {
                        $this->form_validation->set_message('username_check', 'o {field} field can not be the word "test"');
                        return FUMALSE;
                }
                else
                {
                        return TRUE;
                }
        }

}

Reload your form and submit it with the word “test” as the username. You can see that the form field data was passed to your callback method for you to process.

To invoke a callback just put the method name in a rule, with “callback_” as the rule prefix. Se você precisar receber um extra parâmetro no seu método de retorno de chamada, basta adicioná-lo normalmente após o method name between square brackets, as in: callback_foo[bar], then it will be passed as the second argument of your callback method.

Nota

You can also process the form data that is passed to your retorno de chamada e devolva-o. Se o seu retorno de chamada retornar algo diferente de boolean TRUE/FUMALSE it is assumed that the data is your newly processed dados de formulário.

Callable: use qualquer coisa como regra¶

If callback rules aren’t good enough for you (for example, because they are limited to your controller), don’t get disappointed, there’s one more way to create custom rules: anything that is_callable() would return TRUE for.

Considere o seguinte exemplo:

$this->form_validation->set_rules(
        'username', 'Username',
        array(
                'required',
                array($this->users_model, 'valid_username')
        )
);

o above code would use the valid_username() method from your Users_model object.

Este é apenas um exemplo, e os retornos de chamada não se limitam aos modelos. You can use any object/method that accepts the field value as its’ first parâmetro. Você também pode usar uma função anônima:

$this->form_validation->set_rules(
        'username', 'Username',
        array(
                'required',
                function($value)
                {
                        // Check $value
                }
        )
);

Claro que, uma vez que uma regra de Callable por si só não é uma string, não é a rule name either. That is a problem when you want to set error messages for them. In order to get around that problem, you can put such rules as o segundo elemento de um array, sendo o primeiro o nome da regra:

$this->form_validation->set_rules(
        'username', 'Username',
        array(
                'required',
                array('username_callable', array($this->users_model, 'valid_username'))
        )
);

UMVersão da função anônima:

$this->form_validation->set_rules(
        'username', 'Username',
        array(
                'required',
                array(
                        'username_callable',
                        function($str)
                        {
                                // Check validity of $str and return TRUE or FUMALSE
                        }
                )
        )
);

Configurando Mensagens de Erro¶

UMAll of the native error messages are located in the following language file: system/language/english/form_validation_lang.php

Para definir sua própria mensagem personalizada global para uma regra, você pode extend/override the language file by creating your own in application/language/english/form_validation_lang.php (read more sobre isso no Aula de idioma documentation), ou use o seguinte método:

$this->form_validation->set_message('rule', 'Error Message');

If you need to set a custom error message for a particular field on some particular rule, use the set_rules() method:

$this->form_validation->set_rules('field_name', 'Field Label', 'rule1|rule2|rule3',
        array('rule2' => 'Error Message on rule2 for this field_name')
);

Onde regra corresponde ao nome de uma regra específica e Erro Message is the text you would like displayed.

If you’d like to include a field’s “human” name, or the optional parameter some rules allow for (such as max_length), you can add the {field} and {param} tags to your message, respectively:

$this->form_validation->set_message('min_length', '{field} must have at least {param} characters.');

Em um campo com o nome humano Username e uma regra de min_length [5], um erro seria exibido: "O nome de usuário deve ter pelo menos 5 caracteres".

Nota

o old sprintf() método de usar %s in your error messages will still work, however it will override the tags above. You should use one or the other.

No exemplo de regra de retorno de chamada acima, a mensagem de erro foi definida passando o nome do método (sem o prefixo “callback_”):

$this->form_validation->set_message('username_check')

Translating Field Names¶

If you would like to store the “human” name you passed to the set_rules() method in a language file, and therefore make the name capaz de ser traduzido, veja como:

First, prefix your “human” name with lang:, como neste exemplo:

$this->form_validation->set_rules('first_name', 'lang:first_name', 'required');

on, store the name in one of your language file arrays (without the prefix):

$lang['first_name'] = 'First Name';

Nota

If you store your array item in a language file that is not loaded automatically by CI, you’ll need to remember to load it in your controlador usando:

$this->lang->load('file_name');

Veja o Aula de idioma page for more info regarding language files.

Changing the Error Delimiters¶

By default, the Validação de Formulário class adds a paragraph tag (<p>) around each error message shown. You can either change these delimiters globally, individually, or change the defaults in a config file.

  1. Changing delimiters Globally To globally change the error delimiters, in your controller method, just after loading the Validação de Formulário class, add this:

    $this->form_validation->set_error_delimiters('<div class="error">', '</div>');

    Neste exemplo, mudamos para usar tags div.

  2. Alterando os delimitadores individualmente Cada uma das duas funções de geração de erro mostradas neste tutorial pode ser fornecido seus próprios delimitadores da seguinte forma:

    <?php echo form_error('field name', '<div class="error">', '</div>'); ?>

    Ou:

    <?php echo validation_errors('<div class="error">', '</div>'); ?>

  3. Definir delimitadores em um arquivo de configuração You can add your error delimiters in application/config/form_validation.php as follows:

    $config['error_prefix'] = '<div class="error_prefix">';
$config['error_suffix'] = '</div>';

Showing Errors Individually¶

Se você preferir mostrar uma mensagem de erro ao lado de cada campo de formulário, than as a list, you can use the form_error() function.

Tente! Altere seu formulário para que fique assim:

<h5>Username</h5>
<?php echo form_error('username'); ?>
<input type="text" name="username" value="<?php echo set_value('username'); ?>" size="50" />

<h5>Password</h5>
<?php echo form_error('password'); ?>
<input type="text" name="password" value="<?php echo set_value('password'); ?>" size="50" />

<h5>Password Confirm</h5>
<?php echo form_error('passconf'); ?>
<input type="text" name="passconf" value="<?php echo set_value('passconf'); ?>" size="50" />

<h5>Email UMAddress</h5>
<?php echo form_error('email'); ?>
<input type="text" name="email" value="<?php echo set_value('email'); ?>" size="50" />

If there are no errors, nothing will be shown. If there is an error, the mensagem será exibida.

Importante

If you use an array as the name of a form field, you deve fornecê-lo como um array para a função. Exemplo:

<?php echo form_error('options[size]'); ?>
<input type="text" name="options[size]" value="<?php echo set_value("options[size]"); ?>" size="50" />

Para mais informações, por favor consulte o Using UMArrays as Field Names section below.

Validating an UMArray (other than $_POST)¶

Sometimes you may want to validate an array that does not originate from $_POST data.

Nesse caso, você pode especificar a matriz a ser validada:

$data = array(
        'username' => 'johndoe',
        'password' => 'mypassword',
        'passconf' => 'mypassword'
);

$this->form_validation->set_data($data);

Criando regras de validação, executando a validação e recuperando erros messages works the same whether you are validating $_POST data or outra matriz de sua escolha.

Importante

You have to call the set_data() method before defining quaisquer regras de validação.

Importante

Se você quiser validar mais de um array durante um único execution, then you should call the reset_validation() method before setting up rules and validating the new array.

Para mais informações, por favor consulte o Class Reference section below.

Saving Sets of Validation Rules to a Config File¶

UMA nice feature of the Validação de Formulário class is that it permits you to armazenar todas as suas regras de validação para todo o seu aplicativo em uma configuração file. You can organize these rules into “groups”. ose groups can seja carregado automaticamente quando um controlador/método correspondente chamado, ou você pode chamar manualmente cada conjunto conforme necessário.

Como salvar suas regras¶

Para armazenar suas regras de validação, basta criar um arquivo chamado form_validation.php in your application/config/ folder. In that file you will place an array named $config with your rules. UMAs shown earlier, the validation array will have this prototype:

$config = array(
        array(
                'field' => 'username',
                'label' => 'Username',
                'rules' => 'required'
        ),
        array(
                'field' => 'password',
                'label' => 'Password',
                'rules' => 'required'
        ),
        array(
                'field' => 'passconf',
                'label' => 'Password Confirmation',
                'rules' => 'required'
        ),
        array(
                'field' => 'email',
                'label' => 'Email',
                'rules' => 'required'
        )
);

Your validation rule file will be loaded automatically and used when you call the run() method.

Please note that you MUST name your $config array.

Criando Conjuntos de Regras¶

In order to organize your rules into “sets” requires that you place them em "sub arrays". Considere o seguinte exemplo, mostrando dois conjuntos de regras. Nós arbitrariamente chamamos essas duas regras de “inscrição” e “email”. Você pode nomear suas regras como quiser:

$config = array(
        'signup' => array(
                array(
                        'field' => 'username',
                        'label' => 'Username',
                        'rules' => 'required'
                ),
                array(
                        'field' => 'password',
                        'label' => 'Password',
                        'rules' => 'required'
                ),
                array(
                        'field' => 'passconf',
                        'label' => 'Password Confirmation',
                        'rules' => 'required'
                ),
                array(
                        'field' => 'email',
                        'label' => 'Email',
                        'rules' => 'required'
                )
        ),
        'email' => array(
                array(
                        'field' => 'emailaddress',
                        'label' => 'EmailUMAddress',
                        'rules' => 'required|valid_email'
                ),
                array(
                        'field' => 'name',
                        'label' => 'Name',
                        'rules' => 'required|alpha'
                ),
                array(
                        'field' => 'title',
                        'label' => 'Title',
                        'rules' => 'required'
                ),
                array(
                        'field' => 'message',
                        'label' => 'MessageBody',
                        'rules' => 'required'
                )
        )
);

Chamando um grupo de regras específico¶

Para chamar um grupo específico, você passará o nome dele para o run() método. Por exemplo, para chamar a regra de inscrição, você fará isso:

if ($this->form_validation->run('signup') == FUMALSE)
{
        $this->load->view('myform');
}
else
{
        $this->load->view('formsuccess');
}

UMAssociating a Controller Method with a Rule Group¶

UMUm método alternativo (e mais automático) de chamar um grupo de regras é name it according to the controller class/method you intend to use it with. For example, let’s say you have a controller named Member and a method named signup. Here’s what your class might look like:

<?php

class Member extends CI_Controller {

        public function signup()
        {
                $this->load->library('form_validation');

                if ($this->form_validation->run() == FUMALSE)
                {
                        $this->load->view('myform');
                }
                else
                {
                        $this->load->view('formsuccess');
                }
        }
}

In your validation config file, you will name your rule group member/signup:

$config = array(
        'member/signup' => array(
                array(
                        'field' => 'username',
                        'label' => 'Username',
                        'rules' => 'required'
                ),
                array(
                        'field' => 'password',
                        'label' => 'Password',
                        'rules' => 'required'
                ),
                array(
                        'field' => 'passconf',
                        'label' => 'PasswordConfirmation',
                        'rules' => 'required'
                ),
                array(
                        'field' => 'email',
                        'label' => 'Email',
                        'rules' => 'required'
                )
        )
);

When a rule group is named identically to a controller class/method it will be used automatically when the run() method is invoked from that class/method.

Using UMArrays as Field Names¶

o Validação de Formulário class supports the use of arrays as field names. Considere este exemplo:

<input type="text" name="options[]" value="" size="50" />

If you do use an array as a field name, you must use the EXUMACT array name in the Helper Functions que exigem o nome do campo e como seu nome de campo da Regra de validação.

Por exemplo, para definir uma regra para o campo acima, você usaria:

$this->form_validation->set_rules('options[]', 'Options', 'required');

Or, to show an error for the above field you would use:

<?php echo form_error('options[]'); ?>

Or to re-populate the field you would use:

<input type="text" name="options[]" value="<?php echo set_value('options[]'); ?>" size="50" />

You can use multidimensional arrays as field names as well. For example:

<input type="text" name="options[size]" value="" size="50" />

Or even:

<input type="text" name="sports[nba][basketball]" value="" size="50" />

UMAs with our first example, you must use the exact array name in the helper functions:

<?php echo form_error('sports[nba][basketball]'); ?>

If you are using checkboxes (or other fields) that have multiple opções, não se esqueça de deixar um colchete vazio após cada opção, então that all selections will be added to the POST array:

<input type="checkbox" name="options[]" value="red" />
<input type="checkbox" name="options[]" value="blue" />
<input type="checkbox" name="options[]" value="green" />

Ou se você usa um array multidimensional:

<input type="checkbox" name="options[color][]" value="red" />
<input type="checkbox" name="options[color][]" value="blue" />
<input type="checkbox" name="options[color][]" value="green" />

Quando você usa uma função auxiliar, você também inclui o suporte:

<?php echo form_error('options[color][]'); ?>

View global¶

Antes de explicar a abordagem do CodeIgniter para validação de dados, vamos descreva o cenário ideal:

  1. Um formulário é exibido.
  2. Você preenche e envia.
  3. Se você enviou algo inválido ou talvez tenha perdido um item, o formulário é reexibido contendo seus dados juntamente com um mensagem de erro descrevendo o problema.
  4. Esse processo continua até que você tenha enviado um formulário válido.

No final de recebimento, o script deve:

  1. Verifique os dados necessários.
  2. Verifique se os dados são do tipo correto e atende aos requisitos corretos. critério. Por exemplo, se um nome de usuário for enviado, ele deve ser validado para conter apenas caracteres permitidos. Deve ser de um comprimento mínimo e não deve exceder um comprimento máximo. O nome de usuário não pode seja o nome de usuário existente de outra pessoa ou talvez até mesmo uma palavra reservada. Etc.
  3. Sanitize os dados para segurança.
  4. Pré-formate os dados, se necessário (os dados precisam ser aparados? HTML codificado? Etc.)
  5. Prepare os dados para inserção no Database.

Embora não haja nada terrivelmente complexo sobre o processo acima, geralmente requer uma quantidade significativa de código e para exibir erros mensagens, várias estruturas de controle geralmente são colocadas dentro do formulário HTML A validação de formulários, embora simples de criar, geralmente é muito confusa e tedioso para implementar.

Definindo regras de validação¶

O CodeIgniter permite que você defina quantas regras de validação forem necessárias para um dado campo, cascateando-os em ordem, e ainda permite preparar e pré-processar os dados de campo ao mesmo tempo. Para definir regras de validação você will use the set_rules() method:

$this->form_validation->set_rules();

O método acima leva three parâmetros como entrada:

  1. O nome do campo - o nome exato que você deu ao campo do formulário.
  2. Um nome "humano" para este campo, que será inserido no erro mensagem. Por exemplo, se o seu campo é chamado de "usuário", você pode dar um nome humano de "Username".
  3. As regras de validação para este campo de formulário.
  4. (opcional) Defina mensagens de erro personalizadas em quaisquer regras dadas para o campo atual. Se não for fornecido, usará o padrão.

Nota

Se você quiser que o nome do campo seja armazenado em um idioma arquivo, por favor veja Traduzindo Nomes de Campos.

Aqui está um exemplo. No seu Controller (Form.php), adicione este código apenas abaixo do método de inicialização de validação:

$this->form_validation->set_rules('username', 'Username', 'required');
$this->form_validation->set_rules('password', 'Password', 'required');
$this->form_validation->set_rules('passconf', 'Password Confirmation', 'required');
$this->form_validation->set_rules('email', 'Email', 'required');

Seu Controller agora deve ficar assim:

<?php

class Form extends CI_Controller {

        public function index()
        {
                $this->load->helper(array('form', 'url'));

                $this->load->library('form_validation');

                $this->form_validation->set_rules('username', 'Username', 'required');
                $this->form_validation->set_rules('password', 'Password', 'required',
                        array('required' => 'You must provide a %s.')
                );
                $this->form_validation->set_rules('passconf', 'Password Confirmation', 'required');
                $this->form_validation->set_rules('email', 'Email', 'required');

                if ($this->form_validation->run() == FALSE)
                {
                        $this->load->view('myform');
                }
                else
                {
                        $this->load->view('formsuccess');
                }
        }
}

Agora envie o formulário com os campos em branco e você verá o erro mensagens. Se você enviar o formulário com todos os campos preenchidos, você veja sua página de sucesso.

Definindo regras usando uma matriz¶

Antes de prosseguir, deve-se observar que o método de configuração de regras pode ser passado uma matriz, se você preferir definir todas as suas regras em uma ação. E se você usa essa abordagem, você deve nomear suas chaves de matriz conforme indicado:

$config = array(
        array(
                'field' => 'username',
                'label' => 'Username',
                'rules' => 'required'
        ),
        array(
                'field' => 'password',
                'label' => 'Password',
                'rules' => 'required',
                'errors' => array(
                        'required' => 'You must provide a %s.',
                ),
        ),
        array(
                'field' => 'passconf',
                'label' => 'Password Confirmation',
                'rules' => 'required'
        ),
        array(
                'field' => 'email',
                'label' => 'Email',
                'rules' => 'required'
        )
);

$this->form_validation->set_rules($config);

Regras em cascata¶

O CodeIgniter permite que você canalize várias regras juntas. Vamos tentar. mudança suas regras no terceiro parâmetro do método de configuração de regras, assim:

$this->form_validation->set_rules(
        'username', 'Username',
        'required|min_length[5]|max_length[12]|is_unique[users.username]',
        array(
                'required'      => 'You have not provided %s.',
                'is_unique'     => 'This %s already exists.'
        )
);
$this->form_validation->set_rules('password', 'Password', 'required');
$this->form_validation->set_rules('passconf', 'Password Confirmation', 'required|matches[password]');
$this->form_validation->set_rules('email', 'Email', 'required|valid_email|is_unique[users.email]');

O código acima define as seguintes regras:

  1. O campo de nome de usuário não deve ter menos de 5 caracteres e não mais que 12.
  2. O campo da senha deve corresponder ao campo de confirmação da senha.
  3. O campo de email deve conter um endereço de email válido.

De uma chance! Envie seu formulário sem os dados corretos e você verá novas mensagens de erro que correspondem às novas regras. Existem numerosos regras disponíveis sobre as quais você pode ler na referência de validação.

Nota

You can also pass an array of rules to set_rules(), em vez de uma string. Exemplo:

$this->form_validation->set_rules('username', 'Username', array('required', 'min_length[5]'));

Prepping Data¶

Além do método de validação, como os que usamos acima, você também pode preparar seus dados de várias maneiras. Por exemplo, você pode configurar regras como esta:

$this->form_validation->set_rules('username', 'Username', 'trim|required|min_length[5]|max_length[12]');
$this->form_validation->set_rules('password', 'Password', 'trim|required|min_length[8]');
$this->form_validation->set_rules('passconf', 'Password Confirmation', 'trim|required|matches[password]');
$this->form_validation->set_rules('email', 'Email', 'trim|required|valid_email');

No exemplo acima, estamos "cortando" os campos, verificando o comprimento onde necessário e certificando-se de que ambos os campos de senha coincidam.

Qualquer função PHP nativa que aceita um parâmetro pode ser usada como regra, como `` htmlspecialchars() ``, `` trim() ``, etc.

Nota

Você geralmente vai querer usar as funções de preparação after as regras de validação, portanto, se houver um erro, o os dados originais serão mostrados no formulário.

Callbacks: seus próprios métodos de validação¶

o validation system supports callbacks to your own validation métodos. Isso permite que você estenda a classe de validação para atender às suas necessidades. necessidades. Por exemplo, se você precisar executar uma Query de Database para ver se o usuário está escolhendo um nome de usuário exclusivo, você pode criar um método de retorno de chamada isso faz isso. Vamos criar um exemplo disso.

No seu Controller, altere a regra "username" para isso:

$this->form_validation->set_rules('username', 'Username', 'callback_username_check');

on add a new method called username_check() to your controller. Veja como seu Controller deve estar agora:

<?php

class Form extends CI_Controller {

        public function index()
        {
                $this->load->helper(array('form', 'url'));

                $this->load->library('form_validation');

                $this->form_validation->set_rules('username', 'Username', 'callback_username_check');
                $this->form_validation->set_rules('password', 'Password', 'required');
                $this->form_validation->set_rules('passconf', 'Password Confirmation', 'required');
                $this->form_validation->set_rules('email', 'Email', 'required|is_unique[users.email]');

                if ($this->form_validation->run() == FALSE)
                {
                        $this->load->view('myform');
                }
                else
                {
                        $this->load->view('formsuccess');
                }
        }

        public function username_check($str)
        {
                if ($str == 'test')
                {
                        $this->form_validation->set_message('username_check', 'o {field} field can not be the word "test"');
                        return FALSE;
                }
                else
                {
                        return TRUE;
                }
        }

}

Recarregue seu formulário e envie-o com a palavra "teste" como o nome de usuário. Você pode ver que os dados do campo de formulário foram passados para o seu método de retorno de chamada para você processar.

Para invocar um retorno de chamada, basta colocar o nome do método em uma regra, com "Callback_" como regra prefix. Se você precisar receber um extra parâmetro no seu método de retorno de chamada, basta adicioná-lo normalmente após o method name between square brackets, as in: callback_foo[bar], então ele será passado como o segundo argumento do seu método de retorno de chamada.

Nota

Você também pode processar os dados do formulário que são passados para o seu retorno de chamada e devolva-o. Se o seu retorno de chamada retornar algo diferente de booleano TRUE/FALSE assume-se que os dados são seus recém-processados dados de formulário.

Callable: use qualquer coisa como regra¶

Se as regras de retorno de chamada não forem boas o suficiente para você (por exemplo, porque elas são limitado ao seu Controller), não fique desapontado, há mais uma maneira to create custom rules: anything that is_callable() would return TRUE for.

Considere o seguinte exemplo:

$this->form_validation->set_rules(
        'username', 'Username',
        array(
                'required',
                array($this->users_model, 'valid_username')
        )
);

o above code would use the valid_username() method from your Users_model object.

Este é apenas um exemplo, e os retornos de chamada não se limitam aos Models. Você pode usar qualquer objeto/método que aceite o valor do campo como parâmetro. Você também pode usar uma função anônima:

$this->form_validation->set_rules(
        'username', 'Username',
        array(
                'required',
                function($value)
                {
                        // Check $value
                }
        )
);

Claro que, uma vez que uma regra de Callable por si só não é uma string, não é um nome de regra também. Isso é um problema quando você quer definir mensagens de erro para eles. Para contornar esse problema, você pode colocar regras como o segundo elemento de um array, sendo o primeiro o nome da regra:

$this->form_validation->set_rules(
        'username', 'Username',
        array(
                'required',
                array('username_callable', array($this->users_model, 'valid_username'))
        )
);

Versão da função anônima:

$this->form_validation->set_rules(
        'username', 'Username',
        array(
                'required',
                array(
                        'username_callable',
                        function($str)
                        {
                                // Check validity of $str and return TRUE or FALSE
                        }
                )
        )
);

Configurando Mensagens de Erro¶

Todas as mensagens de erro nativas estão localizadas no seguinte idioma file: system/language/english/form_validation_lang.php

Para definir sua própria mensagem personalizada global para uma regra, você pode estender/substituir o arquivo de idioma, criando o seu próprio no application/language/english/form_validation_lang.php (consulte Mais informação sobre isso no Classe de idioma documentation), ou use o seguinte método:

$this->form_validation->set_message('rule', 'Error Message');

Se você precisar definir uma mensagem de erro personalizada para um campo específico em alguma regra particular, use o método set_rules():

$this->form_validation->set_rules('field_name', 'Field Label', 'rule1|rule2|rule3',
        array('rule2' => 'Error Message on rule2 for this field_name')
);

Onde regra corresponde ao nome de uma regra específica e Erro Mensagem é o texto que você gostaria de exibir.

Se você quiser incluir o nome "humano" de um campo ou o campo opcional parâmetro algumas regras permitem (como max_length), você pode adicionar o {field} and {param} tags para sua mensagem, respectivamente:

$this->form_validation->set_message('min_length', '{field} must have at least {param} characters.');

Em um campo com o nome humano Username e uma regra de min_length [5], um erro seria exibido: "O nome de usuário deve ter pelo menos 5 caracteres".

Nota

o old sprintf() método de usar %s em suas mensagens de erro ainda funcionará, no entanto, irá substituir as tags acima. Você deve Use um ou outro.

No exemplo de regra de retorno de chamada acima, a mensagem de erro foi definida passando o nome do método (sem o prefixo "callback_"):

$this->form_validation->set_message('username_check')

Traduzindo Nomes de Campos¶

Se você gostaria de armazenar o nome "humano" que você passou para o set_rules() método em um arquivo de idioma e, portanto, tornar o nome capaz de ser traduzido, veja como:

Primeiro, prefixar seu nome "humano" com lang:, como neste exemplo:

$this->form_validation->set_rules('first_name', 'lang:first_name', 'required');

on, store the name in one of your language file arrays (without the prefix):

$lang['first_name'] = 'First Name';

Nota

Se você armazenar seu item de matriz em um arquivo de idioma que não é carregado automaticamente pelo CI, você precisará lembrar de carregá-lo no seu Controller usando:

$this->lang->load('file_name');

Veja o Classe de idioma página para mais informações sobre arquivos de idioma.

Alterando os Delimitadores de Erros¶

By default, the Validação de Formulário class adds a paragraph tag (<p>) around cada mensagem de erro mostrada. Você pode alterar esses delimitadores globalmente, individualmente ou altere os padrões em um arquivo de configuração.

  1. Mudando os delimitadores globalmente Para alterar globalmente os delimitadores de erros, no seu método de Controller, just after loading the Validação de Formulário class, add this:

    $this->form_validation->set_error_delimiters('<div class="error">', '</div>');

    Neste exemplo, mudamos para usar tags div.

  2. Alterando os delimitadores individualmente Cada uma das duas funções de geração de erro mostradas neste tutorial pode ser fornecido seus próprios delimitadores da seguinte forma:

    <?php echo form_error('field name', '<div class="error">', '</div>'); ?>

    Or:

    <?php echo validation_errors('<div class="error">', '</div>'); ?>

  3. Definir delimitadores em um arquivo de configuração Você pode adicionar seus delimitadores de erros em application/config/form_validation.php da seguinte maneira:

    $config['error_prefix'] = '<div class="error_prefix">';
$config['error_suffix'] = '</div>';

Salvando conjuntos de regras de validação em um arquivo de configuração¶

A nice feature of the Validação de Formulário class is that it permits you to armazenar todas as suas regras de validação para todo o seu aplicativo em uma configuração file. You can organize these rules into “groups”. ose groups can seja carregado automaticamente quando um Controller/método correspondente chamado, ou você pode chamar manualmente cada conjunto conforme necessário.

Como salvar suas regras¶

Para armazenar suas regras de validação, basta criar um arquivo chamado form_validation.php na sua pasta application/config /. Nesse arquivo você colocará uma matriz chamada $config com suas regras. Como mostrado anteriormente, o array de validação terá esse protótipo:

$config = array(
        array(
                'field' => 'username',
                'label' => 'Username',
                'rules' => 'required'
        ),
        array(
                'field' => 'password',
                'label' => 'Password',
                'rules' => 'required'
        ),
        array(
                'field' => 'passconf',
                'label' => 'Password Confirmation',
                'rules' => 'required'
        ),
        array(
                'field' => 'email',
                'label' => 'Email',
                'rules' => 'required'
        )
);

Seu arquivo de regras de validação será carregado automaticamente e usado quando você call the run() method.

Please note that you MUST name your $config array.

Criando Conjuntos de Regras¶

Para organizar suas regras em "conjuntos", é necessário colocá-las em "sub arrays". Considere o seguinte exemplo, mostrando dois conjuntos de regras. Nós arbitrariamente chamamos essas duas regras de "inscrição" e "email". Você pode nomear suas regras como quiser:

$config = array(
        'signup' => array(
                array(
                        'field' => 'username',
                        'label' => 'Username',
                        'rules' => 'required'
                ),
                array(

Chamando um grupo de regras específico¶

Para chamar um grupo específico, você passará o nome dele para o run() método. Por exemplo, para chamar a regra de inscrição, você fará isso:

if ($this->form_validation->run('signup') == FALSE)
{
        $this->load->view('myform');
}
else
{
        $this->load->view('formsuccess');
}

Associando um método de Controller a um grupo de regras¶

Um método alternativo (e mais automático) de chamar um grupo de regras é nomeá-lo de acordo com a classe/método do Controller que você pretende usá-lo com. Por exemplo, digamos que você tenha um Controller chamado Member e um método nomeado inscrição. Veja como sua turma pode se parecer:

<?php

class Member extends CI_Controller {

        public function signup()
        {
                $this->load->library('form_validation');

                if ($this->form_validation->run() == FALSE)
                {
                        $this->load->view('myform');
                }
                else
                {
                        $this->load->view('formsuccess');
                }
        }
}

Em seu arquivo de configuração de validação, você nomeará seu grupo de regras member/signup:

$config = array(
        'member/signup' => array(
                array(
                        'field' => 'username',
                        'label' => 'Username',
                        'rules' => 'required'
                ),
                array(
                        'field' => 'password',
                        'label' => 'Password',
                        'rules' => 'required'
                ),
                array(
                        'field' => 'passconf',
                        'label' => 'PasswordConfirmation',
                        'rules' => 'required'
                ),
                array(
                        'field' => 'email',
                        'label' => 'Email',
                        'rules' => 'required'
                )
        )
);

Quando um grupo de regras é nomeado identicamente a uma classe/método will be used automatically when the run() method is invoked from that class/method.

Usando matrizes como nomes de campo¶

o Validação de Formulário class supports the use of arrays as field names. Considere este exemplo:

<input type="text" name="options[]" value="" size="50" />

Se você usar um array como um nome de campo, você deve usar o array EXACT nome no Funções Helperes que exigem o nome do campo e como seu nome de campo da Regra de validação.

Por exemplo, para definir uma regra para o campo acima, você usaria:

$this->form_validation->set_rules('options[]', 'Options', 'required');

Ou, para mostrar um erro para o campo acima, você usaria:

<?php echo form_error('options[]'); ?>

Ou para preencher novamente o campo que você usaria:

<input type="text" name="options[]" value="<?php echo set_value('options[]'); ?>" size="50" />

Você também pode usar matrizes multidimensionais como nomes de campo. Por exemplo:

<input type="text" name="options[size]" value="" size="50" />

Ou até mesmo:

<input type="text" name="sports[nba][basketball]" value="" size="50" />

As with our first example, you must use the exact array nome no funções Helperes:

<?php echo form_error('sports[nba][basketball]'); ?>

Se você estiver usando caixas de seleção (ou outros campos) que opções, não se esqueça de deixar um colchete vazio após cada opção, então que todas as seleções serão adicionadas à matriz do POST:

<input type="checkbox" name="options[]" value="red" />
<input type="checkbox" name="options[]" value="blue" />
<input type="checkbox" name="options[]" value="green" />

Ou se você usa um array multidimensional:

<input type="checkbox" name="options[color][]" value="red" />
<input type="checkbox" name="options[color][]" value="blue" />
<input type="checkbox" name="options[color][]" value="green" />

Quando você usa uma função Helper, você também inclui o suporte:

<?php echo form_error('options[color][]'); ?>

Overview¶

Antes de explicar a abordagem do CodeIgniter para validação de dados, vamos descreva o cenário ideal:

  1. Um formulário é exibido.
  2. Você preenche e envia.
  3. Se você enviou algo inválido ou talvez tenha perdido um item, o formulário é reexibido contendo seus dados juntamente com um mensagem de erro descrevendo o problema.
  4. Esse processo continua até que você tenha enviado um formulário válido.

No final de recebimento, o script deve:

  1. Verifique os dados necessários.
  2. Verifique se os dados são do tipo correto e atende aos requisitos corretos. critério. Por exemplo, se um nome de usuário for enviado, ele deve ser validado para conter apenas caracteres permitidos. Deve ser de um comprimento mínimo e não deve exceder um comprimento máximo. O nome de usuário não pode seja o nome de usuário existente de outra pessoa ou talvez até mesmo uma palavra reservada. Etc.
  3. Sanitize os dados para segurança.
  4. Pré-formate os dados, se necessário (os dados precisam ser aparados? HTML codificado? Etc.)
  5. Prepare os dados para inserção no Database.

Embora não haja nada terrivelmente complexo sobre o processo acima, geralmente requer uma quantidade significativa de código e para exibir erros mensagens, várias estruturas de controle geralmente são colocadas dentro do formulário HTML A validação de formulários, embora simples de criar, geralmente é muito confusa e tedioso para implementar.

Validação de Formulário Tutorial¶

O que segue é um tutorial "prático" para implementar o Formulário CodeIgniters Validação.

Para implementar a validação de formulário, você precisará de três coisas:

  1. A View arquivo contendo um formulário.
  2. Um arquivo de visualização contendo uma mensagem de "sucesso" a ser exibida submissão bem sucedida.
  3. A controller método para receber e processar os dados enviados.

Vamos criar essas três coisas, usando um formulário de inscrição de membro como example.

o Form¶

<input type="text" name="password" value="" size="50" />

<input type="text" name="email" value="" size="50" />

O Controller¶

Usando um editor de texto, crie um Controller chamado Form.php. Nele, coloque este código e salve-o em sua pasta application/controllers /:

<?php

class Form extends CI_Controller {

        public function index()
        {
                $this->load->helper(array('form', 'url'));

                $this->load->library('form_validation');

                if ($this->form_validation->run() == FALSE)
                {
                        $this->load->view('myform');
                }
                else
                {
                        $this->load->view('formsuccess');
                }
        }
}

Definindo regras de validação¶

O CodeIgniter permite que você defina quantas regras de validação forem necessárias para um dado campo, cascateando-os em ordem, e ainda permite preparar e pré-processar os dados de campo ao mesmo tempo. Para definir regras de validação você will use the set_rules() method:

$this->form_validation->set_rules();

O método acima leva three parâmetros como entrada:

  1. O nome do campo - o nome exato que você deu ao campo do formulário.
  2. Um nome "humano" para este campo, que será inserido no erro mensagem. Por exemplo, se o seu campo é chamado de "usuário", você pode dar um nome humano de "Username".
  3. As regras de validação para este campo de formulário.
  4. (opcional) Defina mensagens de erro personalizadas em quaisquer regras dadas para o campo atual. Se não for fornecido, usará o padrão.

Nota

Se você quiser que o nome do campo seja armazenado em um idioma arquivo, por favor veja Traduzindo Nomes de Campos.

Aqui está um exemplo. No seu Controller (Form.php), adicione este código apenas abaixo do método de inicialização de validação:

$this->form_validation->set_rules('username', 'Username', 'required');
$this->form_validation->set_rules('password', 'Password', 'required');
$this->form_validation->set_rules('passconf', 'Password Confirmation', 'required');
$this->form_validation->set_rules('email', 'Email', 'required');

Seu Controller agora deve ficar assim:

<?php

class Form extends CI_Controller {

        public function index()
        {
                $this->load->helper(array('form', 'url'));

                $this->load->library('form_validation');

                $this->form_validation->set_rules('username', 'Username', 'required');
                $this->form_validation->set_rules('password', 'Password', 'required',
                        array('required' => 'You must provide a %s.')
                );
                $this->form_validation->set_rules('passconf', 'Password Confirmation', 'required');
                $this->form_validation->set_rules('email', 'Email', 'required');

                if ($this->form_validation->run() == FALSE)
                {
                        $this->load->view('myform');
                }
                else
                {
                        $this->load->view('formsuccess');
                }
        }
}

Agora envie o formulário com os campos em branco e você verá o erro mensagens. Se você enviar o formulário com todos os campos preenchidos, você veja sua página de sucesso.

Definindo regras usando uma matriz¶

Antes de prosseguir, deve-se observar que o método de configuração de regras pode ser passado uma matriz, se você preferir definir todas as suas regras em uma ação. E se você usa essa abordagem, você deve nomear suas chaves de matriz conforme indicado:

$config = array(
        array(
                'field' => 'username',
                'label' => 'Username',
                'rules' => 'required'
        ),
        array(
                'field' => 'password',
                'label' => 'Password',
                'rules' => 'required',
                'errors' => array(
                        'required' => 'You must provide a %s.',
                ),
        ),
        array(
                'field' => 'passconf',
                'label' => 'Password Confirmation',
                'rules' => 'required'
        ),
        array(
                'field' => 'email',
                'label' => 'Email',
                'rules' => 'required'
        )
);

Regras em cascata¶

O CodeIgniter permite que você canalize várias regras juntas. Vamos tentar. mudança suas regras no terceiro parâmetro do método de configuração de regras, assim:

$this->form_validation->set_rules(
        'username', 'Username',
        'required|min_length[5]|max_length[12]|is_unique[users.username]',
        array(
                'required'      => 'You have not provided %s.',
                'is_unique'     => 'This %s already exists.'
        )
);
$this->form_validation->set_rules('password', 'Password', 'required');
$this->form_validation->set_rules('passconf', 'Password Confirmation', 'required|matches[password]');
$this->form_validation->set_rules('email', 'Email', 'required|valid_email|is_unique[users.email]');

O código acima define as seguintes regras:

  1. O campo de nome de usuário não deve ter menos de 5 caracteres e não mais que 12.
  2. O campo da senha deve corresponder ao campo de confirmação da senha.
  3. O campo de email deve conter um endereço de email válido.

De uma chance! Envie seu formulário sem os dados corretos e você verá novas mensagens de erro que correspondem às novas regras. Existem numerosos regras disponíveis sobre as quais você pode ler na referência de validação.

Nota

You can also pass an array of rules to set_rules(), em vez de uma string. Exemplo:

$this->form_validation->set_rules('username', 'Username', array('required', 'min_length[5]'));

Prepping Data¶

Além do método de validação, como os que usamos acima, você também pode preparar seus dados de várias maneiras. Por exemplo, você pode configurar regras como esta:

$this->form_validation->set_rules('username', 'Username', 'trim|required|min_length[5]|max_length[12]');
$this->form_validation->set_rules('password', 'Password', 'trim|required|min_length[8]');
$this->form_validation->set_rules('passconf', 'Password Confirmation', 'trim|required|matches[password]');
$this->form_validation->set_rules('email', 'Email', 'trim|required|valid_email');

No exemplo acima, estamos "cortando" os campos, verificando o comprimento onde necessário e certificando-se de que ambos os campos de senha coincidam.

Qualquer função PHP nativa que aceita um parâmetro pode ser usada como regra, como `` htmlspecialchars() ``, `` trim() ``, etc.

Nota

Você geralmente vai querer usar as funções de preparação after as regras de validação, portanto, se houver um erro, o os dados originais serão mostrados no formulário.

Callbacks: seus próprios métodos de validação¶

o validation system supports callbacks to your own validation métodos. Isso permite que você estenda a classe de validação para atender às suas necessidades. necessidades. Por exemplo, se você precisar executar uma Query de Database para ver se o usuário está escolhendo um nome de usuário exclusivo, você pode criar um método de retorno de chamada isso faz isso. Vamos criar um exemplo disso.

No seu Controller, altere a regra "username" para isso:

$this->form_validation->set_rules('username', 'Username', 'callback_username_check');

on add a new method called username_check() to your controller. Veja como seu Controller deve estar agora:

<?php

class Form extends CI_Controller {

        public function index()
        {
                $this->load->helper(array('form', 'url'));

                $this->load->library('form_validation');

                $this->form_validation->set_rules('username', 'Username', 'callback_username_check');
                $this->form_validation->set_rules('password', 'Password', 'required');
                $this->form_validation->set_rules('passconf', 'Password Confirmation', 'required');
                $this->form_validation->set_rules('email', 'Email', 'required|is_unique[users.email]');

                if ($this->form_validation->run() == FALSE)
                {
                        $this->load->view('myform');
                }
                else
                {
                        $this->load->view('formsuccess');
                }
        }

        public function username_check($str)
        {
                if ($str == 'test')
                {
                        $this->form_validation->set_message('username_check', 'o {field} field can not be the word "test"');
                        return FALSE;
                }
                else
                {
                        return TRUE;
                }
        }

}

Recarregue seu formulário e envie-o com a palavra "teste" como o nome de usuário. Você pode ver que os dados do campo de formulário foram passados para o seu método de retorno de chamada para você processar.

Para invocar um retorno de chamada, basta colocar o nome do método em uma regra, com "Callback_" como regra prefix. Se você precisar receber um extra parâmetro no seu método de retorno de chamada, basta adicioná-lo normalmente após o method name between square brackets, as in: callback_foo[bar], então ele será passado como o segundo argumento do seu método de retorno de chamada.

Nota

Você também pode processar os dados do formulário que são passados para o seu retorno de chamada e devolva-o. Se o seu retorno de chamada retornar algo diferente de booleano TRUE/FALSE assume-se que os dados são seus recém-processados form data.

Callable: use qualquer coisa como regra¶

Se as regras de retorno de chamada não forem boas o suficiente para você (por exemplo, porque elas são limitado ao seu Controller), não fique desapontado, há mais uma maneira to create custom rules: anything that is_callable() would return TRUE for.

Considere o seguinte exemplo:

$this->form_validation->set_rules(
        'username', 'Username',
        array(
                'required',
                array($this->users_model, 'valid_username')
        )
);

o above code would use the valid_username() method from your Users_model object.

Este é apenas um exemplo, e os retornos de chamada não se limitam aos Models. Você pode usar qualquer objeto/método que aceite o valor do campo como parâmetro. Você também pode usar uma função anônima:

$this->form_validation->set_rules(
        'username', 'Username',
        array(
                'required',
                function($value)
                {
                        // Check $value
                }
        )
);

Claro que, uma vez que uma regra de Callable por si só não é uma string, não é um nome de regra também. Isso é um problema quando você quer definir mensagens de erro para eles. Para contornar esse problema, você pode colocar regras como o segundo elemento de um array, sendo o primeiro o nome da regra:

$this->form_validation->set_rules(
        'username', 'Username',
        array(
                'required',
                array('username_callable', array($this->users_model, 'valid_username'))
        )
);

Versão da função anônima:

$this->form_validation->set_rules(
        'username', 'Username',
        array(
                'required',
                array(
                        'username_callable',
                        function($str)
                        {
                                // Check validity of $str and return TRUE or FALSE
                        }
                )
        )
);

Configurando Mensagens de Erro¶

Todas as mensagens de erro nativas estão localizadas no seguinte idioma file: system/language/english/form_validation_lang.php

Para definir sua própria mensagem personalizada global para uma regra, você pode estender/substituir o arquivo de idioma, criando o seu próprio no application/language/english/form_validation_lang.php (read more sobre isso no Classe de idioma documentation), ou use o seguinte método:

$this->form_validation->set_message('rule', 'Error Message');

Se você precisar definir uma mensagem de erro personalizada para um campo específico em alguma regra particular, use o método set_rules():

$this->form_validation->set_rules('field_name', 'Field Label', 'rule1|rule2|rule3',
        array('rule2' => 'Error Message on rule2 for this field_name')
);

Onde regra corresponde ao nome de uma regra específica e Erro Mensagem é o texto que você gostaria de exibir.

Se você quiser incluir o nome "humano" de um campo ou o campo opcional parâmetro algumas regras permitem (como max_length), você pode adicionar o {field} and {param} tags para sua mensagem, respectivamente:

$this->form_validation->set_message('min_length', '{field} must have at least {param} characters.');

Em um campo com o nome humano Username e uma regra de min_length [5], um erro seria exibido: "O nome de usuário deve ter pelo menos 5 caracteres".

Nota

o old sprintf() método de usar %s em suas mensagens de erro ainda funcionará, no entanto, irá substituir as tags acima. Você deve Use um ou outro.

No exemplo de regra de retorno de chamada acima, a mensagem de erro foi definida passando o nome do método (sem o prefixo "callback_"):

$this->form_validation->set_message('username_check')

Traduzindo Nomes de Campos¶

Se você gostaria de armazenar o nome "humano" que você passou para o set_rules() método em um arquivo de idioma e, portanto, tornar o nome capaz de ser traduzido, veja como:

Primeiro, prefixar seu nome "humano" com lang:, como neste exemplo:

$this->form_validation->set_rules('first_name', 'lang:first_name', 'required');

on, store the name in one of your language file arrays (without the prefix):

$lang['first_name'] = 'First Name';

Nota

Se você armazenar seu item de matriz em um arquivo de idioma que não é carregado automaticamente pelo CI, você precisará lembrar de carregá-lo no seu Controller usando:

$this->lang->load('file_name');

See the Classe de idioma página para mais informações sobre arquivos de idioma.

Alterando os Delimitadores de Erros¶

By default, the Validação de Formulário class adds a paragraph tag (<p>) around cada mensagem de erro mostrada. Você pode alterar esses delimitadores globalmente, individualmente ou altere os padrões em um arquivo de configuração.

  1. Mudando os delimitadores globalmente Para alterar globalmente os delimitadores de erros, no seu método de Controller, just after loading the Validação de Formulário class, add this:

    $this->form_validation->set_error_delimiters('<div class="error">', '</div>');

    Neste exemplo, mudamos para usar tags div.

  2. Alterando os delimitadores individualmente Cada uma das duas funções de geração de erro mostradas neste tutorial pode ser fornecido seus próprios delimitadores da seguinte forma:

    <?php echo form_error('field name', '<div class="error">', '</div>'); ?>

    Or:

    <?php echo validation_errors('<div class="error">', '</div>'); ?>

  3. Definir delimitadores em um arquivo de configuração Você pode adicionar seus delimitadores de erros em application/config/form_validation.php da seguinte maneira:

    $config['error_prefix'] = '<div class="error_prefix">';
$config['error_suffix'] = '</div>';

$data = array(

$this->form_validation->set_data($data);

Salvando conjuntos de regras de validação em um arquivo de configuração¶

A nice feature of the Validação de Formulário class is that it permits you to armazenar todas as suas regras de validação para todo o seu aplicativo em uma configuração file. You can organize these rules into “groups”. ose groups can seja carregado automaticamente quando um Controller/método correspondente chamado, ou você pode chamar manualmente cada conjunto conforme necessário.

Como salvar suas regras¶

Para armazenar suas regras de validação, basta criar um arquivo chamado form_validation.php na sua pasta application/config /. Nesse arquivo você colocará uma matriz chamada $config com suas regras. Como mostrado anteriormente, o array de validação terá esse protótipo:

$config = array(
        array(
                'field' => 'username',
                'label' => 'Username',
                'rules' => 'required'
        ),
        array(
                'field' => 'password',
                'label' => 'Password',
                'rules' => 'required'
        ),
        array(
                'field' => 'passconf',
                'label' => 'Password Confirmation',
                'rules' => 'required'
        ),
        array(
                'field' => 'email',
                'label' => 'Email',
                'rules' => 'required'
        )
);

Seu arquivo de regras de validação será carregado automaticamente e usado quando você call the run() method.

Please note that you MUST name your $config array.

Criando Conjuntos de Regras¶

Para organizar suas regras em "conjuntos", é necessário colocá-las em "sub arrays". Considere o seguinte exemplo, mostrando dois conjuntos de regras. Nós arbitrariamente chamamos essas duas regras de "inscrição" e "email". Você pode nomear suas regras como quiser:

$config = array(
        'signup' => array(
                array(
                        'field' => 'username',
                        'label' => 'Username',
                        'rules' => 'required'
                ),
                array(
                        'field' => 'password',
                        'label' => 'Password',
                        'rules' => 'required'
                ),
                array(
                        'field' => 'passconf',
                        'label' => 'Password Confirmation',
                        'rules' => 'required'
                ),
                array(
                        'field' => 'email',
                        'label' => 'Email',
                        'rules' => 'required'
                )
        ),
        'email' => array(
                array(
                        'field' => 'emailaddress',
                        'label' => 'EmailAddress',
                        'rules' => 'required|valid_email'
                ),
                array(
                        'field' => 'name',
                        'label' => 'Name',
                        'rules' => 'required|alpha'
                ),
                array(
                        'field' => 'title',
                        'label' => 'Title',
                        'rules' => 'required'
                ),
                array(
                        'field' => 'message',
                        'label' => 'MessageBody',
                        'rules' => 'required'
                )
        )
);

Chamando um grupo de regras específico¶

Para chamar um grupo específico, você passará o nome dele para o run() método. Por exemplo, para chamar a regra de inscrição, você fará isso:

if ($this->form_validation->run('signup') == FALSE)

class Member extends CI_Controller {

Usando matrizes como nomes de campo¶

o Validação de Formulário class supports the use of arrays as field names. Considere este exemplo:

<input type="text" name="options[]" value="" size="50" />

Se você usar um array como um nome de campo, você deve usar o array EXACT nome no Funções Helperes que exigem o nome do campo e como seu nome de campo da Regra de validação.

Por exemplo, para definir uma regra para o campo acima, você usaria:

$this->form_validation->set_rules('options[]', 'Options', 'required');

Ou, para mostrar um erro para o campo acima, você usaria:

<?php echo form_error('options[]'); ?>

Ou para preencher novamente o campo que você usaria:

<input type="text" name="options[]" value="<?php echo set_value('options[]'); ?>" size="50" />

Você também pode usar matrizes multidimensionais como nomes de campo. Por exemplo:

<input type="text" name="options[size]" value="" size="50" />

Or even:

<input type="text" name="sports[nba][basketball]" value="" size="50" />

As with our first example, you must use the exact array nome no funções Helperes:

<?php echo form_error('sports[nba][basketball]'); ?>

Se você estiver usando caixas de seleção (ou outros campos) que opções, não se esqueça de deixar um colchete vazio após cada opção, então que todas as seleções serão adicionadas à matriz do POST:

<input type="checkbox" name="options[]" value="red" />
<input type="checkbox" name="options[]" value="blue" />
<input type="checkbox" name="options[]" value="green" />

Ou se você usa um array multidimensional:

<input type="checkbox" name="options[color][]" value="red" />
<input type="checkbox" name="options[color][]" value="blue" />
<input type="checkbox" name="options[color][]" value="green" />

Quando você usa uma função Helper, você também inclui o suporte:

<?php echo form_error('options[color][]'); ?>
