Cache Driver¶

CodeIgniter features wrappers around some of the most popular forms of cache rápido e dinâmico. Todos, exceto o armazenamento em cache baseado em arquivo, exigem server requirements, and a Fatal Exception will be thrown if server requisitos não são cumpridos.

  • Exemplo de uso
  • Class Reference
  • Drivers
    • Alternative PHP Cache (APC) Cache
    • File-based Cache
    • Memcached Cache
    • WinCache Cache
    • Redis Cache
    • Dummy Cache

Exemplo de uso¶

O exemplo a seguir carregará o driver de cache, especifique APC as the driver to use, and fall back to file-based caching if APC is not disponível no ambiente de hospedagem.

$this->load->driver('cache', array('adapter' => 'apc', 'backup' => 'file'));

if ( ! $foo = $this->cache->get('foo'))
{
        echo 'Saving to the cache!<br />';
        $foo = 'foobarbaz!';

        // Save into the cache for 5 minutes
        $this->cache->save('foo', $foo, 300);
}

echo $foo;

You can also prefix cache item names via the key_prefix setting, which is useful to avoid collisions when you’re running multiple applications on the same environment.

$this->load->driver('cache',
        array('adapter' => 'apc', 'backup' => 'file', 'key_prefix' => 'my_')
);

$this->cache->get('foo'); // Will get the cache entry named 'my_foo'

Class Reference¶

class CI_Cache¶
is_supported($driver)¶
Parâmetros:
  • $driver (string) - o nome do driver de armazenamento em cache
Returns:

TRUE se suportado, FALSE se não for
Return type:

bool

Este método é chamado automaticamente ao acessar drivers via $this->cache->get(). However, if the individual drivers are used, make sure to call this method to ensure the driver is supported in the hosting environment.

if ($this->cache->apc->is_supported())
{
        if ($data = $this->cache->apc->get('my_cache'))
        {
                // do things.
        }
}
get($id)¶
Parâmetros:
  • $id (string) – Cache item name
Returns:

Item value or FALSE if not found
Return type:

mixed

Este método tentará buscar um item do armazenamento de cache. Se o item does not exist, the method will return FALSE.

$foo = $this->cache->get('my_cached_item');
save($id, $data[, $ttl = 60[, $raw = FALSE]])¶
Parâmetros:
  • $id (string) – Cache item name
  • $data (mixed) - os dados para salvar
  • $ttl (int) - Time To Live, em segundos (padrão 60)
  • $raw (bool) – Whether to store the raw value
Returns:

Verdadeiro sucesso, FALSE em falha
Return type:

string

Esse método salvará um item no armazenamento de cache. Se a economia falhar, o method will return FALSE.

$this->cache->save('cache_item_id', 'data_to_cache');

Nota

The $raw parameter is only utilized by APC and Memcache, in order to allow usage of increment() and decrement().

delete($id)¶
Parâmetros:
  • $id (string) - nome do item em cache
Returns:

Verdadeiro sucesso, FALSE em falha
Return type:

bool

This method will delete a specific item from the cache store. If item a exclusão falha, o método retornará FALSE.

$this->cache->delete('cache_item_id');
increment($id[, $offset = 1])¶
Parâmetros:
  • $id (string) - ID do cache
  • $offset (int) - Etapa/valor para adicionar
Returns:

Novo valor no sucesso, FALSE em falha
Return type:

mixed

Executa a incrementação atômica de um valor armazenado bruto.

// 'iterator' has a value of 2

$this->cache->increment('iterator'); // 'iterator' is now 3

$this->cache->increment('iterator', 3); // 'iterator' is now 6
decrement($id[, $offset = 1])¶
Parâmetros:
  • $id (string) - ID do cache
  • $offset (int) - Passo/valor a reduzir por
Returns:

Novo valor no sucesso, FALSE em falha
Return type:

mixed

Performs atomic decrementation of a raw stored value.

// 'iterator' has a value of 6

$this->cache->decrement('iterator'); // 'iterator' is now 5

$this->cache->decrement('iterator', 2); // 'iterator' is now 3
clean()¶
Returns:Verdadeiro sucesso, FALSE em falha
Return type:bool

This method will ‘clean’ the entire cache. If the deletion of the os arquivos de cache falham, o método retornará FALSE.

$this->cache->clean();

Returns:Information on the entire cache database
Return type:mixed

This method will return information on the entire cache.

var_dump($this->cache->cache_info());

Nota

The information returned and the structure of the data is dependent em qual adaptador está sendo usado.

get_metadata($id)¶
Parâmetros:
  • $id (string) – Cache item name
Returns:

Metadata for the cached item
Return type:

mixed

This method will return detailed information on a specific item in the cache.

var_dump($this->cache->get_metadata('my_cached_item'));

Nota

The information returned and the structure of the data is dependent em qual adaptador está sendo usado.

Drivers¶

Alternative PHP Cache (APC) Cache¶

All of the methods listed above can be accessed without passing a specific adapter to the driver loader as follows:

$this->load->driver('cache');
$this->cache->apc->save('foo', 'bar', 10);

Para mais informações sobre a APC, consulte http://php.net/apc.

File-based Cache¶

Unlike caching from the Classe de saída, the driver file-based caching permite que pedaços de arquivos de visualização sejam armazenados em cache. Use isso com cuidado e certifique-se de benchmark sua aplicação, como um ponto pode vir onde o disco AE/S negará ganhos positivos pelo armazenamento em cache.

All of the methods listed above can be accessed without passing a specific adapter to the driver loader as follows:

$this->load->driver('cache');
$this->cache->file->save('foo', 'bar', 10);

Memcached Cache¶

Multiple Memcached servers can be specified in the memcached.php configuration file, located in the _application/config/* directory.

All of the methods listed above can be accessed without passing a specific adapter to the driver loader as follows:

$this->load->driver('cache');
$this->cache->memcached->save('foo', 'bar', 10);

For more information on Memcached, please see http://php.net/memcached.

WinCache Cache¶

Under Windows, you can also utilize the WinCache driver.

All of the methods listed above can be accessed without passing a specific adapter to the driver loader as follows:

$this->load->driver('cache');
$this->cache->wincache->save('foo', 'bar', 10);

For more information on WinCache, please see http://php.net/wincache.

Redis Cache¶

Redis é um armazenamento de valor de chave na memória que pode operar no modo de cache LRU. Para usá-lo, você precisa Servidor Redis e phpredis PHP extension.

Opções de configuração para conectar ao servidor redis devem ser armazenadas no arquivo application/config/redis.php. Available options are:

$config['socket_type'] = 'tcp'; //`tcp` or `unix`
$config['socket'] = '/var/run/redis.sock'; // in case of `unix` socket type
$config['host'] = '127.0.0.1';
$config['password'] = NULL;
$config['port'] = 6379;
$config['timeout'] = 0;

All of the methods listed above can be accessed without passing a specific adapter to the driver loader as follows:

$this->load->driver('cache');
$this->cache->redis->save('foo', 'bar', 10);

Para mais informações sobre Redis, por favor veja http://redis.io.

Dummy Cache¶

This is a caching backend that will always ‘miss.’ It stores no data, but lets you keep your caching code in place in environments that don’t suporte seu cache escolhido.

Cache Driver¶

O CodeIgniter possui wrappers em torno de algumas das formas mais populares de cache rápido e dinâmico. Todos, exceto o armazenamento em cache baseado em arquivo, exigem requisitos do servidor, e uma exceção fatal será lançada se o servidor requisitos não são cumpridos.

  • Exemplo de uso
  • Referência de Classe
  • Drivers
    • Alternative PHP Cache (APC) Cache
    • File-based Cache
    • Memcached Cache
    • WinCache Cache
    • Redis Cache
    • Cache Fictício

Exemplo de uso¶

O exemplo a seguir carregará o driver de cache, especifique APC como o driver para usar, e voltar para o cache baseado em arquivo se APC não é disponível no ambiente de hospedagem.

$this->load->driver('cache', array('adapter' => 'apc', 'backup' => 'file'));

if ( ! $foo = $this->cache->get('foo'))
{
        echo 'Saving to the cache!<br />';
        $foo = 'foobarbaz!';

        // Save into the cache for 5 minutes
        $this->cache->save('foo', $foo, 300);
}

echo $foo;

Você também pode prefixar nomes de itens de cache através do key_prefix definição, o que é útil para evitar colisões quando você está executando vários aplicativos no mesmo ambiente.

$this->load->driver('cache',
        array('adapter' => 'apc', 'backup' => 'file', 'key_prefix' => 'my_')
);

$this->cache->get('foo'); // Will get the cache entry named 'my_foo'

Cache Driver¶

O CodeIgniter possui wrappers em torno de algumas das formas mais populares de cache rápido e dinâmico. Todos, exceto o armazenamento em cache baseado em arquivo, exigem requisitos do servidor, e uma exceção fatal será lançada se o servidor requisitos não são cumpridos.

  • Exemplo de uso

Exemplo de uso¶

O exemplo a seguir carregará o driver de cache, especifique APC como o driver para usar, e voltar para o cache baseado em arquivo se APC não é disponível no ambiente de hospedagem.

$this->load->driver('cache', array('adapter' => 'apc', 'backup' => 'file'));

if ( ! $foo = $this->cache->get('foo'))
{
        echo 'Saving to the cache!<br />';
        $foo = 'foobarbaz!';

        // Save into the cache for 5 minutes
        $this->cache->save('foo', $foo, 300);
}

echo $foo;

Você também pode prefixar nomes de itens de cache através do key_prefix definição, o que é útil para evitar colisões quando você está executando vários aplicativos no mesmo ambiente.

$this->load->driver('cache',
        array('adapter' => 'apc', 'backup' => 'file', 'key_prefix' => 'my_')
);

$this->cache->get('foo'); // Will get the cache entry named 'my_foo'
