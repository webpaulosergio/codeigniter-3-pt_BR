Segurança Class¶

The Segurança Class contains methods that help you create a secure application, processing input data for security.

  • XSS Filtering
  • Cross-site request forgery (CSRF)
  • Class Reference

XSS Filtering¶

O CodeIgniter vem com um filtro de prevenção Cross Site Scripting, que looks for commonly used techniques to trigger JavaScript or other types of code that attempt to hijack cookies or do other malicious things. Se alguma coisa não permitida for encontrada, ela será renderizada com segurança, convertendo the data to character entities.

To filter data through the XSS filter use the xss_clean() method:

$data = $this->security->xss_clean($data);

An optional second parameter, is_image, allows this function to be used to test images for potential XSS attacks, useful for file upload security. When this second parameter is set to TRUE, instead of retornando uma string alterada, a função retorna TRUE se a imagem for safe, and FALSE if it contained potentially malicious information that a navegador pode tentar executar.

if ($this->security->xss_clean($file, TRUE) === FALSE)
{
        // file failed the XSS test
}

Importante

Se você deseja filtrar valores de atributo HTML, use html_escape() instead!

Cross-site request forgery (CSRF)¶

You can enable CSRF protection by altering your application/config/config.php arquivo da seguinte maneira:

$config['csrf_protection'] = TRUE;

Se você usar o Helper de Formulário, então form_open() will automatically insert a hidden csrf field in your forms. If not, então you can use get_csrf_token_name() and get_csrf_hash()

$csrf = array(
        'name' => $this->security->get_csrf_token_name(),
        'hash' => $this->security->get_csrf_hash()
);

...

<input type="hidden" name="<?=$csrf['name'];?>" value="<?=$csrf['hash'];?>" />

Tokens may be either regenerated on every submission (default) or manteve o mesmo ao longo da vida do cookie CSRF. O padrão A regeneração de tokens fornece segurança mais rigorosa, mas pode resultar em preocupações de usabilidade, quando outros tokens se tornam inválidos (back/forward navigation, multiple tabs/windows, asynchronous actions, etc). You may alter this behavior by editing the following config parameter

$config['csrf_regenerate'] = TRUE;

Select URIs can be whitelisted from csrf protection (for example API endpoints expecting externally POSTed content). You can add these URIs by editing the ‘csrf_exclude_uris’ config parameter:

$config['csrf_exclude_uris'] = array('api/person/add');

Expressões regulares também são suportadas (sem distinção entre maiúsculas e minúsculas):

$config['csrf_exclude_uris'] = array(
        'api/record/[0-9]+',
        'api/title/[a-z]+'
);

xss_clean($str[, $is_image = FALSE])¶
Parâmetros:
  • $str (mixed) – Input string or an array of strings
Returns:

XSS-clean data
Return type:

mixed

Tries to remove XSS exploits from the input data and returns the cleaned string. If the optional second parameter is set to true, it will return boolean TRUE if the image is safe to use and FALSE if malicious data was detected in it.

Importante

sanitize_filename($str[, $relative_path = FALSE])¶
Parâmetros:
  • $str (string) - Nome do arquivo/caminho
  • $relative_path (bool) – Whether to preserve any directories in the file path
Returns:

Nome/caminho do arquivo higienizado
Return type:

string

Tries to sanitize filenames in order to prevent directory traversal attempts e outras ameaças de segurança, o que é particularmente útil para arquivos que foram fornecidos via entrada do usuário.

$filename = $this->security->sanitize_filename($this->input->post('filename'));

Return type:string

get_csrf_hash()¶
Returns:CSRF hash
Return type:string

Returns the CSRF hash value. Useful in combination with get_csrf_token_name() for manually building forms or sending valid AJAX POST requests.

entity_decode($str[, $charset = NULL])¶
Parâmetros:
  • $str (string) – Input string
  • $charset (string) – Character set of the input string
Returns:

Entity-decoded string
Return type:

string

This method acts a lot like PHP’s own native html_entity_decode() function in ENT_COMPAT mode, only it tries to detect HTML entities that don’t end in a semicolon because some browsers allow that.

get_random_bytes($length)¶
Parâmetros:
  • $length (int) – Output length
Returns:

A binary stream of random bytes or FALSE on failure
Return type:

string

A convenience method for getting proper random bytes via mcrypt_create_iv(), /dev/urandom or openssl_random_pseudo_bytes() (in that order), if one of them is available.

Usado para gerar tokens CSRF e XSS.

Nota

The output is NOT guaranteed to be cryptographically secure, just the best attempt at that.

Segurança Class¶

The Segurança Class contains methods that help you create a secure aplicação, processando dados de entrada para segurança.

  • Filtragem XSS
  • Falsificação de solicitações entre sites (CSRF)
  • Referência de Classe

Filtragem XSS¶

O CodeIgniter vem com um filtro de prevenção Cross Site Scripting, que procura por técnicas comumente usadas para acionar JavaScript ou outros tipos de código que tenta roubar cookies ou fazer outras coisas maliciosas. Se alguma coisa não permitida for encontrada, ela será renderizada com segurança, convertendo os dados para entidades de caracteres.

To filter data through the XSS filter use the xss_clean() method:

$data = $this->security->xss_clean($data);

An optional second parameter, is_image, allows this function to be used para testar imagens de possíveis ataques XSS, útil para upload de arquivos segurança. Quando este segundo parâmetro é definido como TRUE, em vez de retornando uma string alterada, a função retorna TRUE se a imagem for seguro, e FALSE se continha informações potencialmente maliciosas que um navegador pode tentar executar.

if ($this->security->xss_clean($file, TRUE) === FALSE)
{
        // file failed the XSS test
}

Important

Se você deseja filtrar valores de atributo HTML, use html_escape() instead!

Falsificação de solicitações entre sites (CSRF)¶

Você pode ativar a proteção contra CSRF alterando application/config/config.php arquivo da seguinte maneira:

$config['csrf_protection'] = TRUE;

Se você usar o Helper de Formulário, then form_open() irá inserir automaticamente um campo csrf oculto em seus formulários. Se não, então você pode usar get_csrf_token_name() and get_csrf_hash()

$csrf = array(
        'name' => $this->security->get_csrf_token_name(),
        'hash' => $this->security->get_csrf_hash()
);

...

<input type="hidden" name="<?=$csrf['name'];?>" value="<?=$csrf['hash'];?>" />

Os tokens podem ser regenerados em cada envio (padrão) ou manteve o mesmo ao longo da vida do cookie CSRF. O padrão A regeneração de tokens fornece segurança mais rigorosa, mas pode resultar em preocupações de usabilidade, quando outros tokens se tornam inválidos (back/forward navegação, múltiplas abas/janelas, ações assíncronas, etc). Você pode alterar esse comportamento editando o seguinte parâmetro de configuração

$config['csrf_regenerate'] = TRUE;

URIs selecionados podem ser permitidos na proteção csrf (por exemplo, API endpoints que esperam conteúdo externamente postado). Você pode adicionar esses URIs editando o parâmetro de configuração 'csrf_exclude_uris':

$config['csrf_exclude_uris'] = array('api/person/add');

Expressões regulares também são suportadas (sem distinção entre maiúsculas e minúsculas):

$config['csrf_exclude_uris'] = array(
        'api/record/[0-9]+',
        'api/title/[a-z]+'
);

class CI_Segurança¶
xss_clean($str[, $is_image = FALSE])¶
Parâmetros:
  • $str (mixed) - String de entrada ou um array de strings
Retorna:

Dados limpos de XSS
Tipo de retorno:

mixed

Tenta remover exploits XSS dos dados de entrada e retorna a string limpa. Se o segundo parâmetro opcional for configurado como true, retornará TRUE booleano se a imagem é segura de usar e FALSE se dados maliciosos foram detectados nela.

Important

Este método não é adequado para filtrar valores de atributos HTML! Use html_escape() para isso em vez disso.

sanitize_filename($str[, $relative_path = FALSE])¶
Parâmetros:
  • $str (string) - Nome do arquivo/caminho
  • $relative_path (bool) - Se deseja preservar quaisquer diretórios no caminho do arquivo
Retorna:

Nome/caminho do arquivo higienizado
Tipo de retorno:

string

Tenta limpar nomes de arquivos para impedir tentativas de passagem de diretórios e outras ameaças de segurança, o que é particularmente útil para arquivos que foram fornecidos via entrada do usuário.

$filename = $this->security->sanitize_filename($this->input->post('filename'));

Se for aceitável que a entrada do usuário inclua caminhos relativos, por exemplo file/in/some/approved/folder.txt, you can set the second optional parameter, $relative_path to TRUE.

$filename = $this->security->sanitize_filename($this->input->post('filename'), TRUE);

Retorna o valor de hash do CSRF. Útil em combinação com get_csrf_token_name() para criar formulários manualmente ou enviar solicitações AJAX POST válidas.

entity_decode($str[, $charset = NULL])¶
Parâmetros:
  • $str (string) - string de entrada
  • $charset (string) - Conjunto de caracteres da string de entrada
Retorna:

Cadeia decodificada por entidade
Tipo de retorno:

string

This method acts a lot like PHP’s own native html_entity_decode() function in ENT_COMPAT mode, only ele tenta detectar entidades HTML que não terminam em ponto-e-vírgula porque alguns navegadores permitem isso.

If the $charset parameter is left empty, then your configured $config['charset'] value will be used.
