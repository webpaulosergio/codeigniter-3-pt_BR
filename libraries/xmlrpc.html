Classes de servidor XML-RPC e XML-RPC¶

CodeIgniter’s XML-RPC classes permit you to send requests to another server, or set up your own XML-RPC server to receive requests.

O que é o XML-RPC?¶

Muito simplesmente, é uma maneira de dois computadores se comunicarem internet using XML. One computer, which we will call the client, sends an XML-RPC request to another computer, which we will call the servidor. Uma vez que o servidor receba e processe o pedido, ele enviará de volta um response to the client.

Por exemplo, usando a API MetaWeblog, um cliente XML-RPC (geralmente um desktop publishing tool) will send a request to an XML-RPC Server em execução no seu site. Esta solicitação pode ser uma nova entrada de blog sent for publication, or it could be a request for an existing entry for edição. Quando o servidor XML-RPC receber essa solicitação, ele examinará it to determine which class/method should be called to process the request. Once processed, the server will then send de volta um response message.

For detailed specifications, you can visit the XML-RPC site.

Usando a classe XML-RPC¶

Inicializando a classe¶

Like most other classes in CodeIgniter, the XML-RPC and XML-RPCS classes are initialized in your controller using the $this->load->library function:

To load the XML-RPC class you will use:

$this->load->library('xmlrpc');

Uma vez carregado, o objeto da Library xml-rpc estará disponível usando: $this->xmlrpc

To load the XML-RPC Server class you will use:

$this->load->library('xmlrpc');
$this->load->library('xmlrpcs');

Once loaded, the xml-rpcs library object will be available using: $this->xmlrpcs

Nota

Ao usar a classe Servidor XML-RPC, você deve carregar AMBOS os XML-RPC class and the XML-RPC Server class.

Sending XML-RPC Requests¶

To send a request to an XML-RPC server you must specify the following information:

  • o URL of the server
  • o method on the server you wish to call
  • o request data (explained below).

Here is a basic example that sends a simple Weblogs.com ping to the Ping-o-Matic

$this->load->library('xmlrpc');

$this->xmlrpc->server('http://rpc.pingomatic.com/', 80);
$this->xmlrpc->method('weblogUpdates.ping');

$request = array('My Photoblog', 'http://www.my-site.com/photoblog/');
$this->xmlrpc->request($request);

if ( ! $this->xmlrpc->send_request())
{
        echo $this->xmlrpc->display_error();
}

Explicação¶

O código acima inicializa a classe XML-RPC, define o URL do servidor e method to be called (weblogUpdates.ping). o request (in this case, the título e URL do seu site) é colocado em uma matriz para transporte, and compiled using the request() function. Lastly, the full request is enviei. Se o método send_request() retornar false, exibiremos error message sent back from the XML-RPC Server.

Anatomia de um pedido¶

Uma solicitação XML-RPC é simplesmente os dados que você está enviando para o XML-RPC servidor. Cada peça de dados em uma solicitação é referida como uma solicitação parâmetro. O exemplo acima tem dois parâmetros: o URL e o título do your site. When the XML-RPC server receives your request, it will look para parâmetros que requer.

Request parameters must be placed into an array for transportation, and each parameter can be one of seven data types (strings, numbers, dates, etc.). If your parameters are something other than strings you will have para incluir o tipo de dados na matriz de solicitação.

Aqui está um exemplo de um array simples com três parâmetros:

$request = array('John', 'Doe', 'www.some-site.com');
$this->xmlrpc->request($request);

If you use data types other than strings, or if you have several different data types, you will place each parameter into its own array, with the data type no second position:

$request = array(
        array('John', 'string'),
        array('Doe', 'string'),
        array(FALSE, 'boolean'),
        array(12345, 'int')
);
$this->xmlrpc->request($request);

o Data Types a seção abaixo tem uma lista completa de dados types.

Creating an XML-RPC Server¶

An XML-RPC Server acts as a traffic cop of sorts, waiting for incoming solicitações e redirecionando-as para as funções apropriadas para processing.

To create your own XML-RPC server involves initializing the XML-RPC Classe de servidor no seu controlador onde você espera que a solicitação recebida appear, then setting up an array with mapping instructions so that incoming requests can be sent to the appropriate class and method for processing.

Here is an example to illustrate:

$this->load->library('xmlrpc');
$this->load->library('xmlrpcs');

$config['functions']['new_post'] = array('function' => 'My_blog.new_entry');
$config['functions']['update_post'] = array('function' => 'My_blog.update_entry');
$config['object'] = $this;

$this->xmlrpcs->initialize($config);
$this->xmlrpcs->serve();

o above example contains an array specifying two method requests that the Server allows. o allowed methods are on the left side of the array. Quando qualquer um desses é recebido, eles serão mapeados para o class and method on the right.

o ‘object’ key is a special key that you pass an instantiated class object with, which is necessary when the method you are mapping to is não faz parte do superobjeto CodeIgniter.

Em outras palavras, se um cliente XML-RPC envia uma solicitação para o new_post method, your server will load the My_blog class and call the new_entry function. If the request is for the update_post method, your server will load the My_blog class and call the update_entry() method.

o function names no above example are arbitrary. You’ll decide what they should be called on your server, or if you are using standardized APIs, like the Blogger or MetaWeblog API, you’ll use their function names.

ore are two additional configuration keys you may make use of when initializing the server class: debug can be set to TRUE in order to enable debugging, and xss_clean may be set to FALSE to prevent sending data through the Segurança library’s xss_clean() method.

Solicitações do servidor de processamento¶

When the XML-RPC Server receives a request and loads the class/method para processamento, ele passará um objeto para esse método contendo o data sent by the client.

Using the above example, if the new_post method is requested, the server will expect a class to exist with this prototype:

class My_blog extends CI_Controller {

        public function new_post($request)
        {

        }
}

o $request variable is an object compiled by the Server, which contains the data sent by the XML-RPC Client. Using this object you will have access to the request parameters enabling you to process the request. When you are done you will send a Resposta back to the Client.

Abaixo está um exemplo do mundo real, usando a API do Blogger. Um dos métodos no Blogger API is getUserInfo(). Using this method, an XML-RPC O cliente pode enviar ao servidor um nome de usuário e senha, em troca o servidor envia de volta informações sobre esse usuário em particular (apelido, ID do usuário, endereço de e-mail, etc.). Aqui está como a função de processamento pode parecer:

class My_blog extends CI_Controller {

        public function getUserInfo($request)
        {
                $username = 'smitty';
                $password = 'secretsmittypass';

                $this->load->library('xmlrpc');

                $parameters = $request->output_parameters();

                if ($parameters[1] != $username && $parameters[2] != $password)
                {
                        return $this->xmlrpc->send_error_message('100', 'Invalid Access');
                }

                $response = array(
                        array(
                                'nickname'  => array('Smitty', 'string'),
                                'userid'    => array('99', 'string'),
                                'url'       => array('http://yoursite.com', 'string'),
                                'email'     => array('jsmith@yoursite.com', 'string'),
                                'lastname'  => array('Smith', 'string'),
                                'firstname' => array('John', 'string')
                        ),
                         'struct'
                );

                return $this->xmlrpc->send_response($response);
        }
}

Notas:¶

o output_parameters() method retrieves an indexed array corresponding to the request parameters sent by the client. In the above Por exemplo, os parâmetros de saída serão o nome de usuário e a senha.

Se o nome de usuário e senha enviados pelo cliente não forem válidos, e error message is returned using send_error_message().

If the operation was successful, the client will be sent de volta um response array containing the user’s info.

Formatando uma Resposta¶

Similar to Requests, Respostas must be formatted as an array. However, unlike requests, a response is an array que contém um single item. This item can be an array with several additional arrays, but there can be only one primary array index. In other words, the basic protótipo é este:

$response = array('Resposta data', 'array');

Respostas, however, usually contain multiple pieces of information. In Para realizar isso, devemos colocar a resposta em sua própria matriz para that the primary array continues to contain a single piece of data. Here’s an example showing how this might be accomplished:

$response = array(
        array(
                'first_name' => array('John', 'string'),
                'last_name' => array('Doe', 'string'),
                'member_id' => array(123435, 'int'),
                'todo_list' => array(array('clean house', 'call mom', 'water plants'), 'array'),
        ),
        'struct'
);

Notice that the above array is formatted as a struct. This is the most tipo de dados comum para respostas.

As with Requests, a response can be one of the seven data types listed no Data Types section.

Sending an Error Resposta¶

Se você precisar enviar uma resposta de erro ao cliente, você usará following:

return $this->xmlrpc->send_error_message('123', 'Requested data not available');

o first parameter is the error number while the second parameter is the error message.

Creating Your Own Client and Server¶

To help you understand everything we’ve covered thus far, let’s create a controladores de casal que agem como Cliente e Servidor XML-RPC. Você vai usar o Client to send a request to the Server and receive a response.

o Client¶

Using a text editor, create a controller called Xmlrpc_client.php. In it, place this code and save it to your application/controllers/ folder:

<?php

class Xmlrpc_client extends CI_Controller {

        public function index()
        {
                $this->load->helper('url');
                $server_url = site_url('xmlrpc_server');

                $this->load->library('xmlrpc');

                $this->xmlrpc->server($server_url, 80);
                $this->xmlrpc->method('Greetings');

                $request = array('How is it going?');
                $this->xmlrpc->request($request);

                if ( ! $this->xmlrpc->send_request())
                {
                        echo $this->xmlrpc->display_error();
                }
                else
                {
                        echo '<pre>';
                        print_r($this->xmlrpc->display_response());
                        echo '</pre>';
                }
        }
}
?>

Nota

No código acima, estamos usando um "auxiliar de url". Você pode encontrar mais information no Funções auxiliares page.

o Server¶

Usando um editor de texto, crie um controlador chamado Xmlrpc_server.php. Em it, place this code and save it to your application/controllers/ folder:

<?php

class Xmlrpc_server extends CI_Controller {

        public function index()
        {
                $this->load->library('xmlrpc');
                $this->load->library('xmlrpcs');

                $config['functions']['Greetings'] = array('function' => 'Xmlrpc_server.process');

                $this->xmlrpcs->initialize($config);
                $this->xmlrpcs->serve();
        }


        public function process($request)
        {
                $parameters = $request->output_parameters();

                $response = array(
                        array(
                                'you_said'  => $parameters[0],
                                'i_respond' => 'Not bad at all.'
                        ),
                        'struct'
                );

                return $this->xmlrpc->send_response($response);
        }
}

Try it!¶

Agora visite o seu site usando um URL semelhante a este:

example.com/index.php/xmlrpc_client/

Agora você deve ver a mensagem que você enviou para o servidor e sua resposta back to you.

o client you created sends a message (“How’s is going?”) to the server, along with a request for the “Greetings” method. o Server receives the request and maps it to the process() method, where a response is sent back.

Using Associative Arrays In a Request Parameter¶

Se você deseja usar um array associativo nos parâmetros do seu método, will need to use a struct datatype:

$request = array(
        array(
                // Param 0
                array('name' => 'John'),
                'struct'
        ),
        array(
                // Param 1
                array(
                        'size' => 'large',
                        'shape'=>'round'
                ),
                'struct'
        )
);

$this->xmlrpc->request($request);

You can retrieve the associative array when processing the request in o servidor.

$parameters = $request->output_parameters();
$name = $parameters[0]['name'];
$size = $parameters[1]['size'];
$shape = $parameters[1]['shape'];

Data Types¶

De acordo com XML-RPC spec there are seven types of values that you can send via XML-RPC:

  • int or i4
  • boolean
  • string
  • double
  • dateTime.iso8601
  • base64
  • struct (contains array of values)
  • array (contains array of values)

Class Reference¶

class CI_Xmlrpc¶
initialize([$config = array()])¶
Parâmetros:
  • $config (array) – Configuration data
Return type:

void

Inicializa a library XML-RPC. Aceita um array associativo contendo suas configurações.

server($url[, $port = 80[, $proxy = FALSE[, $proxy_port = 8080]]])¶
Parâmetros:
  • $url (string) – XML-RPC server URL
  • $port (int) – Server port
  • $proxy (string) – Optional proxy
  • $proxy_port (int) - Porta de escuta do proxy
Return type:

void

Sets the URL and port number of the server to which a request is to be sent:

$this->xmlrpc->server('http://www.sometimes.com/pings.php', 80);

A autenticação HTTP básica também é suportada, basta adicioná-la ao URL do servidor:

$this->xmlrpc->server('http://user:pass@localhost/', 80);
timeout($seconds = 5)¶
Parâmetros:
  • $seconds (int) - Tempo limite em segundos
Return type:

void

Set a time out period (in seconds) after which the request will be canceled:

$this->xmlrpc->timeout(6);

Esse período de tempo limite será usado tanto para uma conexão inicial o servidor remoto, bem como obter uma resposta dele. Make sure you set the timeout before calling send_request().

method($function)¶
Parâmetros:
  • $function (string) - nome do método
Return type:

void

Sets the method that will be requested from the XML-RPC server:

$this->xmlrpc->method('method');

Where method is the name of the method.

request($incoming)¶
Parâmetros:
  • $incoming (array) – Request data
Return type:

void

Takes an array of data and builds request to be sent to XML-RPC server:

$request = array(array('My Photoblog', 'string'), 'http://www.yoursite.com/photoblog/');
$this->xmlrpc->request($request);
send_request()¶
Returns:Verdadeiro sucesso, FALSE em falha
Return type:bool

o request sending method. Returns boolean TRUE or FALSE based on success for failure, enabling it to be used conditionally.

display_error()¶
Returns:Error message string
Return type:string

Returns an error message as a string if your request failed for some reason.

echo $this->xmlrpc->display_error();
display_response()¶
Returns:Resposta
Return type:mixed

Returns the response from the remote server once request is received. o response will typically be an associative array.

$this->xmlrpc->display_response();
send_error_message($number, $message)¶
Parâmetros:
  • $number (int) - número do erro
  • $message (string) – Error message
Returns:

XML_RPC_Resposta instance
Return type:

XML_RPC_Resposta

Este método permite enviar uma mensagem de erro do seu servidor para o cliente. Primeiro parâmetro é o número do erro, enquanto o segundo parâmetro é a mensagem de erro.

return $this->xmlrpc->send_error_message(123, 'Requested data not available');

O que é o XML-RPC?¶

Muito simplesmente, é uma maneira de dois computadores se comunicarem internet usando XML. Um computador, que chamaremos de cliente, envia um XML-RPC request para outro computador, que vamos chamar de servidor. Uma vez que o servidor receba e processe o pedido, ele enviará de volta um response para o cliente.

For example, using the MetaWeblog API, um XML-RPC Client (usually a desktop publishing tool) will send a request to um XML-RPC Server em execução no seu site. Esta solicitação pode ser uma nova entrada de blog enviado para publicação, ou pode ser um pedido de uma entrada existente para edição. Quando o servidor XML-RPC receber essa solicitação, ele examinará para determinar qual classe/método deve ser chamado para processar o request. Once processed, the server will then send de volta um response message.

Para especificações detalhadas, você pode visitar o XML-RPC site.

Usando a classe XML-RPC¶

Inicializando a classe¶

Como a maioria das outras classes no CodeIgniter, as classes XML-RPC e XML-RPCS são inicializados em seu Controller usando a Library $this-> load-> function:

Para carregar a classe XML-RPC, você usará:

$this->load->library('xmlrpc');

Uma vez carregado, o objeto da Library xml-rpc estará disponível usando: $this->xmlrpc

Para carregar a classe Servidor XML-RPC, você usará:

$this->load->library('xmlrpc');
$this->load->library('xmlrpcs');

Uma vez carregado, o objeto da Library xml-rpcs estará disponível usando: $this->xmlrpcs

Nota

Ao usar a classe Servidor XML-RPC, você deve carregar AMBOS os Classe XML-RPC e a classe Servidor XML-RPC.

Enviando Solicitações de XML-RPC¶

To send a request to um XML-RPC server you must specify the following information:

  • A URL do servidor
  • O método no servidor que você deseja chamar
  • o request data (explained below).

Aqui está um exemplo básico que envia um ping simples de Weblogs.com para o Ping-o-Matic

$this->load->library('xmlrpc');

$this->xmlrpc->server('http://rpc.pingomatic.com/', 80);
$this->xmlrpc->method('weblogUpdates.ping');

$request = array('My Photoblog', 'http://www.my-site.com/photoblog/');
$this->xmlrpc->request($request);

if ( ! $this->xmlrpc->send_request())
{
        echo $this->xmlrpc->display_error();
}

Explicação¶

O código acima inicializa a classe XML-RPC, define o URL do servidor e método a ser chamado (weblogUpdates.ping). O pedido (neste caso, o título e URL do seu site) é colocado em uma matriz para transporte, e compilado usando a função request(). Por fim, o pedido completo é enviei. Se o método send_request() retornar false, exibiremos mensagem de erro enviada de volta do servidor XML-RPC.

Anatomia de um pedido¶

Uma solicitação XML-RPC é simplesmente os dados que você está enviando para o XML-RPC servidor. Cada peça de dados em uma solicitação é referida como uma solicitação parâmetro. O exemplo acima tem dois parâmetros: o URL e o título do seu site. Quando o servidor XML-RPC recebe sua solicitação, ele para parâmetros que requer.

Os parâmetros de solicitação devem ser colocados em uma matriz para transporte e cada parâmetro pode ser um dos sete tipos de dados (strings, números, datas, etc). Se seus parâmetros são diferentes de strings, você terá para incluir o tipo de dados na matriz de solicitação.

Aqui está um exemplo de um array simples com três parâmetros:

$request = array('John', 'Doe', 'www.some-site.com');
$this->xmlrpc->request($request);

Se você usar tipos de dados diferentes de strings, ou se tiver vários diferentes tipos de dados, você colocará cada parâmetro em sua própria matriz, com o tipo de dados na segunda posição:

$request = array(
        array('John', 'string'),
        array('Doe', 'string'),
        array(FALSE, 'boolean'),
        array(12345, 'int')
);
$this->xmlrpc->request($request);

o Tipos de dados a seção abaixo tem uma lista completa de dados types.

Creating um XML-RPC Server¶

Um servidor XML-RPC atua como um policial de tráfego de tipos, esperando pela entrada solicitações e redirecionando-as para as funções apropriadas para processing.

Para criar seu próprio servidor XML-RPC, é necessário inicializar o XML-RPC Classe de servidor no seu Controller onde você espera que a solicitação recebida aparecer, em seguida, configurar um array com instruções de mapeamento para que as solicitações recebidas podem ser enviadas para a classe e método apropriados para processing.

Aqui está um exemplo para ilustrar:

$this->load->library('xmlrpc');
$this->load->library('xmlrpcs');

$config['functions']['new_post'] = array('function' => 'My_blog.new_entry');
$config['functions']['update_post'] = array('function' => 'My_blog.update_entry');
$config['object'] = $this;

$this->xmlrpcs->initialize($config);
$this->xmlrpcs->serve();

o above example contains an array specifying two method requests that the Server allows. o allowed methods are on the left side of the array. Quando qualquer um desses é recebido, eles serão mapeados para o classe e método à direita.

o ‘object’ key is a special key that you pass an instantiated class objeto com, o que é necessário quando o método que você está mapeando é não faz parte do superobjeto CodeIgniter.

In other words, if um XML-RPC Client sends a request for the new_post método, seu servidor irá carregar a classe My_blog e chamar o new_entry função. Se a solicitação for para o método update_post, seu servidor will load the My_blog class and call the update_entry() method.

o function names no above example are arbitrary. You’ll decide o que eles devem ser chamados em seu servidor, ou se você estiver usando APIs padronizadas, como a API do Blogger ou do MetaWeblog, você usará nomes de funções.

ore are two additional configuration keys you may make use of when inicializando a classe do servidor: debug pode ser configurado para TRUE para habilite a depuração, e xss_clean pode ser definido como FALSE para evitar o envio data through the Segurança library’s xss_clean() method.

Solicitações do servidor de processamento¶

Quando o servidor XML-RPC recebe uma solicitação e carrega a classe/método para processamento, ele passará um objeto para esse método contendo o dados enviados pelo cliente.

Usando o exemplo acima, se o método new_post for solicitado, o servidor irá esperar que uma classe exista com este protótipo:

class My_blog extends CI_Controller {

        public function new_post($request)
        {

        }
}

o $request variable is an object compiled by the Server, which contém os dados enviados pelo cliente XML-RPC. Usando este objeto você irá have access to the request parameters enabling you to process the pedido. Quando terminar, você enviará uma resposta ao cliente.

Abaixo está um exemplo do mundo real, usando a API do Blogger. Um dos métodos no Blogger API is getUserInfo(). Using this method, um XML-RPC O cliente pode enviar ao servidor um nome de usuário e senha, em troca o servidor envia de volta informações sobre esse usuário em particular (apelido, ID do usuário, endereço de e-mail, etc.). Aqui está como a função de processamento pode parecer:

class My_blog extends CI_Controller {

        public function getUserInfo($request)
        {
                $username = 'smitty';
                $password = 'secretsmittypass';

                $this->load->library('xmlrpc');

                $parameters = $request->output_parameters();

                if ($parameters[1] != $username && $parameters[2] != $password)
                {
                        return $this->xmlrpc->send_error_message('100', 'Invalid Access');
                }

                $response = array(
                        array(
                                'nickname'  => array('Smitty', 'string'),
                                'userid'    => array('99', 'string'),
                                'url'       => array('http://yoursite.com', 'string'),
                                'email'     => array('jsmith@yoursite.com', 'string'),
                                'lastname'  => array('Smith', 'string'),
                                'firstname' => array('John', 'string')
                        ),
                         'struct'
                );

                return $this->xmlrpc->send_response($response);
        }
}

Notas:¶

o output_parameters() method retrieves an indexed array correspondente aos parâmetros de solicitação enviados pelo cliente. No acima Por exemplo, os parâmetros de saída serão o nome de usuário e a senha.

Se o nome de usuário e senha enviados pelo cliente não forem válidos, e error message is returned using send_error_message().

If the operation was successful, the client will be sent de volta um response array contendo as informações do usuário.

Formatando uma Resposta¶

Similar to Requests, Respostas must be formatted as an array. No entanto, ao contrário das solicitações, uma resposta é uma matriz que contém um item único. Este item pode ser um array com vários arrays adicionais, mas pode haver apenas um índice de matriz primário. Em outras palavras, o básico protótipo é este:

$response = array('Resposta data', 'array');

As respostas, no entanto, geralmente contêm várias informações. Em Para realizar isso, devemos colocar a resposta em sua própria matriz para que o array primário continua a conter um único pedaço de dados. Aqui está um exemplo mostrando como isso pode ser feito:

$response = array(
        array(
                'first_name' => array('John', 'string'),
                'last_name' => array('Doe', 'string'),
                'member_id' => array(123435, 'int'),
                'todo_list' => array(array('clean house', 'call mom', 'water plants'), 'array'),
        ),
        'struct'
);

Observe que a matriz acima é formatada como uma estrutura. Este é o mais tipo de dados comum para respostas.

Assim como com as solicitações, uma resposta pode ser um dos sete tipos de dados listados no Tipos de dados section.

Enviando uma resposta de erro¶

Se você precisar enviar uma resposta de erro ao cliente, você usará following:

return $this->xmlrpc->send_error_message('123', 'Requested data not available');

o first parameter is the error number while the second parameter is a mensagem de erro.

Criando seu próprio cliente e servidor¶

Para ajudar você a entender tudo o que abordamos até agora, vamos criar um Controlleres de casal que agem como Cliente e Servidor XML-RPC. Você vai usar o Cliente para enviar uma solicitação ao servidor e receber uma resposta.

o Client¶

Usando um editor de texto, crie um Controller chamado Xmlrpc_client.php. Em coloque este código e salve-o em seu aplicativo/controllers / folder:

<?php

class Xmlrpc_client extends CI_Controller {

        public function index()
        {
                $this->load->helper('url');
                $server_url = site_url('xmlrpc_server');

                $this->load->library('xmlrpc');

                $this->xmlrpc->server($server_url, 80);
                $this->xmlrpc->method('Greetings');

                $request = array('How is it going?');
                $this->xmlrpc->request($request);

                if ( ! $this->xmlrpc->send_request())
                {
                        echo $this->xmlrpc->display_error();
                }
                else
                {
                        echo '<pre>';
                        print_r($this->xmlrpc->display_response());
                        echo '</pre>';
                }
        }
}
?>

Nota

No código acima, estamos usando um "Helper de url". Você pode encontrar mais information no Funções Helperes page.

o Server¶

Usando um editor de texto, crie um Controller chamado Xmlrpc_server.php. Em coloque este código e salve-o em seu aplicativo/controllers / folder:

<?php

class Xmlrpc_server extends CI_Controller {

        public function index()
        {
                $this->load->library('xmlrpc');
                $this->load->library('xmlrpcs');

                $config['functions']['Greetings'] = array('function' => 'Xmlrpc_server.process');

                $this->xmlrpcs->initialize($config);
                $this->xmlrpcs->serve();
        }


        public function process($request)
        {
                $parameters = $request->output_parameters();

                $response = array(
                        array(
                                'you_said'  => $parameters[0],
                                'i_respond' => 'Not bad at all.'
                        ),
                        'struct'
                );

                return $this->xmlrpc->send_response($response);
        }
}

Tente!¶

Agora visite o seu site usando um URL semelhante a este:

example.com/index.php/xmlrpc_client/

Agora você deve ver a mensagem que você enviou para o servidor e sua resposta de volta para você.

o client you created sends a message (“How’s is going?”) to the server, along with a request for the “Greetings” method. o Server receives the request and maps it to the process() method, where a resposta é enviada de volta.

Usando matrizes associativas em um parâmetro de solicitação¶

Se você deseja usar um array associativo nos parâmetros do seu método, precisará usar um tipo de dados struct:

$request = array(
        array(
                // Param 0
                array('name' => 'John'),
                'struct'
        ),
        array(
                // Param 1
                array(
                        'size' => 'large',
                        'shape'=>'round'
                ),
                'struct'
        )
);

$this->xmlrpc->request($request);

Você pode recuperar o array associativo ao processar o pedido em o servidor.

$parameters = $request->output_parameters();
$name = $parameters[0]['name'];
$size = $parameters[1]['size'];
$shape = $parameters[1]['shape'];

Tipos de dados¶

De acordo com Especificação XML-RPC há sete tipos de valores que você pode enviar via XML-RPC:

  • int or i4
  • boolean
  • string
  • double
  • dateTime.iso8601
  • base64
  • struct (contém matriz de valores)
  • array (contém matriz de valores)

O que é o XML-RPC?¶

Muito simplesmente, é uma maneira de dois computadores se comunicarem internet usando XML. Um computador, que chamaremos de cliente, envia an XML-RPC request para outro computador, que vamos chamar de servidor. Uma vez que o servidor receba e processe o pedido, ele enviará back a response para o cliente.

Por exemplo, usando a API MetaWeblog, um cliente XML-RPC (geralmente um ferramenta de editoração eletrônica) enviará uma solicitação para um servidor XML-RPC em execução no seu site. Esta solicitação pode ser uma nova entrada de blog enviado para publicação, ou pode ser um pedido de uma entrada existente para edição. Quando o servidor XML-RPC receber essa solicitação, ele examinará para determinar qual classe/método deve ser chamado para processar o pedido. Uma vez processado, o servidor enviará de volta uma resposta message.

Para especificações detalhadas, você pode visitar o XML-RPC site.

Usando a classe XML-RPC¶

Inicializando a classe¶

Como a maioria das outras classes no CodeIgniter, as classes XML-RPC e XML-RPCS são inicializados em seu Controller usando a Library $this-> load-> function:

Para carregar a classe XML-RPC, você usará:

$this->load->library('xmlrpc');

Uma vez carregado, o objeto da Library xml-rpc estará disponível usando: $this->xmlrpc

Para carregar a classe Servidor XML-RPC, você usará:

$this->load->library('xmlrpc');
$this->load->library('xmlrpcs');

Uma vez carregado, o objeto da Library xml-rpcs estará disponível usando: $this->xmlrpcs

Nota

Ao usar a classe Servidor XML-RPC, você deve carregar AMBOS os Classe XML-RPC e a classe Servidor XML-RPC.

Enviando Solicitações de XML-RPC¶

Para enviar uma solicitação para um servidor XML-RPC, você deve especificar o seguinte information:

  • A URL do servidor
  • O método no servidor que você deseja chamar
  • o request data (explained below).

Aqui está um exemplo básico que envia um ping simples de Weblogs.com para o Ping-o-Matic

$this->load->library('xmlrpc');

$this->xmlrpc->server('http://rpc.pingomatic.com/', 80);
$this->xmlrpc->method('weblogUpdates.ping');

$request = array('My Photoblog', 'http://www.my-site.com/photoblog/');
$this->xmlrpc->request($request);

if ( ! $this->xmlrpc->send_request())
{
        echo $this->xmlrpc->display_error();
}

Explicação¶

O código acima inicializa a classe XML-RPC, define o URL do servidor e método a ser chamado (weblogUpdates.ping). O pedido (neste caso, o título e URL do seu site) é colocado em uma matriz para transporte, e compilado usando a função request(). Por fim, o pedido completo é enviei. Se o método send_request() retornar false, exibiremos mensagem de erro enviada de volta do servidor XML-RPC.

Anatomia de um pedido¶

Uma solicitação XML-RPC é simplesmente os dados que você está enviando para o XML-RPC servidor. Cada peça de dados em uma solicitação é referida como uma solicitação parâmetro. O exemplo acima tem dois parâmetros: o URL e o título do seu site. Quando o servidor XML-RPC recebe sua solicitação, ele para parâmetros que requer.

Os parâmetros de solicitação devem ser colocados em uma matriz para transporte e cada parâmetro pode ser um dos sete tipos de dados (strings, números, datas, etc). Se seus parâmetros são diferentes de strings, você terá para incluir o tipo de dados na matriz de solicitação.

Aqui está um exemplo de um array simples com três parâmetros:

$request = array('John', 'Doe', 'www.some-site.com');
$this->xmlrpc->request($request);

Se você usar tipos de dados diferentes de strings, ou se tiver vários diferentes tipos de dados, você colocará cada parâmetro em sua própria matriz, com o tipo de dados na segunda posição:

$request = array(
        array('John', 'string'),
        array('Doe', 'string'),
        array(FALSE, 'boolean'),
        array(12345, 'int')
);
$this->xmlrpc->request($request);

o Data Types a seção abaixo tem uma lista completa de dados types.

Criando um servidor XML-RPC¶

Um servidor XML-RPC atua como um policial de tráfego de tipos, esperando pela entrada solicitações e redirecionando-as para as funções apropriadas para processing.

Para criar seu próprio servidor XML-RPC, é necessário inicializar o XML-RPC Classe de servidor no seu Controller onde você espera que a solicitação recebida aparecer, em seguida, configurar um array com instruções de mapeamento para que as solicitações recebidas podem ser enviadas para a classe e método apropriados para processing.

Aqui está um exemplo para ilustrar:

$this->load->library('xmlrpc');
$this->load->library('xmlrpcs');

$config['functions']['new_post'] = array('function' => 'My_blog.new_entry');
$config['functions']['update_post'] = array('function' => 'My_blog.update_entry');
$config['object'] = $this;

$this->xmlrpcs->initialize($config);
$this->xmlrpcs->serve();

o above example contains an array specifying two method requests that the Server allows. o allowed methods are on the left side of the array. Quando qualquer um desses é recebido, eles serão mapeados para o classe e método à direita.

o ‘object’ key is a special key that you pass an instantiated class objeto com, o que é necessário quando o método que você está mapeando é não faz parte do superobjeto CodeIgniter.

Em outras palavras, se um cliente XML-RPC envia uma solicitação para o new_post método, seu servidor irá carregar a classe My_blog e chamar o new_entry função. Se a solicitação for para o método update_post, seu servidor will load the My_blog class and call the update_entry() method.

o function names in the above example are arbitrary. You’ll decide o que eles devem ser chamados em seu servidor, ou se você estiver usando APIs padronizadas, como a API do Blogger ou do MetaWeblog, você usará nomes de funções.

ore are two additional configuration keys you may make use of when inicializando a classe do servidor: debug pode ser configurado para TRUE para habilite a depuração, e xss_clean pode ser definido como FALSE para evitar o envio data through the Segurança library’s xss_clean() method.

Solicitações do servidor de processamento¶

Quando o servidor XML-RPC recebe uma solicitação e carrega a classe/método para processamento, ele passará um objeto para esse método contendo o dados enviados pelo cliente.

Usando o exemplo acima, se o método new_post for solicitado, o servidor irá esperar que uma classe exista com este protótipo:

class My_blog extends CI_Controller {

        public function new_post($request)
        {

        }
}

o $request variable is an object compiled by the Server, which contém os dados enviados pelo cliente XML-RPC. Usando este objeto você irá have access to the request parameters enabling you to process the pedido. Quando terminar, você enviará uma resposta ao cliente.

Abaixo está um exemplo do mundo real, usando a API do Blogger. Um dos métodos in the Blogger API is getUserInfo(). Using this method, an XML-RPC O cliente pode enviar ao servidor um nome de usuário e senha, em troca o servidor envia de volta informações sobre esse usuário em particular (apelido, ID do usuário, endereço de e-mail, etc.). Aqui está como a função de processamento pode parecer:

class My_blog extends CI_Controller {

        public function getUserInfo($request)
        {
                $username = 'smitty';
                $password = 'secretsmittypass';

                $this->load->library('xmlrpc');

                $parameters = $request->output_parameters();

                if ($parameters[1] != $username && $parameters[2] != $password)
                {
                        return $this->xmlrpc->send_error_message('100', 'Invalid Access');
                }

                $response = array(
                        array(
                                'nickname'  => array('Smitty', 'string'),
                                'userid'    => array('99', 'string'),
                                'url'       => array('http://yoursite.com', 'string'),
                                'email'     => array('jsmith@yoursite.com', 'string'),
                                'lastname'  => array('Smith', 'string'),
                                'firstname' => array('John', 'string')
                        ),
                         'struct'
                );

                return $this->xmlrpc->send_response($response);
        }
}

Notas:¶

o output_parameters() method retrieves an indexed array correspondente aos parâmetros de solicitação enviados pelo cliente. No acima Por exemplo, os parâmetros de saída serão o nome de usuário e a senha.

Se o nome de usuário e senha enviados pelo cliente não forem válidos, e error message is returned using send_error_message().

Se a operação foi bem sucedida, o cliente será enviado de volta uma resposta array contendo as informações do usuário.

Formatando uma Resposta¶

Similar to Requests, Responses must be formatted as an array. No entanto, ao contrário das solicitações, uma resposta é uma matriz que contém um item único. Este item pode ser um array com vários arrays adicionais, mas pode haver apenas um índice de matriz primário. Em outras palavras, o básico protótipo é este:

$response = array('Response data', 'array');

As respostas, no entanto, geralmente contêm várias informações. Em Para realizar isso, devemos colocar a resposta em sua própria matriz para que o array primário continua a conter um único pedaço de dados. Aqui está um exemplo mostrando como isso pode ser feito:

$response = array(
        array(
                'first_name' => array('John', 'string'),
                'last_name' => array('Doe', 'string'),
                'member_id' => array(123435, 'int'),
                'todo_list' => array(array('clean house', 'call mom', 'water plants'), 'array'),
        ),
        'struct'
);

Observe que a matriz acima é formatada como uma estrutura. Este é o mais tipo de dados comum para respostas.

Assim como com as solicitações, uma resposta pode ser um dos sete tipos de dados listados in the Data Types section.

Enviando uma resposta de erro¶

Se você precisar enviar uma resposta de erro ao cliente, você usará following:

return $this->xmlrpc->send_error_message('123', 'Requested data not available');

o first parameter is the error number while the second parameter is a mensagem de erro.

Criando seu próprio cliente e servidor¶

Para ajudar você a entender tudo o que abordamos até agora, vamos criar um Controlleres de casal que agem como Cliente e Servidor XML-RPC. Você vai usar o Cliente para enviar uma solicitação ao servidor e receber uma resposta.

o Client¶

Usando um editor de texto, crie um Controller chamado Xmlrpc_client.php. Em coloque este código e salve-o em seu aplicativo/controllers / folder:

<?php

class Xmlrpc_client extends CI_Controller {

        public function index()
        {
                $this->load->helper('url');
                $server_url = site_url('xmlrpc_server');

                $this->load->library('xmlrpc');

                $this->xmlrpc->server($server_url, 80);
                $this->xmlrpc->method('Greetings');

                $request = array('How is it going?');
                $this->xmlrpc->request($request);

                if ( ! $this->xmlrpc->send_request())
                {
                        echo $this->xmlrpc->display_error();
                }
                else
                {
                        echo '<pre>';
                        print_r($this->xmlrpc->display_response());
                        echo '</pre>';
                }
        }
}
?>

Nota

No código acima, estamos usando um "Helper de url". Você pode encontrar mais informações no Funções Helperes page.

o Server¶

Usando um editor de texto, crie um Controller chamado Xmlrpc_server.php. Em coloque este código e salve-o em seu aplicativo/controllers / folder:

<?php

class Xmlrpc_server extends CI_Controller {

        public function index()
        {
                $this->load->library('xmlrpc');
                $this->load->library('xmlrpcs');

                $config['functions']['Greetings'] = array('function' => 'Xmlrpc_server.process');

                $this->xmlrpcs->initialize($config);
                $this->xmlrpcs->serve();
        }


        public function process($request)
        {
                $parameters = $request->output_parameters();

                $response = array(
                        array(
                                'you_said'  => $parameters[0],
                                'i_respond' => 'Not bad at all.'
                        ),
                        'struct'
                );

                return $this->xmlrpc->send_response($response);
        }
}

Try it!¶

Agora visite o seu site usando um URL semelhante a este:

example.com/index.php/xmlrpc_client/

Agora você deve ver a mensagem que você enviou para o servidor e sua resposta de volta para você.

o client you created sends a message (“How’s is going?”) to the server, along with a request for the “Greetings” method. o Server receives the request and maps it to the process() method, where a resposta é enviada de volta.

Usando matrizes associativas em um parâmetro de solicitação¶

Se você deseja usar um array associativo nos parâmetros do seu método, precisará usar um tipo de dados struct:

$request = array(
        array(
                // Param 0
                array('name' => 'John'),
                'struct'
        ),
        array(
                // Param 1
                array(
                        'size' => 'large',
                        'shape'=>'round'
                ),
                'struct'
        )
);

$this->xmlrpc->request($request);

Você pode recuperar o array associativo ao processar o pedido em o servidor.

$parameters = $request->output_parameters();
$name = $parameters[0]['name'];
$size = $parameters[1]['size'];
$shape = $parameters[1]['shape'];

Data Types¶

De acordo com Especificação XML-RPC there are sete tipos de valores que você pode enviar via XML-RPC:

  • int or i4
  • boolean
  • string
  • double
  • dateTime.iso8601
  • base64
  • struct (contém matriz de valores)
