Classe de Paginação¶

CodeIgniter’s Pagination class is very easy to use, and it is 100% customizable, either dynamically or via stored preferences.

If you are not familiar with the term “pagination”, it refers to links that allows you to navigate from page to page, like this:

« First  < 1 2 3 4 5 >  Last »

Example¶

Aqui está um exemplo simples mostrando como criar paginação em um de seus controller methods:

$this->load->library('pagination');

$config['base_url'] = 'http://example.com/index.php/test/page/';
$config['total_rows'] = 200;
$config['per_page'] = 20;

$this->pagination->initialize($config);

echo $this->pagination->create_links();

Notas¶

The $config array contains your configuration variables. It is passed to the $this->pagination->initialize() method as shown above. Although there are some twenty items you can configure, at minimum you need the three shown. Here is a description of what those items represent:

  • base_url Este é o URL completo para a classe/função do controlador contendo sua paginação. No exemplo acima, está apontando para um controlador chamado "Teste" e uma função chamada "página". Tenha em mente isso você pode re-route your URI Se vocês precisa de uma estrutura diferente.
  • total_rows This number represents the total rows in the result set you are creating pagination for. Typically this number will be the total rows that your database query returned.
  • per_page The number of items you intend to show per page. In the above example, you would be showing 20 items per page.

The create_links() method returns an empty string when there is no pagination to show.

Configurando Preferências em um Arquivo de Configuração¶

If you prefer not to set preferences using the above method, you can em vez disso, coloque-os em um arquivo de configuração. Basta criar um novo arquivo chamado pagination.php, add the $config array in that file. Then save the file in application/config/pagination.php and it will be used automatically. You will NOT need to use $this->pagination->initialize() Se vocês save suas preferências em um arquivo de configuração.

Customizing the Pagination¶

The following is a list of all the preferences you can pass to the função de inicialização para personalizar a exibição.

$config[‘uri_segment’] = 3;

The pagination function automatically determines which segment of your URI contains the page number. If you need something different you can specify it.

$config ['num_links'] = 2;

O número de links "dígitos" que você gostaria antes e depois da seleção número de página. Por exemplo, o número 2 colocará dois dígitos em cada lado, como nos links de exemplo no topo desta página.

$config[‘use_page_numbers’] = TRUE;

Por padrão, o segmento de URI usará o índice inicial para os itens você está paginando. Se você preferir mostrar o número da página atual, set this to TRUE.

$config[‘page_query_string’] = TRUE;

By default, the pagination library assume you are using URI Segmentos, e constrói seus links algo like:

http://example.com/index.php/test/page/20

If you have $config['enable_query_strings'] set to TRUE your links will automatically be re-written using Query Strings. This option can also be explicitly set. Using $config['page_query_string'] set to TRUE, the pagination link will become:

http://example.com/index.php?c=test&m=page&per_page=20

Nota that “per_page” is the default query string passed, however can be configured using $config['query_string_segment'] = 'your_string'

$config[‘reuse_query_string’] = FALSE;

Por padrão, seus argumentos da Cadeia de Consulta (nada a ver com outros query string options) will be ignored. Setting this config to TRUE will add existing query string arguments back into the URL após o segmento URI e antes do sufixo .:

http://example.com/index.php/test/page/20?query=search%term

Isso ajuda você a misturar normal URI Segmentos bem como argumentos de string de consulta, que até 3.0 não era possível.

$config[‘prefix’] = ‘’;

A custom prefix added to the path. The prefix value will be right before o segmento de deslocamento.

$config ['suffix'] = '';

Um sufixo personalizado adicionado ao caminho. O valor do sufixo será logo após o segmento de deslocamento.

$config[‘use_global_url_suffix’] = FALSE;

When set to TRUE, it will override the $config['suffix'] value and instead set it to the one that you have in $config['url_suffix'] in your application/config/config.php file.

Adicionando marcação de inclusão¶

If you would like to surround the entire pagination with some markup you pode fazer isso com estas duas preferências:

$config[‘full_tag_open’] = ‘<p>’;

A tag de abertura colocada no lado esquerdo de todo o resultado.

$config[‘full_tag_close’] = ‘</p>’;

The closing tag placed on the right side of the entire result.

Customizing the First Link¶

$config[‘first_link’] = ‘First’;

O texto que você gostaria de mostrar no "primeiro" link à esquerda. Se você fizer not want this link rendered, you can set its value to FALSE.

Nota

This value can also be translated via a language file.

$config[‘first_tag_open’] = ‘<div>’;

The opening tag for the “first” link.

$config[‘first_tag_close’] = ‘</div>’;

The closing tag for the “first” link.

$config ['first_url'] = '';

Um URL alternativo para usar no link "primeira página".

Customizing the Last Link¶

$config[‘last_link’] = ‘Last’;

The text you would like shown in the “last” link on the right. If you do not want this link rendered, you can set its value to FALSE.

Nota

This value can also be translated via a language file.

$config[‘last_tag_open’] = ‘<div>’;

The opening tag for the “last” link.

$config[‘last_tag_close’] = ‘</div>’;

The closing tag for the “last” link.

Personalizando o link "Próximo"¶

$config[‘next_link’] = ‘&gt;’;

The text you would like shown in the “next” page link. If you do not want this link rendered, you can set its value to FALSE.

Nota

This value can also be translated via a language file.

$config[‘next_tag_open’] = ‘<div>’;

A tag de abertura para o link "próximo".

$config[‘next_tag_close’] = ‘</div>’;

A tag de fechamento do link "próximo".

Customizing the “Previous” Link¶

$config[‘prev_link’] = ‘&lt;’;

O texto que você gostaria de mostrar no link da página "anterior". Se você não want this link rendered, you can set its value to FALSE.

Nota

This value can also be translated via a language file.

$config[‘prev_tag_open’] = ‘<div>’;

The opening tag for the “previous” link.

$config[‘prev_tag_close’] = ‘</div>’;

A tag de fechamento do link "anterior".

Customizing the “Current Page” Link¶

$config[‘cur_tag_open’] = ‘<b>’;

The opening tag for the “current” link.

$config[‘cur_tag_close’] = ‘</b>’;

The closing tag for the “current” link.

Customizing the “Digit” Link¶

$config[‘num_tag_open’] = ‘<div>’;

A tag de abertura do link "dígito".

$config[‘num_tag_close’] = ‘</div>’;

The closing tag for the “digit” link.

Escondendo as páginas¶

Se você não quisesse listar as páginas específicas (por exemplo, você só quer Links "next" e "previous"), você pode suprimir a renderização deles adding:

$config['display_pages'] = FALSE;

Adding attributes to anchors¶

If you want to add an extra attribute to be added to every link rendered by the pagination class, you can set them as key/value pairs in the “attributes” config:

// Produces: class="myclass"
$config['attributes'] = array('class' => 'myclass');

Nota

Uso do antigo método de configuração de classes via “anchor_class” is deprecated.

Desativando o atributo "rel"¶

By default the rel attribute is dynamically generated and appended to the appropriate anchors. If for some reason you want to turn it off, you can pass boolean FALSE as a regular attribute

$config['attributes']['rel'] = FALSE;

Classe de Paginação¶

A classe de paginação do CodeIgniter é muito fácil de usar, e é 100% personalizável, dinamicamente ou através de preferências armazenadas.

Se você não está familiarizado com o termo "paginação", ele se refere a links que permite que você navegue de uma página para outra, assim:

Exemplo¶

Aqui está um exemplo simples mostrando como criar paginação em um de seus controller methods:

$this->load->library('pagination');

$config['base_url'] = 'http://example.com/index.php/test/page/';
$config['total_rows'] = 200;
$config['per_page'] = 20;

$this->pagination->initialize($config);

echo $this->pagination->create_links();

Notas¶

The $config array contains your configuration variables. It is passed to the $this->pagination->initialize() method as shown above. Although existem uns vinte itens que você pode configurar, no mínimo você precisa do três mostrados. Aqui está uma descrição do que esses itens representam:

  • base_url Este é o URL completo para a classe/função do Controller contendo sua paginação. No exemplo acima, está apontando para um Controller chamado "Teste" e uma função chamada "página". Tenha em mente isso você pode redirecionar seu URI Se vocês precisa de uma estrutura diferente.
  • total_rows Este número representa o total de linhas no resultado set você está criando paginação para. Normalmente este número será o total de linhas retornadas pela Query do seu Database.
  • per_page O número de itens que você pretende mostrar por página. No acima exemplo, você estaria mostrando 20 itens por página.

The create_links() method returns an empty string when there is no paginação para mostrar.

Configurando Preferências em um Arquivo de Configuração¶

Se preferir não definir preferências usando o método acima, você pode em vez disso, coloque-os em um arquivo de configuração. Basta criar um novo arquivo chamado pagination.php, add the $config array in that file. Then save the file in application/config/pagination.php and it will be used automatically. You will NOT need to use $this->pagination->initialize() Se vocês save suas preferências em um arquivo de configuração.

Customizando a paginação¶

A seguir está uma lista de todas as preferências que você pode passar para o função de inicialização para personalizar a exibição.

$config ['uri_segment'] = 3;

A função de paginação determina automaticamente qual segmento do seu URI contém o número da página. Se você precisar de algo diferente, você pode especificá-lo.

$config ['num_links'] = 2;

O número de links "dígitos" que você gostaria antes e depois da seleção número de página. Por exemplo, o número 2 colocará dois dígitos em cada lado, como nos links de exemplo no topo desta página.

$config ['use_page_numbers'] = TRUE;

Por padrão, o segmento de URI usará o índice inicial para os itens você está paginando. Se você preferir mostrar o número da página atual, defina isso como TRUE.

$config ['page_query_string'] = TRUE;

Por padrão, a Library de paginação assume que você está usando URI Segmentos, e constrói seus links algo like:

http://example.com/index.php/test/page/20

If you have $config['enable_query_strings'] set to TRUE your links será automaticamente reescrito usando as Cadeias de Query. Esta opção pode also be explicitly set. Using $config['page_query_string'] set to TRUE, o link de paginação se tornará:

http://example.com/index.php?c=test&m=page&per_page=20

Observe que "per_page" é a string de Query padrão aprovada, mas pode ser configurado usando $config['query_string_segment'] = 'your_string'

$config ['reuse_query_string'] = FALSE;

Por padrão, seus argumentos da Cadeia de Query (nada a ver com outros opções de string de Query) serão ignoradas. Configurando esta configuração para TRUE adicionará argumentos de string de Query existentes de volta ao URL após o segmento URI e antes do sufixo .:

http://example.com/index.php/test/page/20?query=search%term

Isso ajuda você a misturar normal URI Segmentos bem como argumentos de string de Query, que até 3.0 não era possível.

$config ['prefixo'] = '';

Um prefixo personalizado adicionado ao caminho. O valor do prefixo será logo antes o segmento de deslocamento.

$config ['suffix'] = '';

Um sufixo personalizado adicionado ao caminho. O valor do sufixo será logo após o segmento de deslocamento.

$config ['use_global_url_suffix'] = FALSE;

Quando definido como TRUE, será override the $config['suffix'] value and instead set it to the one that you have in $config['url_suffix'] in your application/config/config.php file.

A tag de fechamento colocada no lado direito de todo o resultado.

Classe de Paginação¶

A classe de paginação do CodeIgniter é muito fácil de usar, e é 100% personalizável, dinamicamente ou através de preferências armazenadas.

Se você não está familiarizado com o termo "paginação", ele se refere a links que permite que você navegue de uma página para outra, assim:

« First  < 1 2 3 4 5 >  Last »

Exemplo¶

Aqui está um exemplo simples mostrando como criar paginação em um de seus controller methods:

$this->load->library('pagination');

$config['base_url'] = 'http://example.com/index.php/test/page/';
$config['total_rows'] = 200;
$config['per_page'] = 20;

$this->pagination->initialize($config);

echo $this->pagination->create_links();

Notas¶

The $config array contains your configuration variables. It is passed to the $this->pagination->initialize() method as shown above. Although existem uns vinte itens que você pode configurar, no mínimo você precisa do três mostrados. Aqui está uma descrição do que esses itens representam:

  • total_rows Este número representa o total de linhas no resultado set você está criando paginação para. Normalmente este número será o total de linhas retornadas pela Query do seu Database.
  • per_page O número de itens que você pretende mostrar por página. No acima exemplo, você estaria mostrando 20 itens por página.

The create_links() method returns an empty string when there is no paginação para mostrar.
