Classe de analisador de modelo¶

The Classe de analisador de modelo can perform simple text substitution for pseudo-variables contained within your view files. It can parse simple variables or variable tag pairs.

If you’ve never used a template engine, pseudo-variable names are enclosed in braces, like this:

<html>
        <head>
                <title>{blog_title}</title>
        </head>
        <body>
                <h3>{blog_heading}</h3>

        {blog_entries}
                <h5>{title}</h5>
                <p>{body}</p>
        {/blog_entries}

        </body>
</html>

These variables are not actual PHP variables, but rather plain text representations that allow you to eliminate PHP from your templates (ver arquivos).

Nota

CodeIgniter does not exigem que você use esta classe desde Usando PHP puro em suas páginas de visualização permite que eles sejam executados um pouco mais rápido. No entanto, alguns desenvolvedores preferem usar um mecanismo de modelo se they work with designers who they feel would find some confusion working with PHP.

Importante

The Classe de analisador de modelo is not a full-blown template parsing solution. We’ve kept it very lean on purpose in order to maintain maximum performance.

Using the Classe de analisador de modelo¶

Inicializando a classe¶

Like most other classes in CodeIgniter, the Parser class is initialized in your controller using the $this->load->library() method:

$this->load->library('parser');

Once loaded, the Parser library object will be available using: $this->parser

Parsing templates¶

You can use the parse() method to parse (or render) simple templates, like this:

$data = array(
        'blog_title' => 'My Blog Title',
        'blog_heading' => 'My Blog Heading'
);

$this->parser->parse('blog_template', $data);

The first parameter contains the name of the view file (neste exemplo, o arquivo seria chamado blog_template.php), and the second parameter contains an associative matriz de dados a serem substituídos no modelo. No exemplo acima, o modelo conteria duas variáveis: {blog_title} e {blog_heading}

There is no need to “echo” or do something with the data returned by $this->parser->parse(). It is automatically passed to the output class to be sent to the browser. However, if you do want the data returned em vez de enviar para a classe de saída você pode passar TRUE (booleano) como o third parameter:

$string = $this->parser->parse('blog_template', $data, TRUE);

Variable Pairs¶

The above example code allows simple variables to be replaced. What if you would like an entire block of variables to be repeated, with each iteration containing new values? Consider the template example we showed at the top of the page:

<html>
        <head>
                <title>{blog_title}</title>
        </head>
        <body>
                <h3>{blog_heading}</h3>

        {blog_entries}
                <h5>{title}</h5>
                <p>{body}</p>
        {/blog_entries}

        </body>
</html>

No código acima, você notará um par de variáveis: {blog_entries} dados… {/ blog_entries}. Em um caso como este, todo o pedaço de dados between these pairs would be repeated multiple times, corresponding to o número de linhas no elemento "blog_entries" da matriz de parâmetros.

Parsing variable pairs is done using the identical code shown above to parse single variables, except, you will add a multi-dimensional array corresponding to your variable pair data. Consider this example:

$this->load->library('parser');

$data = array(
        'blog_title'   => 'My Blog Title',
        'blog_heading' => 'My Blog Heading',
        'blog_entries' => array(
                array('title' => 'Title 1', 'body' => 'Body 1'),
                array('title' => 'Title 2', 'body' => 'Body 2'),
                array('title' => 'Title 3', 'body' => 'Body 3'),
                array('title' => 'Title 4', 'body' => 'Body 4'),
                array('title' => 'Title 5', 'body' => 'Body 5')
        )
);

$this->parser->parse('blog_template', $data);

Se seus dados de "par" forem provenientes de um resultado de Database, que já é um matriz multi-dimensional, você pode simplesmente usar o database result_array() method:

$query = $this->db->query("SELECT * FROM blog");

$this->load->library('parser');

$data = array(
        'blog_title'   => 'My Blog Title',
        'blog_heading' => 'My Blog Heading',
        'blog_entries' => $query->result_array()
);

$this->parser->parse('blog_template', $data);

Usage Notas¶

If you include substitution parameters that are not referenced in your modelo, eles são ignorados:

$template = 'Hello, {firstname} {lastname}';
$data = array(
        'title' => 'Mr',
        'firstname' => 'John',
        'lastname' => 'Doe'
);
$this->parser->parse_string($template, $data);

// Resultado: Hello, John Doe

If you do not include a substitution parameter that is referenced in your template, the original pseudo-variable is shown in the result:

$template = 'Hello, {firstname} {initials} {lastname}';
$data = array(
        'title' => 'Mr',
        'firstname' => 'John',
        'lastname' => 'Doe'
);
$this->parser->parse_string($template, $data);

// Resultado: Hello, John {initials} Doe

Se você fornecer um parâmetro de substituição de string quando uma matriz for esperada, ou seja, para um par de variáveis, a substituição é feita para a variável de abertura pair tag, but the closing variable pair tag is not rendered properly:

$template = 'Hello, {firstname} {lastname} ({degrees}{degree} {/degrees})';
$data = array(
        'degrees' => 'Mr',
        'firstname' => 'John',
        'lastname' => 'Doe',
        'titles' => array(
                array('degree' => 'BSc'),
                array('degree' => 'PhD')
        )
);
$this->parser->parse_string($template, $data);

// Resultado: Hello, John Doe (Mr{degree} {/degrees})

Se você nomear um dos seus parâmetros de substituição individuais da mesma forma que um usado dentro de um par variável, os resultados podem não ser os esperados:

$template = 'Hello, {firstname} {lastname} ({degrees}{degree} {/degrees})';
$data = array(
        'degree' => 'Mr',
        'firstname' => 'John',
        'lastname' => 'Doe',
        'degrees' => array(
                array('degree' => 'BSc'),
                array('degree' => 'PhD')
        )
);
$this->parser->parse_string($template, $data);

// Resultado: Hello, John Doe (Mr Mr )

Visualizar fragmentos¶

You do not have to use variable pairs to get the effect of iteration in your views. It is possible to use a view fragment for what would be inside a variable pair, and to control the iteration in your controller instead na vista.

An example with the iteration controlled in the view:

$template = '<ul>{menuitems}
        <li><a href="{link}">{title}</a></li>
{/menuitems}</ul>';

$data = array(
        'menuitems' => array(
                array('title' => 'First Link', 'link' => '/first'),
                array('title' => 'Second Link', 'link' => '/second'),
        )
);
$this->parser->parse_string($template, $data);

Resultado:

<ul>
        <li><a href="/first">First Link</a></li>
        <li><a href="/second">Second Link</a></li>
</ul>

An example with the iteration controlled in the controller, using a view fragment:

$temp = '';
$template1 = '<li><a href="{link}">{title}</a></li>';
$data1 = array(
        array('title' => 'First Link', 'link' => '/first'),
        array('title' => 'Second Link', 'link' => '/second'),
);

foreach ($data1 as $menuitem)
{
        $temp .= $this->parser->parse_string($template1, $menuitem, TRUE);
}

$template = '<ul>{menuitems}</ul>';
$data = array(
        'menuitems' => $temp
);
$this->parser->parse_string($template, $data);

Resultado:

<ul>
        <li><a href="/first">First Link</a></li>
        <li><a href="/second">Second Link</a></li>
</ul>

Classe de analisador de Model¶

The Classe de analisador de Model can perform simple text substitution for pseudo-variáveis contidas em seus arquivos de View. Pode analisar variáveis simples ou pares de tags variáveis.

Se você nunca usou um mecanismo de Model, nomes de pseudo-variáveis são colocados entre chaves, assim:

<html>
        <head>
                <title>{blog_title}</title>
        </head>
        <body>
                <h3>{blog_heading}</h3>

        {blog_entries}
                <h5>{title}</h5>
                <p>{body}</p>
        {/blog_entries}

        </body>
</html>

Essas variáveis não são variáveis reais do PHP, mas sim texto simples representações que permitem eliminar o PHP de seus Models (ver arquivos).

Nota

O CodeIgniter faz not exigem que você use esta classe desde Usando PHP puro em suas páginas de visualização permite que eles sejam executados um pouco mais rápido. No entanto, alguns desenvolvedores preferem usar um mecanismo de Model se eles trabalham com designers que eles acham que iria encontrar algum confusão trabalhando com PHP.

Importante

The Classe de analisador de Model is not um full-blown solução de análise de Model. Nós mantivemos isso muito firme de propósito, a fim para manter o desempenho máximo.

Using the Classe de analisador de Model¶

Inicializando a classe¶

Como a maioria das outras classes no CodeIgniter, a classe Parser é inicializada in your controller using the $this->load->library() method:

$this->load->library('parser');

Uma vez carregado, o objeto da Library Parser estará disponível usando: $this->parser

Models de análise¶

You can use the parse() method to parse (or render) simple templates, como isso:

$data = array(
        'blog_title' => 'My Blog Title',
        'blog_heading' => 'My Blog Heading'
);

$this->parser->parse('blog_template', $data);

O primeiro parâmetro contém o nome do view file (neste exemplo, o arquivo seria chamado blog_template.php), e o segundo parâmetro contém uma associação matriz de dados a serem substituídos no Model. No exemplo acima, o Model conteria duas variáveis: {blog_title} e {blog_heading}

Não há necessidade de "ecoar" ou fazer algo com os dados retornados por $this->parser->parse(). It is automatically passed to the output class para ser enviado para o navegador. No entanto, se você quiser que os dados retornados em vez de enviar para a classe de saída você pode passar TRUE (booleano) como o terceiro parâmetro:

$string = $this->parser->parse('blog_template', $data, TRUE);

Pares Variáveis¶

O código de exemplo acima permite que variáveis simples sejam substituídas. E se você gostaria que um bloco inteiro de variáveis fosse repetido, com cada iteração contendo novos valores? Considere o exemplo de Model que mostramos no topo da página:

<html>
        <head>
                <title>{blog_title}</title>
        </head>
        <body>
                <h3>{blog_heading}</h3>

        {blog_entries}
                <h5>{title}</h5>
                <p>{body}</p>
        {/blog_entries}

        </body>
</html>

No código acima, você notará um par de variáveis: {blog_entries} dados {/ blog_entries}. Em um caso como este, todo o pedaço de dados entre estes pares seria repetido várias vezes, correspondendo a o número de linhas no elemento "blog_entries" da matriz de parâmetros.

A análise de pares variáveis é feita usando o código idêntico mostrado acima para analisar variáveis individuais, exceto que você adicionará uma matriz multidimensional correspondente aos seus dados de par variável. Considere este exemplo:

$this->load->library('parser');

$data = array(
        'blog_title'   => 'My Blog Title',
        'blog_heading' => 'My Blog Heading',
        'blog_entries' => array(
                array('title' => 'Title 1', 'body' => 'Body 1'),
                array('title' => 'Title 2', 'body' => 'Body 2'),
                array('title' => 'Title 3', 'body' => 'Body 3'),
                array('title' => 'Title 4', 'body' => 'Body 4'),
                array('title' => 'Title 5', 'body' => 'Body 5')
        )
);

$this->parser->parse('blog_template', $data);

Se seus dados de "par" forem provenientes de um resultado de Database, que já é um matriz multi-dimensional, você pode simplesmente usar o database result_array() method:

$query = $this->db->query("SELECT * FROM blog");

$this->load->library('parser');

$data = array(
        'blog_title'   => 'My Blog Title',
        'blog_heading' => 'My Blog Heading',
        'blog_entries' => $query->result_array()
);

$this->parser->parse('blog_template', $data);

Usage Notas¶

Se você incluir parâmetros de substituição que não são referenciados em seu Model, eles são ignorados:

$template = 'Hello, {firstname} {lastname}';
$data = array(
        'title' => 'Mr',
        'firstname' => 'John',
        'lastname' => 'Doe'
);
$this->parser->parse_string($template, $data);

// Resultado: Hello, John Doe

Se você não incluir um parâmetro de substituição referenciado em seu template, a pseudo-variável original é mostrada no resultado:

$template = 'Hello, {firstname} {initials} {lastname}';
$data = array(
        'title' => 'Mr',
        'firstname' => 'John',
        'lastname' => 'Doe'
);
$this->parser->parse_string($template, $data);

// Resultado: Hello, John {initials} Doe

Se você fornecer um parâmetro de substituição de string quando uma matriz for esperada, ou seja, para um par de variáveis, a substituição é feita para a variável de abertura tag de par, mas a tag de par de variáveis de fechamento não é renderizada corretamente:

$template = 'Hello, {firstname} {lastname} ({degrees}{degree} {/degrees})';
$data = array(
        'degrees' => 'Mr',
        'firstname' => 'John',
        'lastname' => 'Doe',
        'titles' => array(
                array('degree' => 'BSc'),
                array('degree' => 'PhD')
        )
);
$this->parser->parse_string($template, $data);

// Resultado: Hello, John Doe (Mr{degree} {/degrees})

Se você nomear um dos seus parâmetros de substituição individuais da mesma forma que um usado dentro de um par variável, os resultados podem não ser os esperados:

$template = 'Hello, {firstname} {lastname} ({degrees}{degree} {/degrees})';
$data = array(
        'degree' => 'Mr',
        'firstname' => 'John',
        'lastname' => 'Doe',
        'degrees' => array(
                array('degree' => 'BSc'),
                array('degree' => 'PhD')
        )
);
$this->parser->parse_string($template, $data);

// Resultado: Hello, John Doe (Mr Mr )

Visualizar fragmentos¶

Você não precisa usar pares de variáveis para obter o efeito de iteração suas visualizações. É possível usar um fragmento de vista para o que estaria dentro um par variável, e para controlar a iteração em seu Controller na vista.

Um exemplo com a iteração controlada na exibição:

$template = '<ul>{menuitems}
        <li><a href="{link}">{title}</a></li>
{/menuitems}</ul>';

$data = array(
        'menuitems' => array(
                array('title' => 'First Link', 'link' => '/first'),
                array('title' => 'Second Link', 'link' => '/second'),
        )
);
$this->parser->parse_string($template, $data);

Resultado:

<ul>
        <li><a href="/first">First Link</a></li>
        <li><a href="/second">Second Link</a></li>
</ul>

Um exemplo com a iteração controlada no Controller, usando um fragmento de vista:

$temp = '';
$template1 = '<li><a href="{link}">{title}</a></li>';
$data1 = array(
        array('title' => 'First Link', 'link' => '/first'),
        array('title' => 'Second Link', 'link' => '/second'),
);

foreach ($data1 as $menuitem)
{
        $temp .= $this->parser->parse_string($template1, $menuitem, TRUE);
}

$template = '<ul>{menuitems}</ul>';
$data = array(
        'menuitems' => $temp
);
$this->parser->parse_string($template, $data);

Resultado:

<ul>
        <li><a href="/first">First Link</a></li>
        <li><a href="/second">Second Link</a></li>
</ul>

Classe de analisador de Model¶

The Classe de analisador de Model can perform simple text substitution for pseudo-variáveis contidas em seus arquivos de View. Pode analisar variáveis simples ou pares de tags variáveis.

Se você nunca usou um mecanismo de Model, nomes de pseudo-variáveis são colocados entre chaves, assim:

<html>
        <head>
                <title>{blog_title}</title>
        </head>
        <body>
                <h3>{blog_heading}</h3>

        {blog_entries}
                <h5>{title}</h5>
                <p>{body}</p>
        {/blog_entries}

        </body>
</html>

Essas variáveis não são variáveis reais do PHP, mas sim texto simples representações que permitem eliminar o PHP de seus Models (ver arquivos).

Nota

O CodeIgniter faz not exigem que você use esta classe desde Usando PHP puro em suas páginas de visualização permite que eles sejam executados um pouco mais rápido. No entanto, alguns desenvolvedores preferem usar um mecanismo de Model se eles trabalham com designers que eles acham que iria encontrar algum confusão trabalhando com PHP.

Important

The Classe de analisador de Model is not um full-blown solução de análise de Model. Nós mantivemos isso muito firme de propósito, a fim para manter o desempenho máximo.

Using the Classe de analisador de Model¶

Inicializando a classe¶

Como a maioria das outras classes no CodeIgniter, a classe Parser é inicializada in your controller using the $this->load->library() method:

$this->load->library('parser');

Uma vez carregado, o objeto da Library Parser estará disponível usando: $this->parser

Models de análise¶

You can use the parse() method to parse (or render) simple templates, like this:

$data = array(
        'blog_title' => 'My Blog Title',
        'blog_heading' => 'My Blog Heading'
);

$this->parser->parse('blog_template', $data);

O primeiro parâmetro contém o nome do view file (neste exemplo, o arquivo seria chamado blog_template.php), e o segundo parâmetro contém uma associação matriz de dados a serem substituídos no Model. No exemplo acima, o Model conteria duas variáveis: {blog_title} e {blog_heading}

Não há necessidade de "ecoar" ou fazer algo com os dados retornados por $this->parser->parse(). It is automatically passed to the output class para ser enviado para o navegador. No entanto, se você quiser que os dados retornados em vez de enviar para a classe de saída você pode passar TRUE (booleano) como o terceiro parâmetro:

$string = $this->parser->parse('blog_template', $data, TRUE);

Pares Variáveis¶

O código de exemplo acima permite que variáveis simples sejam substituídas. E se você gostaria que um bloco inteiro de variáveis fosse repetido, com cada iteração contendo novos valores? Considere o exemplo de Model que mostramos no topo da página:

<html>
        <head>
                <title>{blog_title}</title>
        </head>
        <body>
                <h3>{blog_heading}</h3>

        {blog_entries}
                <h5>{title}</h5>
                <p>{body}</p>
        {/blog_entries}

        </body>
</html>

No código acima, você notará um par de variáveis: {blog_entries} dados {/ blog_entries}. Em um caso como este, todo o pedaço de dados entre estes pares seria repetido várias vezes, correspondendo a o número de linhas no elemento "blog_entries" da matriz de parâmetros.

A análise de pares variáveis é feita usando o código idêntico mostrado acima para analisar variáveis individuais, exceto que você adicionará uma matriz multidimensional correspondente aos seus dados de par variável. Considere este exemplo:

$this->load->library('parser');

$data = array(
        'blog_title'   => 'My Blog Title',
        'blog_heading' => 'My Blog Heading',
        'blog_entries' => array(
                array('title' => 'Title 1', 'body' => 'Body 1'),
                array('title' => 'Title 2', 'body' => 'Body 2'),
                array('title' => 'Title 3', 'body' => 'Body 3'),
                array('title' => 'Title 4', 'body' => 'Body 4'),
                array('title' => 'Title 5', 'body' => 'Body 5')
        )
);

$this->parser->parse('blog_template', $data);

Se seus dados de "par" forem provenientes de um resultado de Database, que já é um matriz multi-dimensional, você pode simplesmente usar o database result_array() method:

$query = $this->db->query("SELECT * FROM blog");

$this->load->library('parser');

$data = array(
        'blog_title'   => 'My Blog Title',
        'blog_heading' => 'My Blog Heading',
        'blog_entries' => $query->result_array()
);

$this->parser->parse('blog_template', $data);

Usage Notas¶

Se você incluir parâmetros de substituição que não são referenciados em seu Model, eles são ignorados:

$template = 'Hello, {firstname} {lastname}';
$data = array(
        'title' => 'Mr',
        'firstname' => 'John',
        'lastname' => 'Doe'
);
$this->parser->parse_string($template, $data);

// Resultado: Hello, John Doe

Se você não incluir um parâmetro de substituição referenciado em seu template, a pseudo-variável original é mostrada no resultado:

$template = 'Hello, {firstname} {initials} {lastname}';
$data = array(
        'title' => 'Mr',
        'firstname' => 'John',
        'lastname' => 'Doe'
);
$this->parser->parse_string($template, $data);

// Resultado: Hello, John {initials} Doe

Se você fornecer um parâmetro de substituição de string quando uma matriz for esperada, ou seja, para um par de variáveis, a substituição é feita para a variável de abertura tag de par, mas a tag de par de variáveis de fechamento não é renderizada corretamente:

$template = 'Hello, {firstname} {lastname} ({degrees}{degree} {/degrees})';
$data = array(
        'degrees' => 'Mr',
        'firstname' => 'John',
        'lastname' => 'Doe',
        'titles' => array(
                array('degree' => 'BSc'),
                array('degree' => 'PhD')
        )
);
$this->parser->parse_string($template, $data);

// Resultado: Hello, John Doe (Mr{degree} {/degrees})

Se você nomear um dos seus parâmetros de substituição individuais da mesma forma que um usado dentro de um par variável, os resultados podem não ser os esperados:

$template = 'Hello, {firstname} {lastname} ({degrees}{degree} {/degrees})';
$data = array(
        'degree' => 'Mr',
        'firstname' => 'John',
        'lastname' => 'Doe',
        'degrees' => array(
                array('degree' => 'BSc'),
                array('degree' => 'PhD')
        )
);
$this->parser->parse_string($template, $data);

// Resultado: Hello, John Doe (Mr Mr )

Visualizar fragmentos¶

Você não precisa usar pares de variáveis para obter o efeito de iteração suas visualizações. É possível usar um fragmento de vista para o que estaria dentro um par variável, e para controlar a iteração em seu Controller na vista.

Um exemplo com a iteração controlada na exibição:

$template = '<ul>{menuitems}
        <li><a href="{link}">{title}</a></li>
{/menuitems}</ul>';

$data = array(
        'menuitems' => array(
                array('title' => 'First Link', 'link' => '/first'),
                array('title' => 'Second Link', 'link' => '/second'),
        )
);
$this->parser->parse_string($template, $data);

Resultado:

<ul>
        <li><a href="/first">First Link</a></li>
        <li><a href="/second">Second Link</a></li>
</ul>

Um exemplo com a iteração controlada no Controller, usando um fragmento de vista:

$temp = '';
$template1 = '<li><a href="{link}">{title}</a></li>';
$data1 = array(
        array('title' => 'First Link', 'link' => '/first'),
        array('title' => 'Second Link', 'link' => '/second'),
);

foreach ($data1 as $menuitem)
{
        $temp .= $this->parser->parse_string($template1, $menuitem, TRUE);
}

$template = '<ul>{menuitems}</ul>';
$data = array(
        'menuitems' => $temp
);
$this->parser->parse_string($template, $data);

Resultado:

<ul>
        <li><a href="/first">First Link</a></li>
        <li><a href="/second">Second Link</a></li>
</ul>
