Classe de Manipulação de Imagem¶

CodeIgniter’s Image Manipulation class lets you perform the following actions:

  • Redimensionamento de Imagem
  • Thumbnail Creation
  • Recorte de Imagem
  • Rotação de Imagem
  • Image Watermarking

Todas as três principais librarys de imagens são suportadas: GD/GD2, NetPBM e ImageMagick

Nota

A marca d'água só está disponível usando a library GD/GD2. Em addition, even though other libraries are supported, GD is required in order for the script to calculate the image properties. The image processamento, no entanto, será executado com a library que você especificar.

Inicializando a classe¶

Like most other classes in CodeIgniter, the image class is initialized in your controller using the $this->load->library function:

$this->load->library('image_lib');

Once the library is loaded it will be ready for use. The image library object you will use to call all functions is: $this->image_lib

Processing an Image¶

Independentemente do tipo de processamento que você gostaria de realizar (redimensionamento, corte, rotação ou marca d'água), o processo geral é identical. You will set some preferences corresponding to the action you pretendo executar, em seguida, chame uma das quatro funções de processamento disponíveis. For example, to create an image thumbnail you’ll do this:

$config['image_library'] = 'gd2';
$config['source_image'] = '/path/to/image/mypic.jpg';
$config['create_thumb'] = TRUE;
$config['maintain_ratio'] = TRUE;
$config['width']         = 75;
$config['height']       = 50;

$this->load->library('image_lib', $config);

$this->image_lib->resize();

O código acima diz a função image_resize para procurar uma imagem called mypic.jpg located in the source_image folder, then create a thumbnail that is 75 X 50 pixels using the GD2 image_library. Since the maintain_ratio option is enabled, the thumb will be as close to the largura e altura do alvo, preservando o aspecto original ratio. The thumbnail will be called mypic_thumb.jpg and located at the same level as source_image.

Nota

In order for the image class to be allowed to do any processing, the folder containing the image files must have write permissions.

Nota

O processamento de imagens pode exigir uma quantidade considerável de servidores memória para algumas operações. Se você estiver tendo erros de falta de memória while processing images you may need to limit their maximum size, and/or ajustar os limites de memória do PHP.

Processing Methods¶

Existem quatro métodos de processamento disponíveis:

  • $this->image_lib->resize()
  • $this->image_lib->crop()
  • $this->image_lib->rotate()
  • $this->image_lib->watermark()

These methods return boolean TRUE upon success and FALSE for failure. Se eles falharem, você pode recuperar a mensagem de erro usando esta função:

echo $this->image_lib->display_errors();

Uma boa prática é usar a função de processamento condicionalmente, mostrando error upon failure, like this:

if ( ! $this->image_lib->resize())
{
        echo $this->image_lib->display_errors();
}

Nota

Opcionalmente, você pode especificar a formatação HTML a ser aplicada a os erros, enviando as tags de abertura/fechamento na função, like this:

$this->image_lib->display_errors('<p>', '</p>');

Nota que nem todas as preferências estão disponíveis para todas as funções. Para example, the x/y axis preferences are only available for image cropping. Likewise, the width and height preferences have no effect on cropping. The “availability” column indicates which functions support a given preference.

Configurando Preferências em um Arquivo de Configuração¶

If you prefer not to set preferences using the above method, you can em vez disso, coloque-os em um arquivo de configuração. Basta criar um novo arquivo chamado image_lib.php, add the $config array in that file. Then save the file in config/image_lib.php and it will be used automatically. You will NOT need to use the $this->image_lib->initialize() method if you save suas preferências em um arquivo de configuração.

Image Watermarking¶

O recurso de marca d'água requer a library GD/GD2.

Two Types of Watermarking¶

There are two types of watermarking that you can use:

  • Text: The watermark message will be generated using text, either com uma fonte True Type que você especifica ou usando o texto nativo output that the GD library supports. If you use the True Type version sua instalação do GD deve ser compilada com suporte a True Type (a maioria are, but not all).
  • Overlay: A mensagem de marca d'água será gerada sobrepondo uma imagem (geralmente um PNG ou GIF transparente) contendo sua marca d'água over the source image.

Watermarking an Image¶

Just as with the other methods (resizing, cropping, and rotating) the general process for watermarking involves setting the preferences correspondente à ação que você pretende realizar e, em seguida, watermark function. Here is an example:

$config['source_image'] = '/path/to/image/mypic.jpg';
$config['wm_text'] = 'Copyright 2006 - John Doe';
$config['wm_type'] = 'text';
$config['wm_font_path'] = './system/fonts/texb.ttf';
$config['wm_font_size'] = '16';
$config['wm_font_color'] = 'ffffff';
$config['wm_vrt_alignment'] = 'bottom';
$config['wm_hor_alignment'] = 'center';
$config['wm_padding'] = '20';

$this->image_lib->initialize($config);

$this->image_lib->watermark();

O exemplo acima usará uma fonte True Type de 16 pixels para criar o texto “Copyright 2006 - John Doe”. A marca d'água será posicionada no parte inferior/centro da imagem, 20 pixels a partir da parte inferior da imagem.

Nota

In order for the image class to be allowed to do any processing, the image file must have “write” file permissions For example, 777.

Text Preferências¶

This table shows the preferences that are available for the text type of watermarking.

Preferência Default Value Opções Descrição
wm_text None None O texto que você gostaria de mostrar como marca d'água. Normalmente, isso será um copyright notice.
wm_font_path None None O caminho do servidor para a fonte True Type que você gostaria de usar. Se você fizer não use essa opção, a fonte GD nativa será usada.

wm_shadow_distance 3 None The distance (in pixels) from the font that the drop shadow should appear.

This table shows the preferences that are available for the overlay type of watermarking.

Preferência Default Value Opções Descrição
wm_overlay_path None None O caminho do servidor para a imagem que você deseja usar como marca d'água. Requeridos only if you are using the overlay method.

bool

Initializes the class for processing an image.

resize()¶
Returns:Verdadeiro sucesso, FALSE em falha
Return type:bool

O método de redimensionamento de imagem permite redimensionar a imagem original, criar um copy (with or without resizing), or create a thumbnail image.

For practical purposes there is no difference between creating a copy and creating a thumbnail except a thumb will have the thumbnail marker como parte do nome (ou seja, mypic_thumb.jpg).

All preferences listed in the Preferências tabela estão disponíveis para este method except these three: rotation_angle, x_axis and y_axis.

Creating a Thumbnail

The resizing method will create a thumbnail file (and preserve the original) if you set this preference to TRUE:

$config['create_thumb'] = TRUE;

This single preference determines whether a thumbnail is created or not.

Creating a Copy

O método de redimensionamento criará uma cópia do arquivo de imagem (e preservará original) se você definir um caminho e/ou um novo nome de arquivo usando preference:

$config['new_image'] = '/path/to/new_image.jpg';

Notas relacionadas a essa preferência:

  • Se somente o novo nome da imagem for especificado, ele será colocado na mesma folder as the original
  • Se somente o caminho for especificado, a nova imagem será colocada no destino com o mesmo nome do original.
  • If both the path and image name are specified it will placed in its próprio destino e dado o novo nome.

Redimensionando a imagem original

Se nenhuma das duas preferências listadas acima (create_thumb e new_image) are used, the resizing method will instead target the original image for processing.

crop()¶
Returns:Verdadeiro sucesso, FALSE em falha
Return type:bool

O método de corte funciona quase de forma idêntica à função de redimensionamento except it requires that you set preferences for the X and Y axis (in pixels) specifying where to crop, like this:

$config['x_axis'] = 100;
$config['y_axis'] = 40;

All preferences listed in the Preferências tabela estão disponíveis para este method except these: rotation_angle, create_thumb and new_image.

Here’s an example showing how you might crop an image:

$config['image_library'] = 'imagemagick';
$config['library_path'] = '/usr/X11R6/bin/';
$config['source_image'] = '/path/to/image/mypic.jpg';
$config['x_axis'] = 100;
$config['y_axis'] = 60;

$this->image_lib->initialize($config);

if ( ! $this->image_lib->crop())
{
        echo $this->image_lib->display_errors();
}

Nota

Without a visual interface it is difficult to crop images, so this method is not very useful unless you intend to build such an interface. That’s exactly what we did using for the photo gallery module no ExpressionEngine, o CMS que desenvolvemos. Adicionamos uma interface de JavaScript que lets the cropping area be selected.

rotate()¶
Returns:Verdadeiro sucesso, FALSE em falha
Return type:bool

The image rotation method requires that the angle of rotation be set via its preference:

$config['rotation_angle'] = '90';

There are 5 rotation options:

  1. 90 - rotates counter-clockwise by 90 degrees.
  2. 180 - rotates counter-clockwise by 180 degrees.
  3. 270 - gira no sentido anti-horário em 270 graus.
  4. hor - vira a imagem horizontalmente.
  5. vrt - flips the image vertically.

Here’s an example showing how you might rotate an image:

$config['image_library'] = 'netpbm';
$config['library_path'] = '/usr/bin/';
$config['source_image'] = '/path/to/image/mypic.jpg';
$config['rotation_angle'] = 'hor';

$this->image_lib->initialize($config);

if ( ! $this->image_lib->rotate())
{
        echo $this->image_lib->display_errors();
}
watermark()¶
Returns:Verdadeiro sucesso, FALSE em falha
Return type:bool

clear()¶
Return type:void

The clear method resets all of the values used when processing an image. You will want to call this if you are processing images in a loop.

$this->image_lib->clear();

Classe de Manipulação de Imagem¶

A classe Manipulação de Imagens do CodeIgniter permite que você execute o seguinte actions:

  • Redimensionamento de Imagem
  • Criação de miniaturas
  • Recorte de Imagem
  • Rotação de Imagem
  • Marca d'Água da Imagem

Todas as três principais librarys de imagens são suportadas: GD/GD2, NetPBM e ImageMagick

Nota

A marca d'água só está disponível usando a library GD/GD2. Em Além disso, apesar de outras Librarys serem suportadas, o GD é necessário ordem para o script calcular as propriedades da imagem. A imagem processamento, no entanto, será executado com a library que você especificar.

Inicializando a classe¶

Como a maioria das outras classes no CodeIgniter, a classe de imagem é inicializada no seu Controller usando a função $this-> load-> library:

$this->load->library('image_lib');

Depois que a Library for carregada, ela estará pronta para uso. A Library de imagens objeto que você usará para chamar todas as funções é: $this->image_lib

Processando uma imagem¶

Independentemente do tipo de processamento que você gostaria de realizar (redimensionamento, corte, rotação ou marca d'água), o processo geral é idêntico. Você irá definir algumas preferências correspondentes à ação que você pretendo executar, em seguida, chame uma das quatro funções de processamento disponíveis. Por exemplo, para criar uma miniatura de imagem, você fará o seguinte:

$config['image_library'] = 'gd2';
$config['source_image'] = '/path/to/image/mypic.jpg';
$config['create_thumb'] = TRUE;
$config['maintain_ratio'] = TRUE;
$config['width']         = 75;
$config['height']       = 50;

$this->load->library('image_lib', $config);

$this->image_lib->resize();

O código acima diz a função image_resize para procurar uma imagem called mypic.jpg located in the source_image folder, then create a miniatura de 75 x 50 pixels usando a Library de imagens GD2. Desde o A opção maintain_ratio está ativada, o polegar estará tão próximo largura e altura do alvo, preservando o aspecto original ratio. The thumbnail will be called mypic_thumb.jpg and located at the same level as source_image.

Nota

Para que a classe de imagem possa fazer qualquer processamento, a pasta que contém os arquivos de imagem deve ter permissions.

Nota

O processamento de imagens pode exigir uma quantidade considerável de servidores memória para algumas operações. Se você estiver tendo erros de falta de memória durante o processamento de imagens, talvez seja necessário limitar seu tamanho máximo e/ou ajustar os limites de memória do PHP.

Métodos de Processamento¶

Existem quatro métodos de processamento disponíveis:

  • $this->image_lib->resize()
  • $this->image_lib->crop()
  • $this->image_lib->rotate()
  • $this->image_lib->watermark()

Esses métodos retornam o booleano TRUE após o sucesso e FALSE para falha. Se eles falharem, você pode recuperar a mensagem de erro usando esta função:

echo $this->image_lib->display_errors();

Uma boa prática é usar a função de processamento condicionalmente, mostrando erro após falha, assim:

if ( ! $this->image_lib->resize())
{
        echo $this->image_lib->display_errors();
}

Nota

Opcionalmente, você pode especificar a formatação HTML a ser aplicada a os erros, enviando as tags de abertura/fechamento na função, como isso:

$this->image_lib->display_errors('<p>', '</p>');

Configurando Preferências em um Arquivo de Configuração¶

Se preferir não definir preferências usando o método acima, você pode em vez disso, coloque-os em um arquivo de configuração. Basta criar um novo arquivo chamado image_lib.php, adicione o array $config nesse arquivo. Em seguida, salve o arquivo in config/image_lib.php and it will be used automatically. You will NOT need to use the $this->image_lib->initialize() method if you save suas preferências em um arquivo de configuração.

Marca d'Água da Imagem¶

O recurso de marca d'água requer a library GD/GD2.

Dois tipos de marcas de água¶

Existem dois tipos de marcas d'água que você pode usar:

  • Text: A mensagem de marca d'água será gerada usando texto, com uma fonte True Type que você especifica ou usando o texto nativo saída que a Library GD suporta. Se você usa a versão True Type sua instalação do GD deve ser compilada com suporte a True Type (a maioria são, mas não todos).
  • Overlay: A mensagem de marca d'água será gerada sobrepondo uma imagem (geralmente um PNG ou GIF transparente) contendo sua marca d'água sobre a imagem de origem.

Marca d'água de uma imagem¶

Assim como com os outros métodos (redimensionar, cortar e girar), o processo geral de marca d'água envolve a configuração das preferências correspondente à ação que você pretende realizar e, em seguida, função de marca d'água. Aqui está um exemplo:

$config['source_image'] = '/path/to/image/mypic.jpg';
$config['wm_text'] = 'Copyright 2006 - John Doe';
$config['wm_type'] = 'text';
$config['wm_font_path'] = './system/fonts/texb.ttf';
$config['wm_font_size'] = '16';
$config['wm_font_color'] = 'ffffff';
$config['wm_vrt_alignment'] = 'bottom';
$config['wm_hor_alignment'] = 'center';
$config['wm_padding'] = '20';

$this->image_lib->initialize($config);

$this->image_lib->watermark();

O exemplo acima usará uma fonte True Type de 16 pixels para criar o texto "Copyright 2006 - John Doe". A marca d'água será posicionada no parte inferior/centro da imagem, 20 pixels a partir da parte inferior da imagem.

Nota

Para que a classe de imagem possa fazer qualquer processamento, o arquivo de imagem deve ter permissões de arquivo Por exemplo, 777.

resize()¶
Retorna:Verdadeiro sucesso, FALSE em falha
Tipo de retorno:bool

O método de redimensionamento de imagem permite redimensionar a imagem original, criar um cópia (com ou sem redimensionamento) ou crie uma imagem em miniatura.

Para fins práticos, não há diferença entre criar uma cópia e criar uma miniatura, exceto um polegar, terá o marcador de miniatura como parte do nome (ou seja, mypic_thumb.jpg).

Todas as preferências listadas no Preferências Tabela estão disponíveis para este method except these three: rotation_angle, x_axis and y_axis.

Criando uma Miniatura

O método de redimensionamento criará um arquivo de miniatura (e preservará o original) se você definir essa preferência como TRUE:

$config['create_thumb'] = TRUE;

Essa preferência única determina se uma miniatura é criada ou não.

Criando uma cópia

O método de redimensionamento criará uma cópia do arquivo de imagem (e preservará original) se você definir um caminho e/ou um novo nome de arquivo usando preference:

$config['new_image'] = '/path/to/new_image.jpg';

Notas relacionadas a essa preferência:

  • Se somente o novo nome da imagem for especificado, ele será colocado na mesma pasta como o original
  • Se somente o caminho for especificado, a nova imagem será colocada no destino com o mesmo nome do original.
  • Se o caminho e o nome da imagem forem especificados, ele será colocado em sua próprio destino e dado o novo nome.

Redimensionando a imagem original

Se nenhuma das duas preferências listadas acima (create_thumb e new_image) são usados, o método de redimensionamento terá como alvo imagem original para processamento.

crop()¶
Retorna:Verdadeiro sucesso, FALSE em falha
Tipo de retorno:bool

O método de corte funciona quase de forma idêntica à função de redimensionamento exceto que requer que você defina as preferências para os eixos X e Y (em pixels) specifying where to crop, como isso:

$config['x_axis'] = 100;
$config['y_axis'] = 40;

Todas as preferências listadas no Preferências Tabela estão disponíveis para este method except these: rotation_angle, create_thumb and new_image.

Veja um exemplo mostrando como você pode cortar uma imagem:

$config['image_library'] = 'imagemagick';
$config['library_path'] = '/usr/X11R6/bin/';
$config['source_image'] = '/path/to/image/mypic.jpg';
$config['x_axis'] = 100;
$config['y_axis'] = 60;

$this->image_lib->initialize($config);

if ( ! $this->image_lib->crop())
{
        echo $this->image_lib->display_errors();
}

Nota

Sem uma interface visual, é difícil cortar as imagens, por isso método não é muito útil, a menos que você pretenda construir tal interface. Isso é exatamente o que fizemos usando o módulo de galeria de fotos no ExpressionEngine, o CMS que desenvolvemos. Adicionamos uma interface de JavaScript que permite que a área de corte seja selecionada.

rotate()¶
Retorna:Verdadeiro sucesso, FALSE em falha
Tipo de retorno:bool

O método de rotação da imagem requer que o ângulo de rotação seja definido através de sua preferência:

$config['rotation_angle'] = '90';

Existem 5 opções de rotação:

  1. 90 - gira no sentido anti-horário em 90 graus.
  2. 180 - gira no sentido anti-horário em 180 graus.
  3. 270 - gira no sentido anti-horário em 270 graus.
  4. hor - vira a imagem horizontalmente.
  5. vrt - vira a imagem verticalmente.

Veja um exemplo mostrando como você pode girar uma imagem:

$config['image_library'] = 'netpbm';
$config['library_path'] = '/usr/bin/';
$config['source_image'] = '/path/to/image/mypic.jpg';
$config['rotation_angle'] = 'hor';

$this->image_lib->initialize($config);

if ( ! $this->image_lib->rotate())
{
        echo $this->image_lib->display_errors();
}
watermark()¶
Retorna:Verdadeiro sucesso, FALSE em falha
Tipo de retorno:bool

Cria uma marca d'água sobre uma imagem, consulte o Marca d'água de uma imagem seção para mais informações.

clear()¶
Tipo de retorno:void

O método claro redefine todos os valores usados ao processar um imagem. Você vai querer chamar isso se estiver processando imagens em um loop.

$this->image_lib->clear();

Classe de Manipulação de Imagem¶

A classe Manipulação de Imagens do CodeIgniter permite que você execute o seguinte actions:

  • Redimensionamento de Imagem
  • Criação de miniaturas
  • Recorte de Imagem
  • Rotação de Imagem
  • Marca d'Água da Imagem

Todas as três principais librarys de imagens são suportadas: GD/GD2, NetPBM e ImageMagick

Nota

A marca d'água só está disponível usando a library GD/GD2. Em Além disso, apesar de outras Librarys serem suportadas, o GD é necessário ordem para o script calcular as propriedades da imagem. A imagem processamento, no entanto, será executado com a library que você especificar.

Inicializando a classe¶

Como a maioria das outras classes no CodeIgniter, a classe de imagem é inicializada no seu Controller usando a função $this-> load-> library:

$this->load->library('image_lib');

Depois que a Library for carregada, ela estará pronta para uso. A Library de imagens objeto que você usará para chamar todas as funções é: $this->image_lib

Processando uma imagem¶

Independentemente do tipo de processamento que você gostaria de realizar (redimensionamento, corte, rotação ou marca d'água), o processo geral é idêntico. Você irá definir algumas preferências correspondentes à ação que você pretendo executar, em seguida, chame uma das quatro funções de processamento disponíveis. Por exemplo, para criar uma miniatura de imagem, você fará o seguinte:

$config['image_library'] = 'gd2';
$config['source_image'] = '/path/to/image/mypic.jpg';
$config['create_thumb'] = TRUE;
$config['maintain_ratio'] = TRUE;
$config['width']         = 75;
$config['height']       = 50;

$this->load->library('image_lib', $config);

$this->image_lib->resize();

O código acima diz a função image_resize para procurar uma imagem called mypic.jpg located in the source_image folder, then create a miniatura de 75 x 50 pixels usando a Library de imagens GD2. Desde o A opção maintain_ratio está ativada, o polegar estará tão próximo largura e altura do alvo, preservando o aspecto original ratio. The thumbnail will be called mypic_thumb.jpg and located at the same level as source_image.

Nota

Para que a classe de imagem possa fazer qualquer processamento, a pasta que contém os arquivos de imagem deve ter permissions.

Nota

O processamento de imagens pode exigir uma quantidade considerável de servidores memória para algumas operações. Se você estiver tendo erros de falta de memória durante o processamento de imagens, talvez seja necessário limitar seu tamanho máximo e/ou ajustar os limites de memória do PHP.

Métodos de Processamento¶

Existem quatro métodos de processamento disponíveis:

  • $this->image_lib->resize()
  • $this->image_lib->crop()
  • $this->image_lib->rotate()
  • $this->image_lib->watermark()

Esses métodos retornam o booleano TRUE após o sucesso e FALSE para falha. Se eles falharem, você pode recuperar a mensagem de erro usando esta função:

echo $this->image_lib->display_errors();

Uma boa prática é usar a função de processamento condicionalmente, mostrando erro após falha, assim:

if ( ! $this->image_lib->resize())
{
        echo $this->image_lib->display_errors();
}

Nota

Opcionalmente, você pode especificar a formatação HTML a ser aplicada a os erros, enviando as tags de abertura/fechamento na função, like this:

$this->image_lib->display_errors('<p>', '</p>');

Da mesma forma, as preferências de largura e altura não afetam o corte. A coluna "disponibilidade" indica quais funções suportam um dado preference.

Configurando Preferências em um Arquivo de Configuração¶

Se preferir não definir preferências usando o método acima, você pode em vez disso, coloque-os em um arquivo de configuração. Basta criar um novo arquivo chamado image_lib.php, adicione o array $config nesse arquivo. Em seguida, salve o arquivo in config/image_lib.php and it will be used automatically. You will NOT need to use the $this->image_lib->initialize() method if you save suas preferências em um arquivo de configuração.

Marca d'Água da Imagem¶

O recurso de marca d'água requer a library GD/GD2.

Dois tipos de marcas de água¶

Existem dois tipos de marcas d'água que você pode usar:

  • Text: A mensagem de marca d'água será gerada usando texto, com uma fonte True Type que você especifica ou usando o texto nativo saída que a Library GD suporta. Se você usa a versão True Type sua instalação do GD deve ser compilada com suporte a True Type (a maioria são, mas não todos).
  • Overlay: A mensagem de marca d'água será gerada sobrepondo uma imagem (geralmente um PNG ou GIF transparente) contendo sua marca d'água sobre a imagem de origem.

Marca d'água de uma imagem¶

Assim como com os outros métodos (redimensionar, cortar e girar), o processo geral de marca d'água envolve a configuração das preferências correspondente à ação que você pretende realizar e, em seguida, função de marca d'água. Aqui está um exemplo:

$config['source_image'] = '/path/to/image/mypic.jpg';
$config['wm_text'] = 'Copyright 2006 - John Doe';
$config['wm_type'] = 'text';
$config['wm_font_path'] = './system/fonts/texb.ttf';
$config['wm_font_size'] = '16';
$config['wm_font_color'] = 'ffffff';
$config['wm_vrt_alignment'] = 'bottom';
$config['wm_hor_alignment'] = 'center';
$config['wm_padding'] = '20';

$this->image_lib->initialize($config);

$this->image_lib->watermark();

O exemplo acima usará uma fonte True Type de 16 pixels para criar o texto "Copyright 2006 - John Doe". A marca d'água será posicionada no parte inferior/centro da imagem, 20 pixels a partir da parte inferior da imagem.

Nota

Para que a classe de imagem possa fazer qualquer processamento, o arquivo de imagem deve ter permissões de arquivo Por exemplo, 777.

resize()¶
Retorna:Verdadeiro sucesso, FALSE em falha
Tipo de retorno:bool

O método de redimensionamento de imagem permite redimensionar a imagem original, criar um cópia (com ou sem redimensionamento) ou crie uma imagem em miniatura.

Para fins práticos, não há diferença entre criar uma cópia e criar uma miniatura, exceto um polegar, terá o marcador de miniatura como parte do nome (ou seja, mypic_thumb.jpg).

Todas as preferências listadas no Preferências Tabela estão disponíveis para este method except these three: rotation_angle, x_axis and y_axis.

Criando uma Miniatura

O método de redimensionamento criará um arquivo de miniatura (e preservará o original) se você definir essa preferência como TRUE:

$config['create_thumb'] = TRUE;

Essa preferência única determina se uma miniatura é criada ou não.

Criando uma cópia

O método de redimensionamento criará uma cópia do arquivo de imagem (e preservará original) se você definir um caminho e/ou um novo nome de arquivo usando preference:

$config['new_image'] = '/path/to/new_image.jpg';

Notas relacionadas a essa preferência:

  • Se somente o novo nome da imagem for especificado, ele será colocado na mesma pasta como o original
  • Se somente o caminho for especificado, a nova imagem será colocada no destino com o mesmo nome do original.
  • Se o caminho e o nome da imagem forem especificados, ele será colocado em sua próprio destino e dado o novo nome.

Redimensionando a imagem original

Se nenhuma das duas preferências listadas acima (create_thumb e new_image) são usados, o método de redimensionamento terá como alvo imagem original para processamento.

crop()¶
Retorna:Verdadeiro sucesso, FALSE em falha
Tipo de retorno:bool

O método de corte funciona quase de forma idêntica à função de redimensionamento exceto que requer que você defina as preferências para os eixos X e Y (em pixels) especificando onde cortar, assim:

$config['x_axis'] = 100;
$config['y_axis'] = 40;

Todas as preferências listadas no Preferências Tabela estão disponíveis para este method except these: rotation_angle, create_thumb and new_image.

Veja um exemplo mostrando como você pode cortar uma imagem:

$config['image_library'] = 'imagemagick';
$config['library_path'] = '/usr/X11R6/bin/';
$config['source_image'] = '/path/to/image/mypic.jpg';
$config['x_axis'] = 100;
$config['y_axis'] = 60;

$this->image_lib->initialize($config);

if ( ! $this->image_lib->crop())
{
        echo $this->image_lib->display_errors();
}

Nota

Sem uma interface visual, é difícil cortar as imagens, por isso método não é muito útil, a menos que você pretenda construir tal interface. Isso é exatamente o que fizemos usando o módulo de galeria de fotos no ExpressionEngine, o CMS que desenvolvemos. Adicionamos uma interface de JavaScript que permite que a área de corte seja selecionada.

rotate()¶
Retorna:Verdadeiro sucesso, FALSE em falha
Tipo de retorno:bool

O método de rotação da imagem requer que o ângulo de rotação seja definido através de sua preferência:

$config['rotation_angle'] = '90';

Existem 5 opções de rotação:

  1. 90 - gira no sentido anti-horário em 90 graus.
  2. 180 - gira no sentido anti-horário em 180 graus.
  3. 270 - gira no sentido anti-horário em 270 graus.
  4. hor - vira a imagem horizontalmente.
  5. vrt - vira a imagem verticalmente.

Veja um exemplo mostrando como você pode girar uma imagem:

$config['image_library'] = 'netpbm';
$config['library_path'] = '/usr/bin/';
$config['source_image'] = '/path/to/image/mypic.jpg';
$config['rotation_angle'] = 'hor';

$this->image_lib->initialize($config);

if ( ! $this->image_lib->rotate())
{
        echo $this->image_lib->display_errors();
}
