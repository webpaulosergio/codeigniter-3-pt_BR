Classe URI¶

The Classe URI provides methods that help you retrieve information from your String URIs. If you use URI routing, you can also retrieve information about the re-routed segments.

Nota

Esta classe é inicializada automaticamente pelo sistema para que is no need to do it manually.

  • Class Reference

Class Reference¶

class CI_URI¶
segment($n[, $no_result = NULL])¶
Parâmetros:
  • $n (int) – Segment index number
  • $no_result (mixed) – What to return if the searched segment is not found
Returns:

Valor do segmento ou valor $no_result, se não encontrado
Return type:

mixed

Permite que você recupere um segmento específico. Onde n é o segmento number you wish to retrieve. Segments are numbered from left to right. For example, if your full URL is this:

http://example.com/index.php/news/local/metro/crime_is_up

Os números do segmento seriam:

  1. news
  2. local
  3. metro
  4. crime_is_up

The optional second parameter defaults to NULL and allows you to set the return value of this method when the requested URI segment is missing. Por exemplo, isso diria ao método para retornar o número zero no caso de falha:

$product_id = $this->uri->segment(3, 0);

Isso ajuda a evitar ter que escrever código como este:

if ($this->uri->segment(3) === FALSE)
{
        $product_id = 0;
}
else
{
        $product_id = $this->uri->segment(3);
}
rsegment($n[, $no_result = NULL])¶
Parâmetros:
  • $n (int) – Segment index number
  • $no_result (mixed) – What to return if the searched segment is not found
Returns:

Valor do segmento roteado ou valor $no_result, se não encontrado
Return type:

mixed

This method is identical to segment(), except that it lets you retrieve a specific segment from your re-routed URI in the event you are usando CodeIgniter's URI Roteamento feature.

slash_segment($n[, $where = 'trailing'])¶
Parâmetros:
  • $n (int) – Segment index number
  • $where (string) – Where to add the slash (‘trailing’ or ‘leading’)
Returns:

Segment value, prepended/suffixed with a forward slash, or a slash if not found
Return type:

string

This method is almost identical to segment(), except it adds a trailing and/or leading slash based on the second parameter. Se o parâmetro não for usado, será adicionada uma barra à direita. Exemplos:

$this->uri->slash_segment(3);
$this->uri->slash_segment(3, 'leading');
$this->uri->slash_segment(3, 'both');

Returns:

  1. segment/
  2. /segment
  3. /segment/
slash_rsegment($n[, $where = 'trailing'])¶
Parâmetros:
  • $n (int) – Segment index number
  • $where (string) – Where to add the slash (‘trailing’ or ‘leading’)
Returns:

Valor do segmento roteado, prefixado/com sufixo com uma barra invertida ou uma barra se não encontrado
Return type:

string

This method is identical to slash_segment(), except that it lets you add slashes a specific segment from your re-routed URI in the event you estão usando o CodeIgniter URI Roteamento feature.

uri_to_assoc([$n = 3[, $default = array()]])¶
Parâmetros:
  • $n (int) – Segment index number
  • $default (array) – Default values
Returns:

Matriz de segmentos URI associativos
Return type:

array

This method lets you turn URI segments into an associative array of key/value pairs. Consider this URI:

index.php/user/search/name/joe/location/UK/gender/male

Using this method you can turn the URI into an associative array with this prototype:

[array]
(
        'name'          => 'joe'
        'location'      => 'UK'
        'gender'        => 'male'
)

The first parameter lets you set an offset, which defaults to 3 since your O URI normalmente conterá um par controlador/método no primeiro e segundo segmentos. Example:

$array = $this->uri->uri_to_assoc(3);
echo $array['name'];

O segundo parâmetro permite definir nomes de chaves padrão, para que a matriz returned will always contain expected indexes, even if missing from the URI. Example:

$default = array('name', 'gender', 'location', 'type', 'sort');
$array = $this->uri->uri_to_assoc(3, $default);

If the URI does not contain a value in your default, an array index will ser definido para esse nome, com um valor NULL.

Por fim, se um valor correspondente não for encontrado para uma dada chave is an odd number of URI segments) the value will be set to NULL.

ruri_to_assoc([$n = 3[, $default = array()]])¶
Parâmetros:
  • $n (int) – Segment index number
  • $default (array) – Default values
Returns:

Associative routed Matriz de segmentos URI
Return type:

array

This method is identical to uri_to_assoc(), except that it creates an associative array using the re-routed URI in the event you are using CodeIgniter's URI Roteamento feature.

assoc_to_uri($array)¶
Parâmetros:
  • $array (array) - Matriz de entrada de pares chave/valor
Returns:

String URI
Return type:

string

Utiliza um array associativo como entrada e gera uma string URI a partir dele. The array keys will be included in the string. Example:

$array = array('product' => 'shoes', 'size' => 'large', 'color' => 'red');
$str = $this->uri->assoc_to_uri($array);

// Produces: product/shoes/size/large/color/red
uri_string()¶
Returns:String URI
Return type:string

Retorna uma string com o URI completo. Por exemplo, se este for seu URL completo:

http://example.com/index.php/news/local/345

O método retornaria isso:

news/local/345
ruri_string()¶
Returns:Routed String URI
Return type:string

This method is identical to uri_string(), except that it returns the re-routed URI in the event you estão usando o CodeIgniter URI Roteamento feature.

total_segments()¶
Returns:Count of URI segments
Return type:int

Retorna o número total de segmentos.

total_rsegments()¶
Returns:Contagem de segmentos de URI roteados
Return type:int

This method is identical to total_segments(), except that it returns the total number of segments in your re-routed URI in the event you are usando CodeIgniter's URI Roteamento feature.

segment_array()¶
Returns:Matriz de segmentos URI
Return type:array

Returns an array containing the URI segments. For example:

$segs = $this->uri->segment_array();

foreach ($segs as $segment)
{
        echo $segment;
        echo '<br />';
}
rsegment_array()¶
Returns:Routed Matriz de segmentos URI
Return type:array

This method is identical to segment_array(), except that it returns a matriz de segmentos em seu URI redirecionado no caso de você estar usando CodeIgniter's URI Roteamento feature.

The Classe URI provides methods that help you retrieve information from suas strings de URI. Se você usa o roteamento URI, também pode recuperar informações sobre os segmentos reencaminhados.

Nota

Esta classe é inicializada automaticamente pelo sistema para que Não é necessário fazer isso manualmente.

class CI_URI¶
segment($n[, $no_result = NULL])¶
Parâmetros:
  • $n (int) - Número do índice do segmento
  • $no_result (mixed) - O que devolver se o segmento pesquisado não for encontrado
Retorna:

Valor do segmento ou valor $no_result, se não encontrado

mixed

  1. news
  2. local
  3. metro
  4. crime_is_up

O segundo parâmetro opcional é padronizado como NULL e permite que você defina o valor de retorno deste método quando o segmento de URI solicitado está faltando. Por exemplo, isso diria ao método para retornar o número zero no caso de falha:

$product_id = $this->uri->segment(3, 0);

{
        $product_id = 0;
}
else
{
        $product_id = $this->uri->segment(3);
}
rsegment($n[, $no_result = NULL])¶
Parâmetros:
  • $n (int) - Número do índice do segmento

Retorna:

Valor do segmento roteado ou valor $no_result, se não encontrado
Tipo de retorno:

mixed

This method is identical to segment(), except that it lets you retrieve um segmento específico de seu URI redirecionado, caso você esteja usando CodeIgniter's URI Roteamento feature.

slash_segment($n[, $where = 'trailing'])¶
Parâmetros:
  • $n (int) - Número do índice do segmento
  • $where (string) - Onde adicionar a barra ('trailing' ou 'leading')
Retorna:

Valor do segmento, prefixado/sufixado com uma barra ou uma barra, se não for encontrado
Tipo de retorno:

string

This method is almost identical to segment(), except it adiciona uma barra à direita e/ou à esquerda baseada no segundo parâmetro. Se o parâmetro não for usado, será adicionada uma barra à direita. Exemplos:

$this->uri->slash_segment(3);

$this->uri->slash_segment(3, 'both');

Retorna:

  1. segment/
  2. /segment
  3. /segment/
slash_rsegment($n[, $where = 'trailing'])¶
Parâmetros:
  • $n (int) - Número do índice do segmento
  • $where (string) - Onde adicionar a barra ('trailing' ou 'leading')
Retorna:

Valor do segmento roteado, prefixado/com sufixo com uma barra invertida ou uma barra se não encontrado
Tipo de retorno:

string

This method is identical to slash_segment(), except that it lets you adicionar barras a um segmento específico de seu URI redirecionado no caso de você estão usando o CodeIgniter URI Roteamento feature.

uri_to_assoc([$n = 3[, $default = array()]])¶
Parâmetros:
  • $n (int) - Número do índice do segmento
  • $default (array) - Valores padrão
Retorna:

Matriz de segmentos URI associativos
Tipo de retorno:

array

Esse método permite transformar segmentos URI em uma matriz associativa de pares chave/valor. Considere este URI:

index.php/user/search/name/joe/location/UK/gender/male

Usando este método você pode transformar o URI em um array associativo com este protótipo:

[array]
(
        'name'          => 'joe'
        'location'      => 'UK'

Classe URI¶

The Classe URI provides methods that help you retrieve information from suas strings de URI. Se você usa o roteamento URI, também pode recuperar informações sobre os segmentos reencaminhados.

Nota

Esta classe é inicializada automaticamente pelo sistema para que Não é necessário fazer isso manualmente.

  • Referência de Classe

Referência de Classe¶

class CI_URI¶
segment($n[, $no_result = NULL])¶
Parâmetros:
  • $n (int) - Número do índice do segmento
  • $no_result (mixed) - O que devolver se o segmento pesquisado não for encontrado
Retorna:

Valor do segmento ou valor $no_result, se não encontrado
Tipo de retorno:

mixed

Permite que você recupere um segmento específico. Onde n é o segmento número que você deseja recuperar. Os segmentos são numerados da esquerda para a direita. Por exemplo, se o seu URL completo é isto:

http://example.com/index.php/news/local/metro/crime_is_up

Os números do segmento seriam:

  1. news
  2. local
  3. metro
  4. crime_is_up

O segundo parâmetro opcional é padronizado como NULL e permite que você defina o valor de retorno deste método quando o segmento de URI solicitado está faltando. Por exemplo, isso diria ao método para retornar o número zero no caso de falha:

$product_id = $this->uri->segment(3, 0);

Isso ajuda a evitar ter que escrever código como este:

if ($this->uri->segment(3) === FALSE)
{
        $product_id = 0;
}
else
{
        $product_id = $this->uri->segment(3);
}
rsegment($n[, $no_result = NULL])¶
Parâmetros:
  • $n (int) - Número do índice do segmento
  • $no_result (mixed) - O que devolver se o segmento pesquisado não for encontrado
Retorna:

Valor do segmento roteado ou valor $no_result, se não encontrado
Tipo de retorno:

mixed

This method is identical to segment(), except that it lets you retrieve um segmento específico de seu URI redirecionado, caso você esteja usando CodeIgniter's URI Roteamento feature.

slash_segment($n[, $where = 'trailing'])¶
Parâmetros:
  • $n (int) - Número do índice do segmento
  • $where (string) - Onde adicionar a barra ('trailing' ou 'leading')
Retorna:

Valor do segmento, prefixado/sufixado com uma barra ou uma barra, se não for encontrado
Tipo de retorno:

string

This method is almost identical to segment(), except it adiciona uma barra à direita e/ou à esquerda baseada no segundo parâmetro. Se o parâmetro não for usado, será adicionada uma barra à direita. Exemplos:

$this->uri->slash_segment(3);
$this->uri->slash_segment(3, 'leading');
$this->uri->slash_segment(3, 'both');

Retorna:

  1. segment/
  2. /segment
  3. /segment/
