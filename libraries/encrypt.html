Criptografar Classe¶

The Criptografar Classe provides two-way data encryption. It encrypted using the Mcrypt PHP extension, which is required for the Criptografar Classe to run.

Importante

Esta library foi DEPRECADA e só é mantida por compatibilidade com versões anteriores. Por favor use o novo Library de Criptografia.

Usando a library de criptografia¶

Configurando sua chave¶

A key is a piece of information that controls the cryptographic process and permits an encrypted string to be decoded. In fact, the key você escolheu fornecerá o only significa decodificar dados que foram criptografado com essa chave, portanto, não apenas você deve escolher a chave com cuidado, você nunca deve alterá-lo se você pretende usá-lo para dados persistentes.

Escusado será dizer que você deve guardar sua chave com cuidado. Devemos someone gain access to your key, the data will be easily decoded. If your server is not totally under your control it’s impossible to ensure segurança chave para que você possa querer pensar com cuidado antes de usá-lo para anything that requires high security, like storing credit card numbers.

To take maximum advantage of the encryption algorithm, your key should be 32 characters in length (256 bits). The key should be as random a string como você pode inventar, com números e maiúsculas e minúsculas cartas. Sua chave deve not be a simple text string. In order to be criptograficamente seguro, ele precisa ser o mais aleatório possível.

Your key can be either stored in your application/config/config.php, or você pode projetar seu próprio mecanismo de armazenamento e passar a chave dinamicamente when encoding/decoding.

Para salvar sua chave no seu application/config/config.php, open the file and set:

$config['encryption_key'] = "YOUR KEY";

Message Length¶

É importante que você saiba que as mensagens codificadas são criptografadas function generates will be approximately 2.6 times longer than the mensagem original. Por exemplo, se você criptografar a string “my super dados secretos ”, que tem 21 caracteres, você terminará com um encoded string that is roughly 55 characters (we say “roughly” because o comprimento da string codificada é incrementado em clusters de 64 bits, então não é exatamente linear). Mantenha estas informações em mente ao selecionar seus dados mecanismo de armazenamento. Os cookies, por exemplo, só podem conter 4K de information.

Inicializando a classe¶

Like most other classes in CodeIgniter, the Encrypt class is initialized in your controller using the $this->load->library() method:

$this->load->library('encrypt');

Once loaded, the Encrypt library object will be available using:

$this->encrypt

$encrypted_string = $this->encrypt->encode($msg);

$msg = 'My secret message';
$key = 'super-secret-key';

$encrypted_string = $this->encrypt->encode($msg, $key);

encode_from_legacy($string[, $legacy_mode = MCRYPT_MODE_ECB[, $key = '']])¶
Parâmetros:
  • $string (string) – String to encrypt
  • $legacy_mode (int) – Valid PHP MCrypt cipher constant
  • $key (string) – Encryption key
Returns:

Newly encrypted string
Return type:

string

Permite recodificar dados que foram originalmente criptografados com CodeIgniter 1.x to be compatible with the Encrypt library in CodeIgniter 2.x. Só é necessário usar este método se você tiver dados criptografados armazenados permanentemente, como em um arquivo ou database, e em um servidor que suporta Mcrypt. “Light” usa criptografia como dados de sessão criptografados ou flashdata criptografados transitórios não exigem intervention on your part. However, existing encrypted Sessions will be destruído, pois os dados criptografados antes de 2.x não serão decodificados.

Importante

Why only a method to re-encode the data instead of maintaining legacy methods for both encoding and decoding? Os algoritmos no Criptografar library melhorou no CodeIgniter 2.x tanto para desempenho segurança, e não desejamos incentivar o uso continuado do antigo methods. You can of course extend the Encryption library if you wish and substituir os novos métodos com os antigos e manter compatibilidade contínua with CodeIgniter 1.x encrypted data, but this a decision that a desenvolvedor deve fazer com cautela e deliberadamente, se em tudo.

$new_data = $this->encrypt->encode_from_legacy($old_encrypted_string);
Parâmetro Padrão Descrição
$orig_data n/a Os dados criptografados originais da library de criptografia do CodeIgniter 1.x
$legacy_mode MCRYPT_MODE_ECB O modo Mcrypt que foi usado para gerar os dados criptografados originais. CodeIgniter 1.x’s default was MCRYPT_MODE_ECB, and it will assume that para ser o caso, a menos que substituído por este parâmetro.
$key n/a The encryption key. This it typically specified in your config file as outlined above.

Classe de Criptografia¶

The Classe de Criptografia provides two-way data encryption. It encrypted using the Mcrypt PHP extension, which is required for the Classe de Criptografia to run.

Important

Esta library foi DEPRECADA e só é mantida por compatibilidade com versões anteriores. Por favor use o novo Library de Criptografia.

Usando a library de criptografia¶

Configurando sua chave¶

A key is a piece of information that controls the cryptographic processo e permite que uma string criptografada seja decodificada. Na verdade, a chave você escolheu fornecerá o only significa decodificar dados que foram criptografado com essa chave, portanto, não apenas você deve escolher a chave com cuidado, você nunca deve alterá-lo se você pretende usá-lo para dados persistentes.

Escusado será dizer que você deve guardar sua chave com cuidado. Devemos alguém obter acesso à sua chave, os dados serão facilmente decodificados. E se seu servidor não está totalmente sob seu controle, é impossível garantir segurança chave para que você possa querer pensar com cuidado antes de usá-lo para qualquer coisa que exija alta segurança, como armazenar números de cartão de crédito.

Para aproveitar ao máximo o algoritmo de criptografia, sua chave deve ter 32 caracteres de comprimento (256 bits). A chave deve ser tão aleatória string como você pode inventar, com números e maiúsculas e minúsculas cartas. Sua chave deve not seja uma string de texto simples. De forma a criptograficamente seguro, ele precisa ser o mais aleatório possível.

Sua chave pode ser armazenada no seu application/config/config.php, or você pode projetar seu próprio mecanismo de armazenamento e passar a chave dinamicamente quando codificando/decodificando.

Para salvar sua chave no seu application/config/config.phpabra o arquivo and set:

$config['encryption_key'] = "YOUR KEY";

Comprimento da Mensagem¶

É importante que você saiba que as mensagens codificadas são criptografadas função gerada será aproximadamente 2,6 vezes maior que a mensagem original. Por exemplo, se você criptografar a string "my super dados secretos ", que tem 21 caracteres, você terminará com um seqüência codificada que é aproximadamente 55 caracteres (dizemos "aproximadamente" porque o comprimento da string codificada é incrementado em clusters de 64 bits, então não é exatamente linear). Mantenha estas informações em mente ao selecionar seus dados mecanismo de armazenamento. Os cookies, por exemplo, só podem conter 4K de information.

Inicializando a classe¶

Como a maioria das outras classes no CodeIgniter, a classe Encrypt é inicializado em seu Controller usando o $this->load->library() method:

$this->load->library('encrypt');

Uma vez carregado, o objeto Encrypt library estará disponível usando:

$this->encrypt
