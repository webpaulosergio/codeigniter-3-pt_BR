Classe de saída¶

The Output class is a core class with one main function: To send the finalized web page to the requesting browser. It is also responsible for caching your web pages, if you use that feature.

Nota

Esta classe é inicializada automaticamente pelo sistema para que is no need to do it manually.

Under normal circumstances you won’t even notice the Output class since it works transparently without your intervention. For example, when you use o Loader class to load a view file, it’s automatically passed to the Output class, which will be called automatically by CodeIgniter at the end of system execution. It is possível, no entanto, para você intervir manualmente com a saída se você preciso.

  • Class Reference

Class Reference¶

class CI_Output¶
$parse_exec_vars = TRUE;

Enables/disables parsing of the {elapsed_time} and {memory_usage} pseudo-variables.

O CodeIgniter irá analisar esses tokens em sua saída por padrão. Para desabilitar isso, defina esta propriedade para FALSE no seu controlador.

$this->output->parse_exec_vars = FALSE;
set_output($output)¶
Parâmetros:
  • $output (string) - String para definir a saída para
Returns:

CI_Output instance (method chaining)
Return type:

CI_Output

Permits you to manually set the final output string. Usage example:

$this->output->set_output($data);

Importante

Se você definir sua saída manualmente, ela deve ser a última coisa feita in the function you call it from. For example, if you build a page in one dos métodos do seu controlador, não defina a saída até o final.

set_content_type($mime_type[, $charset = NULL])¶
Parâmetros:
  • $mime_type (string) – MIME Type idenitifer string
  • $charset (string) – Character set
Returns:

CI_Output instance (method chaining)
Return type:

CI_Output

Permits you to set the mime-type of your page so you can serve JSON data, JPEG’s, XML, etc easily.

$this->output
        ->set_content_type('application/json')
        ->set_output(json_encode(array('foo' => 'bar')));

$this->output
        ->set_content_type('jpeg') // You could also use ".jpeg" which will have the full stop removed before looking in config/mimes.php
        ->set_output(file_get_contents('files/something.jpg'));

Importante

Make sure any non-mime string you pass to this method exists in application/config/mimes.php or it will have no effect.

You can also set the character set of the document, by passing a second argument:

$this->output->set_content_type('css', 'utf-8');
get_content_type()¶
Returns:String de tipo de conteúdo
Return type:string

Returns the Content-Type HTTP header that’s currently in use, excluding the character set value.

$mime = $this->output->get_content_type();

Nota

Se não definido, o valor de retorno padrão é 'text/html'.

get_header($header)¶
Parâmetros:
  • $header (string) - nome do cabeçalho HTTP
Returns:

HTTP response header or NULL if not found
Return type:

mixed

Returns the requested HTTP header value, or NULL if the requested header is not set. Example:

$this->output->set_content_type('text/plain', 'UTF-8');
echo $this->output->get_header('content-type');
// Outputs: text/plain; charset=utf-8

Nota

O nome do cabeçalho é comparado de maneira não diferencia maiúsculas de minúsculas.

Nota

Raw headers sent via PHP’s native header() function are also detected.

get_output()¶
Returns:Output string
Return type:string

Permits you to manually retrieve any output that has been sent for storage in the output class. Usage example:

$string = $this->output->get_output();

Nota that data will only be retrievable from this function if it has sido enviado anteriormente para a classe de saída por um dos CodeIgniter functions like $this->load->view().

append_output($output)¶
Parâmetros:
  • $output (string) – Additional output data to append
Returns:

CI_Output instance (method chaining)
Return type:

CI_Output

Acrescenta dados na seqüência de saída.

$this->output->append_output($data);
set_header($header[, $replace = TRUE])¶
Parâmetros:
  • $header (string) – HTTP response header
  • $replace (bool) – Whether to replace the old header value, if it is already set
Returns:

CI_Output instance (method chaining)
Return type:

CI_Output

Permite que você defina manualmente cabeçalhos de servidor, que a classe de saída enviar para você ao produzir a exibição renderizada final. Exemplo:

$this->output->set_header('HTTP/1.0 200 OK');
$this->output->set_header('HTTP/1.1 200 OK');
$this->output->set_header('Last-Modified: '.gmdate('D, d M Y H:i:s', $last_update).' GMT');
$this->output->set_header('Cache-Control: no-store, no-cache, must-revalidate');
$this->output->set_header('Cache-Control: post-check=0, pre-check=0');
$this->output->set_header('Pragma: no-cache');
set_status_header([$code = 200[, $text = '']])¶
Parâmetros:
  • $code (int) – HTTP status code
  • $text (string) - Mensagem opcional
Returns:

CI_Output instance (method chaining)
Return type:

CI_Output

Permits you to manually set a server status header. Example:

$this->output->set_status_header(401);
// Sets the header as:  Unauthorized

See here for a full list of headers.

Nota

This method is an alias for Common function set_status_header().

enable_profiler([$val = TRUE])¶
Parâmetros:
  • $val (bool) - Ativar ou desativar o Profiler
Returns:

CI_Output instance (method chaining)
Return type:

CI_Output

Permite que você ative/desative o Profiler, que exibirá benchmark e outros dados na parte inferior de suas páginas para fins de depuração e otimização.

To enable the profiler place the following line anywhere within your Controller methods:

$this->output->enable_profiler(TRUE);

Quando ativado, um relatório será gerado e inserido na parte inferior de suas páginas.

Para desabilitar o profiler, você usaria:

$this->output->enable_profiler(FALSE);

Parâmetros:
  • $sections (array) - seções Profiler
Returns:

CI_Output instance (method chaining)
Return type:

CI_Output

Permite que você ative/desative seções específicas do Profiler quando este estiver ativado. Please refer to the Profiler documentation for further information.

cache($time)¶
Parâmetros:
  • $time (int) - Tempo de expiração do cache em minutos
Returns:

CI_Output instance (method chaining)
Return type:

CI_Output

Caches a página atual para a quantidade especificada de minutos.

For more information, please see the documentação de cache.

_display([$output = ''])¶
Parâmetros:
  • $output (string) - Substituição de dados de saída
Returns:

void
Return type:

void

Envia dados de saída finalizados para o navegador junto com todos os cabeçalhos de servidor. Também pára o benchmark timers.

Nota

This method is called automatically at the end of script execution, you won’t need to call it manually unless you are aborting script execution using exit() or die() in your code.

Example:

$response = array('status' => 'OK');

$this->output
        ->set_status_header(200)
        ->set_content_type('application/json', 'utf-8')
        ->set_output(json_encode($response, JSON_PRETTY_PRINT | JSON_UNESCAPED_UNICODE | JSON_UNESCAPED_SLASHES))
        ->_display();
exit;

Nota

Calling this method manually without aborting script execution will result in duplicated output.

Classe de saída¶

A classe Output é uma classe central com uma função principal: Para enviar o página finalizada para o navegador solicitante. Também é responsável por caching suas páginas da web, se você usar isso feature.

Nota

Esta classe é inicializada automaticamente pelo sistema para que Não é necessário fazer isso manualmente.

Em circunstâncias normais, você nem notará a classe Output desde funciona de forma transparente sem a sua intervenção. Por exemplo, quando você use o Carregador classe para carregar um arquivo de View, é automaticamente passado para a classe Output, que será chamada automaticamente pelo CodeIgniter no final da execução do sistema. Isto é possível, no entanto, para você intervir manualmente com a saída se você preciso.

  • Referência de Classe

Referência de Classe¶

class CI_Output¶
$parse_exec_vars = TRUE;

Ativa/desativa a análise das pseudo-variáveis {elapsed_time} e {memory_usage}.

O CodeIgniter irá analisar esses tokens em sua saída por padrão. Para desabilitar isso, defina esta propriedade para FALSE no seu Controller.

$this->output->parse_exec_vars = FALSE;
set_output($output)¶
Parâmetros:
  • $output (string) - String para definir a saída para
Retorna:

Instância CI_Output (encadeamento do método)
Tipo de retorno:

CI_Output

Permite que você defina manualmente a sequência de saída final. Exemplo de uso:

$this->output->set_output($data);

Importante

Se você definir sua saída manualmente, ela deve ser a última coisa feita na função que você chama de. Por exemplo, se você construir uma página em um dos métodos do seu Controller, não defina a saída até o final.

set_content_type($mime_type[, $charset = NULL])¶
Parâmetros:
  • $mime_type (string) - Cadeia de caracteres do tipo MIME idenitifer
  • $charset (string) - Conjunto de caracteres
Retorna:

Instância CI_Output (encadeamento do método)
Tipo de retorno:

CI_Output

Permite que você defina o tipo MIME de sua página para que você possa exibir dados JSON, JPEG, XML, etc. facilmente.

$this->output
        ->set_content_type('application/json')
        ->set_output(json_encode(array('foo' => 'bar')));

$this->output
        ->set_content_type('jpeg') // You could also use ".jpeg" which will have the full stop removed before looking in config/mimes.php
        ->set_output(file_get_contents('files/something.jpg'));

Importante

Certifique-se de que qualquer string que não seja mime seja passada para este método exists in application/config/mimes.php or it will have no effect.

Você também pode definir o conjunto de caracteres do documento, passando um segundo argumento:

$this->output->set_content_type('css', 'utf-8');
get_content_type()¶
Retorna:String de tipo de conteúdo
Tipo de retorno:string

Retorna o cabeçalho HTTP Content-Type que está atualmente em uso, excluindo o valor do conjunto de caracteres.

$mime = $this->output->get_content_type();

Nota

Se não definido, o valor de retorno padrão é 'text/html'.

get_header($header)¶
Parâmetros:
  • $header (string) - nome do cabeçalho HTTP
Retorna:

Cabeçalho de resposta HTTP ou NULL se não for encontrado

Classe de saída¶

A classe Output é uma classe central com uma função principal: Para enviar o página finalizada para o navegador solicitante. Também é responsável por caching suas páginas da web, se você usar isso feature.

Nota

Esta classe é inicializada automaticamente pelo sistema para que Não é necessário fazer isso manualmente.

Em circunstâncias normais, você nem notará a classe Output desde funciona de forma transparente sem a sua intervenção. Por exemplo, quando você use the Loader classe para carregar um arquivo de View, é automaticamente passado para a classe Output, que será chamada automaticamente pelo CodeIgniter no final da execução do sistema. Isto é possível, no entanto, para você intervir manualmente com a saída se você need to.

  • Referência de Classe

$this->output->parse_exec_vars = FALSE;

  • $charset (string) - Conjunto de caracteres

Permite que você defina o tipo MIME de sua página para que você possa exibir dados JSON, JPEG, XML, etc. facilmente.

$this->output

$this->output->set_content_type('css', 'utf-8');
